## Introduction

Playwright는 테스트 보고서에 표시되는 태그와 어노테이션을 지원합니다.

언제든지 사용자 정의 태그와 어노테이션을 추가할 수 있지만, Playwright에는 몇 가지 기본 제공 어노테이션이 있습니다.
- [test.skip()](/api/class-test.mdx#test-skip)은 테스트를 관련 없는 것으로 표시합니다. Playwright는 이러한 테스트를 실행하지 않습니다. 이 어노테이션은 일부 구성에서 테스트를 적용할 수 없는 경우에 사용하세요.
- [test.fail()](/api/class-test.mdx#test-fail)은 테스트를 실패로 표시합니다. Playwright는 이 테스트를 실행하고 실제로 실패하는지 확인합니다. 테스트가 실패하지 않으면 Playwright에서 불만을 제기합니다.
- [test.fixme()](/api/class-test.mdx#test-fixme)는 테스트를 실패로 표시합니다. Playwright는 `fail` 어노테이션과 달리 이 테스트를 실행하지 않습니다. 테스트 실행이 느리거나 충돌하는 경우 `fixme`를 사용하세요.
- [test.slow()](/api/class-test.mdx#test-slow)는 테스트를 느린 것으로 표시하고 테스트 타임아웃을 3배로 늘립니다.

어노테이션은 단일 테스트 또는 테스트 그룹에 추가할 수 있습니다.

기본 제공 어노테이션은 조건부일 수 있으며, 이 경우 조건이 참일 때 적용되고 테스트 픽스처에 따라 달라질 수 있습니다. 동일한 테스트에 여러 어노테이션이 있을 수 있으며, 구성이 다를 수도 있습니다.

## Focus a test

일부 테스트에 집중할 수 있습니다. 집중된 테스트가 있으면 이러한 테스트만 실행됩니다.

```js
test.only('focus this test', async ({ page }) => {
  // 전체 프로젝트에서 집중된 테스트만 실행합니다.
});
```

## Skip a test

테스트를 건너뛰는 것으로 표시합니다.

```js
test.skip('skip this test', async ({ page }) => {
  // 이 테스트는 실행되지 않습니다.
});
```

## Conditionally skip a test

조건에 따라 특정 테스트를 건너뛸 수 있습니다.

```js
test('skip this test', async ({ page, browserName }) => {
  test.skip(browserName === 'firefox', 'Still working on it');
});
```

## Group tests

테스트를 그룹화하여 논리적 이름을 지정하거나 그룹에 before/after 훅을 적용할 수 있습니다.

```js
import { test, expect } from '@playwright/test';

test.describe('two tests', () => {
  test('one', async ({ page }) => {
    // ...
  });

  test('two', async ({ page }) => {
    // ...
  });
});
```

## Tag tests

때로는 테스트에 `@fast` 또는 `@slow`와 같은 태그를 지정한 다음 테스트 보고서에서 태그별로 필터링하고 싶을 수 있습니다. 또는 특정 태그가 있는 테스트만 실행하고 싶을 수도 있습니다.

테스트에 태그를 지정하려면 테스트를 선언할 때 추가 세부 정보 객체를 제공하거나 테스트 제목에 `@`-토큰을 추가합니다. 태그는 `@` 기호로 시작해야 합니다.

```js
import { test, expect } from '@playwright/test';

test('test login page', {
  tag: '@fast',
}, async ({ page }) => {
  // ...
});

test('test full report @slow', async ({ page }) => {
  // ...
});
```

그룹의 모든 테스트에 태그를 지정하거나 여러 태그를 제공할 수도 있습니다.

```js
import { test, expect } from '@playwright/test';

test.describe('group', {
  tag: '@report',
}, () => {
  test('test report header', async ({ page }) => {
    // ...
  });

  test('test full report', {
    tag: ['@slow', '@vrt'],
  }, async ({ page }) => {
    // ...
  });
});
```

이제 [`--grep`](./test-cli.mdx#reference) 명령줄 옵션을 사용하여 특정 태그가 있는 테스트를 실행할 수 있습니다.


## Annotate tests

태그보다 더 실질적인 것으로 테스트에 주석을 달고 싶다면 테스트를 선언할 때 그렇게 할 수 있습니다. 주석에는 더 많은 컨텍스트를 위한 `type`과 `description`이 있으며 리포터 API에서 사용할 수 있습니다. Playwright의 기본 제공 HTML 리포터는 `type`이 `_` 기호로 시작하는 주석을 제외한 모든 주석을 표시합니다.

예를 들어, 문제 URL로 테스트에 주석을 달려면:

```js
import { test, expect } from '@playwright/test';

test('test login page', {
  annotation: {
    type: 'issue',
    description: 'https://github.com/microsoft/playwright/issues/23180',
  },
}, async ({ page }) => {
  // ...
});
```

그룹의 모든 테스트에 주석을 달거나 여러 주석을 제공할 수도 있습니다.

```js
import { test, expect } from '@playwright/test';

test.describe('report tests', {
  annotation: { type: 'category', description: 'report' },
}, () => {
  test('test report header', async ({ page }) => {
    // ...
  });

  test('test full report', {
    annotation: [
      { type: 'issue', description: 'https://github.com/microsoft/playwright/issues/23180' },
      { type: 'performance', description: 'very slow test!' },
    ],
  }, async ({ page }) => {
    // ...
  });
});
```

## Conditionally skip a group of tests

예를 들어 콜백을 전달하여 Chromium에서만 테스트 그룹을 실행할 수 있습니다.

```js title="example.spec.ts"

test.describe('chromium only', () => {
  test.skip(({ browserName }) => browserName !== 'chromium', 'Chromium only!');

  test.beforeAll(async () => {
    // 이 훅은 Chromium에서만 실행됩니다.
  });

  test('test 1', async ({ page }) => {
    // 이 테스트는 Chromium에서만 실행됩니다.
  });

  test('test 2', async ({ page }) => {
    // 이 테스트는 Chromium에서만 실행됩니다.
  });
});
```

## Use fixme in `beforeEach` hook

`beforeEach` 훅 실행을 방지하려면 훅 자체에 어노테이션을 넣을 수 있습니다.

```js title="example.spec.ts"

test.beforeEach(async ({ page, isMobile }) => {
  test.fixme(isMobile, 'Settings page does not work in mobile yet');

  await page.goto('http://localhost:3000/settings');
});

test('user profile', async ({ page }) => {
  await page.getByText('My Profile').click();
  // ...
});
```

## Runtime annotations

테스트가 이미 실행 중인 동안 [`test.info().annotations`](./api/class-testinfo#test-info-annotations)에 주석을 추가할 수 있습니다.

```js title="example.spec.ts"

test('example test', async ({ page, browser }) => {
  test.info().annotations.push({
    type: 'browser version',
    description: browser.version(),
  });

  // ...
});
```
