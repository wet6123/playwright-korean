## 소개

Playwright로 작성된 테스트는 브라우저 컨텍스트라고 하는 격리된 초기화 환경에서 실행됩니다. 이 격리 모델은 재현성을 향상시키고 테스트 실패의 연쇄 작용을 방지합니다.

## 테스트 격리란 무엇인가요?

테스트 격리는 각 테스트가 다른 테스트와 완전히 격리되는 경우입니다. 모든 테스트는 다른 테스트와 독립적으로 실행됩니다. 즉, 각 테스트는 자체 로컬 스토리지, 세션 스토리지, 쿠키 등을 갖습니다. Playwright는 시크릿 모드와 유사한 프로필과 동일한 [BrowserContext]를 사용하여 이를 달성합니다. 생성 속도가 빠르고 비용이 저렴하며 단일 브라우저에서 실행되는 경우에도 완전히 격리됩니다. Playwright는 각 테스트에 대한 컨텍스트를 생성하고 해당 컨텍스트에서 기본 [Page]를 제공합니다.

## 테스트 격리가 중요한 이유는 무엇인가요?

- 오류 전파 방지. 한 테스트가 실패하더라도 다른 테스트에 영향을 미치지 않습니다.
- 오류 또는 불안정성을 쉽게 디버깅할 수 있습니다. 원하는 만큼 단일 테스트를 실행할 수 있기 때문입니다.
- 병렬 실행, 샤딩 등을 할 때 순서에 대해 생각할 필요가 없습니다.

## 테스트 격리의 두 가지 방법

테스트 격리에는 두 가지 전략이 있습니다. 처음부터 시작하거나 테스트 사이에 정리하는 것입니다. 테스트 사이에 정리하는 방식의 문제점은 정리를 잊어버리기 쉽고 "방문한 링크"와 같은 일부 항목은 정리할 수 없다는 것입니다. 한 테스트의 상태가 다음 테스트로 유출되어 테스트가 실패할 수 있으며 문제가 다른 테스트에서 발생하기 때문에 디버깅이 더 어려워집니다. 처음부터 시작한다는 것은 모든 것이 새롭다는 것을 의미하므로 테스트가 실패하면 해당 테스트 내에서만 디버깅하면 됩니다.

## Playwright는 테스트 격리를 어떻게 달성하는가

Playwright는 브라우저 컨텍스트를 사용하여 테스트 격리를 달성합니다. 각 테스트에는 자체 브라우저 컨텍스트가 있습니다. 테스트를 실행할 때마다 새 브라우저 컨텍스트가 생성됩니다. Playwright를 테스트 러너로 사용하는 경우 브라우저 컨텍스트가 기본적으로 생성됩니다. 그렇지 않으면 브라우저 컨텍스트를 수동으로 생성할 수 있습니다.

<TabItem value="test">

```js
import { test } from '@playwright/test';

test('example test', async ({ page, context }) => {
  // "context"는 이 특정 테스트를 위해 생성된 격리된 BrowserContext입니다.
  // "page"는 이 컨텍스트에 속합니다.
});

test('another test', async ({ page, context }) => {
  // 이 두 번째 테스트의 "context"와 "page"는 첫 번째 테스트와 완전히
  // 격리되어 있습니다.
});
```

</TabItem>
<TabItem value="library">

```js
const browser = await chromium.launch();
const context = await browser.newContext();
const page = await context.newPage();
```

</TabItem>

브라우저 컨텍스트는 모바일 장치, 권한, 로케일 및 색상 구성표와 관련된 다중 페이지 시나리오를 에뮬레이션하는 데에도 사용할 수 있습니다. 자세한 내용은 [에뮬레이션](./emulation.mdx) 가이드를 확인하세요.

## 단일 테스트에서 여러 컨텍스트

Playwright는 단일 시나리오 내에서 여러 브라우저 컨텍스트를 생성할 수 있습니다. 채팅과 같은 다중 사용자 기능을 테스트하려는 경우 유용합니다.

<Tabs
  groupId="js-flavor"
  defaultValue="test"
  values={[
    {label: 'Test', value: 'test'},
    {label: 'Library', value: 'library'}
  ]
}>
<TabItem value="test">

```js
import { test } from '@playwright/test';

test('admin and user', async ({ browser }) => {
  // 두 개의 격리된 브라우저 컨텍스트 생성
  const adminContext = await browser.newContext();
  const userContext = await browser.newContext();

  // 페이지를 생성하고 컨텍스트와 독립적으로 상호 작용
  const adminPage = await adminContext.newPage();
  const userPage = await userContext.newPage();
});
```

</TabItem>
<TabItem value="library">

```js
const { chromium } = require('playwright');

// Chromium 브라우저 인스턴스 생성
const browser = await chromium.launch();

// 두 개의 격리된 브라우저 컨텍스트 생성
const userContext = await browser.newContext();
const adminContext = await browser.newContext();

// 페이지 생성 및 컨텍스트와 독립적으로 상호 작용
const adminPage = await adminContext.newPage();
const userPage = await userContext.newPage();
```

</TabItem>
