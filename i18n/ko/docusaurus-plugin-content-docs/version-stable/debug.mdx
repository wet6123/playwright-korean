## VS Code 디버거

더 나은 개발자 경험을 위해 [VS Code 확장](./getting-started-vscode.mdx)을 사용하여 디버깅하는 것이 좋습니다. VS Code 확장을 사용하면 VS Code에서 바로 테스트를 디버깅하고, 오류 메시지를 확인하고, 중단점을 설정하고, 테스트를 단계별로 실행할 수 있습니다.

<img width="1269" alt="running test in debug mode" src="https://user-images.githubusercontent.com/13063165/212740233-3f278825-13e7-4a88-a118-dd4478d43a16.png" />

### 오류 메시지

테스트가 실패하면 VS Code는 예상된 내용, 수신된 내용 및 전체 호출 로그를 보여주는 편집기에서 바로 오류 메시지를 표시합니다.

<img width="1269" alt="error messaging in vs code" src="https://user-images.githubusercontent.com/13063165/212738654-b573b7c9-05be-476f-ab4c-201bf4265bc0.png" />

### 라이브 디버깅

VS Code에서 테스트를 라이브로 디버깅할 수 있습니다. `브라우저 표시` 옵션을 선택한 상태로 테스트를 실행한 후 VS Code에서 로케이터를 클릭하면 브라우저 창에서 강조 표시됩니다. Playwright는 여러 일치 항목이 있는지 여부도 표시합니다.

<img width="1394" alt="live debugging in VS Code" src="https://user-images.githubusercontent.com/13063165/212884329-0755b007-0d69-4987-b084-38fd5bfb577d.png" />

VS Code에서 로케이터를 편집할 수도 있으며 Playwright는 브라우저 창에서 변경 사항을 실시간으로 보여줍니다.

<img width="1394" alt="live debugging in VS Code" src="https://user-images.githubusercontent.com/13063165/212884772-5022d4b1-6fab-456f-88e3-506f2354e238.png" />

### 로케이터 선택

테스트 사이드바에서 **로케이터 선택** 버튼을 클릭하여 [로케이터](./locators.mdx)를 선택하고 테스트 파일에 복사합니다. 그런 다음 브라우저에서 필요한 요소를 클릭하면 VS Code의 **로케이터 선택** 상자에 표시됩니다. 키보드에서 'enter'를 눌러 로케이터를 클립보드에 복사한 다음 코드의 아무 곳에나 붙여넣습니다. 취소하려면 'escape'를 누릅니다.

<img width="1394" alt="Pick locators" src="https://user-images.githubusercontent.com/13063165/212741666-6479a702-2517-44a3-9eca-e719e13b379c.png" />

Playwright는 페이지를 보고 [역할, 텍스트 및 테스트 ID 로케이터](./locators.mdx)를 우선적으로 고려하여 가장 적합한 로케이터를 파악합니다. Playwright가 로케이터와 일치하는 여러 요소를 찾으면 로케이터를 개선하여 복원력을 높이고 대상 요소를 고유하게 식별하므로 로케이터로 인해 테스트가 실패할 염려가 없습니다.

### 디버그 모드에서 실행

중단점을 설정하려면 빨간색 점이 나타날 때까지 중단점을 설정할 줄 번호 옆을 클릭합니다. 실행하려는 테스트 옆의 줄을 마우스 오른쪽 버튼으로 클릭하여 디버그 모드에서 테스트를 실행합니다.

<img width="1269" alt="setting debug test mode" src="https://user-images.githubusercontent.com/13063165/212739847-ecb7dcfe-8929-45f3-b24e-f9c4b592f430.png" />

브라우저 창이 열리고 테스트가 실행되어 중단점이 설정된 위치에서 일시 중지됩니다. VS Code의 메뉴에서 테스트를 단계별로 실행하고, 테스트를 일시 중지하고, 테스트를 다시 실행할 수 있습니다.

<img width="1269" alt="running test in debug mode" src="https://user-images.githubusercontent.com/13063165/212740233-3f278825-13e7-4a88-a118-dd4478d43a16.png" />

### 다른 브라우저에서 디버깅

기본적으로 디버깅은 Chromium 프로필을 사용하여 수행됩니다. 테스트 사이드바에서 디버그 아이콘을 마우스 오른쪽 버튼으로 클릭하고 드롭다운에서 '기본 프로필 선택' 옵션을 클릭하여 다른 브라우저에서 테스트를 디버깅할 수 있습니다.

<img width="1312" alt="debugging on specific profile" src="https://user-images.githubusercontent.com/13063165/212879469-436f8130-c62a-49e1-9d67-c1903b478d5f.png" />

그런 다음 테스트 디버깅에 사용할 테스트 프로필을 선택합니다. 디버그 모드에서 테스트를 실행할 때마다 선택한 프로필이 사용됩니다. 테스트가 있는 줄 번호를 마우스 오른쪽 버튼으로 클릭하고 메뉴에서 '테스트 디버그'를 선택하여 디버그 모드에서 테스트를 실행할 수 있습니다.

<img width="1312" alt="choosing a profile for debugging" src="https://user-images.githubusercontent.com/13063165/212880198-eac22c3e-68ce-47da-9163-d6b376ae7575.png" />

디버깅에 대한 자세한 내용은 [Visual Studio Code에서 디버깅](https://code.visualstudio.com/docs/editor/debugging)을 참조하세요.

## Playwright Inspector

Playwright Inspector는 Playwright 테스트를 디버깅하는 데 도움이 되는 GUI 도구입니다. 테스트를 단계별로 실행하고, 로케이터를 실시간으로 편집하고, 로케이터를 선택하고, 동작성 로그를 볼 수 있습니다.

<img width="864" alt="Playwright Inspector" src="https://user-images.githubusercontent.com/13063165/212924587-4b84e5f6-b147-40e9-8c75-d7b9ab6b7ca1.png" />

### 디버그 모드에서 실행

`--debug` 플래그와 함께 테스트를 실행하여 Inspector를 엽니다. 이렇게 하면 Playwright가 디버깅용으로 구성되고 Inspector가 열립니다. `--debug`를 사용하면 다음과 같은 유용한 기본값이 추가로 구성됩니다.
- 브라우저가 헤드 모드로 실행됩니다.
- 기본 제한 시간이 0으로 설정됩니다(= 제한 시간 없음).

#### 모든 브라우저에서 모든 테스트 디버깅

모든 테스트를 디버깅하려면 `--debug` 플래그와 함께 테스트 명령을 실행합니다. 이렇게 하면 테스트가 하나씩 실행되고 각 테스트에 대해 Inspector와 브라우저 창이 열립니다.

```bash
npx playwright test --debug
```

#### 모든 브라우저에서 하나의 테스트 디버깅

특정 줄에서 하나의 테스트를 디버깅하려면 테스트 명령 뒤에 디버깅할 테스트 파일 이름과 줄 번호를 차례로 입력하고 `--debug` 플래그를 입력합니다. 이렇게 하면 [`playwright.config`](./test-projects.mdx#configure-projects-for-multiple-browsers)에 구성된 각 브라우저에서 단일 테스트가 실행되고 Inspector가 열립니다.

```bash
npx playwright test example.spec.ts:10 --debug
```

#### 특정 브라우저에서 디버깅

Playwright에서는 [`playwright.config`](./test-projects.mdx#configure-projects-for-multiple-browsers)에서 프로젝트를 구성할 수 있습니다. 구성이 완료되면 `--project` 플래그 뒤에 `playwright.config`에 구성된 프로젝트 이름을 차례로 입력하여 특정 브라우저 또는 모바일 뷰포트에서 테스트를 디버깅할 수 있습니다.

```bash
npx playwright test --project=chromium --debug
npx playwright test --project="Mobile Safari" --debug
npx playwright test --project="Microsoft Edge" --debug
```

#### 특정 브라우저에서 하나의 테스트 디버깅

특정 브라우저에서 하나의 테스트를 실행하려면 테스트 파일 이름과 디버깅할 테스트의 줄 번호와 함께 `--project` 플래그 뒤에 프로젝트 이름을 차례로 추가합니다.

```bash
npx playwright test example.spec.ts:10 --project=webkit --debug
```

### 테스트 단계별 실행

Inspector 상단의 도구 모음을 사용하여 테스트의 각 동작을 재생, 일시 중지 또는 단계별로 실행할 수 있습니다. 테스트 코드에서 강조 표시된 현재 동작과 브라우저 창에서 강조 표시된 일치하는 요소를 볼 수 있습니다.

<img width="1340" alt="Playwright Inspector and browser" src="https://user-images.githubusercontent.com/13063165/212936618-84b87acc-bc2e-46ed-994b-32b2ef742e60.png" />

### 특정 중단점에서 테스트 실행

디버깅 프로세스 속도를 높이려면 테스트에 [page.pause()](/api/class-page.mdx#page-pause) 메서드를 추가할 수 있습니다. 이렇게 하면 디버깅할 지점에 도달하기 위해 테스트의 각 동작을 단계별로 실행할 필요가 없습니다.

```js
await page.pause();
```

`page.pause()` 호출을 추가한 후 디버그 모드에서 테스트를 실행합니다. Inspector에서 "다시 시작" 버튼을 클릭하면 테스트가 실행되고 `page.pause()`에서만 중지됩니다.

<img width="1350" alt="test with page.pause" src="https://user-images.githubusercontent.com/13063165/219473050-122be4c2-31d0-4cbd-aa8b-8588e8b781a6.png" />

### 실시간 로케이터 편집

디버그 모드에서 실행하는 동안 로케이터를 실시간으로 편집할 수 있습니다. '로케이터 선택' 버튼 옆에는 테스트가 일시 중지된 [로케이터](./locators.mdx)를 보여주는 필드가 있습니다. **로케이터 선택** 필드에서 이 로케이터를 직접 편집할 수 있으며 일치하는 요소가 브라우저 창에서 강조 표시됩니다.

<img width="1348" alt="live editing locators" src="https://user-images.githubusercontent.com/13063165/212980815-1cf6ef7b-e69a-496c-898a-ec603a3bc562.png" />

### 로케이터 선택

디버깅하는 동안 더 복원력이 있는 로케이터를 선택해야 할 수도 있습니다. **로케이터 선택** 버튼을 클릭하고 브라우저 창에서 요소를 마우스로 가리키면 됩니다. 요소를 마우스로 가리키면 이 요소를 찾는 데 필요한 코드가 아래에 강조 표시됩니다. 브라우저에서 요소를 클릭하면 로케이터가 필드에 추가되어 코드를 조정하거나 복사할 수 있습니다.

<img width="1392" alt="Picking locators" src="https://user-images.githubusercontent.com/13063165/212968640-ce82a027-9277-4bdf-b0a9-6282fb2becb7.png" />

Playwright는 페이지를 보고 [역할, 텍스트 및 테스트 ID 로케이터](./locators.mdx)를 우선적으로 고려하여 가장 적합한 로케이터를 파악합니다. Playwright가 로케이터와 일치하는 여러 요소를 찾으면 로케이터를 개선하여 복원력을 높이고 대상 요소를 고유하게 식별하므로 로케이터로 인해 테스트가 실패할 염려가 없습니다.

### 동작성 로그

Playwright가 클릭 동작에서 일시 중지될 때까지 로그에서 찾을 수 있는 [동작성 검사](./actionability.mdx)를 이미 수행했습니다. 이렇게 하면 테스트 중에 발생한 상황과 Playwright가 수행하거나 시도한 작업을 이해하는 데 도움이 됩니다. 로그는 요소가 표시되는지, 사용 가능한지, 안정적인지, 로케이터가 요소로 확인되었는지, 뷰로 스크롤되었는지 등을 알려줍니다. 동작성에 도달할 수 없으면 작업을 보류 중으로 표시합니다.

<img width="883" alt="Actionability Logs" src="https://user-images.githubusercontent.com/13063165/212968907-5dede739-e0e3-482a-91cd-726a0f5b0b6d.png" />

## 브라우저 개발자 도구

`PWDEBUG=console`을 사용하여 디버그 모드로 실행하면 개발자 도구 콘솔에서 `playwright` 객체를 사용할 수 있습니다. 개발자 도구는 다음과 같은 도움을 줄 수 있습니다.
- DOM 트리를 검사하고 **요소 선택기 찾기**
- 실행 중에 **콘솔 로그 보기**(또는 [API를 통해 로그를 읽는 방법](./api/class-page.mdx#page-event-console) 알아보기)
- **네트워크 활동** 및 기타 개발자 도구 기능 확인

이렇게 하면 Playwright의 기본 제한 시간이 0(= 제한 시간 없음)으로 설정됩니다.

<img width="1399" alt="Browser Developer Tools with Playwright object" src="https://user-images.githubusercontent.com/13063165/219128002-898f604d-9697-4b7f-95b5-a6a8260b7282.png" />

브라우저 개발자 도구를 사용하여 테스트를 디버깅하려면 먼저 [page.pause()](/api/class-page.mdx#page-pause) 메서드를 사용하여 실행을 일시 중지하도록 테스트에 중단점을 설정합니다.

```js
await page.pause();
```

테스트에 중단점을 설정했으면 `PWDEBUG=console`을 사용하여 테스트를 실행할 수 있습니다.

Playwright가 브라우저 창을 시작하면 개발자 도구를 열 수 있습니다. 콘솔 패널에서 `playwright` 객체를 사용할 수 있습니다.

#### playwright.$(selector)

실제 Playwright 쿼리 엔진을 사용하여 Playwright 선택기를 쿼리합니다. 예를 들면 다음과 같습니다.

```bash
playwright.$('.auth-form >> text=Log in');

<button>Log in</button>
```

#### playwright.$$(selector)

`playwright.$`와 동일하지만 일치하는 모든 요소를 반환합니다.

```bash
playwright.$$('li >> text=John')

[<li>, <li>, <li>, <li>]
```

#### playwright.inspect(selector)

요소 패널에서 요소를 표시합니다.

```bash
playwright.inspect('text=Log in')
```

#### playwright.locator(selector)

로케이터를 만들고 일치하는 요소를 쿼리합니다. 예를 들면 다음과 같습니다.

```bash
playwright.locator('.auth-form', { hasText: 'Log in' });

Locator ()
  - element: button
  - elements: [button]
```

#### playwright.selector(element)

지정된 요소에 대한 선택기를 생성합니다. 예를 들어 요소 패널에서 요소를 선택하고 `$0`을 전달합니다.

```bash
playwright.selector($0)
```

:::

## 헤드 모드

Playwright는 기본적으로 헤드리스 모드로 브라우저를 실행합니다. 이 동작을 변경하려면 `headless: false`를 시작 옵션으로 사용합니다.

[slowMo](/api/class-browsertype.mdx#browser-type-launch-option-slow-mo) 옵션을 사용하여 실행 속도를 늦추고(작업당 N밀리초) 디버깅하는 동안 따라갈 수도 있습니다.

```js
// Chromium, Firefox 또는 WebKit
await chromium.launch({ headless: false, slowMo: 100 });
```
