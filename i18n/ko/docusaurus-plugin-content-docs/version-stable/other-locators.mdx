## 소개

:::note

가장 흔하고 권장되는 로케이터는 주요 [로케이터 가이드](./locators)를 확인하세요.
:::

[page.getByRole()](/api/class-page.mdx#page-get-by-role) 및 [page.getByText()](/api/class-page.mdx#page-get-by-text)와 같은 권장 로케이터 외에도 Playwright는 이 가이드에 설명된 다양한 다른 로케이터를 지원합니다.

## CSS 로케이터

:::note

구현에 종속되어 페이지가 변경될 때 손상될 수 있는 CSS를 사용하는 대신 텍스트 또는 접근성 역할과 같은 [사용자에게 보이는 로케이터](./locators.mdx#quick-guide)를 우선적으로 사용하는 것이 좋습니다.
:::

Playwright는 CSS 선택자로 요소를 찾을 수 있습니다.

```js
await page.locator('css=button').click();
```

Playwright는 표준 CSS 선택자를 다음 두 가지 방법으로 확장합니다.
* CSS 선택자는 섀도우 DOM을 뚫습니다.
* Playwright는 `:visible`, `:has-text()`, `:has()`, `:is()`, `:nth-match()` 등과 같은 사용자 지정 가상 클래스를 추가합니다.

### CSS: 텍스트로 매칭

Playwright는 텍스트 콘텐츠로 요소를 매칭하기 위해 여러 CSS 가상 클래스를 포함합니다.
- `article:has-text("Playwright")` - `:has-text()`는 지정된 텍스트를 내부에 포함하는 모든 요소를 일치시키며, 자식 또는 하위 요소에 있을 수도 있습니다. 매칭은 대소문자를 구분하지 않고 공백을 자르고 부분 문자열을 검색합니다.
  
  예를 들어 `article:has-text("Playwright")`는 `<article><div>Playwright</div></article>`와 일치합니다.
  
  `:has-text()`는 다른 CSS 지정자와 함께 사용해야 합니다. 그렇지 않으면 `<body>`를 포함하여 지정된 텍스트를 포함하는 모든 요소와 일치합니다.
  
  ```js
  // 잘못됨, <body>를 포함하여 많은 요소와 일치합니다.
  await page.locator(':has-text("Playwright")').click();
  // 올바름, <article> 요소만 일치합니다.
  await page.locator('article:has-text("Playwright")').click();
  ```
  
- `#nav-bar :text("Home")` - `:text()` 가상 클래스는 지정된 텍스트를 포함하는 가장 작은 요소를 매칭합니다. 매칭은 대소문자를 구분하지 않고 공백을 자르고 부분 문자열을 검색합니다.
  
  예를 들어, 이것은 `#nav-bar` 요소 내부에 "Home" 텍스트가 있는 요소를 찾습니다.
  
  ```js
  await page.locator('#nav-bar :text("Home")').click();
  ```
  
- `#nav-bar :text-is("Home")` - `:text-is()` 가상 클래스는 정확한 텍스트가 있는 가장 작은 요소를 매칭합니다. 정확한 매칭은 대소문자를 구분하고 공백을 자르고 전체 문자열을 검색합니다.
  
  예를 들어 `:text-is("Log")`는 `<button>Log in</button>`와 일치하지 않습니다. 왜냐하면 `<button>`은 `"Log"`와 같지 않은 단일 텍스트 노드 `"Log in"`을 포함하기 때문입니다. 하지만 `:text-is("Log")`는 `<button> Log <span>in</span></button>`와 일치합니다. 왜냐하면 `<button>`은 `" Log "` 텍스트 노드를 포함하기 때문입니다.
  
  마찬가지로, `:text-is("Download")`는 대소문자를 구분하기 때문에 `<button>download</button>`와 일치하지 않습니다.
* `#nav-bar :text-matches("reg?ex", "i")` - `:text-matches()` 가상 클래스는 [JavaScript와 유사한 정규식](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/RegExp)과 일치하는 텍스트 콘텐츠가 있는 가장 작은 요소를 매칭합니다.
  
  예를 들어 `:text-matches("Log\s*in", "i")`는 `<button>Login</button>` 및 `<button>log IN</button>`과 일치합니다.

:::note

텍스트 매칭은 항상 공백을 정규화합니다. 예를 들어 여러 공백을 하나로 바꾸고, 줄 바꿈을 공백으로 바꾸고, 앞뒤 공백을 무시합니다.
:::

:::note
`button` 및 `submit` 유형의 입력 요소는 텍스트 콘텐츠 대신 `value`로 매칭됩니다. 예를 들어 `:text("Log in")`은 `<input type=button value="Log in">`과 일치합니다.
:::

### CSS: 보이는 요소만 매칭

Playwright는 CSS 선택자에서 `:visible` 가상 클래스를 지원합니다. 예를 들어 `css=button`은 페이지의 모든 버튼과 일치하지만 `css=button:visible`은 보이는 버튼만 일치시킵니다. 이는 매우 유사하지만 가시성이 다른 요소를 구별하는 데 유용합니다.

첫 번째 버튼은 보이지 않고 두 번째 버튼은 보이는 페이지를 고려하십시오.

```html
<button style='display: none'>Invisible</button>
<button>Visible</button>
```

* 이것은 두 버튼을 모두 찾아 [엄격성](./locators.mdx#strictness) 위반 오류를 발생시킵니다.
  
  ```js
  await page.locator('button').click();
  ```
  
* 이것은 두 번째 버튼만 찾고 클릭합니다. 왜냐하면 그것이 보이기 때문입니다.
  
  ```js
  await page.locator('button:visible').click();
  ```
  
### CSS: 다른 요소를 포함하는 요소

`:has()` 가상 클래스는 [실험적인 CSS 가상 클래스](https://developer.mozilla.org/en-US/docs/Web/CSS/:has)입니다. 주어진 요소의 `:scope`에 상대적으로 매개변수로 전달된 선택기가 하나 이상의 요소와 일치하는 경우 요소를 반환합니다.

다음 스니펫은 내부에 `<div class=promo>`가 있는 `<article>` 요소의 텍스트 콘텐츠를 반환합니다.

```js
await page.locator('article:has(div.promo)').textContent();
```

### CSS: 조건 중 하나와 일치하는 요소

쉼표로 구분된 CSS 선택자 목록은 해당 목록의 선택자 중 하나로 선택할 수 있는 모든 요소와 일치합니다.

```js
// "Log in" 또는 "Sign in" 텍스트가 있는 <button>을 클릭합니다.
await page.locator('button:has-text("Log in"), button:has-text("Sign in")').click();
```

`:is()` 가상 클래스는 요소에 대한 추가 조건 목록을 지정하는 데 유용할 수 있는 [실험적인 CSS 가상 클래스](https://developer.mozilla.org/en-US/docs/Web/CSS/:is)입니다.

### CSS: 레이아웃 기반 요소 매칭

:::note

레이아웃 기반 매칭은 예상치 못한 결과를 낼 수 있습니다. 예를 들어 레이아웃이 1픽셀 변경되면 다른 요소가 일치될 수 있습니다.
:::

때로는 대상 요소에 고유한 기능이 없을 때 좋은 선택자를 생각해내기가 어렵습니다. 이 경우 Playwright 레이아웃 CSS 가상 클래스를 사용하면 도움이 될 수 있습니다. 이것들은 일반 CSS와 결합하여 여러 선택지 중 하나를 정확히 찾아낼 수 있습니다.

예를 들어 `input:right-of(:text("Password"))`는 "Password" 텍스트 오른쪽에 있는 입력 필드와 일치합니다. 페이지에 서로 구별하기 어려운 여러 입력이 있는 경우에 유용합니다.

레이아웃 가상 클래스는 `input`과 같은 다른 것과 함께 사용하는 것이 유용합니다. `:right-of(:text("Password"))`와 같이 레이아웃 가상 클래스를 단독으로 사용하면 찾고 있는 입력을 얻지 못할 가능성이 높지만 텍스트와 대상 입력 사이에 있는 일부 빈 요소를 얻을 수 있습니다.

레이아웃 가상 클래스는 [bounding client rect](https://developer.mozilla.org/en-US/docs/Web/API/Element/getBoundingClientRect)를 사용하여 요소의 거리와 상대적 위치를 계산합니다.
* `:right-of(div > button)` - 내부 선택자와 일치하는 요소의 오른쪽에 있는 요소를 모든 수직 위치에서 일치시킵니다.
* `:left-of(div > button)` - 내부 선택자와 일치하는 요소의 왼쪽에 있는 요소를 모든 수직 위치에서 일치시킵니다.
* `:above(div > button)` - 내부 선택자와 일치하는 요소 위에 있는 요소를 모든 수평 위치에서 일치시킵니다.
* `:below(div > button)` - 내부 선택자와 일치하는 요소 아래에 있는 요소를 모든 수평 위치에서 일치시킵니다.
* `:near(div > button)` - 내부 선택자와 일치하는 요소 근처(50 CSS 픽셀 이내)에 있는 요소를 일치시킵니다.

결과 일치 항목은 앵커 요소까지의 거리를 기준으로 정렬되므로 [locator.first()](/api/class-locator.mdx#locator-first)를 사용하여 가장 가까운 것을 선택할 수 있습니다. 이것은 목록과 같이 유사한 요소가 여러 개 있는 경우에만 유용하며, 가장 가까운 요소가 당연히 올바른 요소입니다. 그러나 [locator.first()](/api/class-locator.mdx#locator-first)를 다른 경우에 사용하면 예상대로 작동하지 않을 가능성이 높습니다. 찾고 있는 요소를 대상으로 하지 않고 임의의 빈 `<div>` 또는 스크롤되어 현재 보이지 않는 요소와 같이 가장 가까운 다른 요소를 대상으로 합니다.

```js
// "Username" 오른쪽에 있는 입력을 채웁니다.
await page.locator('input:right-of(:text("Username"))').fill('value');

// 프로모션 카드 근처의 버튼을 클릭합니다.
await page.locator('button:near(.promo-card)').click();

// "Label 3"에 가장 가까운 목록에서 라디오 입력을 클릭합니다.
await page.locator('[type=radio]:left-of(:text("Label 3"))').first().click();
```

모든 레이아웃 가상 클래스는 마지막 인수로 선택적 최대 픽셀 거리를 지원합니다. 예를 들어 `button:near(:text("Username"), 120)`는 "Username" 텍스트가 있는 요소에서 최대 120 CSS 픽셀 떨어져 있는 버튼과 일치합니다.

### CSS: 쿼리 결과에서 n번째 일치 항목 선택

:::note

일반적으로 일부 속성 또는 텍스트 콘텐츠로 요소를 구별할 수 있으며, 이는 페이지 변경에 더 탄력적입니다.
:::

때로는 페이지에 유사한 요소가 여러 개 있고 특정 요소를 선택하기가 어렵습니다. 예를 들어:

```html
<section> <button>Buy</button> </section>
<article><div> <button>Buy</button> </div></article>
<div><div> <button>Buy</button> </div></div>
```

이 경우 `:nth-match(:text("Buy"), 3)`은 위의 스니펫에서 세 번째 버튼을 선택합니다. 인덱스는 1부터 시작합니다.

```js
// 세 번째 "Buy" 버튼을 클릭합니다.
await page.locator(':nth-match(:text("Buy"), 3)').click();
```

`:nth-match()`는 [locator.waitFor()](/api/class-locator.mdx#locator-wait-for)를 사용하여 지정된 수의 요소가 나타날 때까지 기다리는 데 유용합니다.

```js
// 세 개의 버튼이 모두 보일 때까지 기다립니다.
await page.locator(':nth-match(:text("Buy"), 3)').waitFor();
```

:::note
[`:nth-child()`](https://developer.mozilla.org/en-US/docs/Web/CSS/:nth-child)와 달리 요소는 형제일 필요가 없으며 페이지 어디에나 있을 수 있습니다. 위의 스니펫에서 세 개의 버튼은 모두 `:text("Buy")` 선택자와 일치하고 `:nth-match()`는 세 번째 버튼을 선택합니다.
:::

## N번째 요소 로케이터

제로 베이스 인덱스를 전달하는 `nth=` 로케이터를 사용하여 쿼리를 n번째 일치 항목으로 좁힐 수 있습니다.

```js
// 첫 번째 버튼을 클릭합니다.
await page.locator('button').locator('nth=0').click();

// 마지막 버튼을 클릭합니다.
await page.locator('button').locator('nth=-1').click();
```

## 상위 요소 로케이터

다른 요소의 상위 요소를 대상으로 해야 할 때는 대부분 자식 로케이터로 [locator.filter()](/api/class-locator.mdx#locator-filter)해야 합니다. 예를 들어 다음 DOM 구조를 고려하십시오.

```html
<li><label>Hello</label></li>
<li><label>World</label></li>
```

텍스트가 `"Hello"`인 레이블의 상위 `<li>`를 대상으로 하려면 [locator.filter()](/api/class-locator.mdx#locator-filter)를 사용하는 것이 가장 좋습니다.

```js
const child = page.getByText('Hello');
const parent = page.getByRole('listitem').filter({ has: child });
```

또는 상위 요소에 적합한 로케이터를 찾을 수 없는 경우 `xpath=..`를 사용하십시오. DOM 구조의 변경으로 인해 테스트가 중단될 수 있으므로 이 방법은 신뢰할 수 없습니다. 가능한 경우 [locator.filter()](/api/class-locator.mdx#locator-filter)를 사용하는 것이 좋습니다.

```js
const parent = page.getByText('Hello').locator('xpath=..');
```

## React 로케이터

:::note

React 로케이터는 실험적이며 `_` 접두사가 붙습니다. 이 기능은 나중에 변경될 수 있습니다.
:::

React 로케이터를 사용하면 컴포넌트 이름과 속성 값으로 요소를 찾을 수 있습니다. 구문은 [CSS 속성 선택자](https://developer.mozilla.org/en-US/docs/Web/CSS/Attribute_selectors)와 매우 유사하며 모든 CSS 속성 선택자 연산자를 지원합니다.

React 로케이터에서 컴포넌트 이름은 **CamelCase**로 전사됩니다.

```js
await page.locator('_react=BookItem').click();
```

더 많은 예제:
- **컴포넌트**로 매칭: `_react=BookItem`
- 컴포넌트 및 **정확한 속성 값**으로 매칭, 대소문자 구분: `_react=BookItem[author = "Steven King"]`
- 속성 값만으로 매칭, **대소문자 구분 안 함**: `_react=[author = "steven king" i]`
- 컴포넌트 및 **참 속성 값**으로 매칭: `_react=MyButton[enabled]`
- 컴포넌트 및 **부울 값**으로 매칭: `_react=MyButton[enabled = false]`
- 속성 **값 부분 문자열**로 매칭: `_react=[author *= "King"]`
- 컴포넌트 및 **여러 속성**으로 매칭: `_react=BookItem[author *= "king" i][year = 1990]`
- **중첩된** 속성 값으로 매칭: `_react=[some.nested.value = 12]`
- 컴포넌트 및 속성 값 **접두사**로 매칭: `_react=BookItem[author ^= "Steven"]`
- 컴포넌트 및 속성 값 **접미사**로 매칭: `_react=BookItem[author $= "Steven"]`
- 컴포넌트 및 **키**로 매칭: `_react=BookItem[key = '2']`
- 속성 값 **정규식**으로 매칭: `_react=[author = /Steven(\\s+King)?/i]`

트리에서 React 요소 이름을 찾으려면 [React DevTools](https://chrome.google.com/webstore/detail/react-developer-tools/fmkadmapgofadopljbjfkapdkoienihi)를 사용하십시오.

:::note

React 로케이터는 React 15 이상을 지원합니다.
:::

:::note
React 로케이터와 [React DevTools](https://chrome.google.com/webstore/detail/react-developer-tools/fmkadmapgofadopljbjfkapdkoienihi)는 **축소되지 않은** 애플리케이션 빌드에 대해서만 작동합니다.
:::

## Vue 로케이터

:::note

Vue 로케이터는 실험적이며 `_` 접두사가 붙습니다. 이 기능은 나중에 변경될 수 있습니다.
:::

Vue 로케이터를 사용하면 컴포넌트 이름과 속성 값으로 요소를 찾을 수 있습니다. 구문은 [CSS 속성 선택자](https://developer.mozilla.org/en-US/docs/Web/CSS/Attribute_selectors)와 매우 유사하며 모든 CSS 속성 선택자 연산자를 지원합니다.

Vue 로케이터에서 컴포넌트 이름은 **kebab-case**로 전사됩니다.

```js
await page.locator('_vue=book-item').click();
```

더 많은 예제:
- **컴포넌트**로 매칭: `_vue=book-item`
- 컴포넌트 및 **정확한 속성 값**으로 매칭, 대소문자 구분: `_vue=book-item[author = "Steven King"]`
- 속성 값만으로 매칭, **대소문자 구분 안 함**: `_vue=[author = "steven king" i]`
- 컴포넌트 및 **참 속성 값**으로 매칭: `_vue=my-button[enabled]`
- 컴포넌트 및 **부울 값**으로 매칭: `_vue=my-button[enabled = false]`
- 속성 **값 부분 문자열**로 매칭: `_vue=[author *= "King"]`
- 컴포넌트 및 **여러 속성**으로 매칭: `_vue=book-item[author *= "king" i][year = 1990]`
- **중첩된** 속성 값으로 매칭: `_vue=[some.nested.value = 12]`
- 컴포넌트 및 속성 값 **접두사**로 매칭: `_vue=book-item[author ^= "Steven"]`
- 컴포넌트 및 속성 값 **접미사**로 매칭: `_vue=book-item[author $= "Steven"]`
- 속성 값 **정규식**으로 매칭: `_vue=[author = /Steven(\\s+King)?/i]`

트리에서 Vue 요소 이름을 찾으려면 [Vue DevTools](https://chrome.google.com/webstore/detail/vuejs-devtools/nhdogjmejiglipccpnnnanhbledajbpd?hl=en)를 사용하십시오.

:::note

Vue 로케이터는 Vue2 이상을 지원합니다.
:::

:::note
Vue 로케이터와 [Vue DevTools](https://chrome.google.com/webstore/detail/react-developer-tools/fmkadmapgofadopljbjfkapdkoienihi)는 **축소되지 않은** 애플리케이션 빌드에 대해서만 작동합니다.
:::

## XPath 로케이터

:::warning

페이지가 변경될 때 손상되기 쉬운 구현에 종속된 XPath를 사용하는 대신 텍스트 또는 접근성 역할과 같은 [사용자에게 보이는 로케이터](./locators.mdx#quick-guide)를 우선적으로 사용하는 것이 좋습니다.
:::

XPath 로케이터는 [`Document.evaluate`](https://developer.mozilla.org/en/docs/Web/API/Document/evaluate)를 호출하는 것과 같습니다.

```js
await page.locator('xpath=//button').click();
```

:::note
`//` 또는 `..`로 시작하는 모든 선택기 문자열은 xpath 선택기로 간주됩니다. 예를 들어 Playwright는 `'//html/body'`를 `'xpath=//html/body'`로 변환합니다.
:::

:::note
XPath는 섀도우 루트를 뚫지 않습니다.
:::

### XPath union

파이프 연산자(`|`)를 사용하여 XPath에서 여러 선택기를 지정할 수 있습니다. 해당 목록의 선택자 중 하나로 선택할 수 있는 모든 요소와 일치합니다.

```js
// 확인 대화 상자 또는 로드 스피너 중 하나를 기다립니다.
await page.locator(
    `//span[contains(@class, 'spinner__loading')]|//div[@id='confirmation']`
).waitFor();
```

## 양식 컨트롤 재지정을 위한 레이블

:::warning

레이블에서 컨트롤로의 재지정에 의존하는 대신 [레이블 텍스트로 찾기](./locators.mdx#locate-by-label)를 권장합니다.
:::

Playwright의 대상 입력 동작은 레이블과 컨트롤을 자동으로 구별하므로 레이블을 대상으로 지정하여 관련 컨트롤에서 동작을 수행할 수 있습니다.

예를 들어 `<label for="password">Password:</label><input id="password" type="password">`와 같은 DOM 구조를 고려하십시오. [page.getByText()](/api/class-page.mdx#page-get-by-text)를 사용하여 "Password" 텍스트로 레이블을 대상으로 지정할 수 있습니다. 그러나 다음 작업은 레이블 대신 입력에서 수행됩니다.
- [locator.click()](/api/class-locator.mdx#locator-click)은 레이블을 클릭하고 입력 필드에 자동으로 초점을 맞춥니다.
- [locator.fill()](/api/class-locator.mdx#locator-fill)은 입력 필드를 채웁니다.
- [locator.inputValue()](/api/class-locator.mdx#locator-input-value)는 입력 필드의 값을 반환합니다.
- [locator.selectText()](/api/class-locator.mdx#locator-select-text)는 입력 필드에서 텍스트를 선택합니다.
- [locator.setInputFiles()](/api/class-locator.mdx#locator-set-input-files)는 `type=file`이 있는 입력 필드에 파일을 설정합니다.
- [locator.selectOption()](/api/class-locator.mdx#locator-select-option)은 선택 상자에서 옵션을 선택합니다.

```js
// 레이블을 대상으로 하여 입력을 채웁니다.
await page.getByText('Password').fill('secret');
```

그러나 다른 메서드는 레이블 자체를 대상으로 합니다. 예를 들어 [expect(locator).toHaveText()](/api/class-locatorassertions.mdx#locator-assertions-to-have-text)는 입력 필드가 아닌 레이블의 텍스트 콘텐츠를 어설션합니다.

```js
// 레이블을 대상으로 하여 입력을 채웁니다.
await expect(page.locator('label')).toHaveText('Password');
```

## 레거시 텍스트 로케이터

:::warning

최신 [텍스트 로케이터](./locators.mdx#get-by-text)를 대신 권장합니다.
:::

레거시 텍스트 로케이터는 전달된 텍스트를 포함하는 요소와 일치합니다.

```js
await page.locator('text=Log in').click();
```

레거시 텍스트 로케이터에는 몇 가지 변형이 있습니다.
- `text=Log in` - 기본 매칭은 대소문자를 구분하지 않고 공백을 자르고 부분 문자열을 검색합니다. 예를 들어 `text=Log`는 `<button>Log in</button>`과 일치합니다.
  
  ```js
  await page.locator('text=Log in').click();
  ```
  
- `text="Log in"` - 텍스트 본문은 작은따옴표 또는 큰따옴표로 이스케이프하여 공백을 자른 후 정확한 콘텐츠가 있는 텍스트 노드를 검색할 수 있습니다.
  
  예를 들어 `text="Log"`는 `<button>Log in</button>`과 일치하지 않습니다. 왜냐하면 `<button>`은 `"Log"`와 같지 않은 단일 텍스트 노드 `"Log in"`을 포함하기 때문입니다. 하지만 `text="Log"`는 `<button> Log <span>in</span></button>`과 일치합니다. 왜냐하면 `<button>`은 `" Log "` 텍스트 노드를 포함하기 때문입니다. 이 정확한 모드는 대소문자를 구분하는 매칭을 의미하므로 `text="Download"`는 `<button>download</button>`와 일치하지 않습니다.
  
  따옴표로 묶인 본문은 일반적인 이스케이프 규칙을 따릅니다. 예를 들어 큰따옴표로 묶인 문자열에서 큰따옴표를 이스케이프하려면 `\"`를 사용하십시오. `text="foo\"bar"`.
  
  ```js
  await page.locator('text="Log in"').click();
  ```
  
- `/Log\s*in/i` - 본문은 `/` 기호로 묶인 [JavaScript와 유사한 정규식](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/RegExp)일 수 있습니다. 예를 들어 `text=/Log\s*in/i`는 `<button>Login</button>` 및 `<button>log IN</button>`과 일치합니다.
  
  ```js
  await page.locator('text=/Log\\s*in/i').click();
  ```
  
:::note

따옴표(`"` 또는 `'`)로 시작하고 끝나는 문자열 선택기는 레거시 텍스트 로케이터로 간주됩니다. 예를 들어 `"Log in"`은 내부적으로 `text="Log in"`으로 변환됩니다.
:::

:::note
매칭은 항상 공백을 정규화합니다. 예를 들어 여러 공백을 하나로 바꾸고, 줄 바꿈을 공백으로 바꾸고, 앞뒤 공백을 무시합니다.
:::

:::note
`button` 및 `submit` 유형의 입력 요소는 텍스트 콘텐츠 대신 `value`로 매칭됩니다. 예를 들어 `text=Log in`은 `<input type=button value="Log in">`과 일치합니다.
:::

## id, data-testid, data-test-id, data-test 선택자

:::warning

[테스트 ID로 찾기](./locators.mdx#locate-by-test-id)를 대신 권장합니다.
:::

Playwright는 특정 속성을 사용하여 요소를 선택하는 약어를 지원합니다. 현재 다음 속성만 지원됩니다.
- `id`
- `data-testid`
- `data-test-id`
- `data-test`

```js
// id가 "username"인 입력을 채웁니다.
await page.locator('id=username').fill('value');

// data-test-id가 "submit"인 요소를 클릭합니다.
await page.locator('data-test-id=submit').click();
```

:::note
속성 선택자는 CSS 선택자가 아니므로 `:enabled`와 같은 CSS 관련 항목은 지원되지 않습니다. 더 많은 기능을 사용하려면 적절한 [css](#css-locator) 선택자를 사용하십시오. 예를 들어 `css=[data-test="login"]:enabled`.
:::

## 선택자 연결

:::warning

[로케이터 체이닝](./locators.mdx#matching-inside-a-locator)을 대신 권장합니다.
:::

`engine=body`로 정의되거나 약식으로 정의된 선택자는 `>>` 토큰과 결합할 수 있습니다. 예를 들어 `selector1 >> selector2 >> selectors3`. 선택자가 연결되면 다음 선택자는 이전 선택기의 결과를 기준으로 쿼리됩니다.

예를 들어,

```txt
css=article >> css=.bar > .baz >> css=span[attr=value]
```

는 다음과 같습니다.

```js
document
    .querySelector('article')
    .querySelector('.bar > .baz')
    .querySelector('span[attr=value]');
```

선택기에 본문에 `>>`가 포함되어야 하는 경우 연결 구분 기호와 혼동되지 않도록 문자열 내에서 이스케이프해야 합니다. 예를 들어 `text="some >> text"`.

### 중간 일치

:::warning

다른 요소를 포함하는 요소를 찾으려면 [다른 로케이터로 필터링](./locators.mdx#filter-by-childdescendant)하는 것이 좋습니다.
:::

기본적으로 연결된 선택자는 마지막 선택기로 쿼리된 요소로 확인됩니다. 선택기 앞에 `*`를 붙여 중간 선택기로 쿼리된 요소를 캡처할 수 있습니다.

예를 들어 `css=article >> text=Hello`는 `Hello` 텍스트가 있는 요소를 캡처하고 `*css=article >> text=Hello`(`*`에 주목)는 `Hello` 텍스트가 있는 일부 요소를 포함하는 `article` 요소를 캡처합니다.
