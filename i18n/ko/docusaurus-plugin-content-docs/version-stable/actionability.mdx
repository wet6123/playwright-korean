## 소개 (Introduction)

Playwright는 작업을 수행하기 전에 요소에 대해 다양한 실행 가능성 검사(actionability checks)를 수행하여 이러한 작업이 예상대로 동작하도록 보장합니다. 관련된 모든 검사가 통과할 때까지 자동으로 대기(auto-waits)하며, 통과한 후에만 요청된 작업을 수행합니다. 지정된 `timeout` 내에 필요한 검사가 통과하지 못하면, 작업은 `TimeoutError`와 함께 실패합니다.

예를 들어, [locator.click()](/api/class-locator.mdx#locator-click)의 경우 Playwright는 다음을 보장합니다:
- 로케이터(locator)가 정확히 하나의 요소로 확인됩니다.
- 요소가 [Visible] 상태입니다.
- 요소가 [Stable] 상태입니다 (애니메이션 중이 아니거나 애니메이션이 완료됨).
- 요소가 [Receives Events] 상태입니다 (다른 요소에 의해 가려지지 않음).
- 요소가 [Enabled] 상태입니다.

## 강제 작업 (Forcing actions)

[locator.click()](/api/class-locator.mdx#locator-click)와 같은 일부 작업은 `force` 옵션을 지원하여 중요하지 않은 실행 가능성 검사를 비활성화합니다. 예를 들어, [locator.click()](/api/class-locator.mdx#locator-click) 메서드에 truthy `force` 값을 전달하면 대상 요소가 실제로 클릭 이벤트를 수신하는지 확인하지 않습니다.

## 보이는 상태 (Visible)

요소는 비어 있지 않은 경계 상자(bounding box)를 가지고 `visibility:hidden` 계산된 스타일(computed style)이 없을 때 보이는(visible) 것으로 간주됩니다.

이 정의에 따르면 다음 사항에 유의하세요:
* 크기가 0인 요소는 보이는 것으로 간주되지 **않습니다**.
* `display:none`인 요소는 보이는 것으로 간주되지 **않습니다**.
* `opacity:0`인 요소는 보이는 것으로 간주 **됩니다**.

## 안정적인 상태 (Stable)

요소는 최소 두 번의 연속적인 애니메이션 프레임 동안 동일한 경계 상자(bounding box)를 유지했을 때 안정적인(stable) 것으로 간주됩니다.

## 활성화된 상태 (Enabled)

요소는 **비활성화되지 않았을 때** 활성화된(enabled) 것으로 간주됩니다.

다음과 같은 경우 요소는 **비활성화된(disabled)** 상태입니다:
- `[disabled]` 속성을 가진 `<button>`, `<select>`, `<input>`, `<textarea>`, `<option>` 또는 `<optgroup>`인 경우
- `[disabled]` 속성을 가진 `<fieldset>`의 일부인 `<button>`, `<select>`, `<input>`, `<textarea>`, `<option>` 또는 `<optgroup>`인 경우
- `[aria-disabled=true]` 속성을 가진 요소의 자손인 경우

## 편집 가능한 상태 (Editable)

요소는 [Enabled] 상태이고 **읽기 전용(readonly)이 아닐 때** 편집 가능한(editable) 것으로 간주됩니다.

다음과 같은 경우 요소는 **읽기 전용(readonly)** 상태입니다:
- `[readonly]` 속성을 가진 `<select>`, `<input>` 또는 `<textarea>`인 경우
- `[aria-readonly=true]` 속성을 가지고 [이를 지원하는](https://w3c.github.io/aria/#aria-readonly) aria 역할을 가진 경우

## 이벤트 수신 상태 (Receives Events)

요소는 작업 지점(action point)에서 포인터 이벤트의 히트 대상(hit target)일 때 포인터 이벤트를 수신하는(receiving pointer events) 것으로 간주됩니다. 예를 들어, `(10;10)` 지점을 클릭할 때 Playwright는 다른 요소(일반적으로 오버레이)가 대신 `(10;10)`에서 클릭을 가로채는지 확인합니다.

예를 들어, [locator.click()](/api/class-locator.mdx#locator-click) 호출 시점과 관계없이 Playwright가 `Sign Up` 버튼을 클릭하는 시나리오를 생각해 보세요:
- 페이지에서 사용자 이름이 고유한지 확인하고 `Sign Up` 버튼이 비활성화됩니다.
- 서버 확인 후, 비활성화된 `Sign Up` 버튼이 이제 활성화된 다른 버튼으로 교체됩니다.
