## Introduction

Playwright Test는 `await expect(page).toHaveScreenshot()`을 사용하여 스크린샷을 생성하고 시각적으로 비교하는 기능을 포함합니다. 처음 실행 시 Playwright 테스트는 기준 스크린샷을 생성합니다. 후속 실행은 기준과 비교합니다.

```js title="example.spec.ts"
import { test, expect } from '@playwright/test';

test('example test', async ({ page }) => {
  await page.goto('https://playwright.dev');
  await expect(page).toHaveScreenshot();
});
```

:::warning
브라우저 렌더링은 호스트 OS, 버전, 설정, 하드웨어, 전원 (배터리 대 전원 어댑터), 헤드리스 모드 및 기타 요인에 따라 다를 수 있습니다. 일관된 스크린샷을 위해 기준 스크린샷이 생성된 동일한 환경에서 테스트를 실행하십시오.
:::

## Generating screenshots

위의 코드를 처음 실행하면 테스트 러너가 다음과 같이 표시합니다.

```txt
Error: A snapshot doesn't exist at example.spec.ts-snapshots/example-test-1-chromium-darwin.png, writing actual.
```

아직 golden 파일이 없기 때문입니다. 이 메서드는 두 개의 연속된 스크린샷이 일치할 때까지 여러 스크린샷을 캡처하고 마지막 스크린샷을 파일 시스템에 저장했습니다. 이제 리포지토리에 추가할 준비가 되었습니다.

golden 기대 값이 있는 폴더의 이름은 테스트 파일 이름으로 시작합니다.

```bash
drwxr-xr-x  5 user  group  160 Jun  4 11:46 .
drwxr-xr-x  6 user  group  192 Jun  4 11:45 ..
-rw-r--r--  1 user  group  231 Jun  4 11:16 example.spec.ts
drwxr-xr-x  3 user  group   96 Jun  4 11:46 example.spec.ts-snapshots
```

스냅샷 이름 `example-test-1-chromium-darwin.png`은 몇 부분으로 구성됩니다.
- `example-test-1.png` - 자동 생성된 스냅샷 이름. 또는 `toHaveScreenshot()` 메서드의 첫 번째 인수로 스냅샷 이름을 지정할 수 있습니다.
  
  ```js
  await expect(page).toHaveScreenshot('landing.png');
  ```
  
- `chromium-darwin` - 브라우저 이름과 플랫폼. 스크린샷은 렌더링, 글꼴 등이 다르기 때문에 브라우저와 플랫폼 간에 다르므로 다른 스냅샷이 필요합니다. [구성 파일](./test-configuration.mdx)에서 여러 프로젝트를 사용하는 경우 `chromium` 대신 프로젝트 이름이 사용됩니다.

스냅샷 이름 및 경로는 playwright 구성에서 [testConfig.snapshotPathTemplate](/api/class-testconfig.mdx#test-config-snapshot-path-template)로 구성할 수 있습니다.

## Updating screenshots

페이지가 변경된 경우와 같이 기준 스크린샷을 업데이트해야 하는 경우가 있습니다. `--update-snapshots` 플래그를 사용하여 이를 수행하십시오.

```bash
npx playwright test --update-snapshots
```

> `snapshotName`은 `expect().toHaveScreenshot(['relative', 'path', 'to', 'snapshot.png'])`과 같이 스냅샷 파일에 대한 경로 세그먼트 배열도 허용합니다.
> 그러나 이 경로는 각 테스트 파일에 대한 스냅샷 디렉터리(예: `a.spec.js-snapshots`) 내에 유지되어야 하며, 그렇지 않으면 오류가 발생합니다.

## Options

### maxDiffPixels

Playwright Test는 [pixelmatch](https://github.com/mapbox/pixelmatch) 라이브러리를 사용합니다. 동작을 수정하기 위해 [다양한 옵션을 전달](./api/class-pageassertions.mdx#page-assertions-to-have-screenshot-1)할 수 있습니다.

```js title="example.spec.ts"
import { test, expect } from '@playwright/test';

test('example test', async ({ page }) => {
  await page.goto('https://playwright.dev');
  await expect(page).toHaveScreenshot({ maxDiffPixels: 100 });
});
```

프로젝트의 모든 테스트에서 기본값을 공유하려면 playwright 구성에서 전역적으로 또는 프로젝트별로 지정할 수 있습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';
export default defineConfig({
  expect: {
    toHaveScreenshot: { maxDiffPixels: 100 },
  },
});
```

### stylePath

스크린샷을 찍는 동안 페이지에 사용자 정의 스타일시트를 적용할 수 있습니다. 이렇게 하면 동적 또는 휘발성 요소를 필터링하여 스크린샷 결정성을 향상시킬 수 있습니다.

```css title="screenshot.css"
iframe {
  visibility: hidden;
}
```

```js title="example.spec.ts"
import { test, expect } from '@playwright/test';

test('example test', async ({ page }) => {
  await page.goto('https://playwright.dev');
  await expect(page).toHaveScreenshot({ stylePath: path.join(__dirname, 'screenshot.css') });
});
```

프로젝트의 모든 테스트에서 기본값을 공유하려면 playwright 구성에서 전역적으로 또는 프로젝트별로 지정할 수 있습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';
export default defineConfig({
  expect: {
    toHaveScreenshot: {
      stylePath: './screenshot.css'
    },
  },
});
```

## Non-image snapshots

스크린샷 외에도 `expect(value).toMatchSnapshot(snapshotName)`을 사용하여 텍스트 또는 임의의 이진 데이터를 비교할 수 있습니다. Playwright Test는 콘텐츠 유형을 자동으로 감지하고 적절한 비교 알고리즘을 사용합니다.

여기서는 텍스트 콘텐츠를 기준과 비교합니다.

```js title="example.spec.ts"
import { test, expect } from '@playwright/test';

test('example test', async ({ page }) => {
  await page.goto('https://playwright.dev');
  expect(await page.textContent('.hero__title')).toMatchSnapshot('hero.txt');
});
```

스냅샷은 테스트 파일 옆의 별도 디렉토리에 저장됩니다. 예를 들어 `my.spec.ts` 파일은 `my.spec.ts-snapshots` 디렉터리에 스냅샷을 생성하고 저장합니다. 이 디렉터리를 버전 관리(예: `git`)에 커밋하고 변경 사항을 검토해야 합니다.
