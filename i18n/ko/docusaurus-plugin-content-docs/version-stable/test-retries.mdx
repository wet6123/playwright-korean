## Introduction

테스트 재시도는 테스트가 실패했을 때 자동으로 다시 실행하는 방법입니다. 이는 테스트가 불안정하고 간헐적으로 실패할 때 유용합니다. 테스트 재시도는 [구성 파일](./test-configuration.mdx)에서 구성됩니다.

## Failures

Playwright Test는 워커 프로세스에서 테스트를 실행합니다. 이러한 프로세스는 OS 프로세스이며, 독립적으로 실행되고 테스트 러너에 의해 오케스트레이션됩니다. 모든 워커는 동일한 환경을 가지며 각각 자체 브라우저를 시작합니다.

다음 스니펫을 고려하십시오:

```js
import { test } from '@playwright/test';

test.describe('suite', () => {
  test.beforeAll(async () => { /* ... */ });
  test('first good', async ({ page }) => { /* ... */ });
  test('second flaky', async ({ page }) => { /* ... */ });
  test('third good', async ({ page }) => { /* ... */ });
  test.afterAll(async () => { /* ... */ });
});
```

**모든 테스트가 통과**하면 동일한 워커 프로세스에서 순서대로 실행됩니다.
* 워커 프로세스 시작
  * `beforeAll` 훅 실행
  * `first good` 통과
  * `second flaky` 통과
  * `third good` 통과
  * `afterAll` 훅 실행

**테스트가 실패하는 경우**, Playwright Test는 브라우저와 함께 전체 워커 프로세스를 폐기하고 새 워커 프로세스를 시작합니다. 테스팅은 다음 테스트부터 시작하여 새 워커 프로세스에서 계속됩니다.
* 워커 프로세스 #1 시작
  * `beforeAll` 훅 실행
  * `first good` 통과
  * `second flaky` 실패
  * `afterAll` 훅 실행
* 워커 프로세스 #2 시작
  * `beforeAll` 훅 다시 실행
  * `third good` 통과
  * `afterAll` 훅 실행

**[재시도](#retries)를 활성화**하면 두 번째 워커 프로세스가 실패한 테스트를 재시도하여 시작하고 거기에서부터 계속됩니다.
* 워커 프로세스 #1 시작
  * `beforeAll` 훅 실행
  * `first good` 통과
  * `second flaky` 실패
  * `afterAll` 훅 실행
* 워커 프로세스 #2 시작
  * `beforeAll` 훅 다시 실행
  * `second flaky` 재시도 및 통과
  * `third good` 통과
  * `afterAll` 훅 실행

이 방식은 독립적인 테스트에 완벽하게 작동하며 실패한 테스트가 정상적인 테스트에 영향을 미치지 않도록 보장합니다.

## Retries

Playwright는 **테스트 재시도**를 지원합니다. 활성화되면 실패한 테스트는 통과할 때까지 또는 최대 재시도 횟수에 도달할 때까지 여러 번 재시도됩니다. 기본적으로 실패한 테스트는 재시도되지 않습니다.

```bash
# 실패한 테스트에 3번의 재시도 시도 제공
npx playwright test --retries=3
```

구성 파일에서 재시도를 구성할 수 있습니다:

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  // 실패한 테스트에 3번의 재시도 시도 제공
  retries: 3,
});
```

Playwright Test는 테스트를 다음과 같이 분류합니다:
- "passed" - 첫 번째 실행에서 통과한 테스트;
- "flaky" - 첫 번째 실행에서 실패했지만 재시도할 때 통과한 테스트;
- "failed" - 첫 번째 실행에서 실패하고 모든 재시도에서 실패한 테스트.

```bash
Running 3 tests using 1 worker

  ✓ example.spec.ts:4:2 › first passes (438ms)
  x example.spec.ts:5:2 › second flaky (691ms)
  ✓ example.spec.ts:5:2 › second flaky (522ms)
  ✓ example.spec.ts:6:2 › third passes (932ms)

  1 flaky
    example.spec.ts:5:2 › second flaky
  2 passed (4s)
```

[testInfo.retry](/api/class-testinfo.mdx#test-info-retry)를 사용하여 런타임에 재시도를 감지할 수 있습니다. 이는 모든 테스트, 훅 또는 픽스처에서 액세스할 수 있습니다. 다음은 재시도 전에 일부 서버 측 상태를 지우는 예입니다.

```js
import { test, expect } from '@playwright/test';

test('my test', async ({ page }, testInfo) => {
  if (testInfo.retry)
    await cleanSomeCachesOnTheServer();
  // ...
});
```

[test.describe.configure()](/api/class-test.mdx#test-describe-configure)를 사용하여 특정 테스트 그룹 또는 단일 파일에 대한 재시도를 지정할 수 있습니다.

```js
import { test, expect } from '@playwright/test';

test.describe(() => {
  // 이 describe 그룹의 모든 테스트는 2번의 재시도 시도를 받습니다.
  test.describe.configure({ retries: 2 });

  test('test 1', async ({ page }) => {
    // ...
  });

  test('test 2', async ({ page }) => {
    // ...
  });
});
```

## Serial mode

[test.describe.serial()](/api/class-test.mdx#test-describe-serial)을 사용하여 종속된 테스트를 그룹화하여 항상 함께 순서대로 실행되도록 합니다. 테스트 중 하나가 실패하면 후속 테스트는 모두 건너뜁니다. 그룹의 모든 테스트는 함께 재시도됩니다.

`test.describe.serial`을 사용하는 다음 스니펫을 고려하십시오:

```js
import { test } from '@playwright/test';

test.describe.configure({ mode: 'serial' });

test.beforeAll(async () => { /* ... */ });
test('first good', async ({ page }) => { /* ... */ });
test('second flaky', async ({ page }) => { /* ... */ });
test('third good', async ({ page }) => { /* ... */ });
```

[재시도](#retries)없이 실행하면 실패 후의 모든 테스트가 건너뜁니다:
* 워커 프로세스 #1:
  * `beforeAll` 훅 실행
  * `first good` 통과
  * `second flaky` 실패
  * `third good` 전체적으로 건너뜀

[재시도](#retries)를 사용하여 실행하면 모든 테스트가 함께 재시도됩니다:
* 워커 프로세스 #1:
  * `beforeAll` 훅 실행
  * `first good` 통과
  * `second flaky` 실패
  * `third good` 건너뜀
* 워커 프로세스 #2:
  * `beforeAll` 훅 다시 실행
  * `first good` 다시 통과
  * `second flaky` 통과
  * `third good` 통과

:::note

일반적으로 테스트를 격리시켜 독립적으로 효율적으로 실행하고 재시도할 수 있도록 하는 것이 좋습니다.
:::

## Reuse single page between tests

Playwright Test는 각 테스트에 대해 격리된 [Page] 객체를 생성합니다. 그러나 여러 테스트 간에 단일 [Page] 객체를 재사용하려면 [test.beforeAll()](/api/class-test.mdx#test-before-all)에서 직접 생성하고 [test.afterAll()](/api/class-test.mdx#test-after-all)에서 닫을 수 있습니다.

<Tabs
  groupId="js-flavor"
  defaultValue="ts"
  values={[
    {label: 'TypeScript', value: 'ts'},
    {label: 'JavaScript', value: 'js'}
  ]
}>
<TabItem value="ts">

```js title="example.spec.ts"
import { test, type Page } from '@playwright/test';

test.describe.configure({ mode: 'serial' });

let page: Page;

test.beforeAll(async ({ browser }) => {
  page = await browser.newPage();
});

test.afterAll(async () => {
  await page.close();
});

test('runs first', async () => {
  await page.goto('https://playwright.dev/');
});

test('runs second', async () => {
  await page.getByText('Get Started').click();
});
```

</TabItem>
<TabItem value="js">

```js title="example.spec.js"
// @ts-check

const { test } = require('@playwright/test');

test.describe.configure({ mode: 'serial' });

/** @type {import('@playwright/test').Page} */
let page;

test.beforeAll(async ({ browser }) => {
  page = await browser.newPage();
});

test.afterAll(async () => {
  await page.close();
});

test('runs first', async () => {
  await page.goto('https://playwright.dev/');
});

test('runs second', async () => {
  await page.getByText('Get Started').click();
});
```
