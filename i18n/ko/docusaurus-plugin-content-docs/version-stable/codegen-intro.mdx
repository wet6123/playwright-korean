## Introduction

Playwright는 즉시 사용 가능한 테스트 생성 기능을 제공하며, 이는 테스트를 빠르게 시작할 수 있는 좋은 방법입니다. 이 기능은 두 개의 창을 엽니다. 하나는 테스트하려는 웹사이트와 상호 작용하는 브라우저 창이고, 다른 하나는 테스트를 기록하고, 복사하고, 지우고, 테스트 언어를 변경할 수 있는 Playwright Inspector 창입니다.

- [로케이터 생성 방법](/codegen.mdx#generating-locators)

## Running Codegen

`codegen` 명령을 사용하여 테스트 생성기를 실행하고, 그 뒤에 테스트를 생성하려는 웹사이트의 URL을 입력합니다. URL은 선택 사항이며, URL 없이 명령을 실행한 다음 브라우저 창에 직접 URL을 추가할 수도 있습니다.

```bash
npx playwright codegen demo.playwright.dev/todomvc
```

### Recording a test

`codegen`을 실행하고 브라우저에서 작업을 수행합니다. Playwright는 사용자 상호 작용에 대한 코드를 생성합니다. `Codegen`은 렌더링된 페이지를 보고 role, text, test id 로케이터를 우선으로 권장 로케이터를 파악합니다. 생성기가 로케이터와 일치하는 여러 요소를 식별하면 로케이터를 개선하여 복원력을 높이고 대상 요소를 고유하게 식별하므로 로케이터로 인해 테스트가 실패하거나 불안정해지는 것을 제거하고 줄입니다.

테스트 생성기를 사용하면 다음을 기록할 수 있습니다.
* 페이지와 상호 작용하여 클릭 또는 채우기 같은 동작
* 도구 모음의 아이콘 중 하나를 클릭한 다음, 어설션할 페이지의 요소를 클릭하여 어설션합니다. 다음을 선택할 수 있습니다.
  * 요소를 표시할 `'assert visibility'`
  * 요소에 특정 텍스트가 포함되어 있는지 확인하는 `'assert text'`
  * 요소에 특정 값이 있는지 확인하는 `'assert value'`

###### 
![Recording a test](https://github.com/microsoft/playwright/assets/13063165/34a79ea1-639e-4cb3-8115-bfdc78e3d34d)

###### 
페이지와의 상호 작용이 끝나면 `'record'` 버튼을 눌러 기록을 중지하고 `'copy'` 버튼을 사용하여 생성된 코드를 편집기에 복사합니다.

코드를 지우고 다시 기록을 시작하려면 `'clear'` 버튼을 사용합니다. 완료되면 Playwright inspector 창을 닫거나 터미널 명령을 중지합니다.

테스트 생성에 대해 자세히 알아보려면 [Codegen](./codegen.mdx)에 대한 자세한 가이드를 확인하세요.

### Generating locators

테스트 생성기로 [로케이터](/locators.mdx)를 생성할 수 있습니다.
* `'Record'` 버튼을 눌러 기록을 중지하면 `'Pick Locator'` 버튼이 나타납니다.
* `'Pick Locator'` 버튼을 클릭한 다음 브라우저 창에서 요소 위로 마우스를 가져가면 각 요소 아래에 강조 표시된 로케이터가 표시됩니다.
* 로케이터를 선택하려면 찾으려는 요소를 클릭하면 해당 로케이터의 코드가 Pick Locator 버튼 옆의 로케이터 플레이그라운드에 나타납니다.
* 그런 다음 로케이터 플레이그라운드에서 로케이터를 편집하여 미세 조정하고 브라우저 창에서 일치하는 요소가 강조 표시되는지 확인할 수 있습니다.
* 복사 버튼을 사용하여 로케이터를 복사하여 코드에 붙여넣습니다.

###### 
![picking a locator](https://github.com/microsoft/playwright/assets/13063165/2c8a12e2-4e98-4fdd-af92-1d73ae696d86)

### Emulation

에뮬레이션을 사용하여 테스트를 생성하여 특정 뷰포트, 장치, 색 구성표에 대한 테스트를 생성하고 지리적 위치, 언어 또는 시간대를 에뮬레이션할 수도 있습니다. 테스트 생성기는 인증된 상태를 유지하면서 테스트를 생성할 수도 있습니다. 자세한 내용은 [테스트 생성기](./codegen.mdx#emulation) 가이드를 확인하세요.
