## 소개

Node.js용 Playwright는 카나리 릴리스 시스템을 가지고 있습니다.

정식 릴리스를 기다리는 대신 **새로운 미출시 기능을 테스트**할 수 있습니다. 이러한 기능은 Playwright의 `next` NPM 태그로 매일 릴리스됩니다.

새로운 기능이 의도한 대로 작동하는지 확인하여 **유지 관리자에게 피드백을 제공**하는 좋은 방법입니다.

:::note

카나리 릴리스를 프로덕션 환경에서 사용하는 것은 위험해 보일 수 있지만 실제로는 그렇지 않습니다.

카나리 릴리스는 모든 자동 테스트를 통과하며 HTML 보고서, Trace Viewer 또는 Playwright Inspector를 엔드 투 엔드 테스트로 테스트하는 데 사용됩니다.
:::

```txt
npm install -D @playwright/test@next
```

## Next npm Dist Tag

`main` 브랜치의 모든 코드 관련 커밋에 대해 지속적 통합은 `@next` npm dist 태그 아래에 매일 카나리 릴리스를 게시합니다.

[npm](https://www.npmjs.com/package/@playwright/test?activeTab=versions)에서 현재 dist 태그를 확인할 수 있습니다.

- `latest`: 안정적인 릴리스
- `next`: 다음 릴리스, 매일 게시됨
- `beta`: 릴리스 브랜치가 생성된 후, 일반적으로 안정적인 릴리스 일주일 전에 각 커밋이 이 태그 아래에 게시됩니다.

## 카나리 릴리스 사용하기

```txt
npm install -D @playwright/test@next
```
