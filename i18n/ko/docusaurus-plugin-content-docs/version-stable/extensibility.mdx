## Custom selector engines

Playwright는 사용자 정의 선택기 엔진을 지원하며, [selectors.register()](/api/class-selectors.mdx#selectors-register)를 통해 등록됩니다.

선택기 엔진은 다음 속성을 가져야 합니다.
- `selector`와 일치하는 첫 번째 요소를 `root`를 기준으로 쿼리하는 `query` 함수.
- `selector`와 일치하는 모든 요소를 `root`를 기준으로 쿼리하는 `queryAll` 함수.

기본적으로 엔진은 프레임의 JavaScript 컨텍스트에서 직접 실행되며, 예를 들어 애플리케이션 정의 함수를 호출할 수 있습니다. 프레임의 JavaScript로부터 엔진을 격리하되 DOM에 대한 액세스를 유지하려면 `{contentScript: true}` 옵션으로 엔진을 등록하세요. 컨텐츠 스크립트 엔진은 전역 객체 조작 (예: `Node.prototype` 메서드 변경)으로부터 보호되므로 더 안전합니다. 모든 내장 선택기 엔진은 컨텐츠 스크립트로 실행됩니다. 엔진이 다른 사용자 정의 엔진과 함께 사용될 때는 컨텐츠 스크립트로 실행된다는 보장이 없습니다.

선택기는 페이지를 만들기 전에 등록해야 합니다.

태그 이름을 기반으로 요소를 쿼리하는 선택기 엔진 등록 예제:

```js title="baseTest.ts"
import { test as base } from '@playwright/test';

export { expect } from '@playwright/test';

// 선택기 엔진 인스턴스로 평가되는 함수여야 합니다.
const createTagNameEngine = () => ({
  // 루트의 하위 트리에서 주어진 선택기와 일치하는 첫 번째 요소를 반환합니다.
  query(root, selector) {
    return root.querySelector(selector);
  },

  // 루트의 하위 트리에서 주어진 선택기와 일치하는 모든 요소를 반환합니다.
  queryAll(root, selector) {
    return Array.from(root.querySelectorAll(selector));
  }
});

export const test = base.extend<{}, { selectorRegistration: void }>({
  // 작업자 당 한 번 선택기를 등록합니다.
  selectorRegistration: [async ({ playwright }, use) => {
    // 엔진을 등록합니다. 선택기는 "tag=" 접두사가 붙습니다.
    await playwright.selectors.register('tag', createTagNameEngine);
    await use();
  }, { scope: 'worker', auto: true }],
});
```

```js title="example.spec.ts"
import { test, expect } from './baseTest';

test('selector engine test', async ({ page }) => {
  // 이제 'tag=' 선택기를 사용할 수 있습니다.
  const button = page.locator('tag=button');
  await button.click();

  // 내장 로케이터와 결합할 수 있습니다.
  await page.locator('tag=div').getByText('Click me').click();

  // 선택기를 지원하는 모든 메서드에서 사용할 수 있습니다.
  await expect(page.locator('tag=button')).toHaveCount(3);
});
```
