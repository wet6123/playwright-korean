## Migration Principles

이 가이드는 Puppeteer에서 [Playwright Library](./library) 및 [Playwright Test](./intro.mdx)로 마이그레이션하는 방법을 설명합니다. API는 유사하지만 Playwright는 웹 테스트 및 크로스 브라우저 자동화를 위한 훨씬 더 많은 가능성을 제공합니다.
- 대부분의 Puppeteer API를 있는 그대로 사용할 수 있습니다.
- [ElementHandle]의 사용은 권장되지 않으며, 대신 [Locator] 객체와 웹 우선 어설션을 사용하세요.
- Playwright는 크로스 브라우저를 지원합니다.
- 명시적인 대기가 필요하지 않을 수 있습니다.

## Cheat Sheet

| Puppeteer                                          | Playwright Library                          |
|----------------------------------------------------|---------------------------------------------|
| `await puppeteer.launch()`                         | `await playwright.chromium.launch()`        |
| `puppeteer.launch({product: 'firefox'})`           | `await playwright.firefox.launch()`         |
|  WebKit은 Puppeteer에서 지원되지 않습니다.              | `await playwright.webkit.launch()`          |
| `await browser.createIncognitoBrowserContext(...)` | `await browser.newContext(...)`             |
| `await page.setViewport(...)`                      | `await page.setViewportSize(...)`           |
| `await page.waitForXPath(XPathSelector)`           | `await page.waitForSelector(XPathSelector)` |
| `await page.waitForNetworkIdle(...)`               | `await page.waitForLoadState('networkidle')` |
| `await page.$eval(...)`                            | 텍스트, 속성, 클래스를 확인하기 위해 [Assertions](./test-assertions)를 대신 사용할 수 있습니다. |
| `await page.$(...)`                                | 권장되지 않으며, 대신 [Locators](./api/class-locator)를 사용하세요. |
| `await page.$x(xpath_selector)`                    | 권장되지 않으며, 대신 [Locators](./api/class-locator)를 사용하세요. |
| 체크박스 또는 라디오 입력을 위한 전용 메서드가 없음 | `await page.locator(selector).check()`<br/>`await page.locator(selector).uncheck()` |
| `await page.click(selector)`                       | `await page.locator(selector).click()`      |
| `await page.focus(selector)`                       | `await page.locator(selector).focus()`      |
| `await page.hover(selector)`                       | `await page.locator(selector).hover()`      |
| `await page.select(selector, values)`              | `await page.locator(selector).selectOption(values)` |
| `await page.tap(selector)`                         | `await page.locator(selector).tap()`        |
| `await page.type(selector, ...)`                   | `await page.locator(selector).fill(...)` |
| `await page.waitForFileChooser(...)`<br/>`await elementHandle.uploadFile(...)` | `await page.locator(selector).setInputFiles(...)` |
| `await page.cookies([...urls])`                    | `await browserContext.cookies([urls])`      |
| `await page.deleteCookie(...cookies)`              | `await browserContext.clearCookies()`       |
| `await page.setCookie(...cookies)`                 | `await browserContext.addCookies(cookies)`  |
| `page.on(...)`                                     | `page.on(...)`<br/>요청을 가로채고 변경하려면 [page.route()](/api/class-page.mdx#page-route)를 참조하세요. |

`page.waitForNavigation` 및 `page.waitForSelector`는 남아 있지만, [자동 대기](./actionability)로 인해 많은 경우에 필요하지 않습니다.

[ElementHandle]의 사용은 권장되지 않으며, 대신 [Locator] 객체와 웹 우선 어설션을 사용하세요.

Locator는 Playwright의 자동 대기 및 재시도 기능의 핵심입니다. Locator는 엄격합니다. 즉, 특정 DOM 요소를 암시하는 Locator에 대한 모든 작업은 주어진 선택자와 일치하는 요소가 둘 이상인 경우 예외를 발생시킵니다.

## Examples

### Automation example

Puppeteer:

```js
const puppeteer = require('puppeteer');

(async () => {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();
  await page.setViewport({ width: 1280, height: 800 });
  await page.goto('https://playwright.dev/', {
    waitUntil: 'networkidle2',
  });
  await page.screenshot({ path: 'example.png' });
  await browser.close();
})();
```

Playwright로의 한 줄씩 마이그레이션:

```js
const { chromium } = require('playwright'); // 1

(async () => {
  const browser = await chromium.launch();
  const page = await browser.newPage(); // 2
  await page.setViewportSize({ width: 1280, height: 800 }); // 3
  await page.goto('https://playwright.dev/', {
    waitUntil: 'networkidle', // 4
  });
  await page.screenshot({ path: 'example.png' });
  await browser.close();
})();
```

마이그레이션 하이라이트 (Playwright 코드 스니펫의 인라인 주석 참조):
1. 각 Playwright Library 파일은 `chromium`을 명시적으로 가져옵니다. 다른 브라우저인 `webkit` 또는 `firefox`를 사용할 수 있습니다.
1. 브라우저 상태 격리를 위해 [브라우저 컨텍스트](./browser-contexts.mdx)를 고려하십시오.
1. `setViewport`는 `setViewportSize`가 됩니다.
1. `networkidle2`는 `networkidle`이 됩니다. 자동 대기 덕분에 대부분의 경우에 유용하지 않다는 점에 유의하십시오.

### Test example

Jest를 사용하는 Puppeteer:

```js
import puppeteer from 'puppeteer';

describe('Playwright 홈페이지', () => {
  let browser;
  let page;

  beforeAll(async () => {
    browser = await puppeteer.launch();
    page = await browser.newPage();
  });

  it('hero 제목을 포함합니다', async () => {
    await page.goto('https://playwright.dev/');
    await page.waitForSelector('.hero__title');
    const text = await page.$eval('.hero__title', e => e.textContent);
    expect(text).toContain('Playwright enables reliable end-to-end testing'); // 5
  });

  afterAll(() => browser.close());
});
```

Playwright Test로의 한 줄씩 마이그레이션:

```js
import { test, expect } from '@playwright/test'; // 1

test.describe('Playwright 홈페이지', () => {
  test('hero 제목을 포함합니다', async ({ page }) => { // 2, 3
    await page.goto('https://playwright.dev/');
    const titleLocator = page.locator('.hero__title'); // 4
    await expect(titleLocator).toContainText( // 5
        'Playwright enables reliable end-to-end testing'
    );
  });
});
```

1. 각 Playwright Test 파일은 `test` 및 `expect` 함수를 명시적으로 가져옵니다.
1. 테스트 함수는 `async`로 표시됩니다.
1. Playwright Test는 매개변수 중 하나로 `page`를 제공받습니다. 이는 Playwright Test의 많은 [유용한 fixtures](./api/class-fixtures) 중 하나입니다. Playwright Test는 각 테스트에 대해 격리된 [Page] 객체를 생성합니다. 그러나 여러 테스트 간에 단일 [Page] 객체를 재사용하려면 [test.beforeAll()](/api/class-test.mdx#test-before-all)에서 직접 만들고 [test.afterAll()](/api/class-test.mdx#test-after-all)에서 닫을 수 있습니다.
1. [page.locator()](/api/class-page.mdx#page-locator)로 Locator를 생성하는 것은 동기적인 몇 안 되는 메서드 중 하나입니다.
1. `page.$eval()` 대신 [assertions](./test-assertions)를 사용하여 상태를 확인하십시오.

## Testing

테스트를 개선하려면 [Locators](./api/class-locator)와 웹 우선 [Assertions](./test-assertions)를 사용하는 것이 좋습니다. [Writing Tests](./writing-tests)를 참조하세요.

Puppeteer에서는 [ElementHandle]을 검사하고 텍스트 콘텐츠, 속성, 클래스의 값을 추출하기 위해 `page.evaluate()` 또는 `page.$eval()`을 사용하는 것이 일반적입니다. 웹 우선 [Assertions](./test-assertions)는 이를 위한 여러 매처를 제공하며, 이는 더 안정적이고 읽기 쉽습니다.

[Playwright Test](./intro.mdx)는 Playwright와 함께 사용할 수 있도록 권장되는 자체 테스트 러너입니다. Page Object Model, 병렬 처리, fixtures 또는 reporters와 같은 여러 기능을 제공합니다.

## Playwright Test Super Powers

Playwright Test를 사용하면 많은 것을 얻을 수 있습니다!
- 완벽한 제로 구성 TypeScript 지원
- **모든 웹 엔진**(Chrome, Firefox, Safari)에서 **모든 인기 있는 운영 체제**(Windows, macOS, Ubuntu)에서 테스트 실행
- 다중 origin, [(i)frames](./api/class-frame), [탭 및 컨텍스트](./pages)에 대한 완벽한 지원
- 여러 브라우저에서 병렬로 격리된 테스트 실행
- 내장된 테스트 [아티팩트 수집](./test-use-options.mdx#recording-options)

또한 Playwright Test에 번들로 제공되는 이러한 ✨ 멋진 도구 ✨를 모두 얻을 수 있습니다.
- [Playwright Inspector](./debug.mdx)
- [Playwright Test 코드 생성](./codegen-intro.mdx)
- 사후 디버깅을 위한 [Playwright Tracing](./trace-viewer.mdx)
