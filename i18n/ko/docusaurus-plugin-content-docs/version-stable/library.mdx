## Introduction

Playwright Library는 브라우저를 실행하고 상호 작용하기 위한 통합 API를 제공하는 반면, Playwright Test는 이 모든 기능에 더해 완전히 관리되는 엔드 투 엔드 테스트 러너와 경험을 제공합니다.

대부분의 경우 엔드 투 엔드 테스트를 위해서는 `playwright` (Playwright Library)를 직접 사용하는 대신 `@playwright/test` (Playwright Test)를 사용하는 것이 좋습니다. Playwright Test를 시작하려면 [시작 가이드](./intro.mdx)를 따르세요.

## Differences when using library

### Library Example

다음은 Chromium을 실행하고, 페이지로 이동하여 제목을 확인하기 위해 Playwright Library를 직접 사용하는 예제입니다.

<Tabs

  defaultValue="ts"
  values={[
    {label: 'TypeScript', value: 'ts'},
    {label: 'JavaScript', value: 'js'}
  ]
}>
<TabItem value="ts">

```js
import { chromium, devices } from 'playwright';
import assert from 'node:assert';

(async () => {
  // Setup
  const browser = await chromium.launch();
  const context = await browser.newContext(devices['iPhone 11']);
  const page = await context.newPage();

  // The actual interesting bit
  await context.route('**.jpg', route => route.abort());
  await page.goto('https://example.com/');

  assert(await page.title() === 'Example Domain'); // 👎 not a Web First assertion

  // Teardown
  await context.close();
  await browser.close();
})();
```

</TabItem>

### Test Example

비슷한 동작을 수행하는 테스트는 다음과 같습니다.

### Key Differences

주요 차이점은 다음과 같습니다.

| | Library | Test |
| - | - | - |
| Installation | `npm install playwright` | `npm init playwright@latest` - `install`과 `init`의 차이에 주목하세요. |
| Install browsers | `@playwright/browser-chromium`, `@playwright/browser-firefox` 및/또는 `@playwright/browser-webkit` 설치 | `npx playwright install` 또는 하나만 설치하려면 `npx playwright install chromium` |
| `import` from | `playwright` | `@playwright/test` |
| Initialization | 명시적으로 다음을 수행해야 합니다. <ol><li>사용할 브라우저를 선택합니다. 예: `chromium`</li><li>[browserType.launch()](/api/class-browsertype.mdx#browser-type-launch)로 브라우저를 실행합니다.</li><li>[browser.newContext()](/api/class-browser.mdx#browser-new-context)로 컨텍스트를 만들고 컨텍스트 옵션을 명시적으로 전달합니다. 예: `devices['iPhone 11']`</li><li>[browserContext.newPage()](/api/class-browsercontext.mdx#browser-context-new-page)로 페이지를 만듭니다.</li></ol> | 격리된 `page` 및 `context`가 다른 [기본 제공 픽스처](./test-fixtures.mdx#built-in-fixtures)와 함께 각 테스트에 기본적으로 제공됩니다. 명시적인 생성은 없습니다. 테스트에서 인수로 참조되는 경우 Test Runner가 테스트를 위해 생성합니다. (즉, 지연 초기화) |
| Assertions | 내장된 웹 우선 어설션 없음 | 다음과 같은 [웹 우선 어설션](./test-assertions.mdx): <ul><li>[expect(page).toHaveTitle()](/api/class-pageassertions.mdx#page-assertions-to-have-title)</li><li>[expect(page).toHaveScreenshot()](/api/class-pageassertions.mdx#page-assertions-to-have-screenshot-1)</li></ul> 조건을 충족할 때까지 자동 대기 및 재시도합니다.|
| Timeouts | 대부분의 작업에 대해 기본적으로 30초입니다. | 대부분의 작업은 시간 초과되지 않지만 모든 테스트에는 실패하게 만드는 시간 초과(기본적으로 30초)가 있습니다. |
| Cleanup | 명시적으로 다음을 수행해야 합니다. <ol><li>[browserContext.close()](/api/class-browsercontext.mdx#browser-context-close)로 컨텍스트를 닫습니다.</li><li>[browser.close()](/api/class-browser.mdx#browser-close)로 브라우저를 닫습니다.</li></ol> | [기본 제공 픽스처](./test-fixtures.mdx#built-in-fixtures)를 명시적으로 닫을 필요가 없습니다. Test Runner가 처리합니다.
| Running | Library를 사용하는 경우 코드를 노드 스크립트로 실행하고, 필요한 경우 먼저 컴파일합니다. | Test Runner를 사용하는 경우 `npx playwright test` 명령을 사용합니다. [구성](./test-configuration.mdx)과 함께 Test Runner는 모든 컴파일을 처리하고 실행할 항목과 실행 방법을 선택합니다. |

위의 내용 외에도 Playwright Test는 모든 기능을 갖춘 Test Runner로서 다음을 포함합니다.
- [구성 매트릭스 및 프로젝트](./test-configuration.mdx): 위의 예에서 Playwright Library 버전에서 다른 장치 또는 브라우저로 실행하려면 스크립트를 수정하고 정보를 연결해야 합니다. Playwright Test를 사용하면 [구성 매트릭스](./test-configuration.mdx)를 한 곳에서 지정하기만 하면 이러한 각 구성에서 하나의 테스트를 실행합니다.
- [병렬화](./test-parallel.mdx)
- [웹 우선 어설션](./test-assertions.mdx)
- [보고](./test-reporters.mdx)

## Usage

npm 또는 Yarn을 사용하여 Node.js 프로젝트에 Playwright 라이브러리를 설치합니다. [시스템 요구 사항](./intro.mdx#system-requirements)을 참조하세요.

```bash
npm i -D playwright
```

또한 브라우저를 수동으로 설치하거나 자동으로 설치해주는 패키지를 추가해야 합니다.

```bash
# Chromium, Firefox 및 WebKit 브라우저 다운로드
npx playwright install chromium firefox webkit

# 또는 npm 설치 시 브라우저를 다운로드하는 패키지 추가
npm i -D @playwright/browser-chromium @playwright/browser-firefox @playwright/browser-webkit
```

자세한 내용은 [브라우저 관리](./browsers.mdx#managing-browser-binaries)를 참조하세요.

설치가 완료되면 Node.js 스크립트에서 Playwright를 가져와 3개의 브라우저(`chromium`, `firefox` 및 `webkit`) 중 하나를 실행할 수 있습니다.

```js
const { chromium } = require('playwright');

(async () => {
  const browser = await chromium.launch();
  // 페이지를 만들고, UI 요소와 상호 작용하고, 값을 어설션합니다.
  await browser.close();
})();
```

Playwright API는 비동기적이며 Promise 객체를 반환합니다. 코드 예제에서는 가독성을 높이기 위해 [async/await 패턴](https://developer.mozilla.org/en-US/docs/Learn/JavaScript/Asynchronous/Async_await)을 사용합니다. 코드는 자체를 호출하는 이름 없는 비동기 화살표 함수로 래핑됩니다.

```js
(async () => { // 비동기 화살표 함수 시작
  // 함수 코드
  // ...
})(); // 함수 종료 및 자체 호출 ()
```

## First script

첫 번째 스크립트에서는 `https://playwright.dev/`로 이동하여 WebKit에서 스크린샷을 찍습니다.

```js
const { webkit } = require('playwright');

(async () => {
  const browser = await webkit.launch();
  const page = await browser.newPage();
  await page.goto('https://playwright.dev/');
  await page.screenshot({ path: `example.png` });
  await browser.close();
})();
```

기본적으로 Playwright는 브라우저를 헤드리스 모드로 실행합니다. 브라우저 UI를 보려면 브라우저를 실행하는 동안 `headless: false` 플래그를 전달하세요. `slowMo`를 사용하여 실행 속도를 늦출 수도 있습니다. 디버깅 도구 [섹션](./debug.mdx)에서 자세히 알아보세요.

```js
firefox.launch({ headless: false, slowMo: 50 });
```

## Browser downloads

Playwright 브라우저를 다운로드하려면 다음을 실행하세요.

```bash
# 명시적으로 브라우저 다운로드
npx playwright install
```

또는 `@playwright/browser-chromium`, `@playwright/browser-firefox` 및 `@playwright/browser-webkit` 패키지를 추가하여 패키지 설치 중에 해당 브라우저를 자동으로 다운로드할 수 있습니다.

```bash
# npm 설치 시 브라우저를 다운로드하는 헬퍼 패키지 사용
npm install @playwright/browser-chromium
```

## TypeScript support

Playwright는 TypeScript에 대한 기본 제공 지원을 포함합니다. 유형 정의가 자동으로 가져오기됩니다. IDE 환경을 개선하기 위해 유형 검사를 사용하는 것이 좋습니다.

### In JavaScript

VS Code 또는 WebStorm에서 유형 검사를 받으려면 JavaScript 파일의 맨 위에 다음을 추가하세요.

```js
// @ts-check
// ...
```

또는 JSDoc을 사용하여 변수의 유형을 설정할 수 있습니다.

```js
/** @type {import('playwright').Page} */
let page;
```

### In TypeScript

TypeScript 지원은 기본적으로 작동합니다. 유형을 명시적으로 가져올 수도 있습니다.

```js
let page: import('playwright').Page;
```
