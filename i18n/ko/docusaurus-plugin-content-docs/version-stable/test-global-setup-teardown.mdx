## Introduction

전역 설정 및 해제를 구성하는 방법에는 두 가지가 있습니다. 전역 설정 파일을 사용하고 [`globalSetup`](#option-2-configure-globalsetup-and-globalteardown)에서 구성에 설정하거나 [프로젝트 종속성](#option-1-project-dependencies)을 사용하는 것입니다. 프로젝트 종속성을 사용하면 다른 모든 프로젝트보다 먼저 실행되는 프로젝트를 정의합니다. 이는 프로젝트 종속성을 사용하여 전역 설정을 구성하는 권장 방법이며 HTML 보고서에 전역 설정이 표시되고, trace viewer에 설정 기록이 기록되며, fixtures를 사용할 수 있습니다.

## Option 1: Project Dependencies

[프로젝트 종속성](./api/class-testproject#test-project-dependencies)은 다른 프로젝트의 테스트를 실행하기 전에 실행해야 하는 프로젝트 목록입니다. 이러한 종속성은 하나의 프로젝트가 먼저 실행되도록 전역 설정 작업을 구성하는 데 유용할 수 있습니다. 종속성을 사용하면 전역 설정이 trace 및 기타 아티팩트를 생성할 수 있습니다.

### Setup

먼저 'setup db'라는 새 프로젝트를 추가합니다. 그런 다음 `global.setup.ts`라는 파일과 일치하도록 [testProject.testMatch](/api/class-testproject.mdx#test-project-test-match) 속성을 지정합니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  // ...
  projects: [
    {
      name: 'setup db',
      testMatch: /global\.setup\.ts/,
    },
    // {
    //   other project
    // }
  ]
});
```

그런 다음 설정 프로젝트에 종속된 프로젝트에 [testProject.dependencies](/api/class-testproject.mdx#test-project-dependencies) 속성을 추가하고 이전 단계에서 정의한 종속성 프로젝트의 이름을 배열에 전달합니다.

```js title="playwright.config.ts"
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  // ...
  projects: [
    {
      name: 'setup db',
      testMatch: /global\.setup\.ts/,
    },
    {
      name: 'chromium with db',
      use: { ...devices['Desktop Chrome'] },
      dependencies: ['setup db'],
    },
  ]
});
```

이 예제에서 'chromium with db' 프로젝트는 'setup db' 프로젝트에 종속됩니다. 그런 다음 프로젝트의 루트 수준에 저장된 설정 테스트를 만듭니다(설정 및 해제 코드는 [test()](./api/class-test#test-call) 함수를 호출하여 일반 테스트로 정의해야 함).

```js title="tests/global.setup.ts"
import { test as setup } from '@playwright/test';

setup('create new database', async ({ }) => {
  console.log('creating new database...');
  // 데이터베이스 초기화
});
```

```js title="tests/menu.spec.ts"
import { test, expect } from '@playwright/test';

test('menu', async ({ page }) => {
  // 데이터베이스에 종속된 테스트
});
```

### Teardown

설정 프로젝트에 [testProject.teardown](/api/class-testproject.mdx#test-project-teardown) 속성을 추가하여 설정을 해제할 수 있습니다. 이렇게 하면 모든 종속 프로젝트가 실행된 후 실행됩니다.

먼저 이전 단계에서 해제 프로젝트에 부여한 이름인 'cleanup db'라는 이름으로 설정 프로젝트에 [testProject.teardown](/api/class-testproject.mdx#test-project-teardown) 속성을 추가합니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  // ...
  projects: [
    {
      name: 'setup db',
      testMatch: /global\.setup\.ts/,
      teardown: 'cleanup db',
    },
    {
      name: 'cleanup db',
      testMatch: /global\.teardown\.ts/,
    },
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
      dependencies: ['setup db'],
    },
  ]
});
```

그런 다음 프로젝트의 tests 디렉터리에 `global.teardown.ts` 파일을 만듭니다. 이 파일은 모든 테스트가 실행된 후 데이터베이스에서 데이터를 삭제하는 데 사용됩니다.

```js title="tests/global.teardown.ts"
import { test as teardown } from '@playwright/test';

teardown('delete database', async ({ }) => {
  console.log('deleting test database...');
  // 데이터베이스 삭제
});
```

## Option 2: Configure globalSetup and globalTeardown

[구성 파일](./test-configuration.mdx#advanced-configuration)에서 `globalSetup` 옵션을 사용하여 모든 테스트를 실행하기 전에 한 번 설정할 수 있습니다. 전역 설정 파일은 구성 객체를 사용하는 단일 함수를 내보내야 합니다. 이 함수는 모든 테스트 전에 한 번 실행됩니다.

마찬가지로 `globalTeardown`을 사용하여 모든 테스트가 완료된 후 한 번 실행합니다. 또는 `globalSetup`이 전역 해제로 사용될 함수를 반환하도록 합니다. 전역 설정에서 환경 변수를 사용하여 포트 번호, 인증 토큰 등과 같은 데이터를 테스트에 전달할 수 있습니다.

:::note

`globalSetup` 및 `globalTeardown` 주의 사항:
- 이러한 메서드는 [전역 설정 중 오류 발생 시 trace 캡처](#capturing-trace-of-failures-during-global-setup)에 설명된 대로 명시적으로 활성화되지 않는 한 trace 또는 아티팩트를 생성하지 않습니다.
- 구성 파일에 지정된 `headless` 또는 `testIdAttribute`와 같은 옵션은 적용되지 않습니다.
- `globalSetup`에서 throw된 처리되지 않은 예외는 Playwright가 테스트를 실행하지 못하게 하고, reporter에 테스트 결과가 표시되지 않습니다.

trace, 아티팩트를 생성하고, 구성 옵션을 존중하고, 설정 실패 시에도 reporter에서 테스트 결과를 얻으려면 [프로젝트 종속성](#option-1-project-dependencies)을 사용하는 것이 좋습니다.
:::

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  globalSetup: require.resolve('./global-setup'),
  globalTeardown: require.resolve('./global-teardown'),
});
```

### Example

다음은 한 번 인증하고 테스트에서 인증 상태를 재사용하는 전역 설정 예제입니다. 구성 파일에서 `baseURL` 및 `storageState` 옵션을 사용합니다.

```js title="global-setup.ts"
import { chromium, type FullConfig } from '@playwright/test';

async function globalSetup(config: FullConfig) {
  const { baseURL, storageState } = config.projects[0].use;
  const browser = await chromium.launch();
  const page = await browser.newPage();
  await page.goto(baseURL!);
  await page.getByLabel('User Name').fill('user');
  await page.getByLabel('Password').fill('password');
  await page.getByText('Sign in').click();
  await page.context().storageState({ path: storageState as string });
  await browser.close();
}

export default globalSetup;
```

구성 파일에서 `globalSetup`, `baseURL` 및 `storageState`를 지정합니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';
export default defineConfig({
  globalSetup: require.resolve('./global-setup'),
  use: {
    baseURL: 'http://localhost:3000/',
    storageState: 'state.json',
  },
});
```

테스트는 전역 설정에 의해 채워진 `storageState`를 지정하기 때문에 이미 인증된 상태로 시작됩니다.

```js
import { test } from '@playwright/test';

test('test', async ({ page }) => {
  await page.goto('/');
  // 로그인되었습니다!
});
```

`process.env`를 통해 환경 변수로 설정하여 전역 설정 파일에서 테스트에서 임의의 데이터를 사용할 수 있도록 할 수 있습니다.

```js title="global-setup.ts"
import type { FullConfig } from '@playwright/test';

async function globalSetup(config: FullConfig) {
  process.env.FOO = 'some data';
  // 또는 JSON으로 더 복잡한 데이터 구조:
  process.env.BAR = JSON.stringify({ some: 'data' });
}

export default globalSetup;
```

테스트는 전역 설정에 설정된 `process.env` 속성에 액세스할 수 있습니다.

```js
import { test } from '@playwright/test';

test('test', async ({ page }) => {
  // globalSetup에 설정된 환경 변수는 test() 내에서만 사용할 수 있습니다.
  const { FOO, BAR } = process.env;

  // FOO 및 BAR 속성이 채워집니다.
  expect(FOO).toEqual('some data');

  const complexData = JSON.parse(BAR);
  expect(BAR).toEqual({ some: 'data' });
});
```

### Capturing trace of failures during global setup

경우에 따라 전역 설정 중에 발생한 실패의 trace를 캡처하는 것이 유용할 수 있습니다. 이렇게 하려면 설정에서 [trace 시작](./api/class-tracing.mdx#tracing-start)을 해야 하며, 오류가 throw되기 전에 오류가 발생하면 [trace 중지](./api/class-tracing.mdx#tracing-stop)를 해야 합니다. 이는 설정을 `try...catch` 블록으로 래핑하여 달성할 수 있습니다. 다음은 trace를 캡처하도록 전역 설정 예제를 확장하는 예제입니다.

```js title="global-setup.ts"
import { chromium, type FullConfig } from '@playwright/test';

async function globalSetup(config: FullConfig) {
  const { baseURL, storageState } = config.projects[0].use;
  const browser = await chromium.launch();
  const context = await browser.newContext();
  const page = await context.newPage();
  try {
    await context.tracing.start({ screenshots: true, snapshots: true });
    await page.goto(baseURL!);
    await page.getByLabel('User Name').fill('user');
    await page.getByLabel('Password').fill('password');
    await page.getByText('Sign in').click();
    await context.storageState({ path: storageState as string });
    await context.tracing.stop({
      path: './test-results/setup-trace.zip',
    });
    await browser.close();
  } catch (error) {
    await context.tracing.stop({
      path: './test-results/failed-setup-trace.zip',
    });
    await browser.close();
    throw error;
  }
}

export default globalSetup;
```
