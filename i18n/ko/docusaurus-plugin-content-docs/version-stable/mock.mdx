## Introduction

Web API는 일반적으로 HTTP 엔드포인트로 구현됩니다. Playwright는 HTTP 및 HTTPS 네트워크 트래픽을 **모의 실험(mock)**하고 **수정**하는 API를 제공합니다. 페이지가 수행하는 모든 요청(예: [XHR](https://developer.mozilla.org/en-US/docs/Web/API/XMLHttpRequest) 및 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API) 요청)을 추적, 수정 및 모의 실험할 수 있습니다. Playwright를 사용하면 페이지에서 발생한 여러 네트워크 요청이 포함된 HAR 파일을 사용하여 모의 실험할 수도 있습니다.

## API 요청 모의 실험

다음 코드는 `*/**/api/v1/fruits`에 대한 모든 호출을 가로채고 사용자 정의 응답을 대신 반환합니다. API에 대한 요청은 이루어지지 않습니다. 테스트는 모의 실험된 경로를 사용하는 URL로 이동하고 모의 실험 데이터가 페이지에 있는지 확인합니다.

```js
test("mocks a fruit and doesn't call api", async ({ page }) => {
  // 탐색하기 전에 API 호출을 모의 실험합니다.
  await page.route('*/**/api/v1/fruits', async route => {
    const json = [{ name: 'Strawberry', id: 21 }];
    await route.fulfill({ json });
  });
  // 페이지로 이동합니다.
  await page.goto('https://demo.playwright.dev/api-mocking');

  // 딸기 과일이 보이는지 확인합니다.
  await expect(page.getByText('Strawberry')).toBeVisible();
});
```

예제 테스트의 추적에서 API가 호출되지 않았지만 모의 실험 데이터로 완료되었음을 알 수 있습니다. ![api mocking trace](https://github.com/microsoft/playwright/assets/13063165/3dc14cbf-c100-4efc-ac21-d7b52d698b53)

## API 응답 수정

경우에 따라 API 요청을 수행해야 하지만 재현 가능한 테스트를 위해 응답을 패치해야 합니다. 이 경우 요청을 모의 실험하는 대신 요청을 수행하고 수정된 응답으로 완료할 수 있습니다.

아래 예에서는 fruit API에 대한 호출을 가로채고 'Loquat'라는 새 과일을 데이터에 추가합니다. 그런 다음 URL로 이동하여 이 데이터가 있는지 확인합니다.

```js
test('gets the json from api and adds a new fruit', async ({ page }) => {
  // 응답을 가져와서 추가합니다.
  await page.route('*/**/api/v1/fruits', async route => {
    const response = await route.fetch();
    const json = await response.json();
    json.push({ name: 'Loquat', id: 100 });
    // 응답 본문을 지정된 JSON 객체로 패치하면서 원래 응답을 사용하여 완료합니다.
    await route.fulfill({ response, json });
  });

  // 페이지로 이동합니다.
  await page.goto('https://demo.playwright.dev/api-mocking');

  // 새 과일이 보이는지 확인합니다.
  await expect(page.getByText('Loquat', { exact: true })).toBeVisible();
});
```

테스트 추적에서 API가 호출되고 응답이 수정되었음을 알 수 있습니다. ![trace of test showing api being called and fulfilled](https://github.com/microsoft/playwright/assets/13063165/8b8dd82d-1b3e-428e-871b-840581fed439)

응답을 검사하면 새 과일이 목록에 추가되었음을 알 수 있습니다. ![trace of test showing the mock response](https://github.com/microsoft/playwright/assets/13063165/03e6c87c-4ecc-47e8-9ca0-30fface25e9d)

## HAR 파일을 사용한 모의 실험

HAR 파일은 페이지가 로드될 때 이루어지는 모든 네트워크 요청 기록이 포함된 [HTTP Archive](http://www.softwareishard.com/blog/har-12-spec/) 파일입니다. 여기에는 요청 및 응답 헤더, 쿠키, 콘텐츠, 타이밍 등에 대한 정보가 포함되어 있습니다. HAR 파일을 사용하여 테스트에서 네트워크 요청을 모의 실험할 수 있습니다. 다음이 필요합니다.
1. HAR 파일을 기록합니다.
2. 테스트와 함께 HAR 파일을 커밋합니다.
3. 테스트에서 저장된 HAR 파일을 사용하여 요청을 라우팅합니다.

### HAR 파일 기록

HAR 파일을 기록하려면 [page.routeFromHAR()](/api/class-page.mdx#page-route-from-har) 또는 [browserContext.routeFromHAR()](/api/class-browsercontext.mdx#browser-context-route-from-har) 메서드를 사용합니다. 이 메서드는 HAR 파일의 경로와 선택적 옵션 객체를 가져옵니다. 옵션 객체에는 지정된 glob 패턴과 일치하는 URL을 가진 요청만 HAR 파일에서 제공되도록 URL이 포함될 수 있습니다. 지정하지 않으면 모든 요청이 HAR 파일에서 제공됩니다.

`update` 옵션을 true로 설정하면 HAR 파일에서 요청을 제공하는 대신 실제 네트워크 정보로 HAR 파일이 생성되거나 업데이트됩니다. 실제 데이터로 HAR을 채우기 위해 테스트를 만들 때 사용합니다.

```js
test('records or updates the HAR file', async ({ page }) => {
  // HAR 파일에서 응답을 가져옵니다.
  await page.routeFromHAR('./hars/fruit.har', {
    url: '*/**/api/v1/fruits',
    update: true,
  });

  // 페이지로 이동합니다.
  await page.goto('https://demo.playwright.dev/api-mocking');

  // 과일이 보이는지 확인합니다.
  await expect(page.getByText('Strawberry')).toBeVisible();
});
```

### HAR 파일 수정

HAR 파일을 기록한 후에는 'hars' 폴더 안에 있는 해시된 .txt 파일을 열고 JSON을 편집하여 수정할 수 있습니다. 이 파일은 소스 제어에 커밋해야 합니다. `update: true`로 이 테스트를 실행할 때마다 API의 요청으로 HAR 파일이 업데이트됩니다.

```json
[
  {
    "name": "Playwright",
    "id": 100
  },
  // ... 다른 과일
]
```

### HAR에서 재생

이제 HAR 파일을 기록하고 모의 실험 데이터를 수정했으므로 테스트에서 일치하는 응답을 제공하는 데 사용할 수 있습니다. 이를 위해 `update` 옵션을 끄거나 단순히 제거합니다. 이렇게 하면 API를 적중하는 대신 HAR 파일에 대해 테스트가 실행됩니다.

```js
test('gets the json from HAR and checks the new fruit has been added', async ({ page }) => {
  // HAR에서 API 요청을 재생합니다.
  // HAR에서 일치하는 응답을 사용하거나,
  // 일치하는 항목이 없으면 요청을 중단합니다.
  await page.routeFromHAR('./hars/fruit.har', {
    url: '*/**/api/v1/fruits',
    update: false,
  });

  // 페이지로 이동합니다.
  await page.goto('https://demo.playwright.dev/api-mocking');

  // Playwright 과일이 보이는지 확인합니다.
  await expect(page.getByText('Playwright', { exact: true })).toBeVisible();
});
```

테스트 추적에서 경로가 HAR 파일에서 완료되었고 API가 호출되지 않았음을 알 수 있습니다. ![trace showing the HAR file being used](https://github.com/microsoft/playwright/assets/13063165/1bd7ab66-ea4f-43c2-a4e5-ca17d4837ff1)

응답을 검사하면 'hars' 폴더 내에서 해시된 `.txt` 파일을 수동으로 업데이트하여 새 과일이 JSON에 추가되었음을 알 수 있습니다. ![trace showing response from HAR file](https://github.com/microsoft/playwright/assets/13063165/db3117fc-7b02-4973-9a51-29e213261a6a)

HAR 재생은 URL 및 HTTP 메서드와 엄격하게 일치합니다. POST 요청의 경우 POST 페이로드와도 엄격하게 일치합니다. 여러 녹화가 요청과 일치하는 경우 가장 많은 헤더가 일치하는 녹화가 선택됩니다. 리디렉션되는 항목은 자동으로 따릅니다.

기록할 때와 마찬가지로 주어진 HAR 파일 이름이 `.zip`으로 끝나면 네트워크 페이로드가 별도의 항목으로 저장된 HAR 파일이 포함된 아카이브로 간주됩니다. 이 아카이브를 추출하고 페이로드 또는 HAR 로그를 수동으로 편집하고 추출된 har 파일을 가리킬 수도 있습니다. 모든 페이로드는 파일 시스템에서 추출된 har 파일을 기준으로 확인됩니다.

#### CLI로 HAR 기록

테스트를 위해 HAR 파일을 기록하려면 `update` 옵션을 사용하는 것이 좋습니다. 그러나 Playwright CLI를 사용하여 HAR을 기록할 수도 있습니다.

Playwright CLI로 브라우저를 열고 HAR 파일을 생성하려면 `--save-har` 옵션을 전달합니다. 선택적으로 `--save-har-glob`을 사용하여 관심 있는 요청(예: API 엔드포인트)만 저장합니다. har 파일 이름이 `.zip`으로 끝나면 아티팩트가 별도의 파일로 작성되고 모두 단일 `zip`으로 압축됩니다.

```bash
# example.com의 API 요청을 "example.har" 아카이브로 저장합니다.
npx playwright open --save-har=example.har --save-har-glob="**/api/**" https://example.com
```

## WebSocket 모의 실험

다음 코드는 WebSocket 연결을 가로채고 서버에 연결하는 대신 WebSocket을 통한 전체 통신을 모의 실험합니다. 이 예에서는 `"request"`에 `"response"`로 응답합니다.

```js
await page.routeWebSocket('wss://example.com/ws', ws => {
  ws.onMessage(message => {
    if (message === 'request')
      ws.send('response');
  });
});
```

또는 실제 서버에 연결하되 중간에 메시지를 가로채서 수정하거나 차단할 수 있습니다. 다음은 페이지에서 서버로 보낸 일부 메시지를 수정하고 나머지는 수정하지 않은 상태로 두는 예입니다.

```js
await page.routeWebSocket('wss://example.com/ws', ws => {
  const server = ws.connectToServer();
  ws.onMessage(message => {
    if (message === 'request')
      server.send('request2');
    else
      server.send(message);
  });
});
```
