## Introduction

Playwright는 텍스트 입력, 체크 박스, 라디오 버튼, 선택 옵션, 마우스 클릭, 문자 입력, 키 및 단축키, 파일 업로드 및 요소 포커스와 같은 HTML 입력 요소와 상호 작용할 수 있습니다.

## Text input

[locator.fill()](/api/class-locator.mdx#locator-fill)을 사용하는 것이 폼 필드를 채우는 가장 쉬운 방법입니다. 요소에 초점을 맞추고 입력된 텍스트로 `input` 이벤트를 트리거합니다. `<input>`, `<textarea>` 및 `[contenteditable]` 요소에서 작동합니다.

```js
// 텍스트 입력
await page.getByRole('textbox').fill('Peter');

// 날짜 입력
await page.getByLabel('Birth date').fill('2020-02-02');

// 시간 입력
await page.getByLabel('Appointment time').fill('13:15');

// 현지 날짜/시간 입력
await page.getByLabel('Local time').fill('2020-03-02T05:15');
```

## Checkboxes and radio buttons

[locator.setChecked()](/api/class-locator.mdx#locator-set-checked)를 사용하는 것이 체크 박스 또는 라디오 버튼을 체크하고 해제하는 가장 쉬운 방법입니다. 이 메서드는 `input[type=checkbox]`, `input[type=radio]` 및 `[role=checkbox]` 요소와 함께 사용할 수 있습니다.

```js
// 체크 박스 선택
await page.getByLabel('I agree to the terms above').check();

// 선택된 상태 어설션
expect(page.getByLabel('Subscribe to newsletter')).toBeChecked();

// 라디오 버튼 선택
await page.getByLabel('XL').check();
```

## Select options

[locator.selectOption()](/api/class-locator.mdx#locator-select-option)을 사용하여 `<select>` 요소에서 하나 또는 여러 옵션을 선택합니다. 선택할 옵션의 `value` 또는 `label`을 지정할 수 있습니다. 여러 옵션을 선택할 수 있습니다.

```js
// 값 또는 레이블과 일치하는 단일 선택
await page.getByLabel('Choose a color').selectOption('blue');

// 레이블과 일치하는 단일 선택
await page.getByLabel('Choose a color').selectOption({ label: 'Blue' });

// 여러 선택된 항목
await page.getByLabel('Choose multiple colors').selectOption(['red', 'green', 'blue']);
```

## Mouse click

간단한 사용자 클릭을 수행합니다.

```js
// 일반 클릭
await page.getByRole('button').click();

// 더블 클릭
await page.getByText('Item').dblclick();

// 오른쪽 클릭
await page.getByText('Item').click({ button: 'right' });

// Shift + 클릭
await page.getByText('Item').click({ modifiers: ['Shift'] });

// Windows 및 Linux에서 Ctrl + 클릭
// macOS에서 Meta + 클릭
await page.getByText('Item').click({ modifiers: ['ControlOrMeta'] });

// 요소 위에 마우스 커서 올리기
await page.getByText('Item').hover();

// 왼쪽 상단 모서리 클릭
await page.getByText('Item').click({ position: { x: 0, y: 0 } });
```

내부적으로 이 메서드와 다른 포인터 관련 메서드는 다음을 수행합니다.
- 주어진 선택자를 가진 요소가 DOM에 있는지 기다립니다.
- 비어 있지 않고, `display:none`이 아니고, `visibility:hidden`이 아닌지 표시될 때까지 기다립니다.
- CSS 전환이 완료될 때까지 요소가 이동을 멈추기를 기다립니다.
- 요소를 뷰포트로 스크롤합니다.
- 요소가 다른 요소에 의해 가려지지 않을 때와 같이 동작 지점에서 포인터 이벤트를 수신할 때까지 기다립니다.
- 위의 검사 중 요소가 분리되면 다시 시도합니다.

#### Forcing the click

경우에 따라 앱은 요소 위에 마우스를 올리면 클릭을 가로채는 다른 요소로 덮어쓰는 중요하지 않은 로직을 사용합니다. 이 동작은 요소가 덮여지고 클릭이 다른 곳으로 전달되는 버그와 구별할 수 없습니다. 이런 일이 발생하고 있다는 것을 알고 있는 경우 [동작 가능성](./actionability.mdx) 검사를 우회하고 클릭을 강제할 수 있습니다.

```js
await page.getByRole('button').click({ force: true });
```

#### Programmatic click

실제 조건에서 앱을 테스트하는 데 관심이 없고 가능한 모든 수단으로 클릭을 시뮬레이션하려는 경우 [locator.dispatchEvent()](/api/class-locator.mdx#locator-dispatch-event)를 사용하여 요소에서 클릭 이벤트를 간단히 디스패치하여 [`HTMLElement.click()`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLElement/click) 동작을 트리거할 수 있습니다.

```js
await page.getByRole('button').dispatchEvent('click');
```

## Type characters

:::caution

대부분의 경우 [locator.fill()](/api/class-locator.mdx#locator-fill)을 사용하여 텍스트를 입력해야 합니다. 위의 [텍스트 입력](#text-input) 섹션을 참조하십시오. 페이지에 특수한 키보드 처리가 있는 경우에만 문자를 입력해야 합니다.
:::

[locator.pressSequentially()](/api/class-locator.mdx#locator-press-sequentially)를 사용하여 실제 키보드를 가진 사용자처럼 필드에 문자별로 입력합니다.

```js
// 키를 하나씩 누릅니다.
await page.locator('#area').pressSequentially('Hello World!');
```

이 메서드는 필요한 모든 키보드 이벤트를 `keydown`, `keyup`, `keypress` 이벤트와 함께 발생시킵니다. 키 누름 사이의 선택적 `delay`를 지정하여 실제 사용자 동작을 시뮬레이션할 수도 있습니다.

## Keys and shortcuts

```js
// Enter 키 누르기
await page.getByText('Submit').press('Enter');

// Control+Right 디스패치
await page.getByRole('textbox').press('Control+ArrowRight');

// 키보드에서 $ 기호 누르기
await page.getByRole('textbox').press('$');
```

[locator.press()](/api/class-locator.mdx#locator-press) 메서드는 선택된 요소에 초점을 맞추고 단일 키 입력을 생성합니다. 키보드 이벤트의 [keyboardEvent.key](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key) 속성에서 발생하는 논리적 키 이름을 허용합니다.

```txt
Backquote, Minus, Equal, Backslash, Backspace, Tab, Delete, Escape,
ArrowDown, End, Enter, Home, Insert, PageDown, PageUp, ArrowRight,
ArrowUp, F1 - F12, Digit0 - Digit9, KeyA - KeyZ, etc.
```

- 또는 `"a"` 또는 `"#"`와 같이 생성하려는 단일 문자를 지정할 수 있습니다.
- 다음과 같은 수정 단축키도 지원됩니다: `Shift, Control, Alt, Meta`.

간단한 버전은 단일 문자를 생성합니다. 이 문자는 대소문자를 구분하므로 `"a"`와 `"A"`는 다른 결과를 생성합니다.

```js
// <input id=name>
await page.locator('#name').press('Shift+A');

// <input id=name>
await page.locator('#name').press('Shift+ArrowLeft');
```

`"Control+o"` 또는 `"Control+Shift+T"`와 같은 단축키도 지원됩니다. 수정자와 함께 지정하면 수정자가 눌러지고 후속 키가 눌러지는 동안 유지됩니다.

대문자를 생성하려면 `Shift-A`에서 대문자 `A`를 지정해야 합니다. `Shift-a`는 `CapsLock`이 토글된 것처럼 소문자를 생성합니다.

## Upload files

[locator.setInputFiles()](/api/class-locator.mdx#locator-set-input-files) 메서드를 사용하여 업로드할 입력 파일을 선택할 수 있습니다. 첫 번째 인수가 유형이 `"file"`인 [입력 요소](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input)를 가리키도록 예상합니다. 여러 파일을 배열로 전달할 수 있습니다. 일부 파일 경로가 상대적인 경우 현재 작업 디렉토리를 기준으로 확인됩니다. 빈 배열은 선택한 파일을 지웁니다.

```js
// 파일 하나 선택
await page.getByLabel('Upload file').setInputFiles(path.join(__dirname, 'myfile.pdf'));

// 여러 파일 선택
await page.getByLabel('Upload files').setInputFiles([
  path.join(__dirname, 'file1.txt'),
  path.join(__dirname, 'file2.txt'),
]);

// 디렉터리 선택
await page.getByLabel('Upload directory').setInputFiles(path.join(__dirname, 'mydir'));

// 선택한 모든 파일 제거
await page.getByLabel('Upload file').setInputFiles([]);

// 메모리에서 버퍼 업로드
await page.getByLabel('Upload file').setInputFiles({
  name: 'file.txt',
  mimeType: 'text/plain',
  buffer: Buffer.from('this is test')
});
```

입력 요소가 없는 경우 (동적으로 생성됨) [page.on('filechooser')](/api/class-page.mdx#page-event-file-chooser) 이벤트를 처리하거나 작업에 따라 해당 대기 메서드를 사용할 수 있습니다.

```js
// 클릭하기 전에 파일 선택기를 기다리기 시작합니다. await는 없습니다.
const fileChooserPromise = page.waitForEvent('filechooser');
await page.getByLabel('Upload file').click();
const fileChooser = await fileChooserPromise;
await fileChooser.setFiles(path.join(__dirname, 'myfile.pdf'));
```

## Focus element

포커스 이벤트를 처리하는 동적 페이지의 경우 [locator.focus()](/api/class-locator.mdx#locator-focus)를 사용하여 지정된 요소에 초점을 맞출 수 있습니다.

```js
await page.getByLabel('Password').focus();
```

## Drag and Drop

[locator.dragTo()](/api/class-locator.mdx#locator-drag-to)를 사용하여 드래그 앤 드롭 작업을 수행할 수 있습니다. 이 메서드는 다음을 수행합니다.
- 드래그할 요소 위에 마우스를 올려 놓습니다.
- 왼쪽 마우스 버튼을 누릅니다.
- 드롭을 받을 요소로 마우스를 이동합니다.
- 왼쪽 마우스 버튼을 놓습니다.

```js
await page.locator('#item-to-be-dragged').dragTo(page.locator('#item-to-drop-at'));
```

### Dragging manually

드래그 작업을 정확하게 제어하려면 [locator.hover()](/api/class-locator.mdx#locator-hover), [mouse.down()](/api/class-mouse.mdx#mouse-down), [mouse.move()](/api/class-mouse.mdx#mouse-move) 및 [mouse.up()](/api/class-mouse.mdx#mouse-up)과 같은 하위 수준 메서드를 사용하십시오.

```js
await page.locator('#item-to-be-dragged').hover();
await page.mouse.down();
await page.locator('#item-to-drop-at').hover();
await page.mouse.up();
```

:::note
페이지가 `dragover` 이벤트 디스패치에 의존하는 경우 모든 브라우저에서 이벤트를 트리거하려면 마우스 이동을 두 번 이상 수행해야 합니다. 두 번째 마우스 이동을 안정적으로 실행하려면 [mouse.move()](/api/class-mouse.mdx#mouse-move) 또는 [locator.hover()](/api/class-locator.mdx#locator-hover)를 두 번 반복하십시오. 작업 순서는 드래그 요소를 호버링하고, 마우스 다운하고, 드롭 요소를 호버링하고, 드롭 요소를 두 번째로 호버링하고, 마우스 업하는 것입니다.
:::

## Scrolling

대부분의 경우 Playwright는 모든 작업을 수행하기 전에 자동으로 스크롤합니다. 따라서 명시적으로 스크롤할 필요가 없습니다.

```js
// 버튼이 보이도록 자동으로 스크롤합니다.
await page.getByRole('button').click();
```

그러나 드문 경우에 수동으로 스크롤해야 할 수도 있습니다. 예를 들어 "무한 목록"이 더 많은 요소를 로드하도록 강제하거나 특정 스크린샷을 위해 페이지를 배치할 수 있습니다. 이러한 경우 가장 안정적인 방법은 하단에 표시할 요소를 찾고 뷰포트로 스크롤하는 것입니다.

```js
// 바닥글을 뷰포트로 스크롤하여 "무한 목록"이 더 많은 콘텐츠를 로드하도록 강제합니다.
await page.getByText('Footer text').scrollIntoViewIfNeeded();
```

스크롤을 더 정확하게 제어하려면 [mouse.wheel()](/api/class-mouse.mdx#mouse-wheel) 또는 [locator.evaluate()](/api/class-locator.mdx#locator-evaluate)를 사용하십시오.

```js
// 마우스 위치를 지정하고 마우스 휠로 스크롤합니다.
await page.getByTestId('scrolling-container').hover();
await page.mouse.wheel(0, 10);

// 또는 프로그래밍 방식으로 특정 요소를 스크롤합니다.
await page.getByTestId('scrolling-container').evaluate(e => e.scrollTop += 100);
```
