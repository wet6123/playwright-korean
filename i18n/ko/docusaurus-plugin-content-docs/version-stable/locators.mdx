## Introduction

[Locator]는 Playwright 자동 대기 및 재시도 기능의 핵심입니다. 간단히 말해서, 로케이터는 언제든지 페이지에서 요소(들)을 찾는 방법을 나타냅니다.

### Quick Guide

다음은 권장되는 내장 로케이터입니다.
- 명시적 및 암시적 접근성 속성으로 찾으려면 [page.getByRole()](#locate-by-role)를 사용하세요.
- 텍스트 콘텐츠로 찾으려면 [page.getByText()](#locate-by-text)를 사용하세요.
- 연결된 레이블 텍스트로 폼 컨트롤을 찾으려면 [page.getByLabel()](#locate-by-label)를 사용하세요.
- 플레이스홀더로 입력을 찾으려면 [page.getByPlaceholder()](#locate-by-placeholder)를 사용하세요.
- 텍스트 대안으로 요소를 찾으려면 [page.getByAltText()](#locate-by-alt-text)를 사용하세요. 보통 이미지입니다.
- title 속성으로 요소를 찾으려면 [page.getByTitle()](#locate-by-title)를 사용하세요.
- `data-testid` 속성을 기반으로 요소를 찾으려면 [page.getByTestId()](#locate-by-test-id)를 사용하세요 (다른 속성을 구성할 수 있습니다).

```js
await page.getByLabel('User Name').fill('John');

await page.getByLabel('Password').fill('secret-password');

await page.getByRole('button', { name: 'Sign in' }).click();

await expect(page.getByText('Welcome, John!')).toBeVisible();
```

## Locating elements

Playwright는 여러 내장 로케이터와 함께 제공됩니다. 테스트의 복원력을 높이기 위해 사용자에게 보이는 속성 및 [page.getByRole()](/api/class-page.mdx#page-get-by-role)과 같은 명시적 계약에 우선 순위를 둘 것을 권장합니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<button>Sign in</button>
</div>

```html
<button>Sign in</button>
```
</HTMLCard>

이름이 "Sign in"인 `button`의 역할로 요소를 찾습니다.

```js
await page.getByRole('button', { name: 'Sign in' }).click();
```

:::note
[코드 생성기](./codegen.mdx)를 사용하여 로케이터를 생성한 다음 원하는 대로 편집하십시오.
:::

로케이터가 액션에 사용될 때마다 최신 DOM 요소가 페이지에서 찾아집니다. 아래 스니펫에서 기본 DOM 요소는 모든 액션 이전에 한 번씩, 두 번 찾아집니다. 즉, 호출 사이에 DOM이 다시 렌더링되어 변경되는 경우 로케이터에 해당하는 새 요소가 사용됩니다.

```js
const locator = page.getByRole('button', { name: 'Sign in' });

await locator.hover();
await locator.click();
```

[page.getByLabel()](/api/class-page.mdx#page-get-by-label)과 같이 로케이터를 만드는 모든 메서드는 [Locator] 및 [FrameLocator] 클래스에서도 사용할 수 있으므로 이를 연결하고 로케이터를 반복적으로 좁힐 수 있습니다.

```js
const locator = page
    .frameLocator('#my-frame')
    .getByRole('button', { name: 'Sign in' });

await locator.click();
```

### Locate by role

[page.getByRole()](/api/class-page.mdx#page-get-by-role) 로케이터는 사용자와 보조 기술이 페이지를 인식하는 방식(예: 일부 요소가 버튼인지 확인란인지 여부)을 반영합니다. 역할로 찾을 때 일반적으로 접근 가능한 이름도 전달하여 로케이터가 정확한 요소를 정확히 찾아내도록 해야 합니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<h3>Sign up</h3>
<label>
  <input type="checkbox" /> Subscribe
</label>
<br/>
<button>Submit</button>
</div>

```html
<h3>Sign up</h3>
<label>
  <input type="checkbox" /> Subscribe
</label>
<br/>
<button>Submit</button>
```
</HTMLCard>

암시적 역할로 각 요소를 찾을 수 있습니다.

```js
await expect(page.getByRole('heading', { name: 'Sign up' })).toBeVisible();

await page.getByRole('checkbox', { name: 'Subscribe' }).check();

await page.getByRole('button', { name: /submit/i }).click();
```

역할 로케이터에는 [buttons, checkboxes, headings, links, lists, tables 등](https://www.w3.org/TR/html-aria/#docconformance)이 포함되며 [ARIA 역할](https://www.w3.org/TR/wai-aria-1.2/#roles), [ARIA 속성](https://www.w3.org/TR/wai-aria-1.2/#aria-attributes) 및 [접근 가능한 이름](https://w3c.github.io/accname/#dfn-accessible-name)에 대한 W3C 사양을 따릅니다. `<button>`과 같은 많은 html 요소에는 역할 로케이터에서 인식하는 [암시적으로 정의된 역할](https://w3c.github.io/html-aam/#html-element-role-mappings)이 있습니다.

역할 로케이터는 접근성 감사 및 적합성 테스트를 **대체하지 않고** ARIA 지침에 대한 조기 피드백을 제공합니다.

:::note[역할 로케이터를 사용해야 하는 경우]

사용자와 보조 기술이 페이지를 인식하는 방식에 가장 가깝기 때문에 요소 위치를 파악하기 위해 역할 로케이터를 우선적으로 사용하는 것이 좋습니다.
:::

### Locate by label

대부분의 폼 컨트롤에는 일반적으로 폼과 상호 작용하는 데 편리하게 사용할 수 있는 전용 레이블이 있습니다. 이 경우 [page.getByLabel()](/api/class-page.mdx#page-get-by-label)을 사용하여 연결된 레이블로 컨트롤을 찾을 수 있습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<label>Password <input type="password" /></label>

</div>

```html
<label>Password <input type="password" /></label>

```
</HTMLCard>

레이블 텍스트로 찾은 후 입력을 채울 수 있습니다.

```js
await page.getByLabel('Password').fill('secret');
```

:::note[레이블 로케이터를 사용해야 하는 경우]
폼 필드를 찾을 때 이 로케이터를 사용하십시오.
:::

### Locate by placeholder

입력에는 사용자에게 어떤 값을 입력해야 하는지 힌트를 주는 placeholder 속성이 있을 수 있습니다. [page.getByPlaceholder()](/api/class-page.mdx#page-get-by-placeholder)를 사용하여 이러한 입력을 찾을 수 있습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<input type="email" placeholder="name@example.com" />
</div>

```html
<input type="email" placeholder="name@example.com" />
```
</HTMLCard>

플레이스홀더 텍스트로 찾은 후 입력을 채울 수 있습니다.

```js
await page
    .getByPlaceholder('name@example.com')
    .fill('playwright@microsoft.com');
```

:::note[플레이스홀더 로케이터를 사용해야 하는 경우]
레이블은 없지만 플레이스홀더 텍스트가 있는 폼 요소를 찾을 때 이 로케이터를 사용하십시오.
:::

### Locate by text

포함된 텍스트로 요소를 찾습니다. [page.getByText()](/api/class-page.mdx#page-get-by-text)를 사용할 때 부분 문자열, 정확한 문자열 또는 정규식으로 일치시킬 수 있습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<span>Welcome, John</span>
</div>

```html
<span>Welcome, John</span>
```
</HTMLCard>

포함된 텍스트로 요소를 찾을 수 있습니다.

```js
await expect(page.getByText('Welcome, John')).toBeVisible();
```

정확한 일치를 설정합니다.

```js
await expect(page.getByText('Welcome, John', { exact: true })).toBeVisible();
```

정규식과 일치시킵니다.

```js
await expect(page.getByText(/welcome, [A-Za-z]+$/i)).toBeVisible();
```

:::note
텍스트로 일치시키는 것은 정확히 일치하는 경우에도 항상 공백을 정규화합니다. 예를 들어 여러 공백을 하나로 바꾸고, 줄 바꿈을 공백으로 바꾸고, 선행 및 후행 공백을 무시합니다.
:::

:::note[텍스트 로케이터를 사용해야 하는 경우]
`div`, `span`, `p` 등과 같은 비대화형 요소를 찾기 위해 텍스트 로케이터를 사용하는 것이 좋습니다. `button`, `a`, `input` 등과 같은 대화형 요소의 경우 [역할 로케이터](#locate-by-role)를 사용하십시오.
:::

[텍스트로 필터링](#filter-by-text)할 수도 있습니다. 이는 목록에서 특정 항목을 찾으려고 할 때 유용할 수 있습니다.

### Locate by alt text

모든 이미지에는 이미지를 설명하는 `alt` 속성이 있어야 합니다. [page.getByAltText()](/api/class-page.mdx#page-get-by-alt-text)를 사용하여 텍스트 대체를 기반으로 이미지를 찾을 수 있습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<img alt="playwright logo" src="/img/playwright-logo.svg" width="100" />
</div>

```html
<img alt="playwright logo" src="/img/playwright-logo.svg" width="100" />
```
</HTMLCard>

텍스트 대체로 찾은 후 이미지를 클릭할 수 있습니다.

```js
await page.getByAltText('playwright logo').click();
```

:::note[alt 로케이터를 사용해야 하는 경우]
요소가 `img` 및 `area` 요소와 같은 alt 텍스트를 지원하는 경우 이 로케이터를 사용하십시오.
:::

### Locate by title

[page.getByTitle()](/api/class-page.mdx#page-get-by-title)을 사용하여 일치하는 title 속성을 가진 요소를 찾습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<span title='Issues count'>25 issues</span>
</div>

```html
<span title='Issues count'>25 issues</span>
```
</HTMLCard>

title 텍스트로 찾은 후 문제 수를 확인할 수 있습니다.

```js
await expect(page.getByTitle('Issues count')).toHaveText('25 issues');
```

:::note[title 로케이터를 사용해야 하는 경우]
요소에 `title` 속성이 있는 경우 이 로케이터를 사용하십시오.
:::

### Locate by test id

테스트 ID로 테스트하는 것은 텍스트 또는 속성의 역할이 변경되더라도 테스트가 계속 통과되므로 가장 복원력이 뛰어난 테스트 방법입니다. QA와 개발자는 명시적 테스트 ID를 정의하고 [page.getByTestId()](/api/class-page.mdx#page-get-by-test-id)로 쿼리해야 합니다. 그러나 테스트 ID로 테스트하는 것은 사용자에게 표시되지 않습니다. 역할 또는 텍스트 값이 중요한 경우 [역할](#locate-by-role) 및 [텍스트 로케이터](#locate-by-text)와 같은 사용자에게 보이는 로케이터를 사용하는 것이 좋습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<button data-testid="directions">Itinéraire</button>
</div>

```html
<button data-testid="directions">Itinéraire</button>
```
</HTMLCard>

테스트 ID로 요소를 찾을 수 있습니다.

```js
await page.getByTestId('directions').click();
```

:::note[testid 로케이터를 사용해야 하는 경우]
테스트 ID 방법론을 사용하기로 선택했거나 [역할](#locate-by-role) 또는 [텍스트](#locate-by-text)로 찾을 수 없는 경우 테스트 ID를 사용할 수도 있습니다.
:::

#### Set a custom test id attribute

기본적으로 [page.getByTestId()](/api/class-page.mdx#page-get-by-test-id)는 `data-testid` 속성을 기반으로 요소를 찾지만 테스트 구성에서 또는 [selectors.setTestIdAttribute()](/api/class-selectors.mdx#selectors-set-test-id-attribute)를 호출하여 구성할 수 있습니다.

테스트에 사용자 정의 데이터 속성을 사용하도록 테스트 ID를 설정합니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    testIdAttribute: 'data-pw'
  }
});
```

이제 html에서 기본값인 `data-testid` 대신 `data-pw`를 테스트 ID로 사용할 수 있습니다.

<HTMLCard>
<div>
<button data-pw="directions">Itinéraire</button>
</div>

```html
<button data-pw="directions">Itinéraire</button>
```
</HTMLCard>

그런 다음 일반적으로 하는 것처럼 요소를 찾습니다.

```js
await page.getByTestId('directions').click();
```

### Locate by CSS or XPath

CSS 또는 XPath 로케이터를 사용해야 하는 경우 [page.locator()](/api/class-page.mdx#page-locator)를 사용하여 페이지에서 요소를 찾는 방법을 설명하는 선택기를 사용하는 로케이터를 만들 수 있습니다. Playwright는 CSS 및 XPath 선택기를 지원하며 `css=` 또는 `xpath=` 접두사를 생략하면 자동으로 감지합니다.

```js
await page.locator('css=button').click();
await page.locator('xpath=//button').click();

await page.locator('button').click();
await page.locator('//button').click();
```

XPath 및 CSS 선택기는 DOM 구조 또는 구현에 연결될 수 있습니다. 이러한 선택기는 DOM 구조가 변경되면 깨질 수 있습니다. 아래의 긴 CSS 또는 XPath 체인은 불안정한 테스트로 이어지는 **나쁜 습관**의 예입니다.

```js
await page.locator(
    '#tsf > div:nth-child(2) > div.A8SBwf > div.RNNXgb > div > div.a4bIc > input'
).click();

await page
    .locator('//*[@id="tsf"]/div[2]/div[1]/div[1]/div/div[2]/input')
    .click();
```

:::note[이것을 사용해야 하는 경우]
DOM이 자주 변경되어 탄력성이 없는 테스트가 발생할 수 있으므로 CSS 및 XPath는 권장되지 않습니다. 대신 [역할 로케이터](#locate-by-role)와 같이 사용자가 페이지를 인식하는 방식에 가까운 로케이터를 구상하거나 테스트 ID를 사용하여 [명시적 테스트 계약을 정의하십시오](#locate-by-test-id).
:::

## Locate in Shadow DOM

Playwright의 모든 로케이터는 **기본적으로** Shadow DOM의 요소와 함께 작동합니다. 예외는 다음과 같습니다.
- XPath로 찾기는 섀도우 루트를 통과하지 않습니다.
- [Closed-mode shadow roots](https://developer.mozilla.org/en-US/docs/Web/API/Element/attachShadow#parameters)는 지원되지 않습니다.

사용자 정의 웹 구성 요소가 있는 다음 예제를 고려하십시오.

```html
<x-details role=button aria-expanded=true aria-controls=inner-details>
  <div>Title</div>
  #shadow-root
    <div id=inner-details>Details</div>
</x-details>
```

섀도우 루트가 전혀 없는 것처럼 같은 방식으로 찾을 수 있습니다.

`<div>Details</div>`를 클릭하려면:

```js
await page.getByText('Details').click();
```

```html
<x-details role=button aria-expanded=true aria-controls=inner-details>
  <div>Title</div>
  #shadow-root
    <div id=inner-details>Details</div>
</x-details>
```

`<x-details>`를 클릭하려면:

```js
await page.locator('x-details', { hasText: 'Details' }).click();
```

```html
<x-details role=button aria-expanded=true aria-controls=inner-details>
  <div>Title</div>
  #shadow-root
    <div id=inner-details>Details</div>
</x-details>
```

`<x-details>`에 "Details" 텍스트가 포함되어 있는지 확인하려면:

```js
await expect(page.locator('x-details')).toContainText('Details');
```

## Filtering Locators

두 번째 제품 카트의 구매 버튼을 클릭하려는 다음 DOM 구조를 고려하십시오. 올바른 것을 얻기 위해 로케이터를 필터링하는 몇 가지 옵션이 있습니다.

<HTMLCard>
<div>
<ul>
  <li>
    <h3>Product 1</h3>
    <button>Add to cart</button>
  </li>
  <li>
    <h3>Product 2</h3>
    <button>Add to cart</button>
  </li>
</ul>
</div>

```html
<ul>
  <li>
    <h3>Product 1</h3>
    <button>Add to cart</button>
  </li>
  <li>
    <h3>Product 2</h3>
    <button>Add to cart</button>
  </li>
</ul>
```
</HTMLCard>

### Filter by text

로케이터는 [locator.filter()](/api/class-locator.mdx#locator-filter) 메서드를 사용하여 텍스트로 필터링할 수 있습니다. 요소 내 어딘가에(아마도 하위 요소에) 특정 문자열을 대소문자를 구분하지 않고 검색합니다. 정규식을 전달할 수도 있습니다.

```js
await page
    .getByRole('listitem')
    .filter({ hasText: 'Product 2' })
    .getByRole('button', { name: 'Add to cart' })
    .click();
```

정규식을 사용합니다.

```js
await page
    .getByRole('listitem')
    .filter({ hasText: /Product 2/ })
    .getByRole('button', { name: 'Add to cart' })
    .click();
```

### Filter by not having text

또는 텍스트가 **없는** 것으로 필터링합니다.

```js
// 5 in-stock items
await expect(page.getByRole('listitem').filter({ hasNotText: 'Out of stock' })).toHaveCount(5);
```

### Filter by child/descendant

로케이터는 다른 로케이터와 일치하는 하위 항목 또는 하위 항목이 있는 요소만 선택하거나 선택하지 않는 옵션을 지원합니다. 따라서 [locator.getByRole()](/api/class-locator.mdx#locator-get-by-role), [locator.getByTestId()](/api/class-locator.mdx#locator-get-by-test-id), [locator.getByText()](/api/class-locator.mdx#locator-get-by-text) 등과 같은 다른 로케이터로 필터링할 수 있습니다.

<HTMLCard>
<div>
<ul>
  <li>
    <h3>Product 1</h3>
    <button>Add to cart</button>
  </li>
  <li>
    <h3>Product 2</h3>
    <button>Add to cart</button>
  </li>
</ul>
</div>

```html
<ul>
  <li>
    <h3>Product 1</h3>
    <button>Add to cart</button>
  </li>
  <li>
    <h3>Product 2</h3>
    <button>Add to cart</button>
  </li>
</ul>
```
</HTMLCard>

```js
await page
    .getByRole('listitem')
    .filter({ has: page.getByRole('heading', { name: 'Product 2' }) })
    .getByRole('button', { name: 'Add to cart' })
    .click();
```

제품 카드도 어설션하여 하나만 있는지 확인할 수 있습니다.

```js
await expect(page
    .getByRole('listitem')
    .filter({ has: page.getByRole('heading', { name: 'Product 2' }) }))
    .toHaveCount(1);
```

필터링 로케이터는 원래 로케이터를 기준으로 **해야 하며** 문서 루트가 아닌 원래 로케이터 일치로 시작하여 쿼리됩니다. 따라서 필터링 로케이터가 원래 로케이터와 일치하는 `<li>` 목록 항목 외부의 `<ul>` 목록 요소에서 일치를 시작하므로 다음은 작동하지 않습니다.

```js
// ✖ WRONG
await expect(page
    .getByRole('listitem')
    .filter({ has: page.getByRole('list').getByText('Product 2') }))
    .toHaveCount(1);
```

### Filter by not having child/descendant

내부에 일치하는 요소가 **없는** 것으로 필터링할 수도 있습니다.

```js
await expect(page
    .getByRole('listitem')
    .filter({ hasNot: page.getByText('Product 2') }))
    .toHaveCount(1);
```

내부 로케이터는 문서 루트가 아닌 외부 로케이터에서 시작하여 일치합니다.

## Locator operators

### Matching inside a locator

[page.getByText()](/api/class-page.mdx#page-get-by-text) 또는 [locator.getByRole()](/api/class-locator.mdx#locator-get-by-role)과 같은 로케이터를 만드는 메서드를 연결하여 페이지의 특정 부분으로 검색 범위를 좁힐 수 있습니다.

이 예제에서는 먼저 해당 역할의 `listitem`을 찾아 제품이라는 로케이터를 만듭니다. 그런 다음 텍스트로 필터링합니다. 제품 로케이터를 다시 사용하여 버튼의 역할로 가져와 클릭한 다음 어설션을 사용하여 "Product 2"라는 텍스트가 있는 제품이 하나만 있는지 확인합니다.

```js
const product = page.getByRole('listitem').filter({ hasText: 'Product 2' });

await product.getByRole('button', { name: 'Add to cart' }).click();

await expect(product).toHaveCount(1);
```

두 개의 로케이터를 함께 연결하여 예를 들어 특정 대화 상자 안에 있는 "Save" 버튼을 찾을 수도 있습니다.

```js
const saveButton = page.getByRole('button', { name: 'Save' });
// ...
const dialog = page.getByTestId('settings-dialog');
await dialog.locator(saveButton).click();
```

### Matching two locators simultaneously

[locator.and()](/api/class-locator.mdx#locator-and) 메서드는 추가 로케이터를 일치시켜 기존 로케이터를 좁힙니다. 예를 들어 [page.getByRole()](/api/class-page.mdx#page-get-by-role)과 [page.getByTitle()](/api/class-page.mdx#page-get-by-title)을 결합하여 역할과 title로 모두 일치시킬 수 있습니다.

```js
const button = page.getByRole('button').and(page.getByTitle('Subscribe'));
```

### Matching one of the two alternative locators

둘 이상의 요소를 대상으로 지정하고 어떤 요소인지 모르는 경우 [locator.or()](/api/class-locator.mdx#locator-or)를 사용하여 대안 중 하나 또는 둘 다와 일치하는 로케이터를 만듭니다.

예를 들어 "New email" 버튼을 클릭하고 싶지만 보안 설정 대화 상자가 대신 표시되는 시나리오를 생각해 보십시오. 이 경우 "New email" 버튼 또는 대화 상자를 기다리고 그에 따라 행동할 수 있습니다.

:::note

"New email" 버튼과 보안 대화 상자가 모두 화면에 나타나면 "or" 로케이터는 둘 다 일치하여 ["엄격 모드 위반" 오류](#strictness)가 발생할 수 있습니다. 이 경우 [locator.first()](/api/class-locator.mdx#locator-first)를 사용하여 둘 중 하나만 일치시킬 수 있습니다.
:::

```js
const newEmail = page.getByRole('button', { name: 'New' });
const dialog = page.getByText('Confirm security settings');
await expect(newEmail.or(dialog).first()).toBeVisible();
if (await dialog.isVisible())
  await page.getByRole('button', { name: 'Dismiss' }).click();
await newEmail.click();
```

### Matching only visible elements

:::note

일반적으로 가시성을 확인하는 대신 요소를 고유하게 식별하는 [더 안정적인 방법](./locators.mdx#quick-guide)을 찾는 것이 좋습니다.
:::

보이지 않는 첫 번째 버튼과 [표시되는](./actionability.mdx#visible) 두 번째 버튼이 있는 페이지를 생각해 보십시오.

```html
<button style='display: none'>Invisible</button>
<button>Visible</button>
```

* 이렇게 하면 두 버튼이 모두 발견되고 [엄격](./locators.mdx#strictness) 위반 오류가 발생합니다.
  
  ```js
  await page.locator('button').click();
  ```
  
* 이렇게 하면 표시되기 때문에 두 번째 버튼만 찾은 다음 클릭합니다.
  
  ```js
  await page.locator('button').filter({ visible: true }).click();
  ```
  
## Lists

### Count items in a list

목록의 항목 수를 세기 위해 로케이터를 어설션할 수 있습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<ul>
  <li>apple</li>
  <li>banana</li>
  <li>orange</li>
</ul>
</div>

```html
<ul>
  <li>apple</li>
  <li>banana</li>
  <li>orange</li>
</ul>
```
</HTMLCard>

목록에 3개의 항목이 있는지 확인하려면 count 어설션을 사용하십시오.

```js
await expect(page.getByRole('listitem')).toHaveCount(3);
```

### Assert all text in a list

목록에서 모든 텍스트를 찾기 위해 로케이터를 어설션할 수 있습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<ul>
  <li>apple</li>
  <li>banana</li>
  <li>orange</li>
</ul>
</div>

```html
<ul>
  <li>apple</li>
  <li>banana</li>
  <li>orange</li>
</ul>
```
</HTMLCard>

목록에 "apple", "banana" 및 "orange" 텍스트가 있는지 확인하려면 [expect(locator).toHaveText()](/api/class-locatorassertions.mdx#locator-assertions-to-have-text)를 사용하십시오.

```js
await expect(page
    .getByRole('listitem'))
    .toHaveText(['apple', 'banana', 'orange']);
```

### Get a specific item

목록에서 특정 항목을 가져오는 방법에는 여러 가지가 있습니다.

#### Get by text

[page.getByText()](/api/class-page.mdx#page-get-by-text) 메서드를 사용하여 텍스트 콘텐츠로 목록에서 요소를 찾은 다음 클릭합니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<ul>
  <li>apple</li>
  <li>banana</li>
  <li>orange</li>
</ul>
</div>

```html
<ul>
  <li>apple</li>
  <li>banana</li>
  <li>orange</li>
</ul>
```
</HTMLCard>

텍스트 콘텐츠로 항목을 찾아 클릭합니다.

```js
await page.getByText('orange').click();
```

#### Filter by text

[locator.filter()](/api/class-locator.mdx#locator-filter)를 사용하여 목록에서 특정 항목을 찾습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<ul>
  <li>apple</li>
  <li>banana</li>
  <li>orange</li>
</ul>
</div>

```html
<ul>
  <li>apple</li>
  <li>banana</li>
  <li>orange</li>
</ul>
```
</HTMLCard>

"listitem" 역할로 항목을 찾은 다음 "orange" 텍스트로 필터링한 다음 클릭합니다.

```js
await page
    .getByRole('listitem')
    .filter({ hasText: 'orange' })
    .click();
```

#### Get by test id

[page.getByTestId()](/api/class-page.mdx#page-get-by-test-id) 메서드를 사용하여 목록에서 요소를 찾습니다. 아직 테스트 ID가 없는 경우 html을 수정하고 테스트 ID를 추가해야 할 수 있습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<ul>
  <li data-testid='apple'>apple</li>
  <li data-testid='banana'>banana</li>
  <li data-testid='orange'>orange</li>
</ul>
</div>

```html
<ul>
  <li data-testid='apple'>apple</li>
  <li data-testid='banana'>banana</li>
  <li data-testid='orange'>orange</li>
</ul>
```
</HTMLCard>

테스트 ID "orange"로 항목을 찾아 클릭합니다.

```js
await page.getByTestId('orange').click();
```

#### Get by nth item

동일한 요소 목록이 있고 요소 간을 구분하는 유일한 방법이 순서인 경우 [locator.first()](/api/class-locator.mdx#locator-first), [locator.last()](/api/class-locator.mdx#locator-last) 또는 [locator.nth()](/api/class-locator.mdx#locator-nth)를 사용하여 목록에서 특정 요소를 선택할 수 있습니다.

```js
const banana = await page.getByRole('listitem').nth(1);
```

그러나 이 방법은 주의해서 사용하십시오. 페이지가 변경되어 로케이터가 예상한 요소와 완전히 다른 요소를 가리키는 경우가 많습니다. 대신 [엄격 기준](#strictness)을 통과하는 고유한 로케이터를 구상해 보십시오.

### Chaining filters

다양한 유사점을 가진 요소가 있는 경우 [locator.filter()](/api/class-locator.mdx#locator-filter) 메서드를 사용하여 올바른 요소를 선택할 수 있습니다. 여러 필터를 연결하여 선택 범위를 좁힐 수도 있습니다.

예를 들어 다음 DOM 구조를 고려하십시오.

<HTMLCard>
<div>
<ul>
  <li>
    <div>John</div>
    <div><button>Say hello</button></div>
  </li>
  <li>
    <div>Mary</div>
    <div><button>Say hello</button></div>
  </li>
  <li>
    <div>John</div>
    <div><button>Say goodbye</button></div>
  </li>
  <li>
    <div>Mary</div>
    <div><button>Say goodbye</button></div>
  </li>
</ul>
</div>

```html
<ul>
  <li>
    <div>John</div>
    <div><button>Say hello</button></div>
  </li>
  <li>
    <div>Mary</div>
    <div><button>Say hello</button></div>
  </li>
  <li>
    <div>John</div>
    <div><button>Say goodbye</button></div>
  </li>
  <li>
    <div>Mary</div>
    <div><button>Say goodbye</button></div>
  </li>
</ul>
```
</HTMLCard>

"Mary"와 "Say goodbye"가 있는 행의 스크린샷을 찍으려면:

```js
const rowLocator = page.getByRole('listitem');

await rowLocator
    .filter({ hasText: 'Mary' })
    .filter({ has: page.getByRole('button', { name: 'Say goodbye' }) })
    .screenshot({ path: 'screenshot.png' });
```

이제 프로젝트의 루트 디렉터리에 "screenshot.png" 파일이 있어야 합니다.

### Rare use cases

#### Do something with each element in the list

요소 반복:

```js
for (const row of await page.getByRole('listitem').all())
  console.log(await row.textContent());
```

일반 for 루프를 사용하여 반복:

```js
const rows = page.getByRole('listitem');
const count = await rows.count();
for (let i = 0; i < count; ++i)
  console.log(await rows.nth(i).textContent());
```

#### Evaluate in the page

[locator.evaluateAll()](/api/class-locator.mdx#locator-evaluate-all) 내부의 코드는 페이지에서 실행되므로 모든 DOM API를 호출할 수 있습니다.

```js
const rows = page.getByRole('listitem');
const texts = await rows.evaluateAll(
    list => list.map(element => element.textContent));
```

## Strictness

로케이터는 엄격합니다. 즉, 일부 대상 DOM 요소를 의미하는 로케이터에 대한 모든 작업은 둘 이상의 요소가 일치하는 경우 예외를 발생시킵니다. 예를 들어 다음 호출은 DOM에 여러 버튼이 있는 경우 예외를 발생시킵니다.

#### Throws an error if more than one

```js
await page.getByRole('button').click();
```

반면에 Playwright는 여러 요소 작업을 수행할 때 이해하므로 로케이터가 여러 요소로 확인될 때 다음 호출은 완벽하게 작동합니다.

#### Works fine with multiple elements

```js
await page.getByRole('button').count();
```

[locator.first()](/api/class-locator.mdx#locator-first), [locator.last()](/api/class-locator.mdx#locator-last) 및 [locator.nth()](/api/class-locator.mdx#locator-nth)를 통해 여러 요소가 일치할 때 사용할 요소를 Playwright에 알려 엄격도 검사에서 명시적으로 선택 해제할 수 있습니다. 이러한 방법은 페이지가 변경될 때 Playwright가 의도하지 않은 요소를 클릭할 수 있으므로 **권장되지 않습니다**. 대신 위의 모범 사례에 따라 대상 요소를 고유하게 식별하는 로케이터를 만드십시오.
