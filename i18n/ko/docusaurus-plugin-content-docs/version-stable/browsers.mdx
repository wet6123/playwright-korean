## 소개

각 Playwright 버전은 작동하기 위해 특정 버전의 브라우저 바이너리가 필요합니다. Playwright CLI를 사용하여 이러한 브라우저를 설치해야 합니다.

Playwright는 릴리스마다 지원하는 브라우저 버전을 업데이트하므로, 최신 Playwright는 언제나 최신 브라우저를 지원합니다. 이는 Playwright를 업데이트할 때마다 `install` CLI 명령을 다시 실행해야 할 수도 있음을 의미합니다.

## 브라우저 설치

Playwright는 지원되는 브라우저를 설치할 수 있습니다. 인자 없이 명령을 실행하면 기본 브라우저가 설치됩니다.

```bash
npx playwright install
```

인자를 제공하여 특정 브라우저를 설치할 수도 있습니다:

```bash
npx playwright install webkit
```

지원되는 모든 브라우저 보기:

```bash
npx playwright install --help
```

## 시스템 의존성 설치

시스템 의존성은 자동으로 설치될 수 있습니다. 이는 CI 환경에 유용합니다.

```bash
npx playwright install-deps
```

인수로 전달하여 단일 브라우저에 대한 의존성을 설치할 수도 있습니다:

```bash
npx playwright install-deps chromium
```

`install-deps`와 `install`을 결합하여 브라우저와 OS 의존성을 단일 명령으로 설치하는 것도 가능합니다.

```bash
npx playwright install --with-deps chromium
```

공식적으로 지원되는 운영 체제는 [시스템 요구 사항](./intro.mdx#system-requirements)을 참조하세요.

## Playwright 정기적 업데이트

Playwright 버전을 최신 상태로 유지하면 새로운 기능을 사용하고 최신 브라우저 버전에서 앱을 테스트하며, 최신 브라우저 버전이 일반에 공개되기 전에 오류를 발견할 수 있습니다.

```bash
# Playwright 업데이트
npm install -D @playwright/test@latest

# 새 브라우저 설치
npx playwright install
```

최신 버전과 릴리스된 변경 사항을 확인하려면 [릴리스 노트](./release-notes.mdx)를 확인하세요.

```bash
# 다음 명령을 실행하여 사용 중인 Playwright 버전 확인
npx playwright --version
```

## 브라우저 구성

Playwright는 Chromium, WebKit, Firefox 브라우저뿐만 아니라 Google Chrome, Microsoft Edge와 같은 브랜드 브라우저에서도 테스트를 실행할 수 있습니다. 또한 에뮬레이션된 태블릿 및 모바일 장치에서도 실행할 수 있습니다. 선택된 데스크톱, 태블릿 및 모바일 장치의 전체 목록은 [장치 매개변수 레지스트리](https://github.com/microsoft/playwright/blob/main/packages/playwright-core/src/server/deviceDescriptorsSource.json)를 참조하세요.

### 다른 브라우저에서 테스트 실행

Playwright는 설정 파일에서 **프로젝트**를 설정하여 여러 브라우저와 구성에서 테스트를 실행할 수 있습니다. 각 프로젝트에 [다른 옵션](./test-configuration)을 추가할 수도 있습니다.

```js
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  projects: [
    /* 데스크톱 브라우저 대상 테스트 */
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
    {
      name: 'firefox',
      use: { ...devices['Desktop Firefox'] },
    },
    {
      name: 'webkit',
      use: { ...devices['Desktop Safari'] },
    },
    /* 모바일 뷰포트 대상 테스트 */
    {
      name: 'Mobile Chrome',
      use: { ...devices['Pixel 5'] },
    },
    {
      name: 'Mobile Safari',
      use: { ...devices['iPhone 12'] },
    },
    /* 브랜드 브라우저 대상 테스트 */
    {
      name: 'Google Chrome',
      use: { ...devices['Desktop Chrome'], channel: 'chrome' }, // 또는 'chrome-beta'
    },
    {
      name: 'Microsoft Edge',
      use: { ...devices['Desktop Edge'], channel: 'msedge' }, // 또는 'msedge-dev'
    },
  ],
});
```

Playwright는 기본적으로 모든 프로젝트를 실행합니다.

```bash
npx playwright test

Running 7 tests using 5 workers

  ✓ [chromium] › example.spec.ts:3:1 › basic test (2s)
  ✓ [firefox] › example.spec.ts:3:1 › basic test (2s)
  ✓ [webkit] › example.spec.ts:3:1 › basic test (2s)
  ✓ [Mobile Chrome] › example.spec.ts:3:1 › basic test (2s)
  ✓ [Mobile Safari] › example.spec.ts:3:1 › basic test (2s)
  ✓ [Google Chrome] › example.spec.ts:3:1 › basic test (2s)
  ✓ [Microsoft Edge] › example.spec.ts:3:1 › basic test (2s)
```

`--project` 명령줄 옵션을 사용하여 단일 프로젝트를 실행합니다.

```bash
npx playwright test --project=firefox

Running 1 test using 1 worker

  ✓ [firefox] › example.spec.ts:3:1 › basic test (2s)
```

VS Code 확장 프로그램을 사용하면 Playwright 사이드바에서 브라우저 이름 옆의 확인란을 선택하여 다른 브라우저에서 테스트를 실행할 수 있습니다. 이 이름들은 Playwright 설정 파일의 projects 섹션에 정의되어 있습니다. Playwright를 설치할 때의 기본 설정은 Chromium, Firefox, WebKit의 3개 프로젝트를 제공합니다. 첫 번째 프로젝트가 기본적으로 선택됩니다.

여러 프로젝트(브라우저)에서 테스트를 실행하려면 프로젝트 이름 옆의 확인란을 선택하여 각 프로젝트를 선택합니다.

![테스트를 실행할 프로젝트 선택](https://github.com/microsoft/playwright/assets/13063165/6dc86ef4-6097-481c-9cab-b6e053ec7ea6)

### Chromium

Google Chrome, Microsoft Edge 및 기타 Chromium 기반 브라우저의 경우, 기본적으로 Playwright는 오픈 소스 Chromium 빌드를 사용합니다. Chromium 프로젝트는 브랜드 브라우저보다 앞서 나가므로, 전 세계적으로 Google Chrome N 버전을 사용할 때 Playwright는 이미 몇 주 후에 Google Chrome 및 Microsoft Edge에서 출시될 Chromium N+1 버전을 지원합니다.

### Chromium: 헤드리스 셸

Playwright는 헤드(headed) 모드 작동을 위한 일반 Chromium 빌드와 헤드리스(headless) 모드를 위한 별도의 [chromium 헤드리스 셸](https://developer.chrome.com/blog/chrome-headless-shell)을 제공합니다.

헤드리스 셸에서만 테스트를 실행하는 경우(즉, `channel` 옵션이 지정되지 **않은** 경우), 예를 들어 CI 환경에서는 설치 중 `--only-shell`을 전달하여 전체 Chromium 브라우저 다운로드를 피할 수 있습니다.

```bash
# 헤드리스 모드로만 테스트 실행
npx playwright install --with-deps --only-shell
```

### Chromium: 새로운 헤드리스 모드

`'chromium'` 채널을 사용하여 새로운 헤드리스 모드를 선택할 수 있습니다. [공식 Chrome 문서](https://developer.chrome.com/blog/chrome-headless-shell)에 따르면:

> 반면 새로운 헤드리스는 실제 Chrome 브라우저이므로 더 정통하고 신뢰할 수 있으며 더 많은 기능을 제공합니다. 따라서 고정밀 엔드 투 엔드 웹 앱 테스트 또는 브라우저 확장 프로그램 테스트에 더 적합합니다.

자세한 내용은 [이슈 #33566](https://github.com/microsoft/playwright/issues/33566)을 참조하세요.

```js
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'], channel: 'chromium' },
    },
  ],
});
```

새로운 헤드리스 모드를 사용하면 `--no-shell` 옵션을 사용하여 브라우저 설치 중 헤드리스 셸 다운로드를 건너뛸 수 있습니다:

```bash
# 헤드리스 모드로만 테스트 실행
npx playwright install --with-deps --no-shell
```

### Google Chrome & Microsoft Edge

Playwright는 최신 Chromium 빌드를 다운로드하여 사용할 수 있지만, 컴퓨터에 설치된 브랜드 Google Chrome 및 Microsoft Edge 브라우저를 대상으로 작동할 수도 있습니다 (Playwright는 기본적으로 이들을 설치하지 않음). 특히 현재 Playwright 버전은 이러한 브라우저의 Stable 및 Beta 채널을 지원합니다.

사용 가능한 채널은 `chrome`, `msedge`, `chrome-beta`, `msedge-beta`, `chrome-dev`, `msedge-dev`, `chrome-canary`, `msedge-canary`입니다.

:::warning

특정 엔터프라이즈 브라우저 정책은 Playwright가 Google Chrome 및 Microsoft Edge를 실행하고 제어하는 기능에 영향을 줄 수 있습니다. 브라우저 정책이 있는 환경에서 실행하는 것은 Playwright 프로젝트의 범위를 벗어납니다.
:::

:::warning
Google Chrome과 Microsoft Edge는 일반 헤드 모드에 더 가까운 [새로운 헤드리스 모드](https://developer.chrome.com/docs/chromium/headless) 구현으로 전환했습니다. 이는 Playwright가 헤드리스로 실행될 때 기본적으로 사용하는 [chromium 헤드리스 셸](https://developer.chrome.com/blog/chrome-headless-shell)과 다르므로 경우에 따라 다른 동작을 예상해야 합니다. 자세한 내용은 [이슈 #33566](https://github.com/microsoft/playwright/issues/33566)을 참조하세요.
:::

```js
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  projects: [
    /* 브랜드 브라우저 대상 테스트 */
    {
      name: 'Google Chrome',
      use: { ...devices['Desktop Chrome'], channel: 'chrome' }, // 또는 'chrome-beta'
    },
    {
      name: 'Microsoft Edge',
      use: { ...devices['Desktop Edge'], channel: 'msedge' }, // 또는 "msedge-beta" 또는 'msedge-dev'
    },
  ],
});
```

#### Google Chrome & Microsoft Edge 설치

컴퓨터에 Google Chrome 또는 Microsoft Edge가 없는 경우 Playwright 명령줄 도구를 사용하여 설치할 수 있습니다:

```bash
npx playwright install msedge
```

:::warning
Google Chrome 또는 Microsoft Edge 설치는 운영 체제의 기본 전역 위치에 설치되어 현재 브라우저 설치를 덮어씁니다.
:::

`--help` 옵션으로 실행하여 설치할 수 있는 브라우저의 전체 목록을 확인하세요.

#### Google Chrome & Microsoft Edge 사용 시점과 사용하지 않아야 할 시점

##### 기본값

최신 Chromium을 사용하는 기본 Playwright 구성을 사용하는 것이 대부분의 경우 좋은 생각입니다. Playwright는 브라우저의 Stable 채널보다 앞서 있기 때문에, 향후 Google Chrome 또는 Microsoft Edge 릴리스가 사이트를 손상시키지 않을 것이라는 안심감을 줍니다. 문제를 조기에 발견하고 공식 Chrome 업데이트 전에 수정할 시간이 충분합니다.

##### 회귀 테스트

그렇긴 하지만, 테스트 정책은 종종 현재 공개적으로 사용 가능한 브라우저에 대해 회귀 테스트를 수행하도록 요구합니다. 이 경우 안정적인 채널 중 하나인 `"chrome"` 또는 `"msedge"`를 선택할 수 있습니다.

##### 미디어 코덱

공식 바이너리를 사용하여 테스트하는 또 다른 이유는 미디어 코덱과 관련된 기능을 테스트하기 위함입니다. Chromium에는 다양한 라이선스 고려 사항 및 계약으로 인해 Google Chrome 또는 Microsoft Edge가 번들로 제공하는 모든 코덱이 포함되어 있지 않습니다. 사이트가 이러한 종류의 코덱에 의존하는 경우(드문 경우지만), 공식 채널을 사용해야 합니다.

##### 엔터프라이즈 정책

Google Chrome과 Microsoft Edge는 엔터프라이즈 정책을 준수하며, 여기에는 기능 제한, 네트워크 프록시, 테스트를 방해하는 필수 확장 프로그램 등이 포함됩니다. 따라서 이러한 정책을 사용하는 조직의 일원이라면 로컬 테스트에는 번들된 Chromium을 사용하는 것이 가장 쉽습니다. 일반적으로 이러한 제한이 없는 봇에서는 여전히 안정적인 채널을 선택할 수 있습니다.

### Firefox

Playwright의 Firefox 버전은 최신 [Firefox Stable](https://www.mozilla.org/en-US/firefox/new/) 빌드와 일치합니다. Playwright는 패치에 의존하기 때문에 브랜드 버전의 Firefox에서는 작동하지 않습니다.

기본 플랫폼에 크게 의존하는 특정 기능의 가용성은 운영 체제마다 다를 수 있습니다. 예를 들어, 사용 가능한 미디어 코덱은 Linux, macOS 및 Windows 간에 상당히 다릅니다.

### WebKit

Playwright의 WebKit은 최신 WebKit 메인 브랜치 소스에서 파생되며, 종종 이러한 업데이트가 Apple Safari 및 기타 WebKit 기반 브라우저에 통합되기 전에 이루어집니다. 이는 잠재적인 브라우저 업데이트 문제에 대응할 수 있는 충분한 시간을 제공합니다. Playwright는 패치에 의존하기 때문에 브랜드 버전의 Safari에서는 작동하지 않습니다. 대신 최신 WebKit 빌드를 사용하여 테스트할 수 있습니다.

기본 플랫폼에 크게 의존하는 특정 기능의 가용성은 운영 체제마다 다를 수 있습니다. 예를 들어, 사용 가능한 미디어 코덱은 Linux, macOS 및 Windows 간에 상당히 다릅니다. Linux CI에서 WebKit을 실행하는 것이 일반적으로 가장 저렴한 옵션이지만, Safari에 가장 가까운 경험을 원한다면 macOS에서 WebKit을 실행해야 합니다(예: 비디오 재생 시).

## 방화벽 또는 프록시 뒤에서 설치

기본적으로 Playwright는 Microsoft의 CDN에서 브라우저를 다운로드합니다.

때로는 회사에서 공용 리소스에 대한 직접적인 액세스를 차단하는 내부 프록시를 유지 관리합니다. 이 경우 프록시 서버를 통해 브라우저를 다운로드하도록 Playwright를 구성할 수 있습니다.

## 브라우저 바이너리 관리

Playwright는 Chromium, WebKit, Firefox 브라우저를 OS별 캐시 폴더에 다운로드합니다:
- Windows: `%USERPROFILE%\AppData\Local\ms-playwright`
- macOS: `~/Library/Caches/ms-playwright`
- Linux: `~/.cache/ms-playwright`

이 브라우저들은 설치 시 수백 메가바이트의 디스크 공간을 차지합니다:

```bash
du -hs ~/Library/Caches/ms-playwright/*
281M  chromium-XXXXXX
187M  firefox-XXXX
180M  webkit-XXXX
```

환경 변수를 사용하여 기본 동작을 재정의할 수 있습니다. Playwright를 설치할 때 특정 위치에 브라우저를 다운로드하도록 요청하세요:

<Tabs

    {label: 'Batch', value: 'batch'}


</TabItem>

Playwright 스크립트를 실행할 때 공유 위치에서 브라우저를 검색하도록 요청하세요.

### 오래된 브라우저 제거

Playwright는 해당 브라우저를 사용하는 클라이언트를 추적합니다. 특정 버전의 브라우저를 요구하는 클라이언트가 더 이상 없으면 해당 버전은 시스템에서 삭제됩니다. 이렇게 하면 다른 버전의 Playwright 인스턴스를 안전하게 사용하면서 동시에 더 이상 사용되지 않는 브라우저를 위해 디스크 공간을 낭비하지 않을 수 있습니다.

사용되지 않는 브라우저 제거를 옵트아웃하려면 `PLAYWRIGHT_SKIP_BROWSER_GC=1` 환경 변수를 설정하면 됩니다.

### 브라우저 제거

이렇게 하면 현재 Playwright 설치의 브라우저(chromium, firefox, webkit)가 제거됩니다:

```bash
npx playwright uninstall
```

다른 Playwright 설치의 브라우저도 제거하려면 `--all` 플래그를 전달하세요:

```bash
npx playwright uninstall --all
```
