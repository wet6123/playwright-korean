## Introduction

[Dockerfile.noble]은 Docker 환경에서 Playwright 스크립트를 실행하는 데 사용할 수 있습니다. 이 이미지는 [Playwright 브라우저](./browsers.mdx#install-browsers) 및 [브라우저 시스템 종속성](./browsers.mdx#install-system-dependencies)을 포함합니다. Playwright 패키지/종속성은 이미지에 포함되어 있지 않으므로 별도로 설치해야 합니다.

## Usage

이 Docker 이미지는 [Microsoft Artifact Registry]에 게시됩니다.

:::info

이 Docker 이미지는 테스트 및 개발 목적으로만 사용하기 위한 것입니다. 이 Docker 이미지를 사용하여 신뢰할 수 없는 웹 사이트를 방문하는 것은 권장되지 않습니다.
:::

### Pull the image

```bash
docker pull mcr.microsoft.com/playwright:v1.51.1-noble
```

### Run the image

기본적으로 Docker 이미지는 `root` 사용자를 사용하여 브라우저를 실행합니다. 이렇게 하면 루트에서는 사용할 수 없는 Chromium 샌드박스가 비활성화됩니다. 신뢰할 수 있는 코드(예: 엔드 투 엔드 테스트)를 실행하고 별도의 사용자 관리에 대한 번거로움을 피하고 싶다면 루트 사용자가 적합할 수 있습니다. 웹 스크래핑 또는 크롤링의 경우 Docker 컨테이너 내부에 별도의 사용자를 만들고 seccomp 프로필을 사용하는 것이 좋습니다.

#### End-to-end tests

신뢰할 수 있는 웹 사이트에서는 별도의 사용자를 만들지 않고 루트를 사용할 수 있습니다. 브라우저에서 실행될 코드를 신뢰하기 때문입니다.

```bash
docker run -it --rm --ipc=host mcr.microsoft.com/playwright:v1.51.1-noble /bin/bash
```

#### Crawling and scraping

신뢰할 수 없는 웹 사이트에서는 seccomp 프로필과 함께 브라우저를 시작하는 데 별도의 사용자를 사용하는 것이 좋습니다. 컨테이너 내부에서 또는 Docker 이미지를 기본 이미지로 사용하는 경우 `adduser`를 사용해야 합니다.

```bash
docker run -it --rm --ipc=host --user pwuser --security-opt seccomp=seccomp_profile.json mcr.microsoft.com/playwright:v1.51.1-noble /bin/bash
```

[`seccomp_profile.json`](https://github.com/microsoft/playwright/blob/main/utils/docker/seccomp_profile.json)은 샌드박스와 함께 Chromium을 실행하는 데 필요합니다. 이것은 추가 사용자 네임스페이스 복제 권한이 있는 [기본 Docker seccomp 프로필](https://github.com/docker/engine/blob/d0d99b04cf6e00ed3fc27e81fc3d94e7eda70af3/profiles/seccomp/default.json)입니다.

```json
{
  "comment": "Allow create user namespaces",
  "names": [
    "clone",
    "setns",
    "unshare"
  ],
  "action": "SCMP_ACT_ALLOW",
  "args": [],
  "includes": {},
  "excludes": {}
}
```

:::note
Chrome을 사용할 때는 `--ipc=host`를 사용하는 것이 좋습니다([Docker docs](https://docs.docker.com/engine/reference/run/#ipc-settings---ipc)). 이 플래그가 없으면 Chrome의 메모리가 부족할 수 있습니다.
:::

### Using on CI

샘플 구성은 [CI (지속적 통합) 가이드](./ci.mdx)를 참조하십시오.

### Remote Connection

호스트 시스템 또는 다른 시스템에서 테스트를 실행하는 동안 Docker에서 Playwright 서버를 실행할 수 있습니다. 이는 지원되지 않는 Linux 배포판 또는 원격 실행 시나리오에서 테스트를 실행하는 데 유용합니다.

#### Running the Playwright Server

Docker에서 Playwright 서버를 시작합니다.

```bash
docker run -p 3000:3000 --rm --init -it --workdir /home/pwuser --user pwuser mcr.microsoft.com/playwright:v1.51.1-noble /bin/sh -c "npx -y playwright@1.51.1 run-server --port 3000 --host 0.0.0.0"
```

#### Connecting to the Server

원격 Playwright 서버에 연결하는 방법은 두 가지가 있습니다.
1. `@playwright/test`와 함께 환경 변수 사용:

```bash
PW_TEST_CONNECT_WS_ENDPOINT=ws://127.0.0.1:3000/ npx playwright test
```

2. 다른 애플리케이션의 경우 [browserType.connect()](/api/class-browsertype.mdx#browser-type-connect) API 사용:

```js
const browser = await playwright['chromium'].connect('ws://127.0.0.1:3000/');
```

#### Network Configuration

Docker 컨테이너 내에서 로컬 서버에 액세스해야 하는 경우:

```bash
docker run --add-host=hostmachine:host-gateway -p 3000:3000 --rm --init -it --workdir /home/pwuser --user pwuser mcr.microsoft.com/playwright:v1.51.1-noble /bin/sh -c "npx -y playwright@1.51.1 run-server --port 3000 --host 0.0.0.0"
```

이렇게 하면 `hostmachine`이 호스트의 localhost를 가리키게 됩니다. 테스트에서 로컬 서버에 액세스할 때 `localhost` 대신 `hostmachine`을 사용해야 합니다.

:::note

원격으로 테스트를 실행할 때는 테스트의 Playwright 버전이 Docker 컨테이너에서 실행되는 버전과 일치하는지 확인하십시오.
:::

## Image tags

[사용 가능한 모든 이미지 태그]를 참조하십시오.

현재 다음과 같은 태그가 있는 이미지를 게시합니다.
- `:v1.51.1` - Ubuntu 24.04 LTS (Noble Numbat)를 기반으로 하는 Playwright v1.51.1 릴리스 Docker 이미지.
- `:v1.51.1-noble` - Ubuntu 24.04 LTS (Noble Numbat)를 기반으로 하는 Playwright v1.51.1 릴리스 Docker 이미지.
- `:v1.51.1-jammy` - Ubuntu 22.04 LTS (Jammy Jellyfish)를 기반으로 하는 Playwright v1.51.1 릴리스 Docker 이미지.

:::note

가능하면 항상 Docker 이미지를 특정 버전에 고정하는 것이 좋습니다. Docker 이미지의 Playwright 버전이 프로젝트/테스트의 버전과 일치하지 않으면 Playwright는 브라우저 실행 파일을 찾을 수 없습니다.
:::

### Base images

현재 다음 [Ubuntu](https://hub.docker.com/_/ubuntu) 버전을 기반으로 이미지를 게시합니다.
- **Ubuntu 24.04 LTS** (Noble Numbat), 이미지 태그에 `noble` 포함
- **Ubuntu 22.04 LTS** (Jammy Jellyfish), 이미지 태그에 `jammy` 포함

#### Alpine

Firefox 및 WebKit용 브라우저 빌드는 [glibc](https://en.wikipedia.org/wiki/Glibc) 라이브러리용으로 빌드되었습니다. [musl](https://en.wikipedia.org/wiki/Musl) 표준 라이브러리를 기반으로 하는 Alpine Linux 및 기타 배포판은 지원되지 않습니다.

## Build your own image

Docker 내에서 Playwright를 실행하려면 Node.js, [Playwright 브라우저](./browsers.mdx#install-browsers) 및 [브라우저 시스템 종속성](./browsers.mdx#install-system-dependencies)이 설치되어 있어야 합니다. 다음 Dockerfile을 참조하십시오.

```Dockerfile
FROM node:20-bookworm

RUN npx -y playwright@1.51.1 install --with-deps
```
