## Introduction

페이지에서 다운로드되는 모든 첨부 파일에 대해 [page.on('download')](/api/class-page.mdx#page-event-download) 이벤트가 발생합니다. 이러한 모든 첨부 파일은 임시 폴더에 다운로드됩니다. 이벤트에서 [Download] 객체를 사용하여 다운로드 URL, 파일 이름 및 페이로드 스트림을 얻을 수 있습니다.

[browserType.launch()](/api/class-browsertype.mdx#browser-type-launch)에서 [downloadsPath](/api/class-browsertype.mdx#browser-type-launch-option-downloads-path) 옵션을 사용하여 다운로드된 파일을 유지할 위치를 지정할 수 있습니다.

:::note

다운로드된 파일은 해당 파일을 생성한 브라우저 컨텍스트가 닫힐 때 삭제됩니다.
:::

다음은 파일 다운로드를 처리하는 가장 간단한 방법입니다.

```js
// 클릭하기 전에 다운로드를 기다리기 시작합니다. await는 없습니다.
const downloadPromise = page.waitForEvent('download');
await page.getByText('Download file').click();
const download = await downloadPromise;

// 다운로드 프로세스가 완료될 때까지 기다렸다가 다운로드한 파일을 어딘가에 저장합니다.
await download.saveAs('/path/to/save/at/' + download.suggestedFilename());
```

#### Variations

다운로드를 시작하는 것이 무엇인지 모르는 경우에도 이벤트를 처리할 수 있습니다.

```js
page.on('download', download => download.path().then(console.log));
```

이벤트를 처리하면 제어 흐름이 분기되어 스크립트를 따르기가 더 어려워집니다. 기본 제어 흐름이 이 작업이 해결되기를 기다리지 않기 때문에 파일을 다운로드하는 동안 시나리오가 종료될 수 있습니다.

:::note

파일 업로드에 대해서는 [파일 업로드](./input.mdx#upload-files) 섹션을 참조하세요.
:::
