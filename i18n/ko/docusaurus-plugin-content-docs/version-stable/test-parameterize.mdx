## Introduction

테스트 수준 또는 프로젝트 수준에서 테스트를 파라미터화할 수 있습니다.

## Parameterized Tests

```js title="example.spec.ts"
[
  { name: 'Alice', expected: 'Hello, Alice!' },
  { name: 'Bob', expected: 'Hello, Bob!' },
  { name: 'Charlie', expected: 'Hello, Charlie!' },
].forEach(({ name, expected }) => {
  // test.describe() 또는 테스트 이름이 고유한 경우 여러 테스트를 사용하여 수행할 수도 있습니다.
  test(`testing with ${name}`, async ({ page }) => {
    await page.goto(`https://example.com/greet?name=${name}`);
    await expect(page.getByRole('heading')).toHaveText(expected);
  });
});
```

### Before and after hooks

대부분의 경우 `beforeEach`, `beforeAll`, `afterEach` 및 `afterAll` 훅을 `forEach` 외부에 배치하여 훅이 한 번만 실행되도록 해야 합니다.

```js title="example.spec.ts"
test.beforeEach(async ({ page }) => {
  // ...
});

test.afterEach(async ({ page }) => {
  // ...
});

[
  { name: 'Alice', expected: 'Hello, Alice!' },
  { name: 'Bob', expected: 'Hello, Bob!' },
  { name: 'Charlie', expected: 'Hello, Charlie!' },
].forEach(({ name, expected }) => {
  test(`testing with ${name}`, async ({ page }) => {
    await page.goto(`https://example.com/greet?name=${name}`);
    await expect(page.getByRole('heading')).toHaveText(expected);
  });
});
```

각 테스트에 대한 훅을 가지려면 `describe()` 내부에 배치하여 각 반복/개별 테스트에 대해 실행되도록 할 수 있습니다.

```js title="example.spec.ts"
[
  { name: 'Alice', expected: 'Hello, Alice!' },
  { name: 'Bob', expected: 'Hello, Bob!' },
  { name: 'Charlie', expected: 'Hello, Charlie!' },
].forEach(({ name, expected }) => {
  test.describe(() => {
    test.beforeEach(async ({ page }) => {
      await page.goto(`https://example.com/greet?name=${name}`);
    });
    test(`testing with ${expected}`, async ({ page }) => {
      await expect(page.getByRole('heading')).toHaveText(expected);
    });
  });
});
```

## Parameterized Projects

Playwright Test는 여러 테스트 프로젝트를 동시에 실행하는 것을 지원합니다. 다음 예제에서는 서로 다른 옵션으로 두 개의 프로젝트를 실행합니다.

`person` 옵션을 선언하고 구성에서 값을 설정합니다. 첫 번째 프로젝트는 `Alice` 값으로 실행되고 두 번째 프로젝트는 `Bob` 값으로 실행됩니다.

## Passing Environment Variables

명령줄에서 환경 변수를 사용하여 테스트를 구성할 수 있습니다.

예를 들어 사용자 이름과 암호가 필요한 다음 테스트 파일을 고려하십시오. 일반적으로 비밀을 소스 코드에 저장하지 않는 것이 좋으므로 외부에서 비밀을 전달하는 방법이 필요합니다.

```js title="example.spec.ts"
test(`example test`, async ({ page }) => {
  // ...
  await page.getByLabel('User Name').fill(process.env.USER_NAME);
  await page.getByLabel('Password').fill(process.env.PASSWORD);
});
```

명령줄에서 비밀 사용자 이름과 암호를 설정하여 이 테스트를 실행할 수 있습니다.

### .env files

환경 변수를 더 쉽게 관리하려면 `.env` 파일과 같은 것을 고려하십시오. 다음은 [`dotenv`](https://www.npmjs.com/package/dotenv) 패키지를 사용하여 구성 파일에서 직접 환경 변수를 읽는 예제입니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';
import dotenv from 'dotenv';
import path from 'path';

// ".env" 파일에서 읽습니다.
dotenv.config({ path: path.resolve(__dirname, '.env') });

// 또는 "../my.env" 파일에서 읽습니다.
dotenv.config({ path: path.resolve(__dirname, '..', 'my.env') });

export default defineConfig({
  use: {
    baseURL: process.env.STAGING === '1' ? 'http://staging.example.test/' : 'http://example.test/',
  }
});
```

이제 `.env` 파일을 편집하여 원하는 변수를 설정할 수 있습니다.

```bash
# .env 파일
STAGING=0
USER_NAME=me
PASSWORD=secret
```

평소처럼 테스트를 실행하면 환경 변수가 선택됩니다.

```bash
npx playwright test
```

## Create tests via a CSV file

Playwright 테스트 러너는 Node.js에서 실행됩니다. 즉, 파일 시스템에서 직접 파일을 읽고 원하는 CSV 라이브러리를 사용하여 구문 분석할 수 있습니다.

예를 들어 다음 CSV 파일을 참조하십시오. 이 예제에서는 `input.csv`입니다.

```txt
"test_case","some_value","some_other_value"
"value 1","value 11","foobar1"
"value 2","value 22","foobar21"
"value 3","value 33","foobar321"
"value 4","value 44","foobar4321"
```

이를 기반으로 NPM의 [csv-parse](https://www.npmjs.com/package/csv-parse) 라이브러리를 사용하여 일부 테스트를 생성합니다.

```js title="test.spec.ts"
import fs from 'fs';
import path from 'path';
import { test } from '@playwright/test';
import { parse } from 'csv-parse/sync';

const records = parse(fs.readFileSync(path.join(__dirname, 'input.csv')), {
  columns: true,
  skip_empty_lines: true
});

for (const record of records) {
  test(`foo: ${record.test_case}`, async ({ page }) => {
    console.log(record.test_case, record.some_value, record.some_other_value);
  });
}
```
