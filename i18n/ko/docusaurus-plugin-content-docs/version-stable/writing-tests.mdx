## Introduction

Playwright 테스트는 간단합니다.
- **액션을 수행**하고,
- **기대에 따라 상태를 단언**합니다.

액션을 수행하기 전에 아무것도 기다릴 필요가 없습니다. Playwright는 각 액션을 수행하기 전에 광범위한 [actionability](./actionability.mdx) 검사를 자동으로 기다립니다.

또한 검사를 수행할 때 경쟁 조건을 처리할 필요도 없습니다. Playwright 단언은 궁극적으로 충족되어야 하는 기대를 설명하는 방식으로 설계되었습니다.

그게 전부입니다! 이러한 설계 선택을 통해 Playwright 사용자는 테스트에서 불안정한 시간 초과 및 경쟁적인 검사를 완전히 잊을 수 있습니다.

## First test

다음 예제를 통해 테스트 작성 방법을 확인하세요.

```js title="tests/example.spec.ts"
import { test, expect } from '@playwright/test';

test('has title', async ({ page }) => {
  await page.goto('https://playwright.dev/');

  // 제목이 부분 문자열을 "포함"하는지 확인합니다.
  await expect(page).toHaveTitle(/Playwright/);
});

test('get started link', async ({ page }) => {
  await page.goto('https://playwright.dev/');

  // 시작하기 링크를 클릭합니다.
  await page.getByRole('link', { name: 'Get started' }).click();

  // 페이지에 Installation 이름의 제목이 있는지 확인합니다.
  await expect(page.getByRole('heading', { name: 'Installation' })).toBeVisible();
});
```

:::note
VS Code에서 JavaScript를 사용하는 경우 각 테스트 파일 시작 부분에 `// @ts-check`를 추가하여 자동 형식 검사를 받으세요.
:::

## Actions

### Navigation

대부분의 테스트는 페이지를 URL로 탐색하는 것으로 시작합니다. 그런 다음 테스트는 페이지 요소와 상호 작용할 수 있습니다.

```js
await page.goto('https://playwright.dev/');
```

Playwright는 페이지가 로드 상태에 도달할 때까지 기다린 후 진행합니다. [page.goto()](/api/class-page.mdx#page-goto) 옵션에 대해 자세히 알아보세요.

### Interactions

액션 수행은 요소를 찾는 것으로 시작합니다. Playwright는 이를 위해 [Locators API](./locators.mdx)를 사용합니다. 로케이터는 언제든지 페이지에서 요소(들)를 찾는 방법을 나타내며, 사용 가능한 [다양한 유형](./locators.mdx)의 로케이터에 대해 자세히 알아보세요. Playwright는 요소가 [actionable](./actionability.mdx)해질 때까지 기다린 후 액션을 수행하므로 요소가 사용 가능해질 때까지 기다릴 필요가 없습니다.

```js
// 로케이터를 만듭니다.
const getStarted = page.getByRole('link', { name: 'Get started' });

// 클릭합니다.
await getStarted.click();
```

대부분의 경우 한 줄로 작성됩니다.

```js
await page.getByRole('link', { name: 'Get started' }).click();
```

## Assertions

Playwright에는 `expect` 함수 형식의 [테스트 단언](./test-assertions.mdx)이 포함되어 있습니다. 단언을 하려면 `expect(value)`를 호출하고 기대를 반영하는 일치자를 선택합니다.

`toEqual`, `toContain`, `toBeTruthy`와 같은 많은 일반적인 일치자를 사용하여 모든 조건을 단언할 수 있습니다.

```js
expect(success).toBeTruthy();
```

Playwright에는 예상 조건이 충족될 때까지 기다리는 비동기 일치자도 포함되어 있습니다. 이러한 일치자를 사용하면 테스트를 안정적이고 탄력적으로 만들 수 있습니다. 예를 들어 이 코드는 페이지가 "Playwright"를 포함하는 제목을 얻을 때까지 기다립니다.

```js
await expect(page).toHaveTitle(/Playwright/);
```

다음은 가장 인기 있는 비동기 단언 목록입니다. 숙지해야 할 [더 많은 내용](./test-assertions.mdx)이 있습니다.

| Assertion | Description |
| :- | :- |
| [expect(locator).toBeChecked()](/api/class-locatorassertions.mdx#locator-assertions-to-be-checked) | 확인란이 선택되었습니다. |
| [expect(locator).toBeEnabled()](/api/class-locatorassertions.mdx#locator-assertions-to-be-enabled) | 컨트롤이 활성화되었습니다. |
| [expect(locator).toBeVisible()](/api/class-locatorassertions.mdx#locator-assertions-to-be-visible) | 요소가 보입니다. |
| [expect(locator).toContainText()](/api/class-locatorassertions.mdx#locator-assertions-to-contain-text) | 요소에 텍스트가 포함되어 있습니다. |
| [expect(locator).toHaveAttribute()](/api/class-locatorassertions.mdx#locator-assertions-to-have-attribute) | 요소에 속성이 있습니다. |
| [expect(locator).toHaveCount()](/api/class-locatorassertions.mdx#locator-assertions-to-have-count) | 요소 목록의 길이가 지정되었습니다. |
| [expect(locator).toHaveText()](/api/class-locatorassertions.mdx#locator-assertions-to-have-text) | 요소가 텍스트와 일치합니다. |
| [expect(locator).toHaveValue()](/api/class-locatorassertions.mdx#locator-assertions-to-have-value) | 입력 요소에 값이 있습니다. |
| [expect(page).toHaveTitle()](/api/class-pageassertions.mdx#page-assertions-to-have-title) | 페이지에 제목이 있습니다. |
| [expect(page).toHaveURL()](/api/class-pageassertions.mdx#page-assertions-to-have-url) | 페이지에 URL이 있습니다. |

### Test Isolation

Playwright 테스트는 테스트에 전달되는 [기본 제공 페이지 픽스처](./test-fixtures#built-in-fixtures)와 같은 [테스트 픽스처](./test-fixtures.mdx) 개념을 기반으로 합니다. 페이지는 [브라우저 컨텍스트로 인해 테스트 간에 격리됩니다](./browser-contexts). 이는 모든 테스트가 단일 브라우저에서 여러 테스트를 실행하는 경우에도 새로운 환경을 얻는 새로운 브라우저 프로필과 동일합니다.

```js title="tests/example.spec.ts"
import { test } from '@playwright/test';

test('example test', async ({ page }) => {
  // "page"는 이 특정 테스트를 위해 생성된 격리된 BrowserContext에 속합니다.
});

test('another test', async ({ page }) => {
  // 이 두 번째 테스트의 "page"는 첫 번째 테스트와 완전히 격리되어 있습니다.
});
```

### Using Test Hooks

`test.describe`를 사용하여 테스트 그룹을 선언하고 각 테스트 전/후에 실행되는 `test.beforeEach` 및 `test.afterEach`와 같은 다양한 [테스트 훅](./api/class-test.mdx)을 사용할 수 있습니다. 다른 훅에는 모든 테스트 전/후에 작업자당 한 번 실행되는 `test.beforeAll` 및 `test.afterAll`이 포함됩니다.

```js title="tests/example.spec.ts"
import { test, expect } from '@playwright/test';

test.describe('navigation', () => {
  test.beforeEach(async ({ page }) => {
    // 각 테스트 전에 시작 URL로 이동합니다.
    await page.goto('https://playwright.dev/');
  });

  test('main navigation', async ({ page }) => {
    // 단언은 expect API를 사용합니다.
    await expect(page).toHaveURL('https://playwright.dev/');
  });
});
```
