## Introduction

Playwright는 테스트를 실행하기 전에 로컬 개발 서버를 시작할 수 있는 config 파일에 `webserver` 옵션이 함께 제공됩니다. 이는 개발 중에 테스트를 작성할 때와 테스트할 스테이징 또는 프로덕션 URL이 없을 때 이상적입니다.

## Configuring a web server

Playwright config의 `webserver` 속성을 사용하여 테스트 중에 개발 웹 서버를 시작합니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  // 테스트를 시작하기 전에 로컬 개발 서버를 실행합니다.
  webServer: {
    command: 'npm run start',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
    stdout: 'ignore',
    stderr: 'pipe',
  },
});
```

| Property | Description |
| :- | :- |
| [testConfig.webServer](/api/class-testconfig.mdx#test-config-web-server) | 테스트 중에 개발 웹 서버 (또는 여러 개)를 시작합니다. |
| `command`| 앱의 로컬 개발 서버를 시작하는 셸 명령어입니다. |
| `url`| 서버가 연결을 수락할 준비가 되었을 때 2xx, 3xx, 400, 401, 402 또는 403 상태 코드를 반환할 것으로 예상되는 http 서버의 URL입니다. |
| `reuseExistingServer`| `true`인 경우 사용 가능한 경우 URL에서 기존 서버를 다시 사용합니다. 해당 URL에서 실행 중인 서버가 없으면 명령을 실행하여 새 서버를 시작합니다. `false`인 경우 기존 프로세스가 URL에서 수신 대기 중이면 오류가 발생합니다. stdout을 보려면 `DEBUG=pw:webserver` 환경 변수를 설정하면 됩니다. |
| `ignoreHTTPSErrors` | `url`을 가져올 때 HTTPS 오류를 무시할지 여부입니다. 기본값은 `false`입니다. |
| `cwd` | 생성된 프로세스의 현재 작업 디렉토리이며, 기본값은 구성 파일의 디렉토리입니다. |
| `stdout` | `"pipe"`인 경우 명령어의 stdout을 프로세스 stdout으로 파이프합니다. `"ignore"`인 경우 명령어의 stdout을 무시합니다. 기본값은 `"ignore"`입니다. |
| `stderr` | 명령어의 stderr을 프로세스 stderr으로 파이프할지 또는 무시할지 여부입니다. 기본값은 `"pipe"`입니다. |
| `timeout` | 프로세스가 시작되어 사용 가능해질 때까지 기다리는 시간 (밀리초)입니다. 기본값은 60000입니다. |
| `gracefulShutdown` | 프로세스를 종료하는 방법입니다. 지정하지 않으면 프로세스 그룹이 강제로 `SIGKILL`됩니다. `{ signal: 'SIGTERM', timeout: 500 }`으로 설정하면 프로세스 그룹에 `SIGTERM` 신호가 전송되고 500ms 이내에 종료되지 않으면 `SIGKILL`이 이어집니다. `SIGINT`를 신호로 대신 사용할 수도 있습니다. `0` 제한 시간은 `SIGKILL`이 전송되지 않음을 의미합니다. Windows는 `SIGTERM` 및 `SIGINT` 신호를 지원하지 않으므로 이 옵션은 Windows에서 무시됩니다. Docker 컨테이너를 종료하려면 `SIGTERM`이 필요합니다. |

## Adding a server timeout

웹 서버는 부팅하는 데 시간이 더 오래 걸릴 수 있습니다. 이 경우 서버 시작을 기다리는 제한 시간을 늘릴 수 있습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  // 나머지 구성...

  // 테스트를 시작하기 전에 로컬 개발 서버를 실행합니다.
  webServer: {
    command: 'npm run start',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
    timeout: 120 * 1000,
  },
});
```

## Adding a baseURL

테스트에서 상대 URL을 사용하고 전체 URL을 반복해서 지정할 필요가 없도록 구성의 `use: {}` 섹션에 `baseURL`을 지정하는 것이 좋습니다.

[page.goto()](/api/class-page.mdx#page-goto), [page.route()](/api/class-page.mdx#page-route), [page.waitForURL()](/api/class-page.mdx#page-wait-for-url), [page.waitForRequest()](/api/class-page.mdx#page-wait-for-request) 또는 [page.waitForResponse()](/api/class-page.mdx#page-wait-for-response)를 사용하는 경우 해당 URL을 빌드하기 위해 [`URL()`](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) 생성자를 사용하여 기본 URL을 고려합니다. 예를 들어 baseURL을 `http://localhost:3000`으로 설정하고 테스트에서 `/login`으로 이동하면 Playwright는 `http://localhost:3000/login`을 사용하여 테스트를 실행합니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  // 나머지 구성...

  // 테스트를 시작하기 전에 로컬 개발 서버를 실행합니다.
  webServer: {
    command: 'npm run start',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
  },
  use: {
    baseURL: 'http://localhost:3000',
  },
});
```

이제 페이지를 탐색할 때 상대 경로를 사용할 수 있습니다.

```js title="test.spec.ts"
import { test } from '@playwright/test';

test('test', async ({ page }) => {
  // 이렇게 하면 http://localhost:3000/login으로 이동합니다.
  await page.goto('./login');
});
```

## Multiple web servers

`webServer` 구성의 배열을 제공하여 여러 웹 서버 (또는 백그라운드 프로세스)를 동시에 시작할 수 있습니다. 자세한 내용은 [testConfig.webServer](/api/class-testconfig.mdx#test-config-web-server)를 참조하세요.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  webServer: [
    {
      command: 'npm run start',
      url: 'http://localhost:3000',
      timeout: 120 * 1000,
      reuseExistingServer: !process.env.CI,
    },
    {
      command: 'npm run backend',
      url: 'http://localhost:3333',
      timeout: 120 * 1000,
      reuseExistingServer: !process.env.CI,
    }
  ],
  use: {
    baseURL: 'http://localhost:3000',
  },
});
```
