## Introduction

Playwright는 동일한 기본 구현을 공유하는 여러 언어로 제공됩니다. 브라우저 자동화를 위한 모든 핵심 기능은 모든 언어에서 지원되지만, 테스트 생태계 통합은 다릅니다. 여러분의 경험, 테스트 생태계에 대한 친숙도 및 프로젝트 제약 조건을 기반으로 언어를 선택하세요. 최상의 경험을 위해 각 언어에 대해 권장하는 테스트 러너를 선택하세요.

## JavaScript and TypeScript

Node.js용 Playwright는 훌륭한 병렬화 메커니즘, 스크린샷 어설션, html 리포터, 자동 추적 등을 제공하는 자체 [테스트 러너](https://playwright.dev/docs/running-tests)와 함께 제공됩니다.
* [Documentation](https://playwright.dev/docs/intro)
* [GitHub repo](https://github.com/microsoft/playwright)

## Python

Playwright [Pytest plugin](https://playwright.dev/python/docs/test-runners)은 엔드 투 엔드 테스트를 실행하는 권장 방법입니다. 컨텍스트 격리, 여러 브라우저 구성에서 실행 등을 즉시 제공합니다.
* [Documentation](https://playwright.dev/python/docs/intro)
* [GitHub repo](https://github.com/microsoft/playwright-python)

## Java

프로젝트 요구 사항에 따라 JUnit 또는 TestNG와 같은 모든 테스트 프레임워크를 선택할 수 있습니다.
* [Documentation](https://playwright.dev/java/docs/intro)
* [GitHub repo](https://github.com/microsoft/playwright-java)

## .NET

.NET용 Playwright는 엔드 투 엔드 테스트 작성을 위한 MSTest, NUnit 및 xUnit [기본 클래스](https://playwright.dev/dotnet/docs/test-runners)와 함께 제공됩니다.
* [Documentation](https://playwright.dev/dotnet/docs/intro)
* [GitHub repo](https://github.com/microsoft/playwright-dotnet)
