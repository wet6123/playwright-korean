## Introduction

Playwright를 사용하면 단일 테스트, 테스트 집합 또는 모든 테스트를 실행할 수 있습니다. `--project` 플래그를 사용하여 테스트를 하나의 브라우저 또는 여러 브라우저에서 실행할 수 있습니다. 테스트는 기본적으로 병렬로 실행되며 헤드리스 방식으로 실행됩니다. 즉, 테스트를 실행하는 동안 브라우저 창이 열리지 않고 터미널에서 결과가 표시됩니다. 그러나 `--headed` CLI 인수를 사용하여 헤드 모드로 테스트를 실행하거나 `--ui` 플래그를 사용하여 [UI 모드](./test-ui-mode.mdx)에서 테스트를 실행할 수 있습니다. 감시 모드, 시간 여행 디버깅 등과 함께 테스트의 전체 추적을 확인하세요.

## Running tests

### Command line

`playwright test` 명령으로 테스트를 실행할 수 있습니다. 그러면 `playwright.config` 파일에 구성된 대로 모든 브라우저에서 테스트가 실행됩니다. 테스트는 기본적으로 헤드리스 모드에서 실행됩니다. 즉, 테스트를 실행하는 동안 브라우저 창이 열리지 않고 터미널에서 결과가 표시됩니다.

```bash
npx playwright test
```

![tests running in command line](https://github.com/microsoft/playwright/assets/13063165/981c1b2b-dc7e-4b85-b241-272b44da6628)

### Run tests in UI mode

각 테스트 단계를 쉽게 진행하고 각 단계 전, 도중 및 후에 어떤 일이 일어났는지 시각적으로 확인할 수 있는 더 나은 개발자 경험을 위해 [UI 모드](./test-ui-mode.mdx)에서 테스트를 실행하는 것이 좋습니다. UI 모드에는 로케이터 선택기, 감시 모드 등과 같은 많은 다른 기능도 함께 제공됩니다.

```bash
npx playwright test --ui
```

![UI Mode](https://github.com/microsoft/playwright/assets/13063165/c5b501cc-4f5d-485a-87cc-66044c651786)

기능에 대해 자세히 알아보려면 [UI 모드에 대한 자세한 가이드](./test-ui-mode.mdx)를 확인하세요.

### Run tests in headed mode

헤드 모드에서 테스트를 실행하려면 `--headed` 플래그를 사용하세요. 그러면 Playwright가 웹 사이트와 상호 작용하는 방식을 시각적으로 확인할 수 있습니다.

```bash
npx playwright test --headed
```

### Run tests on different browsers

테스트를 실행할 브라우저를 지정하려면 `--project` 플래그 뒤에 브라우저 이름을 사용하세요.

```bash
npx playwright test --project webkit
```

테스트를 실행할 여러 브라우저를 지정하려면 `--project` 플래그를 여러 번 사용하고 각 브라우저 이름을 차례로 지정하세요.

```bash
npx playwright test --project webkit --project firefox
```

### Run specific tests

단일 테스트 파일을 실행하려면 실행할 테스트 파일 이름을 전달하세요.

```bash
npx playwright test landing-page.spec.ts
```

서로 다른 디렉터리의 테스트 파일 집합을 실행하려면 테스트를 실행할 디렉터리 이름을 전달하세요.

```bash
npx playwright test tests/todo-page/ tests/landing-page/
```

파일 이름에 `landing` 또는 `login`이 있는 파일을 실행하려면 이러한 키워드를 CLI에 전달하기만 하면 됩니다.

```bash
npx playwright test landing login
```

특정 제목의 테스트를 실행하려면 `-g` 플래그 뒤에 테스트 제목을 사용하세요.

```bash
npx playwright test -g "add a todo item"
```

### Run last failed tests

마지막 테스트 실행에서 실패한 테스트만 실행하려면 먼저 테스트를 실행한 다음 `--last-failed` 플래그를 사용하여 다시 실행하세요.

```bash
npx playwright test --last-failed
```

### Run tests in VS Code

[VS Code 확장 프로그램](https://marketplace.visualstudio.com/items?itemName=ms-playwright.playwright)을 사용하여 VS Code에서 바로 테스트를 실행할 수 있습니다. 설치가 완료되면 실행할 테스트 옆에 있는 녹색 삼각형을 클릭하거나 테스트 사이드바에서 모든 테스트를 실행할 수 있습니다. 자세한 내용은 [VS Code 시작하기](./getting-started-vscode.mdx#running-tests) 가이드를 확인하세요.

![Playwright VS Code extension](https://github.com/microsoft/playwright/assets/13063165/47726e70-683b-4bd5-94de-7d03dd45c30f)

## Debugging tests

Playwright는 Node.js에서 실행되므로 `console.log`를 사용하거나 IDE 내부 또는 [VS Code 확장 프로그램](./getting-started-vscode.mdx)을 사용하여 VS Code에서 직접 디버거를 선택하여 디버깅할 수 있습니다. Playwright에는 테스트의 각 단계를 쉽게 진행하고 로그, 오류, 네트워크 요청을 확인하고 DOM 스냅샷 등을 검사할 수 있는 [UI 모드](./test-ui-mode.mdx)가 함께 제공됩니다. [Playwright Inspector](./debug.mdx#playwright-inspector)를 사용할 수도 있습니다. 이 기능을 사용하면 Playwright API 호출을 단계별로 실행하고 디버그 로그를 확인하고 [로케이터](./locators.mdx)를 탐색할 수 있습니다.

### Debug tests in UI mode

각 테스트 단계를 쉽게 진행하고 각 단계 전, 도중 및 후에 어떤 일이 일어났는지 시각적으로 확인할 수 있는 더 나은 개발자 경험을 위해 [UI 모드](./test-ui-mode.mdx)에서 테스트를 디버깅하는 것이 좋습니다. UI 모드에는 로케이터 선택기, 감시 모드 등과 같은 많은 다른 기능도 함께 제공됩니다.

```bash
npx playwright test --ui
```

![showing errors in ui mode](https://github.com/microsoft/playwright/assets/13063165/ffca2fd1-5349-41fb-ade9-ace143bb2c58)

디버깅하는 동안 로케이터 선택 버튼을 사용하여 페이지에서 요소를 선택하고 Playwright가 해당 요소를 찾는 데 사용할 로케이터를 확인할 수 있습니다. 로케이터 플레이그라운드에서 로케이터를 편집하고 브라우저 창에서 실시간으로 강조 표시되는 것을 볼 수도 있습니다. 로케이터 복사 버튼을 사용하여 로케이터를 클립보드에 복사한 다음 테스트에 붙여넣으세요.

![pick locator in ui mode](https://github.com/microsoft/playwright/assets/13063165/9e7eeb84-bd26-4010-8614-75e24b56c716)

기능에 대해 자세히 알아보려면 [UI 모드에 대한 자세한 가이드](./test-ui-mode.mdx)를 확인하세요.

### Debug tests with the Playwright Inspector

모든 테스트를 디버깅하려면 Playwright 테스트 명령 뒤에 `--debug` 플래그를 사용하세요.

```bash
npx playwright test --debug
```

![Debugging Tests with the Playwright inspector](https://github.com/microsoft/playwright/assets/13063165/6b3b3caa-d258-4cb8-aa05-cd407f501626)

이 명령은 브라우저 창과 Playwright Inspector를 엽니다. Inspector 상단의 step over 버튼을 사용하여 테스트를 단계별로 진행할 수 있습니다. 또는 재생 버튼을 눌러 처음부터 끝까지 테스트를 실행하세요. 테스트가 완료되면 브라우저 창이 닫힙니다.

하나의 테스트 파일을 디버깅하려면 디버깅할 테스트 파일 이름 뒤에 `--debug` 플래그를 사용하여 Playwright 테스트 명령을 실행하세요.

```bash
npx playwright test example.spec.ts --debug
```

`test(..`가 정의된 줄 번호에서 특정 테스트를 디버깅하려면 테스트 파일 이름 끝에 콜론과 줄 번호를 추가한 다음 `--debug` 플래그를 사용하세요.

```bash
npx playwright test example.spec.ts:10 --debug
```

디버깅하는 동안 로케이터 선택 버튼을 사용하여 페이지에서 요소를 선택하고 Playwright가 해당 요소를 찾는 데 사용할 로케이터를 확인할 수 있습니다. 로케이터를 편집하고 브라우저 창에서 실시간으로 강조 표시되는 것을 볼 수도 있습니다. 로케이터 복사 버튼을 사용하여 로케이터를 클립보드에 복사한 다음 테스트에 붙여넣으세요.

![Locator picker in the Playwright Inspector](https://github.com/microsoft/playwright/assets/13063165/013d5edb-583e-423a-bb53-9f57bab7f3e1)

[VS Code 디버거](./debug.mdx#vs-code-debugger), UI 모드 및 [Playwright Inspector](./debug.mdx#playwright-inspector)를 사용한 디버깅뿐만 아니라 [브라우저 개발자 도구](./debug.mdx#browser-developer-tools)를 사용한 디버깅에 대해 자세히 알아보려면 [디버깅 가이드](./debug.mdx)를 확인하세요.

## Test reports

[HTML Reporter](./test-reporters.mdx#html-reporter)는 브라우저, 통과한 테스트, 실패한 테스트, 건너뛴 테스트 및 불안정한 테스트별로 보고서를 필터링할 수 있도록 테스트의 전체 보고서를 보여줍니다. 기본적으로 HTML 보고서는 일부 테스트가 실패한 경우 자동으로 열리고, 그렇지 않으면 다음 명령으로 열 수 있습니다.

```bash
npx playwright show-report
```

![HTML Report](https://github.com/microsoft/playwright/assets/13063165/c5f60e56-fb75-4a2d-a4b6-054b8c5d69c1)

테스트를 필터링하고 검색할 수 있을 뿐만 아니라 각 테스트를 클릭하여 테스트 오류를 확인하고 테스트의 각 단계를 탐색할 수 있습니다.

![HTML Reporter detail view](https://github.com/microsoft/playwright/assets/13063165/f36ada14-4701-4534-a3be-ed22c2b16cf5)
