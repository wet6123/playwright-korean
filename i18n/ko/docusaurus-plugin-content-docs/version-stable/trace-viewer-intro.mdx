## Introduction

Playwright Trace Viewer는 테스트의 기록된 Playwright 트레이스를 탐색할 수 있는 GUI 도구입니다. 즉, 테스트의 각 동작을 앞뒤로 이동하면서 각 동작 중에 발생한 상황을 시각적으로 확인할 수 있습니다.

## Recording a Trace

기본적으로 [playwright.config](./trace-viewer.mdx#tracing-on-ci) 파일에는 각 테스트에 대한 `trace.zip` 파일을 생성하는 데 필요한 구성이 포함됩니다. 트레이스는 `on-first-retry`로 실행되도록 설정됩니다. 즉, 실패한 테스트의 첫 번째 재시도 시에 실행됩니다. 또한 `retries`는 CI에서 실행할 때는 2로, 로컬에서는 0으로 설정됩니다. 이는 트레이스가 첫 번째 실행이 아닌 실패한 테스트의 첫 번째 재시도 시에 기록되고 두 번째 재시도 시에는 기록되지 않음을 의미합니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';
export default defineConfig({
  retries: process.env.CI ? 2 : 0, // CI에서 실행할 때는 2로 설정
  // ...
  use: {
    trace: 'on-first-retry', // 각 테스트의 첫 번째 재시도 시에 트레이스 기록
  },
});
```

트레이스 기록에 사용할 수 있는 옵션에 대해 자세히 알아보려면 [Trace Viewer](/trace-viewer.mdx)에 대한 자세한 가이드를 확인하세요.

트레이스는 일반적으로 Continuous Integration(CI) 환경에서 실행됩니다. 로컬에서는 테스트 개발 및 디버깅을 위해 [UI 모드](/test-ui-mode.mdx)를 사용할 수 있기 때문입니다. 그러나 [UI 모드](/test-ui-mode.mdx)를 사용하지 않고 로컬에서 트레이스를 실행하려면 `--trace on`을 사용하여 트레이스를 강제로 켤 수 있습니다.

```bash
npx playwright test --trace on
```

## Opening the HTML report

HTML 보고서는 실행된 모든 테스트와 브라우저, 소요 시간을 보여줍니다. 테스트는 통과한 테스트, 실패한 테스트, 불안정한 테스트 또는 건너뛴 테스트별로 필터링할 수 있습니다. 특정 테스트를 검색할 수도 있습니다. 테스트를 클릭하면 오류, 테스트 단계 및 트레이스와 같은 테스트에 대한 자세한 정보를 볼 수 있는 자세한 보기가 열립니다.

```bash
npx playwright show-report
```

## Opening the trace

HTML 보고서에서 필요한 테스트의 트레이스를 직접 열려면 테스트 이름 파일 이름 옆에 있는 트레이스 아이콘을 클릭하세요.

![playwright html report](https://github.com/microsoft/playwright/assets/13063165/a3da1fb5-6619-4c03-98aa-adf65c376525)

테스트의 자세한 보기를 열고 `'Traces'` 탭으로 스크롤한 다음 트레이스 스크린샷을 클릭하여 트레이스를 열 수도 있습니다.

![playwright html report detailed view](https://github.com/microsoft/playwright/assets/13063165/2b583d6f-5241-4ecf-83a8-650072d4a201)

리포터에 대한 자세한 내용은 [HTML Reporter](/test-reporters.mdx#html-reporter)를 포함한 리포터에 대한 자세한 가이드를 확인하세요.

## Viewing the trace

각 동작을 클릭하거나 타임라인을 사용하여 마우스를 올려 테스트의 트레이스를 보고 동작 전후의 페이지 상태를 확인하세요. 테스트의 각 단계에서 로그, 소스 및 네트워크, 오류 및 콘솔을 검사합니다. 트레이스 뷰어는 DOM 스냅샷을 생성하므로 완전히 상호 작용하고 브라우저 DevTools를 열어 HTML, CSS 등을 검사할 수 있습니다.

![playwright trace viewer](https://github.com/microsoft/playwright/assets/13063165/10fe3585-8401-4051-b1c2-b2e92ac4c274)

트레이스에 대한 자세한 내용은 [Trace Viewer](/trace-viewer.mdx)에 대한 자세한 가이드를 확인하세요.
