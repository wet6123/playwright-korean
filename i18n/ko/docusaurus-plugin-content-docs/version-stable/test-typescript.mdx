## Introduction

Playwright는 TypeScript를 기본적으로 지원합니다. TypeScript로 테스트를 작성하기만 하면 Playwright가 이를 읽고 JavaScript로 변환하여 실행합니다.

Playwright는 유형을 검사하지 않으며 중요하지 않은 TypeScript 컴파일 오류가 있더라도 테스트를 실행합니다. Playwright와 함께 TypeScript 컴파일러를 실행하는 것이 좋습니다. 예를 들어 GitHub Actions에서는 다음과 같습니다.

```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    ...
    - name: Run type checks
      run: npx tsc -p tsconfig.json --noEmit
    - name: Run Playwright tests
      run: npx playwright test
```

로컬 개발의 경우 다음과 같이 [watch](https://www.typescriptlang.org/docs/handbook/configuring-watch.html) 모드에서 `tsc`를 실행할 수 있습니다.

```sh
npx tsc -p tsconfig.json --noEmit -w
```

## tsconfig.json

Playwright는 로드하는 각 소스 파일에 대해 `tsconfig.json`을 선택합니다. Playwright는 `allowJs`, `baseUrl`, `paths` 및 `references`의 tsconfig 옵션만 **지원합니다**.

테스트를 위해 일부 설정을 변경할 수 있도록 테스트 디렉토리에 별도의 `tsconfig.json`을 설정하는 것이 좋습니다. 다음은 디렉터리 구조의 예입니다.

```txt
src/
    source.ts

tests/
    tsconfig.json  # test-specific tsconfig (테스트 전용 tsconfig)
    example.spec.ts

tsconfig.json  # generic tsconfig for all typescript sources (모든 typescript 소스에 대한 일반 tsconfig)

playwright.config.ts
```

### tsconfig path mapping

Playwright는 `tsconfig.json`에 선언된 [경로 매핑](https://www.typescriptlang.org/docs/handbook/module-resolution.html#path-mapping)을 지원합니다. `baseUrl`도 설정해야 합니다.

다음은 Playwright에서 작동하는 `tsconfig.json`의 예입니다.

```json title="tsconfig.json"
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@myhelper/*": ["packages/myhelper/*"] // This mapping is relative to "baseUrl". (이 매핑은 "baseUrl"을 기준으로 합니다.)
    }
  }
}
```

이제 매핑된 경로를 사용하여 가져올 수 있습니다.

```js title="example.spec.ts"
import { test, expect } from '@playwright/test';
import { username, password } from '@myhelper/credentials';

test('example', async ({ page }) => {
  await page.getByLabel('User Name').fill(username);
  await page.getByLabel('Password').fill(password);
});
```

### tsconfig resolution

기본적으로 Playwright는 디렉터리 구조를 따라 올라가면서 `tsconfig.json` 또는 `jsconfig.json`을 찾아 각 가져온 파일에 대해 가장 가까운 tsconfig를 찾습니다. 이렇게 하면 테스트에만 사용되는 `tests/tsconfig.json` 파일을 만들 수 있으며 Playwright가 자동으로 선택합니다.

```sh
# Playwright will choose tsconfig automatically (Playwright가 tsconfig를 자동으로 선택합니다.)
npx playwright test
```

또는 명령줄에서 사용할 단일 tsconfig 파일을 지정할 수 있으며 Playwright는 테스트 파일뿐만 아니라 모든 가져온 파일에 대해 이 파일을 사용합니다.

```sh
# Pass a specific tsconfig (특정 tsconfig 전달)
npx playwright test --tsconfig=tsconfig.test.json
```

테스트 파일, 리포터 등을 로드하는 데 사용될 구성 파일에 단일 tsconfig 파일을 지정할 수 있습니다. 그러나 playwright 구성 자체 또는 해당 구성에서 가져온 파일을 로드하는 동안에는 사용되지 않습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  tsconfig: './tsconfig.test.json',
});
```

## Manually compile tests with TypeScript

경우에 따라 Playwright Test는 `tsconfig.json`에서 구성된 실험적이거나 최신 TypeScript 기능을 사용하는 경우와 같이 TypeScript 코드를 올바르게 변환하지 못할 수 있습니다.

이 경우 Playwright로 테스트를 보내기 전에 TypeScript 컴파일을 직접 수행할 수 있습니다.

먼저 테스트 디렉토리 내부에 `tsconfig.json` 파일을 추가합니다.

```json
{
    "compilerOptions": {
        "target": "ESNext",
        "module": "commonjs",
        "moduleResolution": "Node",
        "sourceMap": true,
        "outDir": "../tests-out",
    }
}
```

`package.json`에서 두 개의 스크립트를 추가합니다.

```json
{
  "scripts": {
    "pretest": "tsc --incremental -p tests/tsconfig.json",
    "test": "playwright test -c tests-out"
  }
}
```

`pretest` 스크립트는 테스트에서 typescript를 실행합니다. `test`는 `tests-out` 디렉토리에 생성된 테스트를 실행합니다. `-c` 인수는 `tests-out` 디렉토리 내부에서 테스트를 찾도록 테스트 러너를 구성합니다.

그러면 `npm run test`는 테스트를 빌드하고 실행합니다.
