[WebSocket] 클래스는 페이지 내의 WebSocket 연결을 나타냅니다. 이 클래스는 전송 및 수신되는 데이터를 검사하고 조작하는 기능을 제공합니다.

WebSocket 프레임을 가로채거나 수정하려면 [WebSocketRoute]를 사용하는 것이 좋습니다.

### waitForEvent {#web-socket-wait-for-event}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>webSocket.waitForEvent</x-search>

이벤트가 발생할 때까지 기다렸다가 해당 값을 predicate 함수에 전달합니다. predicate가 truthy 값을 반환하면 반환합니다. 이벤트가 발생하기 전에 webSocket이 닫히면 오류가 발생합니다. 이벤트 데이터 값을 반환합니다.

**Usage**

```js
await webSocket.waitForEvent(event);
await webSocket.waitForEvent(event, optionsOrPredicate, options);
```

**Arguments**
- `event` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-wait-for-event-option-event"/><a href="#web-socket-wait-for-event-option-event" class="list-anchor">#</a>
  
  이벤트 이름, `webSocket.on(event)`에 전달하는 것과 동일합니다.
- `optionsOrPredicate` [function] | [Object] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-wait-for-event-option-options-or-predicate"/><a href="#web-socket-wait-for-event-option-options-or-predicate" class="list-anchor">#</a>
  - `predicate` [function]
    
    이벤트 데이터를 수신하고 대기가 해결되어야 할 때 truthy 값으로 확인합니다.
  - `timeout` [number] *(optional)*
    
    대기할 최대 시간(밀리초)입니다. 기본값은 `0` - 제한 시간 없음입니다. 기본값은 구성에서 `actionTimeout` 옵션을 통해 변경하거나 [browserContext.setDefaultTimeout()](/api/class-browsercontext.mdx#browser-context-set-default-timeout) 또는 [page.setDefaultTimeout()](/api/class-page.mdx#page-set-default-timeout) 메서드를 사용하여 변경할 수 있습니다.
  
  이벤트를 수신하는 predicate 또는 options 객체입니다. 선택 사항입니다.
- `options` [Object] *(optional)*
  - `predicate` [function] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-wait-for-event-option-predicate"/><a href="#web-socket-wait-for-event-option-predicate" class="list-anchor">#</a>
    
    이벤트 데이터를 수신하고 대기가 해결되어야 할 때 truthy 값으로 확인합니다.

**Returns**
- [Promise]&lt;[Object]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-wait-for-event-return"/><a href="#web-socket-wait-for-event-return" class="list-anchor">#</a>
