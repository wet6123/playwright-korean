## Properties

### attachments {#test-result-attachments}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>testResult.attachments</x-search>

[testInfo.attachments](/api/class-testinfo.mdx#test-info-attachments)를 통해 테스트 실행 중에 첨부된 파일 또는 버퍼 목록입니다.

**Usage**

```js
testResult.attachments
```

**Type**
- [Array]&lt;[Object]&gt;
  - `name` [string]
    
    첨부 파일 이름.
  - `contentType` [string]
    
    리포트에서 올바르게 표시하기 위한 이 첨부 파일의 콘텐츠 유형입니다(예: `'application/json'` 또는 `'image/png'`).
  - `path` [string] *(optional)*
    
    첨부된 파일에 대한 파일 시스템의 선택적 경로입니다.
  - `body` [Buffer] *(optional)*
    
    파일 대신 사용되는 선택적 첨부 파일 본문입니다.

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.30</font><x-search>testResult.parallelIndex</x-search>

### workerIndex {#test-result-worker-index}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>testResult.workerIndex</x-search>

테스트가 실행된 워커의 인덱스입니다. 예를 들어 사용자가 테스트를 중단하는 등 테스트가 한 번도 실행되지 않은 경우 유일한 결과는 `workerIndex`가 `-1`인 것입니다.

Playwright Test를 사용한 [병렬 처리 및 샤딩](../test-parallel.mdx)에 대해 자세히 알아보세요.

**Usage**

```js
testResult.workerIndex
```

**Type**
- [number]
