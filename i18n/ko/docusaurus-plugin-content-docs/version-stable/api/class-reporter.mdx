테스트 러너는 테스트 실행 중 다양한 이벤트에 대해 보고자에게 알립니다. 보고자의 모든 메서드는 선택 사항입니다.

보고자 메서드 중 일부를 사용하여 클래스를 구현하여 사용자 지정 보고자를 만들 수 있습니다. 이 클래스를 기본값으로 내보내야 합니다.

<TabItem value="ts">

```js title="my-awesome-reporter.ts"
import type {
  Reporter, FullConfig, Suite, TestCase, TestResult, FullResult
} from '@playwright/test/reporter';

class MyReporter implements Reporter {
  constructor(options: { customOption?: string } = {}) {
    console.log(`my-awesome-reporter setup with customOption set to ${options.customOption}`);
  }

  onBegin(config: FullConfig, suite: Suite) {
    console.log(`Starting the run with ${suite.allTests().length} tests`);
  }

  onTestBegin(test: TestCase) {
    console.log(`Starting test ${test.title}`);
  }

  onTestEnd(test: TestCase, result: TestResult) {
    console.log(`Finished test ${test.title}: ${result.status}`);
  }

  onEnd(result: FullResult) {
    console.log(`Finished the run: ${result.status}`);
  }
}
export default MyReporter;
```

이제 [testConfig.reporter](/api/class-testconfig.mdx#test-config-reporter)와 함께 이 보고자를 사용하십시오. [보고자 사용](../test-reporters.mdx)에 대해 자세히 알아보십시오.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: [['./my-awesome-reporter.ts', { customOption: 'some value' }]],
});
```

다음은 일반적인 보고자 호출 순서입니다.
* [reporter.onBegin()](/api/class-reporter.mdx#reporter-on-begin)은 다른 모든 스위트와 테스트를 포함하는 루트 스위트와 함께 한 번 호출됩니다. [스위트 계층 구조][Suite]에 대해 자세히 알아보십시오.
* [reporter.onTestBegin()](/api/class-reporter.mdx#reporter-on-test-begin)은 각 테스트 실행에 대해 호출됩니다. 실행되는 [TestCase]와 거의 비어 있는 [TestResult]가 제공됩니다. 테스트 결과는 테스트 실행 중에 채워지고(예: 단계 및 stdio 포함) 테스트가 완료되면 최종 `status`를 얻습니다.
* [reporter.onStepBegin()](/api/class-reporter.mdx#reporter-on-step-begin) 및 [reporter.onStepEnd()](/api/class-reporter.mdx#reporter-on-step-end)는 테스트 내부에서 실행되는 각 단계에 대해 호출됩니다. 단계가 실행되면 테스트 실행이 아직 완료되지 않았습니다.
* [reporter.onTestEnd()](/api/class-reporter.mdx#reporter-on-test-end)는 테스트 실행이 완료되면 호출됩니다. 이때 [TestResult]가 완료되고 [testResult.status](/api/class-testresult.mdx#test-result-status), [testResult.error](/api/class-testresult.mdx#test-result-error) 등을 사용할 수 있습니다.
* [reporter.onEnd()](/api/class-reporter.mdx#reporter-on-end)는 실행해야 하는 모든 테스트가 완료된 후 한 번 호출됩니다.
* [reporter.onExit()](/api/class-reporter.mdx#reporter-on-exit)는 테스트 러너가 종료되기 직전에 호출됩니다.

또한 [reporter.onStdOut()](/api/class-reporter.mdx#reporter-on-std-out) 및 [reporter.onStdErr()](/api/class-reporter.mdx#reporter-on-std-err)은 워커 프로세스에서 표준 출력이 생성될 때 호출되고(테스트 실행 중일 수 있음) [reporter.onError()](/api/class-reporter.mdx#reporter-on-error)는 테스트 실행 외부에서 문제가 발생할 때 호출됩니다.

사용자 지정 보고자가 터미널에 아무 것도 출력하지 않으면 [reporter.printsToStdio()](/api/class-reporter.mdx#reporter-prints-to-stdio)를 구현하고 `false`를 반환합니다. 이렇게 하면 Playwright는 사용자 경험을 향상시키기 위해 사용자 지정 보고자 외에도 표준 터미널 보고자 중 하나를 사용합니다.

**병합된 보고서 API 참고 사항**

[`blob`](../test-reporters#blob-reporter) 보고서를 여러 개 병합할 때 [`merge-reports`](../test-sharding#merge-reports-cli) CLI 명령을 통해 동일한 [Reporter] API가 호출되어 최종 보고서를 생성하고 기존의 모든 보고자가 변경 없이 작동해야 합니다. 그러나 일부 사용자 지정 보고자에 영향을 줄 수 있는 약간의 미묘한 차이가 있습니다.
* 다른 샤드의 프로젝트는 항상 별도의 [TestProject] 객체로 유지됩니다. 예를 들어 'Desktop Chrome' 프로젝트가 5대의 장비에 걸쳐 샤딩된 경우 [reporter.onBegin()](/api/class-reporter.mdx#reporter-on-begin)에 전달된 구성에 동일한 이름을 가진 프로젝트의 인스턴스가 5개 있습니다.

## 메서드

### onBegin {#reporter-on-begin}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>reporter.onBegin</x-search>

테스트를 실행하기 전에 한 번 호출됩니다. 모든 테스트가 이미 검색되어 [Suite]의 계층 구조로 배치되었습니다.

**사용법**

```js
reporter.onBegin(config, suite);
```

**인수**
- `config` [FullConfig]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="reporter-on-begin-option-config"/><a href="#reporter-on-begin-option-config" class="list-anchor">#</a>
  
  해결된 구성입니다.
- `suite` [Suite]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="reporter-on-begin-option-suite"/><a href="#reporter-on-begin-option-suite" class="list-anchor">#</a>
  
  모든 프로젝트, 파일 및 테스트 케이스를 포함하는 루트 스위트입니다.

### onEnd {#reporter-on-end}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>reporter.onEnd</x-search>

모든 테스트가 실행되었거나 테스트가 중단된 후에 호출됩니다. 이 메서드는 [Promise]를 반환할 수 있으며 Playwright Test는 이를 기다립니다. 보고자는 상태를 재정의하고 테스트 러너의 종료 코드에 영향을 줄 수 있습니다.

**사용법**

```js
await reporter.onEnd(result);
```

**인수**
- `result` [Object]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="reporter-on-end-option-result"/><a href="#reporter-on-end-option-result" class="list-anchor">#</a>
  - `status` "passed" | "failed" | "timedout" | "interrupted"
    
    테스트 실행 상태입니다.
  - `startTime` [Date]
    
    테스트 실행 시작 시간입니다.
  - `duration` [number]
    
    테스트 실행 기간(밀리초)입니다.
  
  전체 테스트 실행 결과입니다. `status`는 다음 중 하나일 수 있습니다.
  * `'passed'` - 모든 것이 예상대로 진행되었습니다.
  * `'failed'` - 테스트가 실패했습니다.
  * `'timedout'` - [testConfig.globalTimeout](/api/class-testconfig.mdx#test-config-global-timeout)에 도달했습니다.
  * `'interrupted'` - 사용자에 의해 중단되었습니다.

**반환 값**
- [Promise]&lt;[Object]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="reporter-on-end-return"/><a href="#reporter-on-end-return" class="list-anchor">#</a>
  - `status` "passed" | "failed" | "timedout" | "interrupted" *(선택 사항)*

```js
reporter.onError(error);
```

**인수**

  오류입니다.

### onExit {#reporter-on-exit}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.33</font><x-search>reporter.onExit</x-search>

테스트 러너가 종료되기 직전에 호출됩니다. 이 시점에서 모든 보고자는 [reporter.onEnd()](/api/class-reporter.mdx#reporter-on-end) 신호를 받았으므로 모든 보고서가 빌드되어야 합니다. 이 후크에서 보고서를 업로드하는 코드를 실행할 수 있습니다.

**사용법**

**반환 값**

### printsToStdio {#reporter-prints-to-stdio}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>reporter.printsToStdio</x-search>

**사용법**

```js
reporter.printsToStdio();
```

**반환 값**
- [boolean]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="reporter-prints-to-stdio-return"/><a href="#reporter-prints-to-stdio-return" class="list-anchor">#</a>
