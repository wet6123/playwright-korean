BrowserContexts는 여러 독립적인 브라우저 세션을 운영하는 방법을 제공합니다.

페이지가 `window.open` 호출 등으로 다른 페이지를 열면, 팝업은 부모 페이지의 브라우저 컨텍스트에 속하게 됩니다.

Playwright는 [browser.newContext()](/api/class-browser.mdx#browser-new-context) 메서드를 사용하여 격리된 비영구적 브라우저 컨텍스트를 생성할 수 있습니다. 비영구적 브라우저 컨텍스트는 디스크에 브라우징 데이터를 쓰지 않습니다.

```js
// 새 시크릿 브라우저 컨텍스트 생성
const context = await browser.newContext();
// 컨텍스트 내부에 새 페이지 생성
const page = await context.newPage();
await page.goto('https://example.com');
// 더 이상 필요하지 않으면 컨텍스트 폐기
await context.close();
```

### addInitScript {#browser-context-add-init-script}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browserContext.addInitScript</x-search>

다음 시나리오 중 하나에서 평가될 스크립트를 추가합니다:
* 브라우저 컨텍스트에서 페이지가 생성되거나 탐색될 때마다.
* 브라우저 컨텍스트의 모든 페이지에서 하위 프레임이 첨부되거나 탐색될 때마다. 이 경우 스크립트는 새로 첨부된 프레임의 컨텍스트에서 평가됩니다.

스크립트는 문서가 생성된 후, 해당 스크립트가 실행되기 전에 평가됩니다. 이는 JavaScript 환경을 수정하는 데 유용합니다(예: `Math.random` 시드 설정).

**사용법**

페이지 로드 전에 `Math.random`을 재정의하는 예제:

```js
// preload.js
Math.random = () => 42;
```

```js
// Playwright 스크립트에서, preload.js 파일이 같은 디렉토리에 있다고 가정합니다.
await browserContext.addInitScript({
  path: 'preload.js'
});
```

:::note
[browserContext.addInitScript()](/api/class-browsercontext.mdx#browser-context-add-init-script) 및 [page.addInitScript()](/api/class-page.mdx#page-add-init-script)를 통해 설치된 여러 스크립트의 평가 순서는 정의되지 않습니다.
:::

**인수**
- `script` [function] | [string] | [Object]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-add-init-script-option-script"/><a href="#browser-context-add-init-script-option-script" class="list-anchor">#</a>
  - `path` [string] *(선택 사항)*

    JavaScript 파일 경로. `path`가 상대 경로인 경우 현재 작업 디렉토리를 기준으로 확인됩니다. 선택 사항입니다.
  - `content` [string] *(선택 사항)*

    원시 스크립트 내용. 선택 사항입니다.

  브라우저 컨텍스트의 모든 페이지에서 평가될 스크립트입니다.
- `arg` [Serializable] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-add-init-script-option-arg"/><a href="#browser-context-add-init-script-option-arg" class="list-anchor">#</a>

  [script](/api/class-browsercontext.mdx#browser-context-add-init-script-option-script)에 전달할 선택적 인수 (함수를 전달할 때만 지원됨).

**반환값**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-add-init-script-return"/><a href="#browser-context-add-init-script-return" class="list-anchor">#</a>

### exposeBinding {#browser-context-expose-binding}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browserContext.exposeBinding</x-search>

이 메서드는 컨텍스트 내 모든 페이지의 모든 프레임에 있는 `window` 객체에 [name](/api/class-browsercontext.mdx#browser-context-expose-binding-option-name)이라는 함수를 추가합니다. 호출되면 이 함수는 [callback](/api/class-browsercontext.mdx#browser-context-expose-binding-option-callback)을 실행하고 [callback](/api/class-browsercontext.mdx#browser-context-expose-binding-option-callback)의 반환 값으로 확인되는 [Promise]를 반환합니다. [callback](/api/class-browsercontext.mdx#browser-context-expose-binding-option-callback)이 [Promise]를 반환하면 기다립니다.

[callback](/api/class-browsercontext.mdx#browser-context-expose-binding-option-callback) 함수의 첫 번째 인수에는 호출자에 대한 정보가 포함됩니다: `{ browserContext: BrowserContext, page: Page, frame: Frame }`.

페이지 전용 버전은 [page.exposeBinding()](/api/class-page.mdx#page-expose-binding)을 참조하십시오.

**사용법**

컨텍스트의 모든 페이지에 있는 모든 프레임에 페이지 URL을 노출하는 예제:

```js
const { webkit } = require('playwright');  // 또는 'chromium' 또는 'firefox'.

(async () => {
  const browser = await webkit.launch({ headless: false });
  const context = await browser.newContext();
  await context.exposeBinding('pageURL', ({ page }) => page.url());
  const page = await context.newPage();
  await page.setContent(`
    <script>
      async function onClick() {
        document.querySelector('div').textContent = await window.pageURL();
      }
    </script>
    <button onclick="onClick()">Click me</button>
    <div></div>
  `);
  await page.getByRole('button').click();
})();
```

**인수**
- `name` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-expose-binding-option-name"/><a href="#browser-context-expose-binding-option-name" class="list-anchor">#</a>

  window 객체에 있는 함수의 이름입니다.
- `callback` [function]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-expose-binding-option-callback"/><a href="#browser-context-expose-binding-option-callback" class="list-anchor">#</a>

  Playwright 컨텍스트에서 호출될 콜백 함수입니다.
- `options` [Object] *(선택 사항)*
  - `handle` [boolean] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-expose-binding-option-handle"/><a href="#browser-context-expose-binding-option-handle" class="list-anchor">#</a>

    :::warning[지원 중단됨]
    이 옵션은 향후 제거될 예정입니다.
    :::


    값을 전달하는 대신 핸들로 인수를 전달할지 여부입니다. 핸들을 전달할 때는 하나의 인수만 지원됩니다. 값으로 전달할 때는 여러 인수가 지원됩니다.

**반환값**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-expose-binding-return"/><a href="#browser-context-expose-binding-return" class="list-anchor">#</a>

### exposeFunction {#browser-context-expose-function}

**사용법**

```js

const crypto = require('crypto');

(async () => {
  const browser = await webkit.launch({ headless: false });
  const context = await browser.newContext();
  await context.exposeFunction('sha256', text =>
    crypto.createHash('sha256').update(text).digest('hex'),
  );
  const page = await context.newPage();
  await page.setContent(`
    <script>
      async function onClick() {
        document.querySelector('div').textContent = await window.sha256('PLAYWRIGHT');
      }
    </script>
    <button onclick="onClick()">Click me</button>
    <div></div>
  `);
  await page.getByRole('button').click();
})();
```

- `name` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-expose-function-option-name"/><a href="#browser-context-expose-function-option-name" class="list-anchor">#</a>

### route {#browser-context-route}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browserContext.route</x-search>

라우팅은 브라우저 컨텍스트의 모든 페이지에서 이루어지는 네트워크 요청을 수정하는 기능을 제공합니다. 라우트가 활성화되면 url 패턴과 일치하는 모든 요청은 계속되거나, 처리되거나, 중단될 때까지 지연됩니다.

:::note

[browserContext.route()](/api/class-browsercontext.mdx#browser-context-route)는 서비스 워커에 의해 가로채진 요청을 가로채지 않습니다. [이 이슈](https://github.com/microsoft/playwright/issues/1090)를 참조하십시오. 요청 가로채기를 사용할 때는 [serviceWorkers](/api/class-browser.mdx#browser-new-context-option-service-workers)를 `'block'`으로 설정하여 서비스 워커를 비활성화하는 것을 권장합니다.
:::

**사용법**

모든 이미지 요청을 중단하는 간단한 핸들러 예제:

```js
const context = await browser.newContext();
await context.route('**/*.{png,jpg,jpeg}', route => route.abort());
const page = await context.newPage();
await page.goto('https://example.com');
await browser.close();
```

또는 대신 정규식 패턴을 사용하는 동일한 스니펫:

```js
const context = await browser.newContext();
await context.route(/(\.png$)|(\.jpg$)/, route => route.abort());
const page = await context.newPage();
await page.goto('https://example.com');
await browser.close();
```

라우트 작업을 결정하기 위해 요청을 검사할 수 있습니다. 예를 들어, 특정 post 데이터를 포함하는 모든 요청을 모의 처리하고 다른 모든 요청은 그대로 둡니다:

```js
await context.route('/api/**', async route => {
  if (route.request().postData().includes('my-string'))
    await route.fulfill({ body: 'mocked-data' });
  else
    await route.continue();
});
```

페이지 라우트([page.route()](/api/class-page.mdx#page-route)로 설정됨)는 요청이 두 핸들러 모두와 일치할 때 브라우저 컨텍스트 라우트보다 우선합니다.

핸들러가 있는 라우트를 제거하려면 [browserContext.unroute()](/api/class-browsercontext.mdx#browser-context-unroute)를 사용할 수 있습니다.

:::note

라우팅을 활성화하면 HTTP 캐시가 비활성화됩니다.
:::

**인수**
- `url` [string] | [RegExp] | [function]\([URL]\):[boolean]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-route-option-url"/><a href="#browser-context-route-option-url" class="list-anchor">#</a>

  라우팅하는 동안 일치시킬 glob 패턴, 정규식 패턴 또는 [URL]을 수신하는 조건자입니다. 컨텍스트 옵션을 통해 [baseURL](/api/class-browser.mdx#browser-new-context-option-base-url)이 제공되고 전달된 URL이 경로인 경우 [`new URL()`](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) 생성자를 통해 병합됩니다.
- `handler` [function]\([Route], [Request]\):[Promise]&lt;[Object]&gt; | [Object]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-route-option-handler"/><a href="#browser-context-route-option-handler" class="list-anchor">#</a>

  요청을 라우팅하는 핸들러 함수입니다.
- `options` [Object] *(선택 사항)*
  - `times` [number] *(선택 사항)* <font size="2">v1.15에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-route-option-times"/><a href="#browser-context-route-option-times" class="list-anchor">#</a>

    라우트를 몇 번 사용해야 하는지 지정합니다. 기본적으로 매번 사용됩니다.

**반환값**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-context-route-return"/><a href="#browser-context-route-return" class="list-anchor">#</a>

### on('page') {#browser-context-event-page}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browserContext.on('page')</x-search>

이 이벤트는 BrowserContext에서 새 Page가 생성될 때 발생합니다. 페이지는 여전히 로딩 중일 수 있습니다. 이 이벤트는 팝업 페이지에 대해서도 발생합니다. 특정 페이지와 관련된 팝업에 대한 이벤트를 수신하려면 [page.on('popup')](/api/class-page.mdx#page-event-popup)을 참조하십시오.

페이지를 사용할 수 있는 가장 빠른 시점은 초기 URL로 이동했을 때입니다. 예를 들어, `window.open('http://example.com')`으로 팝업을 열 때 이 이벤트는 "http://example.com"에 대한 네트워크 요청이 완료되고 해당 응답이 팝업에서 로드되기 시작할 때 발생합니다. 이 네트워크 요청을 라우팅하거나 수신하려면 [Page]의 유사한 메서드 대신 각각 [browserContext.route()](/api/class-browsercontext.mdx#browser-context-route) 및 [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request)를 사용하십시오.

```js
const newPagePromise = context.waitForEvent('page');
await page.getByText('open new page').click();
const newPage = await newPagePromise;
console.log(await newPage.evaluate('location.href'));
```

:::note
페이지가 특정 상태에 도달할 때까지 기다리려면 [page.waitForLoadState()](/api/class-page.mdx#page-wait-for-load-state)를 사용하십시오 (대부분의 경우 필요하지 않습니다).
:::

**사용법**

```js
browserContext.on('page', data => {});
```

**이벤트 데이터**
- [Page]

### on('request') {#browser-context-event-request}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.12에 추가됨</font><x-search>browserContext.on('request')</x-search>

이 컨텍스트를 통해 생성된 모든 페이지에서 요청이 발생할 때 발생합니다. [request] 객체는 읽기 전용입니다. 특정 페이지의 요청만 수신하려면 [page.on('request')](/api/class-page.mdx#page-event-request)를 사용하십시오.

요청을 가로채고 수정하려면 [browserContext.route()](/api/class-browsercontext.mdx#browser-context-route) 또는 [page.route()](/api/class-page.mdx#page-route)를 참조하십시오.

**사용법**

```js
browserContext.on('request', data => {});
```

**이벤트 데이터**
- [Request]

### on('requestfailed') {#browser-context-event-request-failed}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.12에 추가됨</font><x-search>browserContext.on('requestfailed')</x-search>

요청이 실패할 때(예: 타임아웃) 발생합니다. 특정 페이지의 실패한 요청만 수신하려면 [page.on('requestfailed')](/api/class-page.mdx#page-event-request-failed)를 사용하십시오.

:::note

404 또는 503과 같은 HTTP 오류 응답은 HTTP 관점에서 여전히 성공적인 응답이므로 요청은 [browserContext.on('requestfailed')](/api/class-browsercontext.mdx#browser-context-event-request-failed) 이벤트가 아닌 [browserContext.on('requestfinished')](/api/class-browsercontext.mdx#browser-context-event-request-finished) 이벤트로 완료됩니다.
:::

**사용법**

```js
browserContext.on('requestfailed', data => {});
```

**이벤트 데이터**
- [Request]

browserContext.on('requestfinished', data => {});
