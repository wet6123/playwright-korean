[GenericAssertions] 클래스는 테스트에서 모든 값에 대한 어설션을 만드는 데 사용할 수 있는 어설션 메서드를 제공합니다. [GenericAssertions]의 새 인스턴스는 [expect()](/api/class-playwrightassertions.mdx#playwright-assertions-expect-generic)를 호출하여 생성됩니다.

```js
import { test, expect } from '@playwright/test';

test('assert a value', async ({ page }) => {
  const value = 1;
  expect(value).toBe(2);
});
```

## Methods

### any {#generic-assertions-any}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.any</x-search>

`expect.any()`는 [constructor](/api/class-genericassertions.mdx#generic-assertions-any-option-constructor)에서 생성된 모든 객체 인스턴스 또는 해당 기본 형식을 일치시킵니다. 패턴 매칭을 수행하려면 [expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal) 내에서 사용하세요.

**Usage**

```js
// Match instance of a class. (클래스의 인스턴스 일치)
class Example {}
expect(new Example()).toEqual(expect.any(Example));

// Match any number. (모든 숫자 일치)
expect({ prop: 1 }).toEqual({ prop: expect.any(Number) });

// Match any string. (모든 문자열 일치)
expect('abc').toEqual(expect.any(String));
```

**Arguments**
- `constructor` [Function]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="generic-assertions-any-option-constructor"/><a href="#generic-assertions-any-option-constructor" class="list-anchor">#</a>
  
  `ExampleClass`와 같은 예상 객체의 생성자 또는 `Number`와 같은 기본 박스형 형식입니다.

### anything {#generic-assertions-anything}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.anything</x-search>

`expect.anything()`은 `null` 및 `undefined`를 제외한 모든 것을 일치시킵니다. 패턴 매칭을 수행하려면 [expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal) 내에서 사용하세요.

**Usage**

```js

expect(value).toEqual({ prop: expect.anything() });
expect(value).not.toEqual({ otherProp: expect.anything() });
```

### arrayContaining {#generic-assertions-array-containing}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.arrayContaining</x-search>

`expect.arrayContaining()`은 예상 배열의 모든 요소를 임의의 순서로 포함하는 배열을 일치시킵니다. 수신된 배열은 예상 배열의 상위 집합일 수 있으며 일부 추가 요소를 포함할 수 있습니다.

패턴 매칭을 수행하려면 이 메서드를 [expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal) 내에서 사용하세요.

**Usage**

```js
expect([1, 2, 3]).toEqual(expect.arrayContaining([3, 1]));
expect([1, 2, 3]).not.toEqual(expect.arrayContaining([1, 4]));
```

**Arguments**
- `expected` [Array]&lt;[Object]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="generic-assertions-array-containing-option-expected"/><a href="#generic-assertions-array-containing-option-expected" class="list-anchor">#</a>

### closeTo {#generic-assertions-close-to}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.closeTo</x-search>

부동 소수점 숫자를 대략적인 같음으로 비교합니다. 패턴 매칭을 수행하려면 이 메서드를 [expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal) 내에서 사용하세요. 두 숫자만 비교할 때는 [expect(value).toBeCloseTo()](/api/class-genericassertions.mdx#generic-assertions-to-be-close-to)를 사용하는 것이 좋습니다.

**Usage**


### objectContaining {#generic-assertions-object-containing}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.objectContaining</x-search>

`expect.objectContaining()`은 예상 객체의 모든 속성을 포함하고 일치시키는 객체를 일치시킵니다. 수신된 객체는 예상 객체의 상위 집합일 수 있으며 일부 추가 속성을 포함할 수 있습니다.

패턴 매칭을 수행하려면 이 메서드를 [expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal) 내에서 사용하세요. 객체 속성은 예상치를 더욱 완화하기 위해 일치자가 될 수 있습니다. 예시를 참조하세요.

**Usage**

```js
// Assert some of the properties. (일부 속성 어설션)
expect({ foo: 1, bar: 2 }).toEqual(expect.objectContaining({ foo: 1 }));

// Matchers can be used on the properties as well. (속성에도 일치자를 사용할 수 있습니다.)
expect({ foo: 1, bar: 2 }).toEqual(expect.objectContaining({ bar: expect.any(Number) }));

// Complex matching of sub-properties. (하위 속성의 복잡한 일치)
expect({
  list: [1, 2, 3],
  obj: { prop: 'Hello world!', another: 'some other value' },
  extra: 'extra',
}).toEqual(expect.objectContaining({
  list: expect.arrayContaining([2, 3]),
  obj: expect.objectContaining({ prop: expect.stringContaining('Hello') }),
}));
```

**Arguments**
- `expected` [Object]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="generic-assertions-object-containing-option-expected"/><a href="#generic-assertions-object-containing-option-expected" class="list-anchor">#</a>
  
  속성의 하위 집합을 포함하는 예상 객체 패턴입니다.

### stringMatching {#generic-assertions-string-matching}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.stringMatching</x-search>

`expect.stringMatching()`은 차례로 예상 패턴과 일치하는 수신된 문자열을 일치시킵니다. 패턴 매칭을 수행하려면 이 메서드를 [expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal) 내에서 사용하세요.

**Usage**

```js

// Inside another matcher. (다른 일치자 내부)

  status: 'passed',
  time: '123ms',
}).toEqual({
  status: expect.stringMatching(/passed|failed/),
  time: expect.stringMatching(/\d+m?s/),
});
```

**Arguments**
- `expected` [string] | [RegExp]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="generic-assertions-string-matching-option-expected"/><a href="#generic-assertions-string-matching-option-expected" class="list-anchor">#</a>
  
  예상 문자열이 일치해야 하는 패턴입니다.

### toEqual {#generic-assertions-to-equal}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.toEqual</x-search>

"깊은 같음" 검사를 수행하여 값의 내용을 [expected](/api/class-genericassertions.mdx#generic-assertions-to-equal-option-expected)의 내용과 비교합니다.

객체의 경우 이 메서드는 [expect(value).toBe()](/api/class-genericassertions.mdx#generic-assertions-to-be)에서 수행하는 것처럼 객체를 참조로 비교하는 대신 모든 필드의 같음을 재귀적으로 확인합니다.

기본 값의 경우 이 메서드는 [expect(value).toBe()](/api/class-genericassertions.mdx#generic-assertions-to-be)과 같습니다.

**Usage**

```js
const value = { prop: 1 };
expect(value).toEqual({ prop: 1 });
```

**Non-strict equality (엄격하지 않은 같음)**

[expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal)은 수신된 값과 예상 값의 내용을 비교하는 깊은 같음 검사를 수행합니다. 두 객체가 동일한 인스턴스를 참조하는지 확인하려면 대신 [expect(value).toBe()](/api/class-genericassertions.mdx#generic-assertions-to-be)를 사용하세요.

[expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal)은 `undefined` 속성 및 배열 항목을 무시하고 객체 유형이 동일해야 한다고 주장하지 않습니다. 보다 엄격한 일치의 경우 [expect(value).toStrictEqual()](/api/class-genericassertions.mdx#generic-assertions-to-strict-equal)을 사용하세요.

**Pattern matching (패턴 매칭)**

[expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal)은 다음 일치자를 사용하여 객체, 배열 및 기본 형식에 대한 패턴 매칭을 수행하는 데 사용할 수도 있습니다.
* [expect(value).any()](/api/class-genericassertions.mdx#generic-assertions-any)
* [expect(value).anything()](/api/class-genericassertions.mdx#generic-assertions-anything)
* [expect(value).arrayContaining()](/api/class-genericassertions.mdx#generic-assertions-array-containing)
* [expect(value).closeTo()](/api/class-genericassertions.mdx#generic-assertions-close-to)
* [expect(value).objectContaining()](/api/class-genericassertions.mdx#generic-assertions-object-containing)
* [expect(value).stringContaining()](/api/class-genericassertions.mdx#generic-assertions-string-containing)
* [expect(value).stringMatching()](/api/class-genericassertions.mdx#generic-assertions-string-matching)

다음은 복잡한 객체 내부의 일부 값을 어설션하는 예입니다.

```js
expect({
  list: [1, 2, 3],
  obj: { prop: 'Hello world!', another: 'some other value' },
  extra: 'extra',
}).toEqual(expect.objectContaining({
  list: expect.arrayContaining([2, 3]),
  obj: expect.objectContaining({ prop: expect.stringContaining('Hello') }),
}));
```

**Arguments**
- `expected` [Object]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="generic-assertions-to-equal-option-expected"/><a href="#generic-assertions-to-equal-option-expected" class="list-anchor">#</a>
  
  예상 값입니다.

### toHaveProperty {#generic-assertions-to-have-property}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.toHaveProperty</x-search>

제공된 `keyPath`의 속성이 객체에 있는지 확인하고 선택적으로 속성이 [expected](/api/class-genericassertions.mdx#generic-assertions-to-have-property-option-expected)와 같은지 확인합니다. 같음은 [expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal)과 유사하게 재귀적으로 확인됩니다.

```js
const value = {
```

- `expected` [Object] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="generic-assertions-to-have-property-option-expected"/><a href="#generic-assertions-to-have-property-option-expected" class="list-anchor">#</a>

### toMatchObject {#generic-assertions-to-match-object}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.toMatchObject</x-search>

"깊은 같음" 검사를 수행하여 값의 내용을 [expected](/api/class-genericassertions.mdx#generic-assertions-to-match-object-option-expected)의 내용과 비교합니다. [expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal)과 달리 값에 추가 속성이 있을 수 있으므로 객체 속성의 하위 집합만 확인할 수 있습니다.

배열을 비교할 때 항목 수가 일치해야 하며 각 항목은 재귀적으로 확인됩니다.

```js
expect([{ a: 1, b: 2 }]).toMatchObject([{ a: 1 }]);
```

**Arguments**
- `expected` [Object] | [Array]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="generic-assertions-to-match-object-option-expected"/><a href="#generic-assertions-to-match-object-option-expected" class="list-anchor">#</a>

### toStrictEqual {#generic-assertions-to-strict-equal}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.toStrictEqual</x-search>

값의 내용과 [expected](/api/class-genericassertions.mdx#generic-assertions-to-strict-equal-option-expected)의 내용 **및** 해당 유형을 비교합니다.

[expect(value).toEqual()](/api/class-genericassertions.mdx#generic-assertions-to-equal)과의 차이점:
* undefined 속성이 있는 키가 확인됩니다. 예를 들어 `{ a: undefined, b: 2 }`는 `{ b: 2 }`와 일치하지 않습니다.
* 배열 희소성이 확인됩니다. 예를 들어 `[, 1]`은 `[undefined, 1]`과 일치하지 않습니다.
* 객체 유형이 동일한지 확인됩니다. 예를 들어 필드 `a` 및 `b`가 있는 클래스 인스턴스는 필드 `a` 및 `b`가 있는 리터럴 객체와 같지 않습니다.

**Usage**

```js
const value = { prop: 1 };
expect(value).toStrictEqual({ prop: 1 });
```

**Arguments**
- `expected` [Object]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="generic-assertions-to-strict-equal-option-expected"/><a href="#generic-assertions-to-strict-equal-option-expected" class="list-anchor">#</a>
  
  예상 값입니다.

### toThrow {#generic-assertions-to-throw}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>genericAssertions.toThrow</x-search>

함수를 호출하고 오류가 발생하는지 확인합니다.

선택적으로 오류를 [예상된 값](/api/class-genericassertions.mdx#generic-assertions-to-throw-option-expected)과 비교합니다. 허용되는 예상 값:
* 정규 표현식 - 오류 메시지는 패턴과 **일치**해야 합니다.
* 문자열 - 오류 메시지는 부분 문자열을 **포함**해야 합니다.
* Error 객체 - 오류 메시지는 객체의 message 속성과 **같아야** 합니다.
* Error 클래스 - 오류 객체는 해당 클래스의 **인스턴스**여야 합니다.

**사용법**

```js
expect(() => {
  throw new Error('Something bad');
}).toThrow();

expect(() => {
  throw new Error('Something bad');
}).toThrow(/something/);

expect(() => {
  throw new Error('Something bad');
}).toThrow(Error);
```

**인수**
- `expected` [Object] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="generic-assertions-to-throw-option-expected"/><a href="#generic-assertions-to-throw-option-expected" class="list-anchor">#</a>
  
  예상되는 오류 메시지 또는 오류 객체.
