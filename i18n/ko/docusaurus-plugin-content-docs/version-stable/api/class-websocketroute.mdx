[`WebSocket`](https://developer.mozilla.org/en-US/docs/Web/API/WebSocket) 경로가 [page.routeWebSocket()](/api/class-page.mdx#page-route-web-socket) 또는 [browserContext.routeWebSocket()](/api/class-browsercontext.mdx#browser-context-route-web-socket)으로 설정될 때마다 `WebSocketRoute` 객체는 실제 서버처럼 WebSocket을 처리할 수 있게 해줍니다.

**모의(Mocking)**

기본적으로 라우팅된 WebSocket은 서버에 연결되지 않습니다. 이렇게 하면 WebSocket을 통해 전체 통신을 모의할 수 있습니다. 다음은 `"request"`에 `"response"`로 응답하는 예제입니다.

```js
await page.routeWebSocket('wss://example.com/ws', ws => {
  ws.onMessage(message => {
    if (message === 'request')
      ws.send('response');
  });
});
```

WebSocket 경로 핸들러 내에서 [webSocketRoute.connectToServer()](/api/class-websocketroute.mdx#web-socket-route-connect-to-server)를 호출하지 않으므로 Playwright는 WebSocket이 모의될 것으로 가정하고 페이지 내에서 자동으로 WebSocket을 엽니다.

다음은 JSON 메시지를 처리하는 또 다른 예제입니다.

```js
await page.routeWebSocket('wss://example.com/ws', ws => {
  ws.onMessage(message => {
    const json = JSON.parse(message);
    if (json.request === 'question')
      ws.send(JSON.stringify({ response: 'answer' }));
  });
});
```

**가로채기(Intercepting)**

또는 실제 서버에 연결하되 중간에 메시지를 가로채서 수정하거나 차단할 수 있습니다. [webSocketRoute.connectToServer()](/api/class-websocketroute.mdx#web-socket-route-connect-to-server)를 호출하면 메시지를 보내거나 수신 메시지를 처리할 수 있는 서버 측 `WebSocketRoute` 인스턴스가 반환됩니다.

다음은 페이지에서 서버로 전송된 일부 메시지를 수정하는 예제입니다. 서버에서 페이지로 전송된 메시지는 기본 전달에 의존하여 그대로 유지됩니다.

```js
await page.routeWebSocket('/ws', ws => {
  const server = ws.connectToServer();
  ws.onMessage(message => {
    if (message === 'request')
      server.send('request2');
    else
      server.send(message);
  });
});
```

서버에 연결되면 모든 **메시지가 기본적으로 페이지와 서버 간에 전달**됩니다.

그러나 원래 경로에서 [webSocketRoute.onMessage()](/api/class-websocketroute.mdx#web-socket-route-on-message)를 호출하면 페이지에서 서버로의 메시지는 **더 이상 전달되지 않으며** 대신 [handler](/api/class-websocketroute.mdx#web-socket-route-on-message-option-handler)에서 처리해야 합니다.

마찬가지로 서버 측 WebSocket에서 [webSocketRoute.onMessage()](/api/class-websocketroute.mdx#web-socket-route-on-message)를 호출하면 서버에서 페이지로의 **메시지 전달이 중지**되고 [handler](/api/class-websocketroute.mdx#web-socket-route-on-message-option-handler)가 처리해야 합니다.

다음 예제는 양방향으로 일부 메시지를 차단합니다. [webSocketRoute.onMessage()](/api/class-websocketroute.mdx#web-socket-route-on-message)를 양방향으로 호출하므로 자동 전달이 전혀 없습니다.

```js
await page.routeWebSocket('/ws', ws => {
  const server = ws.connectToServer();
  ws.onMessage(message => {
    if (message !== 'blocked-from-the-page')
      server.send(message);
  });
  server.onMessage(message => {
    if (message !== 'blocked-from-the-server')
      ws.send(message);
  });
});
```

### close {#web-socket-route-close}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.48</font><x-search>webSocketRoute.close</x-search>

WebSocket 연결의 한쪽을 닫습니다.

**사용법**

```js
await webSocketRoute.close();
await webSocketRoute.close(options);
```

**인수**
- `options` [Object] *(선택 사항)*
  - `code` [number] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-route-close-option-code"/><a href="#web-socket-route-close-option-code" class="list-anchor">#</a>
    
    선택적 [종료 코드](https://developer.mozilla.org/en-US/docs/Web/API/WebSocket/close#code).
  - `reason` [string] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-route-close-option-reason"/><a href="#web-socket-route-close-option-reason" class="list-anchor">#</a>
    
    선택적 [종료 사유](https://developer.mozilla.org/en-US/docs/Web/API/WebSocket/close#reason).

**반환 값**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-route-close-return"/><a href="#web-socket-route-close-return" class="list-anchor">#</a>

### connectToServer {#web-socket-route-connect-to-server}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.48</font><x-search>webSocketRoute.connectToServer</x-search>

기본적으로 라우팅된 WebSocket은 서버에 연결되지 않으므로 전체 WebSocket 통신을 모의할 수 있습니다. 이 메서드는 실제 WebSocket 서버에 연결하고 서버 측 [WebSocketRoute] 인스턴스를 반환하여 서버에서 메시지를 보내고 받을 수 있도록 합니다.

서버에 연결되면:
* 서버에서 받은 메시지는 서버 측 `WebSocketRoute`에서 [webSocketRoute.onMessage()](/api/class-websocketroute.mdx#web-socket-route-on-message)가 호출되지 않는 한 **자동으로 페이지의 WebSocket으로 전달**됩니다.
* 페이지의 [`WebSocket.send()`](https://developer.mozilla.org/en-US/docs/Web/API/WebSocket/send) 호출에 의해 전송된 메시지는 원래 `WebSocketRoute`에서 [webSocketRoute.onMessage()](/api/class-websocketroute.mdx#web-socket-route-on-message)가 호출되지 않는 한 **자동으로 서버로 전달**됩니다.

자세한 내용은 맨 위의 예제를 참조하세요.

**사용법**

```js
webSocketRoute.connectToServer();
```

**반환 값**
- [WebSocketRoute]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-route-connect-to-server-return"/><a href="#web-socket-route-connect-to-server-return" class="list-anchor">#</a>

### onClose {#web-socket-route-on-close}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.48</font><x-search>webSocketRoute.onClose</x-search>

[`WebSocket.close`](https://developer.mozilla.org/en-US/docs/Web/API/WebSocket/close)를 처리할 수 있습니다.

기본적으로 페이지나 서버에서 연결의 한쪽을 닫으면 다른 쪽이 닫힙니다. 그러나 [webSocketRoute.onClose()](/api/class-websocketroute.mdx#web-socket-route-on-close) 핸들러가 설정되면 기본적으로 종료 전달이 비활성화되고 핸들러가 이를 처리해야 합니다.

**사용법**

```js
webSocketRoute.onClose(handler);
```

**인수**
- `handler` [function]\([number] | [undefined]\):[Promise]&lt;[Object]&gt; | [Object]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-route-on-close-option-handler"/><a href="#web-socket-route-on-close-option-handler" class="list-anchor">#</a>
  
  WebSocket 종료를 처리할 함수입니다. 선택적 [종료 코드](https://developer.mozilla.org/en-US/docs/Web/API/WebSocket/close#code)와 선택적 [종료 사유](https://developer.mozilla.org/en-US/docs/Web/API/WebSocket/close#reason)를 받습니다.

### onMessage {#web-socket-route-on-message}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.48</font><x-search>webSocketRoute.onMessage</x-search>

이 메서드를 사용하면 페이지 또는 서버에서 WebSocket에서 보낸 메시지를 처리할 수 있습니다.

원래 WebSocket 경로에서 호출되면 이 메서드는 페이지에서 보낸 메시지를 처리합니다. [webSocketRoute.send()](/api/class-websocketroute.mdx#web-socket-route-send)로 응답하거나, [webSocketRoute.connectToServer()](/api/class-websocketroute.mdx#web-socket-route-connect-to-server)에서 반환된 서버 측 연결로 전달하거나, 다른 작업을 수행하여 이 메시지를 처리할 수 있습니다.

이 메서드가 호출되면 메시지는 자동으로 서버 또는 페이지로 전달되지 않습니다. [webSocketRoute.send()](/api/class-websocketroute.mdx#web-socket-route-send)를 호출하여 수동으로 수행해야 합니다. 자세한 내용은 맨 위의 예제를 참조하세요.

이 메서드를 다시 호출하면 핸들러가 새 핸들러로 재정의됩니다.

**사용법**

```js
webSocketRoute.onMessage(handler);
```

**인수**
- `handler` [function]\([string]\):[Promise]&lt;[Object]&gt; | [Object]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-route-on-message-option-handler"/><a href="#web-socket-route-on-message-option-handler" class="list-anchor">#</a>
  
  메시지를 처리할 함수입니다.

### send {#web-socket-route-send}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.48</font><x-search>webSocketRoute.send</x-search>

WebSocket에 메시지를 보냅니다. 원래 WebSocket에서 호출되면 페이지로 메시지를 보냅니다. [webSocketRoute.connectToServer()](/api/class-websocketroute.mdx#web-socket-route-connect-to-server)의 결과에서 호출되면 서버로 메시지를 보냅니다. 자세한 내용은 맨 위의 예제를 참조하세요.

**사용법**

```js
webSocketRoute.send(message);
```

**인수**
- `message` [string] | [Buffer]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="web-socket-route-send-option-message"/><a href="#web-socket-route-send-option-message" class="list-anchor">#</a>
  
  보낼 메시지입니다.
