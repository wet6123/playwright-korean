[Browser]는 [browserType.launch()](/api/class-browsertype.mdx#browser-type-launch)를 통해 생성됩니다. [Browser]를 사용하여 [Page]를 생성하는 예제입니다:

```js
const { firefox } = require('playwright');  // 또는 'chromium' 또는 'webkit'.

(async () => {
  const browser = await firefox.launch();
  const page = await browser.newPage();
  await page.goto('https://example.com');
  await browser.close();
})();
```

### close {#browser-close}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browser.close</x-search>

이 브라우저가 [browserType.launch()](/api/class-browsertype.mdx#browser-type-launch)를 사용하여 얻어진 경우, 브라우저와 모든 페이지(열려 있는 경우)를 닫습니다.

이 브라우저가 연결된 경우, 이 브라우저에 속하는 모든 생성된 컨텍스트를 지우고 브라우저 서버에서 연결을 끊습니다.

:::note

이는 브라우저를 강제 종료하는 것과 유사합니다. 페이지를 정상적으로 닫고 페이지 닫기 이벤트를 받으려면, [browser.close()](/api/class-browser.mdx#browser-close)를 호출하기 **전에** [browser.newContext()](/api/class-browser.mdx#browser-new-context)를 사용하여 명시적으로 생성한 모든 [BrowserContext] 인스턴스에서 [browserContext.close()](/api/class-browsercontext.mdx#browser-context-close)를 호출하십시오.
:::

[Browser] 객체 자체는 폐기된 것으로 간주되며 더 이상 사용할 수 없습니다.

**사용법**

```js
await browser.close();
await browser.close(options);
```

**인수**
- `options` [Object] *(선택)*
  - `reason` [string] *(선택)* <font size="2">추가된 버전: v1.40</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-close-option-reason"/><a href="#browser-close-option-reason" class="list-anchor">#</a>

    브라우저 종료로 인해 중단된 작업에 보고될 이유입니다.

**반환값**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-close-return"/><a href="#browser-close-return" class="list-anchor">#</a>

### contexts {#browser-contexts}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browser.contexts</x-search>

열려 있는 모든 브라우저 컨텍스트의 배열을 반환합니다. 새로 생성된 브라우저에서는 0개의 브라우저 컨텍스트를 반환합니다.

**사용법**

```js
const browser = await pw.webkit.launch();
console.log(browser.contexts().length); // `0` 출력

const context = await browser.newContext();
console.log(browser.contexts().length); // `1` 출력
```

**반환값**
- [Array]&lt;[BrowserContext]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-contexts-return"/><a href="#browser-contexts-return" class="list-anchor">#</a>

### newContext {#browser-new-context}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browser.newContext</x-search>

새 브라우저 컨텍스트를 생성합니다. 다른 브라우저 컨텍스트와 쿠키/캐시를 공유하지 않습니다.

:::note

이 메서드를 직접 사용하여 [BrowserContext]를 생성하는 경우, 코드가 [BrowserContext] 사용을 마쳤을 때 반환된 컨텍스트를 [browserContext.close()](/api/class-browsercontext.mdx#browser-context-close)를 통해 명시적으로 닫고, [browser.close()](/api/class-browser.mdx#browser-close)를 호출하기 전에 수행하는 것이 가장 좋습니다. 이렇게 하면 `context`가 정상적으로 닫히고 HAR 및 비디오와 같은 아티팩트가 완전히 플러시되고 저장됩니다.
:::

**사용법**

```js
(async () => {
  const browser = await playwright.firefox.launch();  // 또는 'chromium' 또는 'webkit'.
  // 새 시크릿 브라우저 컨텍스트 생성.
  const context = await browser.newContext();
  // 초기 상태 컨텍스트에서 새 페이지 생성.
  const page = await context.newPage();
  await page.goto('https://example.com');

  // 모든 것을 정상적으로 닫습니다.
  await context.close();
  await browser.close();
})();
```

**인수**
- `options` [Object] *(선택)*
  - `acceptDownloads` [boolean] *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-context-option-accept-downloads"/><a href="#browser-new-context-option-accept-downloads" class="list-anchor">#</a>

    모든 첨부 파일을 자동으로 다운로드할지 여부입니다. 기본값은 `true`이며 모든 다운로드가 수락됩니다.
  - `baseURL` [string] *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-context-option-base-url"/><a href="#browser-new-context-option-base-url" class="list-anchor">#</a>

    [page.goto()](/api/class-page.mdx#page-goto), [page.route()](/api/class-page.mdx#page-route), [page.waitForURL()](/api/class-page.mdx#page-wait-for-url), [page.waitForRequest()](/api/class-page.mdx#page-wait-for-request) 또는 [page.waitForResponse()](/api/class-page.mdx#page-wait-for-response)를 사용할 때, 해당 URL을 구축하기 위해 [`URL()`](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) 생성자를 사용하여 기본 URL을 고려합니다. 기본적으로 설정되지 않습니다. 예시:
    * baseURL: `http://localhost:3000` 이고 `/bar.html`로 이동하면 `http://localhost:3000/bar.html`이 됩니다.
    * baseURL: `http://localhost:3000/foo/` 이고 `./bar.html`로 이동하면 `http://localhost:3000/foo/bar.html`이 됩니다.
    * baseURL: `http://localhost:3000/foo` (후행 슬래시 없음) 이고 `./bar.html`로 이동하면 `http://localhost:3000/bar.html`이 됩니다.
  - `bypassCSP` [boolean] *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-context-option-bypass-csp"/><a href="#browser-new-context-option-bypass-csp" class="list-anchor">#</a>

    페이지의 Content-Security-Policy를 우회할지 여부를 토글합니다. 기본값은 `false`입니다.
  - `clientCertificates` [Array]&lt;[Object]&gt; *(선택)* <font size="2">추가된 버전: 1.46</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-context-option-client-certificates"/><a href="#browser-new-context-option-client-certificates" class="list-anchor">#</a>
    - `origin` [string]

      인증서가 유효한 정확한 출처입니다. 출처에는 `https` 프로토콜, 호스트 이름 및 선택적으로 포트가 포함됩니다.
    - `certPath` [string] *(선택)*

      PEM 형식의 인증서 파일 경로입니다.
    - `cert` [Buffer] *(선택)*

      PEM 형식의 인증서 직접 값입니다.
    - `keyPath` [string] *(선택)*

      PEM 형식의 개인 키 파일 경로입니다.
    - `key` [Buffer] *(선택)*

      PEM 형식의 개인 키 직접 값입니다.
    - `pfxPath` [string] *(선택)*

      PFX 또는 PKCS12로 인코딩된 개인 키 및 인증서 체인 파일 경로입니다.
    - `pfx` [Buffer] *(선택)*

      PFX 또는 PKCS12로 인코딩된 개인 키 및 인증서 체인의 직접 값입니다.
    - `passphrase` [string] *(선택)*

      개인 키(PEM 또는 PFX)의 암호입니다.

    TLS 클라이언트 인증은 서버가 클라이언트 인증서를 요청하고 검증할 수 있도록 합니다.

    **상세 정보**

    사용할 클라이언트 인증서 배열입니다. 각 인증서 객체는 `certPath`와 `keyPath` 둘 다, 단일 `pfxPath`, 또는 해당 직접 값(`cert`와 `key`, 또는 `pfx`)을 가져야 합니다. 선택적으로 인증서가 암호화된 경우 `passphrase` 속성을 제공해야 합니다. `origin` 속성은 인증서가 유효한 요청 출처와 정확히 일치하도록 제공해야 합니다.

    :::note

    macOS에서 WebKit을 사용할 때 `localhost`에 접근하면 클라이언트 인증서를 선택하지 않습니다. `localhost`를 `local.playwright`로 대체하여 작동하도록 할 수 있습니다.
    :::

  - `colorScheme` [null] | "light" | "dark" | "no-preference" *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-context-option-color-scheme"/><a href="#browser-new-context-option-color-scheme" class="list-anchor">#</a>

    [prefers-colors-scheme](https://developer.mozilla.org/en-US/docs/Web/CSS/@media/prefers-color-scheme) 미디어 기능을 에뮬레이트합니다. 지원되는 값은 `'light'` 및 `'dark'`입니다. 자세한 내용은 [page.emulateMedia()](/api/class-page.mdx#page-emulate-media)를 참조하세요. `null`을 전달하면 시스템 기본값으로 에뮬레이션을 재설정합니다. 기본값은 `'light'`입니다.
  - `contrast` [null] | "no-preference" | "more" *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-context-option-contrast"/><a href="#browser-new-context-option-contrast" class="list-anchor">#</a>

    `'prefers-contrast'` 미디어 기능을 에뮬레이트합니다. 지원되는 값은 `'no-preference'`, `'more'`입니다. 자세한 내용은 [page.emulateMedia()](/api/class-page.mdx#page-emulate-media)를 참조하세요. `null`을 전달하면 시스템 기본값으로 에뮬레이션을 재설정합니다. 기본값은 `'no-preference'`입니다.
  - `deviceScaleFactor` [number] *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-context-option-device-scale-factor"/><a href="#browser-new-context-option-device-scale-factor" class="list-anchor">#</a>

    장치 스케일 팩터(dpr로 생각할 수 있음)를 지정합니다. 기본값은 `1`입니다. [장치 스케일 팩터로 장치 에뮬레이트하기](../emulation.mdx#devices)에 대해 자세히 알아보세요.
  - `extraHTTPHeaders` [Object]&lt;[string], [string]&gt; *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-context-option-extra-http-headers"/><a href="#browser-new-context-option-extra-http-headers" class="list-anchor">#</a>

    모든 요청과 함께 보낼 추가 HTTP 헤더를 포함하는 객체입니다. 기본값은 없습니다.
  - `forcedColors` [null] | "active" | "none" *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-context-option-forced-colors"/><a href="#browser-new-context-option-forced-colors" class="list-anchor">#</a>

    `'forced-colors'` 미디어 기능을 에뮬레이트합니다. 지원되는 값은 `'active'`, `'none'`입니다. 자세한 내용은 [page.emulateMedia()](/api/class-page.mdx#page-emulate-media)를 참조하세요. `null`을 전달하면 시스템 기본값으로 에뮬레이션을 재설정합니다. 기본값은 `'none'`입니다.

### newPage {#browser-new-page}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browser.newPage</x-search>

새 브라우저 컨텍스트에서 새 페이지를 생성합니다. 이 페이지를 닫으면 컨텍스트도 닫힙니다.

이는 단일 페이지 시나리오와 짧은 스니펫에만 사용해야 하는 편의 API입니다. 프로덕션 코드와 테스트 프레임워크는 정확한 수명을 제어하기 위해 명시적으로 [browser.newContext()](/api/class-browser.mdx#browser-new-context)를 생성한 다음 [browserContext.newPage()](/api/class-browsercontext.mdx#browser-context-new-page)를 사용해야 합니다.

**사용법**

```js
await browser.newPage();
await browser.newPage(options);
```

**인수**
- `options` [Object] *(선택)*
  - `acceptDownloads` [boolean] *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-page-option-accept-downloads"/><a href="#browser-new-page-option-accept-downloads" class="list-anchor">#</a>

    모든 첨부 파일을 자동으로 다운로드할지 여부입니다. 기본값은 `true`이며 모든 다운로드가 수락됩니다.
  - `baseURL` [string] *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-page-option-base-url"/><a href="#browser-new-page-option-base-url" class="list-anchor">#</a>

    [page.goto()](/api/class-page.mdx#page-goto), [page.route()](/api/class-page.mdx#page-route), [page.waitForURL()](/api/class-page.mdx#page-wait-for-url), [page.waitForRequest()](/api/class-page.mdx#page-wait-for-request) 또는 [page.waitForResponse()](/api/class-page.mdx#page-wait-for-response)를 사용할 때, 해당 URL을 구축하기 위해 [`URL()`](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) 생성자를 사용하여 기본 URL을 고려합니다. 기본적으로 설정되지 않습니다. 예시:
    * baseURL: `http://localhost:3000` 이고 `/bar.html`로 이동하면 `http://localhost:3000/bar.html`이 됩니다.
    * baseURL: `http://localhost:3000/foo/` 이고 `./bar.html`로 이동하면 `http://localhost:3000/foo/bar.html`이 됩니다.
    * baseURL: `http://localhost:3000/foo` (후행 슬래시 없음) 이고 `./bar.html`로 이동하면 `http://localhost:3000/bar.html`이 됩니다.
  - `bypassCSP` [boolean] *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-page-option-bypass-csp"/><a href="#browser-new-page-option-bypass-csp" class="list-anchor">#</a>

    페이지의 Content-Security-Policy를 우회할지 여부를 토글합니다. 기본값은 `false`입니다.
  - `clientCertificates` [Array]&lt;[Object]&gt; *(선택)* <font size="2">추가된 버전: 1.46</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-new-page-option-client-certificates"/><a href="#browser-new-page-option-client-certificates" class="list-anchor">#</a>
    - `origin` [string]

      인증서가 유효한 정확한 출처입니다. 출처에는 `https` 프로토콜, 호스트 이름 및 선택적으로 포트가 포함됩니다.
    - `certPath` [string] *(선택)*

      PEM 형식의 인증서 파일 경로입니다.
    - `cert` [Buffer] *(선택)*

      PEM 형식의 인증서 직접 값입니다.
    - `keyPath` [string] *(선택)*

      PEM 형식의 개인 키 파일 경로입니다.
    - `key` [Buffer] *(선택)*

      PEM 형식의 개인 키 직접 값입니다.
    - `pfxPath` [string] *(선택)*

      PFX 또는 PKCS12로 인코딩된 개인 키 및 인증서 체인 파일 경로입니다.
    - `pfx` [Buffer] *(선택)*

      PFX 또는 PKCS12로 인코딩된 개인 키 및 인증서 체인의 직접 값입니다.
    - `passphrase` [string] *(선택)*

      개인 키(PEM 또는 PFX)의 암호입니다.

    TLS 클라이언트 인증은 서버가 클라이언트 인증서를 요청하고 검증할 수 있도록 합니다.

    **상세 정보**

    사용할 클라이언트 인증서 배열입니다. 각 인증서 객체는 `certPath`와 `keyPath` 둘 다, 단일 `pfxPath`, 또는 해당 직접 값(`cert`와 `key`, 또는 `pfx`)을 가져야 합니다. 선택적으로 인증서가 암호화된 경우 `passphrase` 속성을 제공해야 합니다. `origin` 속성은 인증서가 유효한 요청 출처와 정확히 일치하도록 제공해야 합니다.

    :::note

### removeAllListeners {#browser-remove-all-listeners}

<font size="2" style={{position: "relative", top: "-20px"}}>추가된 버전: v1.47</font><x-search>browser.removeAllListeners</x-search>

주어진 유형의 모든 리스너(또는 유형이 주어지지 않은 경우 등록된 모든 리스너)를 제거합니다. 비동기 리스너가 완료될 때까지 기다리거나 이러한 리스너에서 발생하는 후속 오류를 무시할 수 있습니다.

**사용법**

```js
await browser.removeAllListeners();
await browser.removeAllListeners(type, options);
```

**인수**
- `type` [string] *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-remove-all-listeners-option-type"/><a href="#browser-remove-all-listeners-option-type" class="list-anchor">#</a>
- `options` [Object] *(선택)*
  - `behavior` "wait" | "ignoreErrors" | "default" *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-remove-all-listeners-option-behavior"/><a href="#browser-remove-all-listeners-option-behavior" class="list-anchor">#</a>

    이미 실행 중인 리스너를 기다릴지 여부와 오류 발생 시 수행할 작업을 지정합니다:
    * `'default'` - 현재 리스너 호출(있는 경우)이 완료될 때까지 기다리지 않음. 리스너가 오류를 발생시키면 처리되지 않은 오류가 발생할 수 있음
    * `'wait'` - 현재 리스너 호출(있는 경우)이 완료될 때까지 기다림
    * `'ignoreErrors'` - 현재 리스너 호출(있는 경우)이 완료될 때까지 기다리지 않음. 제거 후 리스너에서 발생한 모든 오류는 자동으로 처리됨

**반환값**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-remove-all-listeners-return"/><a href="#browser-remove-all-listeners-return" class="list-anchor">#</a>

  - `categories` [Array]&lt;[string]&gt; *(선택)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-start-tracing-option-categories"/><a href="#browser-start-tracing-option-categories" class="list-anchor">#</a>
