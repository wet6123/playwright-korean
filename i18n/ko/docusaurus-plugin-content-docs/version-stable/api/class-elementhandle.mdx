* extends: [JSHandle]

ElementHandle은 페이지 내 DOM 요소를 나타냅니다. ElementHandle은 [page.$()](/api/class-page.mdx#page-query-selector) 메서드로 만들 수 있습니다.

:::warning[Discouraged]

ElementHandle 사용은 권장되지 않으며, 대신 [Locator] 객체 및 웹 우선 어설션을 사용하세요.
:::

```js
const hrefElement = await page.$('a');
await hrefElement.click();
```

ElementHandle은 [jsHandle.dispose()](/api/class-jshandle.mdx#js-handle-dispose)로 핸들을 삭제하지 않는 한 DOM 요소가 가비지 컬렉션되지 않도록 합니다. ElementHandle은 원본 프레임이 탐색될 때 자동으로 삭제됩니다.

ElementHandle 인스턴스는 [page.$eval()](/api/class-page.mdx#page-eval-on-selector) 및 [page.evaluate()](/api/class-page.mdx#page-evaluate) 메서드에서 인수로 사용할 수 있습니다.

[Locator]와 ElementHandle의 차이점은 ElementHandle은 특정 요소를 가리키는 반면, [Locator]는 요소를 검색하는 방법을 캡처한다는 것입니다.

아래 예제에서 handle은 페이지의 특정 DOM 요소를 가리킵니다. 해당 요소의 텍스트가 변경되거나 React에서 완전히 다른 컴포넌트를 렌더링하는 데 사용되더라도 handle은 여전히 해당 DOM 요소를 가리킵니다. 이로 인해 예기치 않은 동작이 발생할 수 있습니다.

```js
const handle = await page.$('text=Submit');
// ...
await handle.hover();
await handle.click();
```

로케이터를 사용하면 `element`가 사용될 때마다 선택기를 사용하여 최신 DOM 요소가 페이지에서 검색됩니다. 따라서 아래 코드 조각에서 기본 DOM 요소는 두 번 검색됩니다.

```js
const locator = page.getByText('Submit');
// ...
await locator.hover();
await locator.click();
```

### boundingBox {#element-handle-bounding-box}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>elementHandle.boundingBox</x-search>

이 메서드는 요소의 경계 상자를 반환하고, 요소가 보이지 않으면 `null`을 반환합니다. 경계 상자는 기본 프레임 뷰포트를 기준으로 계산됩니다. 이는 일반적으로 브라우저 창과 동일합니다.

스크롤은 [Element.getBoundingClientRect](https://developer.mozilla.org/en-US/docs/Web/API/Element/getBoundingClientRect)와 유사하게 반환된 경계 상자에 영향을 줍니다. 즉, `x` 및/또는 `y`가 음수일 수 있습니다.

자식 프레임의 요소는 [Element.getBoundingClientRect](https://developer.mozilla.org/en-US/docs/Web/API/Element/getBoundingClientRect)와 달리 기본 프레임을 기준으로 경계 상자를 반환합니다.

페이지가 정적이라고 가정하면 경계 상자 좌표를 사용하여 입력을 수행하는 것이 안전합니다. 예를 들어 다음 코드 조각은 요소의 가운데를 클릭해야 합니다.

**사용법**

```js
const box = await elementHandle.boundingBox();
await page.mouse.click(box.x + box.width / 2, box.y + box.height / 2);
```

**반환 값**
- [Promise]&lt;[null] | [Object]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-bounding-box-return"/><a href="#element-handle-bounding-box-return" class="list-anchor">#</a>
  - `x` [number]
    
    픽셀 단위의 요소의 x 좌표입니다.
  - `y` [number]
    
    픽셀 단위의 요소의 y 좌표입니다.
  - `width` [number]
    
    픽셀 단위의 요소의 너비입니다.
  - `height` [number]
    
    픽셀 단위의 요소의 높이입니다.

### waitForElementState {#element-handle-wait-for-element-state}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>elementHandle.waitForElementState</x-search>

요소가 [state](/api/class-elementhandle.mdx#element-handle-wait-for-element-state-option-state)를 만족할 때 반환합니다.

[state](/api/class-elementhandle.mdx#element-handle-wait-for-element-state-option-state) 매개변수에 따라 이 메서드는 [동작 가능성](../actionability.mdx) 검사 중 하나가 통과될 때까지 기다립니다. 이 메서드는 `"hidden"` 상태를 기다리는 경우가 아니면 기다리는 동안 요소가 분리되면 예외를 발생시킵니다.
* `"visible"` 요소가 [표시될 때까지](../actionability.mdx#visible) 기다립니다.
* `"hidden"` 요소가 [표시되지 않거나](../actionability.mdx#visible) 연결되지 않을 때까지 기다립니다. 숨김을 기다리는 것은 요소가 분리될 때 예외를 발생시키지 않습니다.
* `"stable"` 요소가 [표시](../actionability.mdx#visible)되고 [안정적](../actionability.mdx#stable)일 때까지 기다립니다.
* `"enabled"` 요소가 [활성화될 때까지](../actionability.mdx#enabled) 기다립니다.
* `"disabled"` 요소가 [활성화되지 않을 때까지](../actionability.mdx#enabled) 기다립니다.
* `"editable"` 요소가 [편집 가능할 때까지](../actionability.mdx#editable) 기다립니다.

요소가 [timeout](/api/class-elementhandle.mdx#element-handle-wait-for-element-state-option-timeout) 밀리초 동안 조건을 만족하지 않으면 이 메서드는 예외를 발생시킵니다.

**사용법**

```js
await elementHandle.waitForElementState(state);
await elementHandle.waitForElementState(state, options);
```

**인수**
- `state` "visible" | "hidden" | "stable" | "enabled" | "disabled" | "editable"<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-wait-for-element-state-option-state"/><a href="#element-handle-wait-for-element-state-option-state" class="list-anchor">#</a>
  
  대기할 상태입니다. 자세한 내용은 아래를 참조하세요.
- `options` [Object] *(선택 사항)*
  - `timeout` [number] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-wait-for-element-state-option-timeout"/><a href="#element-handle-wait-for-element-state-option-timeout" class="list-anchor">#</a>
    
    최대 시간(밀리초)입니다. 기본값은 `0`이며, 제한 시간이 없습니다. 기본값은 구성에서 `actionTimeout` 옵션을 사용하거나 [browserContext.setDefaultTimeout()](/api/class-browsercontext.mdx#browser-context-set-default-timeout) 또는 [page.setDefaultTimeout()](/api/class-page.mdx#page-set-default-timeout) 메서드를 사용하여 변경할 수 있습니다.

**반환 값**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-wait-for-element-state-return"/><a href="#element-handle-wait-for-element-state-return" class="list-anchor">#</a>

### $eval {#element-handle-eval-on-selector}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>elementHandle.$eval</x-search>

:::warning[Discouraged]

이 메서드는 요소가 동작 가능성 검사를 통과할 때까지 기다리지 않으므로 테스트가 불안정해질 수 있습니다. 대신 [locator.evaluate()](/api/class-locator.mdx#locator-evaluate), 다른 [Locator] 도우미 메서드 또는 웹 우선 어설션을 사용하세요.

이 메서드는 `ElementHandle`의 하위 트리에서 지정된 선택기와 일치하는 요소를 찾아 [pageFunction](/api/class-elementhandle.mdx#element-handle-eval-on-selector-option-expression)에 첫 번째 인수로 전달합니다. 선택기와 일치하는 요소가 없으면 이 메서드는 오류를 발생시킵니다.

**사용법**

```js
const tweetHandle = await page.$('.tweet');
```

- `arg` [EvaluationArgument] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-eval-on-selector-option-arg"/><a href="#element-handle-eval-on-selector-option-arg" class="list-anchor">#</a>

### check {#element-handle-check}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>elementHandle.check</x-search>

:::warning[Discouraged]

대신 로케이터 기반 [locator.check()](/api/class-locator.mdx#locator-check)를 사용하세요. [로케이터](../locators.mdx)에 대해 자세히 알아보세요.

:::


이 메서드는 다음 단계를 수행하여 요소를 확인합니다.
1. 요소가 확인란 또는 라디오 입력인지 확인합니다. 그렇지 않으면 이 메서드는 예외를 발생시킵니다. 요소가 이미 선택되어 있으면 이 메서드는 즉시 반환됩니다.
1. [force](/api/class-elementhandle.mdx#element-handle-check-option-force) 옵션이 설정되지 않은 경우 요소에 대한 [동작 가능성](../actionability.mdx) 검사를 기다립니다.
1. 필요한 경우 요소를 뷰로 스크롤합니다.
1. [page.mouse](/api/class-page.mdx#page-mouse)를 사용하여 요소의 가운데를 클릭합니다.
1. 이제 요소가 선택되었는지 확인합니다. 그렇지 않으면 이 메서드는 예외를 발생시킵니다.

작업 중 언제라도 요소가 DOM에서 분리되면 이 메서드는 예외를 발생시킵니다.

지정된 [timeout](/api/class-elementhandle.mdx#element-handle-check-option-timeout) 동안 모든 단계가 완료되지 않으면 이 메서드는 [TimeoutError]를 발생시킵니다. 제한 시간을 0으로 전달하면 이 기능이 비활성화됩니다.

**사용법**

```js
await elementHandle.check();
await elementHandle.check(options);
```

  - `noWaitAfter` [boolean] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-check-option-no-wait-after"/><a href="#element-handle-check-option-no-wait-after" class="list-anchor">#</a>

### dispatchEvent {#element-handle-dispatch-event}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>elementHandle.dispatchEvent</x-search>

:::warning[Discouraged]

대신 로케이터 기반 [locator.dispatchEvent()](/api/class-locator.mdx#locator-dispatch-event)를 사용하세요. [로케이터](../locators.mdx)에 대해 자세히 알아보세요.

:::


아래 코드 조각은 요소에 `click` 이벤트를 디스패치합니다. 요소의 가시성 상태에 관계없이 `click`이 디스패치됩니다. 이는 [element.click()](https://developer.mozilla.org/en-US/docs/Web/API/HTMLElement/click)을 호출하는 것과 동일합니다.

**사용법**

```js
await elementHandle.dispatchEvent('click');
```

내부적으로 주어진 [type](/api/class-elementhandle.mdx#element-handle-dispatch-event-option-type)을 기반으로 이벤트 인스턴스를 생성하고, [eventInit](/api/class-elementhandle.mdx#element-handle-dispatch-event-option-event-init) 속성으로 초기화한 다음 요소에 디스패치합니다. 이벤트는 기본적으로 `composed`, `cancelable`이며 버블링됩니다.

[eventInit](/api/class-elementhandle.mdx#element-handle-dispatch-event-option-event-init)은 이벤트별로 다르므로 초기 속성 목록은 이벤트 문서를 참조하세요:
* [DeviceMotionEvent](https://developer.mozilla.org/en-US/docs/Web/API/DeviceMotionEvent/DeviceMotionEvent)
* [DeviceOrientationEvent](https://developer.mozilla.org/en-US/docs/Web/API/DeviceOrientationEvent/DeviceOrientationEvent)
* [DragEvent](https://developer.mozilla.org/en-US/docs/Web/API/DragEvent/DragEvent)
* [Event](https://developer.mozilla.org/en-US/docs/Web/API/Event/Event)
* [FocusEvent](https://developer.mozilla.org/en-US/docs/Web/API/FocusEvent/FocusEvent)
* [KeyboardEvent](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/KeyboardEvent)
* [MouseEvent](https://developer.mozilla.org/en-US/docs/Web/API/MouseEvent/MouseEvent)
* [PointerEvent](https://developer.mozilla.org/en-US/docs/Web/API/PointerEvent/PointerEvent)
* [TouchEvent](https://developer.mozilla.org/en-US/docs/Web/API/TouchEvent/TouchEvent)
* [WheelEvent](https://developer.mozilla.org/en-US/docs/Web/API/WheelEvent/WheelEvent)

이벤트에 실시간 객체를 전달하려면 속성 값으로 `JSHandle`을 지정할 수도 있습니다:

```js
// Note you can only create DataTransfer in Chromium and Firefox
const dataTransfer = await page.evaluateHandle(() => new DataTransfer());
await elementHandle.dispatchEvent('dragstart', { dataTransfer });
```

**인수**
- `type` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-dispatch-event-option-type"/><a href="#element-handle-dispatch-event-option-type" class="list-anchor">#</a>
  
  DOM 이벤트 유형: `"click"`, `"dragstart"` 등.
- `eventInit` [EvaluationArgument] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-dispatch-event-option-event-init"/><a href="#element-handle-dispatch-event-option-event-init" class="list-anchor">#</a>
  
  선택적 이벤트별 초기화 속성.

**반환 값**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-dispatch-event-return"/><a href="#element-handle-dispatch-event-return" class="list-anchor">#</a>

### press {#element-handle-press}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>elementHandle.press</x-search>

:::warning[Discouraged]

대신 로케이터 기반 [locator.press()](/api/class-locator.mdx#locator-press)를 사용하세요. [로케이터](../locators.mdx)에 대해 자세히 알아보세요.

:::


요소에 포커스를 맞춘 다음 [keyboard.down()](/api/class-keyboard.mdx#keyboard-down)과 [keyboard.up()](/api/class-keyboard.mdx#keyboard-up)을 사용합니다.

[key](/api/class-elementhandle.mdx#element-handle-press-option-key)는 의도된 [keyboardEvent.key](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key) 값이나 텍스트를 생성하기 위한 단일 문자를 지정할 수 있습니다. [key](/api/class-elementhandle.mdx#element-handle-press-option-key) 값의 상위 집합은 [여기](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key/Key_Values)에서 찾을 수 있습니다. 키의 예는 다음과 같습니다:

`F1` - `F12`, `Digit0`- `Digit9`, `KeyA`- `KeyZ`, `Backquote`, `Minus`, `Equal`, `Backslash`, `Backspace`, `Tab`, `Delete`, `Escape`, `ArrowDown`, `End`, `Enter`, `Home`, `Insert`, `PageDown`, `PageUp`, `ArrowRight`, `ArrowUp` 등.

다음과 같은 수정 단축키도 지원됩니다: `Shift`, `Control`, `Alt`, `Meta`, `ShiftLeft`, `ControlOrMeta`.

`Shift`를 누르고 있으면 [key](/api/class-elementhandle.mdx#element-handle-press-option-key)에 해당하는 텍스트가 대문자로 입력됩니다.

[key](/api/class-elementhandle.mdx#element-handle-press-option-key)가 단일 문자인 경우 대소문자를 구분하므로 `a`와 `A` 값은 각각 다른 텍스트를 생성합니다.

`key: "Control+o"`, `key: "Control++` 또는 `key: "Control+Shift+T"`와 같은 단축키도 지원됩니다. 수정자와 함께 지정하면 수정자가 눌려지고 있는 동안 후속 키가 눌려집니다.

**사용법**

```js
await elementHandle.press(key);
await elementHandle.press(key, options);
```

**인수**
- `key` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-press-option-key"/><a href="#element-handle-press-option-key" class="list-anchor">#</a>
  
  눌러야 할 키의 이름 또는 생성할 문자 (예: `ArrowLeft` 또는 `a`).
- `options` [Object] *(선택 사항)*

이 메서드는 [동작 가능성](../actionability.mdx) 검사를 기다린 다음, 스크린샷을 찍기 전에 요소를 뷰로 스크롤합니다. 요소가 DOM에서 분리된 경우 메서드는 오류를 발생시킵니다.

캡처된 스크린샷이 있는 버퍼를 반환합니다.

**사용법**

```js
await elementHandle.screenshot();
```

### waitForSelector {#element-handle-wait-for-selector}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>elementHandle.waitForSelector</x-search>

:::warning[Discouraged]

가시성을 확인하는 웹 어설션이나 로케이터 기반 [locator.waitFor()](/api/class-locator.mdx#locator-wait-for)를 대신 사용하세요.

:::


[state](/api/class-elementhandle.mdx#element-handle-wait-for-selector-option-state) 옵션을 만족하는 선택기로 지정된 요소를 반환합니다. `hidden` 또는 `detached`를 기다리는 경우 `null`을 반환합니다.

요소 핸들을 기준으로 [selector](/api/class-elementhandle.mdx#element-handle-wait-for-selector-option-selector)가 [state](/api/class-elementhandle.mdx#element-handle-wait-for-selector-option-state) 옵션(DOM에서 나타나거나 사라지거나, 보이거나 숨겨짐)을 만족할 때까지 기다립니다. 메서드를 호출하는 순간 [selector](/api/class-elementhandle.mdx#element-handle-wait-for-selector-option-selector)가 이미 조건을 만족하면 메서드는 즉시 반환됩니다. 선택기가 [timeout](/api/class-elementhandle.mdx#element-handle-wait-for-selector-option-timeout) 밀리초 동안 조건을 만족하지 않으면 함수는 예외를 발생시킵니다.

**사용법**

```js
await page.setContent(`<div><span></span></div>`);
const div = await page.$('div');
// div를 기준으로 'span' 선택기를 기다립니다.
const span = await div.waitForSelector('span', { state: 'attached' });
```

:::note
이 메서드는 페이지 이동 간에 작동하지 않습니다. 대신 [page.waitForSelector()](/api/class-page.mdx#page-wait-for-selector)를 사용하세요.
:::

**인수**
- `selector` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-wait-for-selector-option-selector"/><a href="#element-handle-wait-for-selector-option-selector" class="list-anchor">#</a>
  
  쿼리할 선택기입니다.
- `options` [Object] *(선택 사항)*
  - `state` "attached" | "detached" | "visible" | "hidden" *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-wait-for-selector-option-state"/><a href="#element-handle-wait-for-selector-option-state" class="list-anchor">#</a>
    
    기본값은 `'visible'`입니다. 다음 중 하나일 수 있습니다:
    * `'attached'` - 요소가 DOM에 존재할 때까지 기다립니다.
    * `'detached'` - 요소가 DOM에 존재하지 않을 때까지 기다립니다.
    * `'visible'` - 요소가 비어 있지 않은 경계 상자를 가지고 `visibility:hidden`이 없을 때까지 기다립니다. 내용이 없거나 `display:none`이 있는 요소는 빈 경계 상자를 가지며 보이는 것으로 간주되지 않습니다.
    * `'hidden'` - 요소가 DOM에서 분리되거나 빈 경계 상자를 가지거나 `visibility:hidden`일 때까지 기다립니다. 이것은 `'visible'` 옵션의 반대입니다.
  - `strict` [boolean] *(선택 사항)* <font size="2">Added in: v1.15</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-wait-for-selector-option-strict"/><a href="#element-handle-wait-for-selector-option-strict" class="list-anchor">#</a>
    
    true인 경우, 호출은 선택기가 단일 요소로 해결되어야 합니다. 주어진 선택기가 두 개 이상의 요소로 해결되면 호출은 예외를 발생시킵니다.
  - `timeout` [number] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-wait-for-selector-option-timeout"/><a href="#element-handle-wait-for-selector-option-timeout" class="list-anchor">#</a>
    
    최대 시간(밀리초)입니다. 기본값은 `0`이며, 제한 시간이 없습니다. 기본값은 구성에서 `actionTimeout` 옵션을 사용하거나 [browserContext.setDefaultTimeout()](/api/class-browsercontext.mdx#browser-context-set-default-timeout) 또는 [page.setDefaultTimeout()](/api/class-page.mdx#page-set-default-timeout) 메서드를 사용하여 변경할 수 있습니다.

**반환 값**
- [Promise]&lt;[null] | [ElementHandle]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="element-handle-wait-for-selector-return"/><a href="#element-handle-wait-for-selector-return" class="list-anchor">#</a>
