`TestCase`는 테스트 파일의 모든 [test()](/api/class-test.mdx#test-call) 호출에 해당합니다. 단일 [test()](/api/class-test.mdx#test-call)가 여러 프로젝트에서 실행되거나 여러 번 반복되는 경우, 해당 프로젝트의 스위트에서 여러 `TestCase` 객체를 가집니다.


---

## Methods

### ok {#test-case-ok}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>testCase.ok</x-search>

테스트가 정상적으로 실행되는 것으로 간주되는지 여부입니다. ok가 아닌 테스트는 0이 아닌 종료 코드로 테스트 실행에 실패합니다.

**Usage**

```js
testCase.ok();
```

**Returns**
- [boolean]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-case-ok-return"/><a href="#test-case-ok-return" class="list-anchor">#</a>

---

### outcome {#test-case-outcome}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>testCase.outcome</x-search>

이 테스트의 결과입니다. 결과는 [testResult.status](/api/class-testresult.mdx#test-result-status)와 동일하지 않습니다.
* 실패할 것으로 예상되고 실제로 실패한 테스트는 `'expected'`입니다.
* 두 번째 재시도에서 통과한 테스트는 `'flaky'`입니다.

**Usage**

```js
testCase.outcome();
```

**Returns**
- "skipped" | "expected" | "unexpected" | "flaky"<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-case-outcome-return"/><a href="#test-case-outcome-return" class="list-anchor">#</a>

## Properties

### annotations {#test-case-annotations}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>testCase.annotations</x-search>

현재 테스트에 적용 가능한 주석 목록입니다. 다음을 포함합니다.
* [test()](/api/class-test.mdx#test-call) 및 [test.describe()](/api/class-test.mdx#test-describe)를 통해 테스트 또는 스위트에서 정의된 주석
* 메서드 [test.skip()](/api/class-test.mdx#test-skip), [test.fixme()](/api/class-test.mdx#test-fixme) 및 [test.fail()](/api/class-test.mdx#test-fail)에 의해 암시적으로 추가된 주석
* 테스트 실행 중 [testInfo.annotations](/api/class-testinfo.mdx#test-info-annotations)에 추가된 주석

주석은 [testInfo.annotations](/api/class-testinfo.mdx#test-info-annotations)를 통해 테스트 실행 중에 사용할 수 있습니다.

[테스트 주석](../test-annotations.mdx)에 대해 자세히 알아보세요.

**Usage**

```js
testCase.annotations
```

**Type**
- [Array]&lt;[Object]&gt;
  - `type` [string]
    
    주석 유형, 예를 들어 `'skip'` 또는 `'fail'`입니다.
  - `description` [string] *(선택 사항)*
    
    선택적 설명입니다.

### expectedStatus {#test-case-expected-status}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>testCase.expectedStatus</x-search>

예상되는 테스트 상태입니다.
* [test.skip()](/api/class-test.mdx#test-skip) 또는 [test.fixme()](/api/class-test.mdx#test-fixme)로 표시된 테스트는 `'skipped'`로 예상됩니다.
* [test.fail()](/api/class-test.mdx#test-fail)로 표시된 테스트는 `'failed'`로 예상됩니다.
* 다른 테스트는 `'passed'`로 예상됩니다.

실제 상태에 대해서는 [testResult.status](/api/class-testresult.mdx#test-result-status)도 참조하세요.

**Usage**

```js
testCase.expectedStatus
```

**Type**
- "passed" | "failed" | "timedOut" | "skipped" | "interrupted"
