이 API는 웹 API 테스트에 사용됩니다. API 엔드포인트를 트리거하거나, 마이크로서비스를 구성하거나, e2e 테스트를 위한 환경 또는 서비스를 준비하는 데 사용할 수 있습니다.

각 Playwright 브라우저 컨텍스트에는 연결된 [APIRequestContext] 인스턴스가 있으며, 이는 브라우저 컨텍스트와 쿠키 저장소를 공유하며 [browserContext.request](/api/class-browsercontext.mdx#browser-context-request) 또는 [page.request](/api/class-page.mdx#page-request)를 통해 접근할 수 있습니다. **쿠키 관리**

[browserContext.request](/api/class-browsercontext.mdx#browser-context-request) 및 [page.request](/api/class-page.mdx#page-request)에서 반환된 [APIRequestContext]는 해당 [BrowserContext]와 쿠키 저장소를 공유합니다. 각 API 요청의 `Cookie` 헤더는 브라우저 컨텍스트의 값으로 채워집니다. API 응답에 `Set-Cookie` 헤더가 포함된 경우, 이는 자동으로 [BrowserContext] 쿠키를 업데이트하고 페이지에서 수행된 요청에서 해당 쿠키를 사용하게 됩니다. 즉, 이 API를 사용하여 로그인하면 e2e 테스트도 로그인 상태가 되며, 그 반대의 경우도 마찬가지입니다.

API 요청이 브라우저 쿠키를 방해하지 않도록 하려면 [apiRequest.newContext()](/api/class-apirequest.mdx#api-request-new-context)를 호출하여 새 [APIRequestContext]를 생성해야 합니다. 이러한 `APIRequestContext` 객체는 자체적인 격리된 쿠키 저장소를 갖게 됩니다.

### delete {#api-request-context-delete}

### get {#api-request-context-get}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.get</x-search>

HTTP(S) [GET](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods/GET) 요청을 보내고 응답을 반환합니다. 이 메서드는 컨텍스트에서 요청 쿠키를 채우고 응답에서 컨텍스트 쿠키를 업데이트합니다. 이 메서드는 자동으로 리디렉션을 따릅니다.

**사용법**

요청 매개변수는 `params` 옵션으로 구성할 수 있으며, URL 검색 매개변수로 직렬화됩니다:

```js
// 객체로 파라미터 전달
await request.get('https://example.com/api/getText', {
  params: {
    'isbn': '1234',
    'page': 23,
  }
});

// URLSearchParams로 파라미터 전달
const searchParams = new URLSearchParams();
searchParams.set('isbn', '1234');
searchParams.append('page', 23);
```

### post {#api-request-context-post}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.post</x-search>

HTTP(S) [POST](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods/POST) 요청을 보내고 응답을 반환합니다. 이 메서드는 컨텍스트에서 요청 쿠키를 채우고 응답에서 컨텍스트 쿠키를 업데이트합니다. 이 메서드는 자동으로 리디렉션을 따릅니다.

**사용법**

JSON 객체는 요청에 직접 전달될 수 있습니다:

```js
await request.post('https://example.com/api/createBook', {
  data: {
    title: 'Book Title',
    author: 'John Doe',
  }
});
```

서버에 폼 데이터를 보내려면 `form` 옵션을 사용합니다. 이 값은 `application/x-www-form-urlencoded` 인코딩으로 요청 본문에 인코딩됩니다(파일을 보내기 위해 `multipart/form-data` 폼 인코딩을 사용하는 방법은 아래 참조):

```js
await request.post('https://example.com/api/findBook', {
  form: {
    title: 'Book Title',
    author: 'John Doe',
  }
});
```

요청 본문에 파일(들)을 보내는 일반적인 방법은 `multipart/form-data` 인코딩으로 폼 필드로 업로드하는 것입니다. [FormData]를 사용하여 요청 본문을 구성하고 이를 `multipart` 매개변수로 요청에 전달합니다:

```js
const form = new FormData();
form.set('name', 'John');
form.append('name', 'Doe');
// 동일한 이름으로 두 개의 파일 필드를 보냅니다.
form.append('file', new File(['console.log(2024);'], 'f1.js', { type: 'text/javascript' }));
form.append('file', new File(['hello'], 'f2.txt', { type: 'text/plain' }));
await request.post('https://example.com/api/uploadForm', {
  multipart: form
});
```
