[PageAssertions] 클래스는 테스트에서 [Page] 상태에 대한 어설션을 수행하는 데 사용할 수 있는 어설션 메서드를 제공합니다.

```js
import { test, expect } from '@playwright/test';

test('navigates to login', async ({ page }) => {
  // ...
  await page.getByText('Sign in').click();
  await expect(page).toHaveURL(/.*\/login/);
});
```

## Methods

### toHaveScreenshot(name) {#page-assertions-to-have-screenshot-1}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.23</font><x-search>pageAssertions.toHaveScreenshot(name)</x-search>

이 함수는 두 개의 연속된 페이지 스크린샷이 동일한 결과를 낼 때까지 기다린 다음, 마지막 스크린샷을 예상 결과와 비교합니다.

**Usage**

```js
await expect(page).toHaveScreenshot('image.png');
```

스크린샷 어설션은 Playwright 테스트 러너에서만 작동합니다.

**Arguments**
- `name` [string] | [Array]&lt;[string]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-name"/><a href="#page-assertions-to-have-screenshot-1-option-name" class="list-anchor">#</a>
  
  스냅샷 이름.
- `options` [Object] *(optional)*
  - `animations` "disabled" | "allow" *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-animations"/><a href="#page-assertions-to-have-screenshot-1-option-animations" class="list-anchor">#</a>
    
    `"disabled"`로 설정하면 CSS 애니메이션, CSS 전환 및 웹 애니메이션이 중지됩니다. 애니메이션은 지속 시간에 따라 다르게 처리됩니다.
    * 유한 애니메이션은 완료될 때까지 빨리 감기되어 `transitionend` 이벤트가 발생합니다.
    * 무한 애니메이션은 초기 상태로 취소된 다음 스크린샷을 찍은 후에 다시 재생됩니다.
    
    기본값은 애니메이션을 비활성화하는 `"disabled"`입니다.
  - `caret` "hide" | "initial" *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-caret"/><a href="#page-assertions-to-have-screenshot-1-option-caret" class="list-anchor">#</a>
    
    `"hide"`로 설정하면 스크린샷에서 텍스트 캐럿을 숨깁니다. `"initial"`로 설정하면 텍스트 캐럿 동작이 변경되지 않습니다. 기본값은 `"hide"`입니다.
  - `clip` [Object] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-clip"/><a href="#page-assertions-to-have-screenshot-1-option-clip" class="list-anchor">#</a>
    - `x` [number]
      
      클립 영역의 왼쪽 상단 모서리의 x 좌표
    - `y` [number]
      
      클립 영역의 왼쪽 상단 모서리의 y 좌표
    - `width` [number]
      
      클리핑 영역의 너비
    - `height` [number]
      
      클리핑 영역의 높이
    
    결과 이미지의 클리핑을 지정하는 객체입니다.
  - `fullPage` [boolean] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-full-page"/><a href="#page-assertions-to-have-screenshot-1-option-full-page" class="list-anchor">#</a>
    
    true인 경우 현재 보이는 뷰포트 대신 스크롤 가능한 전체 페이지의 스크린샷을 찍습니다. 기본값은 `false`입니다.
  - `mask` [Array]&lt;[Locator]&gt; *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-mask"/><a href="#page-assertions-to-have-screenshot-1-option-mask" class="list-anchor">#</a>
    
    스크린샷을 찍을 때 마스크 처리해야 하는 로케이터를 지정합니다. 마스크 처리된 요소는 해당 경계 상자를 완전히 덮는 분홍색 상자 `#FF00FF`([maskColor](/api/class-pageassertions.mdx#page-assertions-to-have-screenshot-1-option-mask-color)로 사용자 정의됨)로 덮어씌워집니다. 마스크는 보이지 않는 요소에도 적용됩니다. 이를 비활성화하려면 [보이는 요소만 일치](../locators.mdx#matching-only-visible-elements)를 참조하세요.
  - `maskColor` [string] *(optional)* <font size="2">Added in: v1.35</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-mask-color"/><a href="#page-assertions-to-have-screenshot-1-option-mask-color" class="list-anchor">#</a>
    
    마스크 처리된 요소에 대한 오버레이 상자의 색상을 [CSS 색상 형식](https://developer.mozilla.org/en-US/docs/Web/CSS/color_value)으로 지정합니다. 기본 색상은 분홍색 `#FF00FF`입니다.
  - `maxDiffPixelRatio` [number] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-max-diff-pixel-ratio"/><a href="#page-assertions-to-have-screenshot-1-option-max-diff-pixel-ratio" class="list-anchor">#</a>
    
    전체 픽셀 수에 대해 다른 픽셀의 허용 가능한 비율입니다. `0`과 `1` 사이의 값입니다. 기본값은 `TestConfig.expect`로 구성할 수 있습니다. 기본적으로 설정되지 않습니다.
  - `maxDiffPixels` [number] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-max-diff-pixels"/><a href="#page-assertions-to-have-screenshot-1-option-max-diff-pixels" class="list-anchor">#</a>
    
    다를 수 있는 픽셀의 허용 가능한 양입니다. 기본값은 `TestConfig.expect`로 구성할 수 있습니다. 기본적으로 설정되지 않습니다.
  - `omitBackground` [boolean] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-omit-background"/><a href="#page-assertions-to-have-screenshot-1-option-omit-background" class="list-anchor">#</a>
    
    기본 흰색 배경을 숨기고 투명도가 있는 스크린샷을 캡처할 수 있습니다. `jpeg` 이미지에는 적용되지 않습니다. 기본값은 `false`입니다.
  - `scale` "css" | "device" *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-scale"/><a href="#page-assertions-to-have-screenshot-1-option-scale" class="list-anchor">#</a>
    
    `"css"`로 설정하면 스크린샷은 페이지의 각 CSS 픽셀당 하나의 픽셀을 갖습니다. 고해상도 장치의 경우 스크린샷 크기를 작게 유지합니다. `"device"` 옵션을 사용하면 각 장치 픽셀당 하나의 픽셀이 생성되므로 고해상도 장치의 스크린샷이 두 배 이상 커집니다.
    
    기본값은 `"css"`입니다.
  - `stylePath` [string] | [Array]&lt;[string]&gt; *(optional)* <font size="2">Added in: v1.41</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-style-path"/><a href="#page-assertions-to-have-screenshot-1-option-style-path" class="list-anchor">#</a>
    
    스크린샷을 만들 때 적용할 스타일시트가 포함된 파일 이름입니다. 여기에서 동적 요소를 숨기거나, 요소를 보이지 않게 만들거나, 속성을 변경하여 반복 가능한 스크린샷을 만들 수 있습니다. 이 스타일시트는 Shadow DOM을 관통하여 내부 프레임에 적용됩니다.
  - `threshold` [number] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-threshold"/><a href="#page-assertions-to-have-screenshot-1-option-threshold" class="list-anchor">#</a>
    
    비교된 이미지에서 동일한 픽셀 간의 [YIQ 색 공간](https://en.wikipedia.org/wiki/YIQ)에서 허용 가능한 인지 색상 차이입니다. 0(엄격)과 1(관대) 사이의 값이며, 기본값은 `TestConfig.expect`로 구성할 수 있습니다. 기본값은 `0.2`입니다.
  - `timeout` [number] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-option-timeout"/><a href="#page-assertions-to-have-screenshot-1-option-timeout" class="list-anchor">#</a>
    
    어설션을 다시 시도하는 시간(밀리초)입니다. 기본값은 `TestConfig.expect`의 `timeout`입니다.

**Returns**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-1-return"/><a href="#page-assertions-to-have-screenshot-1-return" class="list-anchor">#</a>

### toHaveURL {#page-assertions-to-have-url}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.20</font><x-search>pageAssertions.toHaveURL</x-search>

페이지가 지정된 URL로 이동되었는지 확인합니다.

**Usage**

```js
// Check for the page URL to be 'https://playwright.dev/docs/intro' (including query string)
// 페이지 URL이 'https://playwright.dev/docs/intro'인지 확인합니다 (쿼리 문자열 포함).
await expect(page).toHaveURL('https://playwright.dev/docs/intro');

// Check for the page URL to contain 'doc', followed by an optional 's', followed by '/'
// 페이지 URL에 'doc'이 포함되어 있는지 확인하고, 그 뒤에 선택적 's'와 '/'가 오는지 확인합니다.
await expect(page).toHaveURL(/docs?\//);

// Check for the predicate to be satisfied
// 예를 들어, 쿼리 문자열을 확인합니다.
await expect(page).toHaveURL(url => {
  const params = url.searchParams;
  return params.has('search') && params.has('options') && params.get('id') === '5';
});
```

**Arguments**
- `url` [string] | [RegExp] | [function]\([URL]\):[boolean] <font size="2">Added in: v1.18</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-url-option-url"/><a href="#page-assertions-to-have-url-option-url" class="list-anchor">#</a>
  
  일치시킬 예상 URL 문자열, RegExp 또는 [URL]을 받는 predicate. [baseURL](/api/class-browser.mdx#browser-new-context-option-base-url)이 컨텍스트 옵션을 통해 제공되고 `url` 인수가 문자열인 경우, 두 값은 [`new URL()`](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) 생성자를 통해 병합되고 현재 브라우저 URL과 비교하는 데 사용됩니다.
- `options` [Object] *(optional)*
  - `ignoreCase` [boolean] *(optional)* <font size="2">Added in: v1.44</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-url-option-ignore-case"/><a href="#page-assertions-to-have-url-option-ignore-case" class="list-anchor">#</a>
    
    대소문자를 구분하지 않고 일치시킬지 여부입니다. [ignoreCase](/api/class-pageassertions.mdx#page-assertions-to-have-url-option-ignore-case) 옵션은 지정된 경우 해당 정규식 매개변수보다 우선합니다. 제공된 predicate는 이 플래그를 무시합니다.
  - `timeout` [number] *(optional)* <font size="2">Added in: v1.18</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="page-assertions-to-have-url-option-timeout"/><a href="#page-assertions-to-have-url-option-timeout" class="list-anchor">#</a>
