Web API 테스트에 사용할 수 있는 API를 노출합니다. 이 클래스는 웹 요청을 보내는 데 사용할 수 있는 [APIRequestContext] 인스턴스를 만드는 데 사용됩니다. 이 클래스의 인스턴스는 [playwright.request](/api/class-playwright.mdx#playwright-request)를 통해 얻을 수 있습니다. 자세한 내용은 [APIRequestContext]를 참조하세요.


---

## 메소드 (Methods)

### newContext {#api-request-new-context}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequest.newContext</x-search>

[APIRequestContext]의 새 인스턴스를 생성합니다.

**사용법**

```js
await apiRequest.newContext();
await apiRequest.newContext(options);
```

**인수**
- `options` [Object] *(optional)*
  - `baseURL` [string] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-base-url"/><a href="#api-request-new-context-option-base-url" class="list-anchor">#</a>

    [apiRequestContext.get()](/api/class-apirequestcontext.mdx#api-request-context-get)와 같은 메서드는 해당 URL을 빌드하기 위해 [`URL()`](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) 생성자를 사용하여 기본 URL을 고려합니다. 예시:
    * baseURL: `http://localhost:3000` 이고 `/bar.html`로 요청을 보내면 `http://localhost:3000/bar.html`이 됩니다.
    * baseURL: `http://localhost:3000/foo/` 이고 `./bar.html`로 요청을 보내면 `http://localhost:3000/foo/bar.html`이 됩니다.
    * baseURL: `http://localhost:3000/foo` (후행 슬래시 없음) 이고 `./bar.html`로 이동하면 `http://localhost:3000/bar.html`이 됩니다.
  - `clientCertificates` [Array]&lt;[Object]&gt; *(optional)* <font size="2">Added in: 1.46</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-client-certificates"/><a href="#api-request-new-context-option-client-certificates" class="list-anchor">#</a>
    - `origin` [string]

      인증서가 유효한 정확한 출처(origin). 출처에는 `https` 프로토콜, 호스트 이름 및 선택적으로 포트가 포함됩니다.
    - `certPath` [string] *(optional)*

      PEM 형식의 인증서가 있는 파일 경로입니다.
    - `cert` [Buffer] *(optional)*

      PEM 형식의 인증서 직접 값입니다.
    - `keyPath` [string] *(optional)*

      PEM 형식의 개인 키가 있는 파일 경로입니다.
    - `key` [Buffer] *(optional)*

      PEM 형식의 개인 키 직접 값입니다.
    - `pfxPath` [string] *(optional)*

      PFX 또는 PKCS12로 인코딩된 개인 키 및 인증서 체인이 있는 파일 경로입니다.
    - `pfx` [Buffer] *(optional)*

      PFX 또는 PKCS12로 인코딩된 개인 키 및 인증서 체인의 직접 값입니다.
    - `passphrase` [string] *(optional)*

      개인 키(PEM 또는 PFX)의 암호 구문입니다.

    TLS 클라이언트 인증을 통해 서버는 클라이언트 인증서를 요청하고 확인할 수 있습니다.

    **상세 정보**

    사용할 클라이언트 인증서 배열입니다. 각 인증서 객체는 `certPath`와 `keyPath`를 모두 갖거나, 단일 `pfxPath`, 또는 해당 직접 값 (`cert`와 `key`, 또는 `pfx`)을 가져야 합니다. 선택적으로, 인증서가 암호화된 경우 `passphrase` 속성을 제공해야 합니다. `origin` 속성은 인증서가 유효한 요청 출처와 정확히 일치하도록 제공되어야 합니다.

    :::note

    macOS에서 WebKit을 사용할 때 `localhost`에 액세스하면 클라이언트 인증서가 선택되지 않습니다. `localhost`를 `local.playwright`로 바꾸면 작동하게 할 수 있습니다.
    :::

  - `extraHTTPHeaders` [Object]&lt;[string], [string]&gt; *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-extra-http-headers"/><a href="#api-request-new-context-option-extra-http-headers" class="list-anchor">#</a>

    모든 요청과 함께 보낼 추가 HTTP 헤더를 포함하는 객체입니다. 기본값은 없습니다.
  - `failOnStatusCode` [boolean] *(optional)* <font size="2">Added in: v1.51</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-fail-on-status-code"/><a href="#api-request-new-context-option-fail-on-status-code" class="list-anchor">#</a>

    2xx 및 3xx 이외의 응답 코드에서 오류를 발생시킬지 여부입니다. 기본적으로 모든 상태 코드에 대해 응답 객체가 반환됩니다.
  - `httpCredentials` [Object] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-http-credentials"/><a href="#api-request-new-context-option-http-credentials" class="list-anchor">#</a>
    - `username` [string]


    - `password` [string]


    - `origin` [string] *(optional)*

      특정 출처(scheme://host:port)에서 HTTP 자격 증명 전송을 제한합니다.
    - `send` "unauthorized" | "always" *(optional)*

      이 옵션은 해당 [APIRequestContext]에서 보낸 요청에만 적용되며 브라우저에서 보낸 요청에는 영향을 미치지 않습니다. `'always'` - 기본 인증 자격 증명이 포함된 `Authorization` 헤더가 각 API 요청과 함께 전송됩니다. `'unauthorized'` - `WWW-Authenticate` 헤더가 포함된 401(Unauthorized) 응답을 수신했을 때만 자격 증명이 전송됩니다. 기본값은 `'unauthorized'`입니다.

    [HTTP 인증](https://developer.mozilla.org/en-US/docs/Web/HTTP/Authentication)을 위한 자격 증명입니다. 출처가 지정되지 않은 경우 사용자 이름과 암호는 인증되지 않은 응답 시 모든 서버로 전송됩니다.
  - `ignoreHTTPSErrors` [boolean] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-ignore-https-errors"/><a href="#api-request-new-context-option-ignore-https-errors" class="list-anchor">#</a>

    네트워크 요청을 보낼 때 HTTPS 오류를 무시할지 여부입니다. 기본값은 `false`입니다.
  - `proxy` [Object] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-proxy"/><a href="#api-request-new-context-option-proxy" class="list-anchor">#</a>
    - `server` [string]

      모든 요청에 사용할 프록시입니다. HTTP 및 SOCKS 프록시가 지원됩니다. 예를 들어 `http://myproxy.com:3128` 또는 `socks5://myproxy.com:3128`입니다. 약식 `myproxy.com:3128`은 HTTP 프록시로 간주됩니다.
    - `bypass` [string] *(optional)*

      프록시를 우회할 선택적 쉼표로 구분된 도메인입니다. 예를 들어 `".com, chromium.org, .domain.com"`입니다.
    - `username` [string] *(optional)*

      HTTP 프록시에 인증이 필요한 경우 사용할 선택적 사용자 이름입니다.
    - `password` [string] *(optional)*

      HTTP 프록시에 인증이 필요한 경우 사용할 선택적 암호입니다.

    네트워크 프록시 설정입니다.
  - `storageState` [string] | [Object] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-storage-state"/><a href="#api-request-new-context-option-storage-state" class="list-anchor">#</a>
    - `cookies` [Array]&lt;[Object]&gt;
      - `name` [string]


      - `value` [string]


      - `domain` [string]


      - `path` [string]


      - `expires` [number]

        초 단위의 유닉스 시간입니다.
      - `httpOnly` [boolean]


      - `secure` [boolean]


      - `sameSite` "Strict" | "Lax" | "None"


    - `origins` [Array]&lt;[Object]&gt;
      - `origin` [string]


      - `localStorage` [Array]&lt;[Object]&gt;
        - `name` [string]


        - `value` [string]


    주어진 저장소 상태로 컨텍스트를 채웁니다. 이 옵션은 [browserContext.storageState()](/api/class-browsercontext.mdx#browser-context-storage-state) 또는 [apiRequestContext.storageState()](/api/class-apirequestcontext.mdx#api-request-context-storage-state)를 통해 얻은 로그인 정보로 컨텍스트를 초기화하는 데 사용할 수 있습니다. 저장된 저장소가 있는 파일 경로 또는 [browserContext.storageState()](/api/class-browsercontext.mdx#browser-context-storage-state) 또는 [apiRequestContext.storageState()](/api/class-apirequestcontext.mdx#api-request-context-storage-state) 메서드 중 하나가 반환한 값입니다.
  - `timeout` [number] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-timeout"/><a href="#api-request-new-context-option-timeout" class="list-anchor">#</a>

    응답을 기다리는 최대 시간(밀리초)입니다. 기본값은 `30000`(30초)입니다. 시간 제한을 비활성화하려면 `0`을 전달합니다.
  - `userAgent` [string] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-user-agent"/><a href="#api-request-new-context-option-user-agent" class="list-anchor">#</a>

    이 컨텍스트에서 사용할 특정 사용자 에이전트입니다.
