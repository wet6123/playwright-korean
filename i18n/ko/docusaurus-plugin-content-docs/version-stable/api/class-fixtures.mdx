Playwright Test는 [테스트 픽스처](../test-fixtures.mdx) 개념을 기반으로 합니다. 테스트 픽스처는 각 테스트에 대한 환경을 설정하는 데 사용되며, 테스트에 필요한 모든 것을 제공하고 그 외에는 아무것도 제공하지 않습니다.

Playwright Test는 각 테스트 선언을 살펴보고, 테스트에 필요한 픽스처 집합을 분석하고, 해당 픽스처를 테스트에 맞게 특별히 준비합니다. 픽스처에 의해 준비된 값은 `test`, hooks, annotations 및 기타 픽스처에서 첫 번째 매개변수로 사용할 수 있는 단일 객체로 병합됩니다.

```js
import { test, expect } from '@playwright/test';

test('basic test', async ({ page }) => {
  // ...
});
```

위의 테스트가 주어지면 Playwright Test는 테스트를 실행하기 전에 `page` 픽스처를 설정하고 테스트가 완료된 후 해체합니다. `page` 픽스처는 테스트에서 사용할 수 있는 [Page] 객체를 제공합니다.

Playwright Test는 아래 나열된 기본 제공 픽스처와 함께 제공되며, 사용자 정의 픽스처를 추가할 수도 있습니다. Playwright Test는 또한 [fixtures.browser](/api/class-fixtures.mdx#fixtures-browser), [fixtures.context](/api/class-fixtures.mdx#fixtures-context) 및 [fixtures.page](/api/class-fixtures.mdx#fixtures-page)를 구성할 수 있는 [옵션][TestOptions]을 제공합니다.

## Properties

### browser {#fixtures-browser}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>fixtures.browser</x-search>

[Browser] 인스턴스는 [동일한 워커](../test-parallel.mdx)의 모든 테스트 간에 공유되므로 테스트가 효율적입니다. 그러나 각 테스트는 격리된 [BrowserContext]에서 실행되며 새로운 환경을 제공받습니다.

[브라우저 구성](../test-configuration.mdx) 방법과 [사용 가능한 옵션][TestOptions]을 확인하세요.

**Usage**

```js
test.beforeAll(async ({ browser }) => {
  const page = await browser.newPage();
  // ...
});
```

**Type**
- [Browser]

### browserName {#fixtures-browser-name}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>fixtures.browserName</x-search>

테스트를 실행하는 브라우저의 이름입니다. 기본값은 `'chromium'`입니다. 브라우저를 기반으로 [테스트에 주석을 달 때](../test-annotations.mdx) 유용합니다.

**Usage**

```js
test('skip this test in Firefox', async ({ page, browserName }) => {
  test.skip(browserName === 'firefox', 'Still working on it');
  // 아직 작업 중입니다.
  // ...
});
```

**Type**
- "chromium" | "firefox" | "webkit"

---

### context {#fixtures-context}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>fixtures.context</x-search>

각 테스트에 대해 생성된 격리된 [BrowserContext] 인스턴스입니다. 컨텍스트는 서로 격리되어 있기 때문에 여러 테스트가 최대 효율을 위해 단일 [Browser]에서 실행되더라도 모든 테스트는 새로운 환경을 제공받습니다.

[컨텍스트 구성](../test-configuration.mdx) 방법과 [사용 가능한 옵션][TestOptions]을 확인하세요.

기본 [fixtures.page](/api/class-fixtures.mdx#fixtures-page)는 이 컨텍스트에 속합니다.

**Usage**

```js
test('example test', async ({ page, context }) => {
  await context.route('*external.com/*', route => route.abort());
  // ...
});
```

**Type**
- [BrowserContext]

---

### page {#fixtures-page}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>fixtures.page</x-search>

각 테스트에 대해 생성된 격리된 [Page] 인스턴스입니다. 페이지는 [fixtures.context](/api/class-fixtures.mdx#fixtures-context) 격리로 인해 테스트 간에 격리됩니다.

이것은 테스트에서 가장 일반적으로 사용되는 픽스처입니다.

**Usage**

```js
import { test, expect } from '@playwright/test';

test('basic test', async ({ page }) => {
  await page.goto('/signin');
  await page.getByLabel('User Name').fill('user');
  await page.getByLabel('Password').fill('password');
  await page.getByText('Sign in').click();
  // ...
});
```

**Type**
- [Page]

---

### request {#fixtures-request}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>fixtures.request</x-search>

각 테스트에 대한 격리된 [APIRequestContext] 인스턴스입니다.

**Usage**

```js
import { test, expect } from '@playwright/test';

test('basic test', async ({ request }) => {
  await request.post('/signin', {
    data: {
      username: 'user',
      password: 'password'
    }
  });
  // ...
});
```

**Type**
- [APIRequestContext]
