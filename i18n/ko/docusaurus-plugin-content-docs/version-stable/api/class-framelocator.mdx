FrameLocator는 페이지의 `iframe`에 대한 뷰를 나타냅니다. `iframe`을 검색하고 해당 iframe에서 요소를 찾는 데 충분한 로직을 캡처합니다. FrameLocator는 [locator.contentFrame()](/api/class-locator.mdx#locator-content-frame), [page.frameLocator()](/api/class-page.mdx#page-frame-locator) 또는 [locator.frameLocator()](/api/class-locator.mdx#locator-frame-locator) 메서드로 생성할 수 있습니다.

```js
const locator = page.locator('#my-frame').contentFrame().getByText('Submit');
await locator.click();
```

**엄격성**

Frame locator는 엄격합니다. 즉, frame locator에 대한 모든 작업은 주어진 선택기와 일치하는 요소가 두 개 이상인 경우 오류를 발생시킵니다.

```js
// DOM에 여러 프레임이 있는 경우 오류 발생:
await page.locator('.result-frame').contentFrame().getByRole('button').click();

// locator가 첫 번째 프레임을 선택하도록 명시적으로 지정했기 때문에 작동합니다.
await page.locator('.result-frame').contentFrame().first().getByRole('button').click();
```

**Locator를 FrameLocator로 변환**

`iframe`을 가리키는 [Locator] 객체가 있는 경우 [locator.contentFrame()](/api/class-locator.mdx#locator-content-frame)을 사용하여 [FrameLocator]로 변환할 수 있습니다.

**FrameLocator를 Locator로 변환**

[FrameLocator] 객체가 있는 경우 [frameLocator.owner()](/api/class-framelocator.mdx#frame-locator-owner)를 사용하여 동일한 `iframe`을 가리키는 [Locator]로 변환할 수 있습니다.

```html
<br/>
<button>Submit</button>
```

```js
await expect(page.getByRole('heading', { name: 'Sign up' })).toBeVisible();
```

**세부 정보**

많은 html 요소에는 역할 선택기에서 인식하는 암시적으로 [정의된 역할](https://w3c.github.io/html-aam/#html-element-role-mappings)이 있습니다. [여기](https://www.w3.org/TR/wai-aria-1.2/#role_definitions)에서 지원되는 모든 역할을 찾을 수 있습니다. ARIA 지침은 `role` 및/또는 `aria-*` 속성을 기본값으로 설정하여 암시적 역할 및 속성을 복제하는 것을 **권장하지 않습니다**.

### getByText {#frame-locator-get-by-text}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.27</font><x-search>frameLocator.getByText</x-search>

지정된 텍스트를 포함하는 요소를 찾을 수 있습니다.

접근 가능한 역할과 같은 다른 기준으로 일치시킨 다음 텍스트 콘텐츠로 필터링할 수 있는 [locator.filter()](/api/class-locator.mdx#locator-filter)도 참조하십시오.

**사용법**

다음 DOM 구조를 고려하십시오.

```html
<div>Hello <span>world</span></div>
<div>Hello</div>
```

텍스트 하위 문자열, 정확한 문자열 또는 정규식으로 찾을 수 있습니다.

```js
// <span>와 일치합니다.
page.getByText('world');

// 첫 번째 <div>와 일치합니다.
page.getByText('Hello world');

// 두 번째 <div>와 일치합니다.
page.getByText('Hello', { exact: true });

// 두 <div>와 모두 일치합니다.
page.getByText(/Hello/);

// 두 번째 <div>와 일치합니다.
page.getByText(/^hello$/i);
```

**인수**
- `text` [string] | [RegExp]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="frame-locator-get-by-text-option-text"/><a href="#frame-locator-get-by-text-option-text" class="list-anchor">#</a>
  
  요소를 찾을 텍스트입니다.

### locator {#frame-locator-locator}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.17</font><x-search>frameLocator.locator</x-search>

이 메서드는 locator의 하위 트리에서 지정된 선택기와 일치하는 요소를 찾습니다. 또한 [locator.filter()](/api/class-locator.mdx#locator-filter) 메서드와 유사한 필터 옵션을 허용합니다.

[locator에 대해 자세히 알아보세요](../locators.mdx).

**사용법**

```js
frameLocator.locator(selectorOrLocator);
frameLocator.locator(selectorOrLocator, options);
```

**인수**
- `selectorOrLocator` [string] | [Locator]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="frame-locator-locator-option-selector-or-locator"/><a href="#frame-locator-locator-option-selector-or-locator" class="list-anchor">#</a>
  
  DOM 요소를 확인할 때 사용할 선택기 또는 locator입니다.
- `options` [Object] *(선택 사항)*
  - `has` [Locator] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="frame-locator-locator-option-has"/><a href="#frame-locator-locator-option-has" class="list-anchor">#</a>
    
    이 상대 locator와 일치하는 요소를 포함하는 메서드의 결과로 범위를 좁힙니다. 예를 들어 `text=Playwright`가 있는 `article`은 `<article><div>Playwright</div></article>`와 일치합니다.
    
    내부 locator는 외부 locator에 **상대적이어야** 하며 문서 루트가 아니라 외부 locator 일치로 시작하여 쿼리됩니다. 예를 들어 `<article><content><div>Playwright</div></content></article>`에서 `div`가 있는 `content`를 찾을 수 있습니다. 그러나 `article div`가 있는 `content`를 찾으면 내부 locator가 상대적이어야 하고 `content` 외부의 요소를 사용해서는 안 되기 때문에 실패합니다.
    
    외부 및 내부 locator는 동일한 프레임에 속해야 합니다. 내부 locator는 [FrameLocator]를 포함해서는 안 됩니다.
  - `hasNot` [Locator] *(선택 사항)* <font size="2">Added in: v1.33</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="frame-locator-locator-option-has-not"/><a href="#frame-locator-locator-option-has-not" class="list-anchor">#</a>
    
    내부 locator와 일치하는 요소를 포함하지 않는 요소와 일치합니다. 내부 locator는 외부 locator에 대해 쿼리됩니다. 예를 들어 `div`가 없는 `article`은 `<article><span>Playwright</span></article>`와 일치합니다.
    
    외부 및 내부 locator는 동일한 프레임에 속해야 합니다. 내부 locator는 [FrameLocator]를 포함해서는 안 됩니다.
  - `hasNotText` [string] | [RegExp] *(선택 사항)* <font size="2">Added in: v1.33</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="frame-locator-locator-option-has-not-text"/><a href="#frame-locator-locator-option-has-not-text" class="list-anchor">#</a>
    
    자식 또는 하위 요소에 있을 수 있는 지정된 텍스트를 내부에 포함하지 않는 요소와 일치합니다. [string]이 전달되면 대소문자를 구분하지 않고 하위 문자열을 검색합니다.
  - `hasText` [string] | [RegExp] *(선택 사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="frame-locator-locator-option-has-text"/><a href="#frame-locator-locator-option-has-text" class="list-anchor">#</a>
    
    자식 또는 하위 요소에 있을 수 있는 지정된 텍스트를 내부에 포함하는 요소와 일치합니다. [string]이 전달되면 대소문자를 구분하지 않고 하위 문자열을 검색합니다. 예를 들어 `"Playwright"`는 `<article><div>Playwright</div></article>`와 일치합니다.

**반환 값**
- [Locator]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="frame-locator-locator-return"/><a href="#frame-locator-locator-return" class="list-anchor">#</a>
