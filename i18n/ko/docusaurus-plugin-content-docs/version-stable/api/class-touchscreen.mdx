Touchscreen 클래스는 뷰포트의 왼쪽 상단 모서리를 기준으로 메인 프레임 CSS 픽셀에서 작동합니다. 터치스크린의 메서드는 `hasTouch`가 true로 설정된 상태로 초기화된 브라우저 컨텍스트에서만 사용할 수 있습니다.

이 클래스는 탭 제스처 에뮬레이션으로 제한됩니다. 터치 이벤트를 수동으로 디스패치하여 시뮬레이션된 다른 제스처의 예는 [레거시 터치 이벤트 에뮬레이션](../touch-events.mdx) 페이지를 참조하십시오.


---

## Methods

### tap {#touchscreen-tap}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>touchscreen.tap</x-search>

([x](/api/class-touchscreen.mdx#touchscreen-tap-option-x),[y](/api/class-touchscreen.mdx#touchscreen-tap-option-y)) 위치에서 단일 터치로 `touchstart` 및 `touchend` 이벤트를 디스패치합니다.

:::note

[page.tap()](/api/class-page.mdx#page-tap) 메서드는 브라우저 컨텍스트의 [hasTouch](/api/class-browser.mdx#browser-new-context-option-has-touch) 옵션이 false이면 오류를 발생시킵니다.
:::

**Usage**

```js
await touchscreen.tap(x, y);
```

**Arguments**
- `x` [number]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="touchscreen-tap-option-x"/><a href="#touchscreen-tap-option-x" class="list-anchor">#</a>
  
  CSS 픽셀 단위로 메인 프레임의 뷰포트를 기준으로 하는 X 좌표입니다.
- `y` [number]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="touchscreen-tap-option-y"/><a href="#touchscreen-tap-option-y" class="list-anchor">#</a>
  
  CSS 픽셀 단위로 메인 프레임의 뷰포트를 기준으로 하는 Y 좌표입니다.

**Returns**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="touchscreen-tap-return"/><a href="#touchscreen-tap-return" class="list-anchor">#</a>
