`TestInfo`는 현재 실행 중인 테스트에 대한 정보를 포함합니다. 이는 테스트 함수, [test.beforeEach()](/api/class-test.mdx#test-before-each), [test.afterEach()](/api/class-test.mdx#test-after-each), [test.beforeAll()](/api/class-test.mdx#test-before-all) 및 [test.afterAll()](/api/class-test.mdx#test-after-all) 후크, 그리고 테스트 범위의 픽스처에서 사용할 수 있습니다. `TestInfo`는 파일 첨부, 테스트 타임아웃 업데이트, 현재 실행 중인 테스트 결정 및 재시도 여부 등 테스트 실행을 제어하는 유틸리티를 제공합니다.

```js
import { test, expect } from '@playwright/test';

test('basic test', async ({ page }, testInfo) => {
  expect(testInfo.title).toBe('basic test');
  await page.screenshot(testInfo.outputPath('screenshot.png'));
});
```

## Methods

### attach {#test-info-attach}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>testInfo.attach</x-search>

현재 테스트에 값 또는 디스크의 파일을 첨부합니다. 일부 보고서는 테스트 첨부 파일을 표시합니다. [path](/api/class-testinfo.mdx#test-info-attach-option-path) 또는 [body](/api/class-testinfo.mdx#test-info-attach-option-body) 중 하나를 지정해야 하지만 둘 다 지정할 수는 없습니다.

예를 들어 스크린샷을 테스트에 첨부할 수 있습니다.

```js
import { test, expect } from '@playwright/test';

test('basic test', async ({ page }, testInfo) => {
  await page.goto('https://playwright.dev');
  const screenshot = await page.screenshot();
  await testInfo.attach('screenshot', { body: screenshot, contentType: 'image/png' });
});
```

또는 API에서 반환된 파일을 첨부할 수 있습니다.

```js
import { test, expect } from '@playwright/test';
import { download } from './my-custom-helpers';

test('basic test', async ({}, testInfo) => {
  const tmpPath = await download('a');
  await testInfo.attach('downloaded', { path: tmpPath });
});
```

:::note
[testInfo.attach()](/api/class-testinfo.mdx#test-info-attach)는 첨부된 파일을 보고서에서 액세스할 수 있는 위치로 복사하는 작업을 자동으로 처리합니다. attach 호출을 기다린 후 첨부 파일을 안전하게 제거할 수 있습니다.
:::

**Usage**

```js
await testInfo.attach(name);
await testInfo.attach(name, options);
```

**Arguments**
- `name` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-info-attach-option-name"/><a href="#test-info-attach-option-name" class="list-anchor">#</a>
  
  첨부 파일 이름입니다. 이름은 삭제되고 디스크에 저장할 때 파일 이름의 접두사로 사용됩니다.
- `options` [Object] *(optional)*
  - `body` [string] | [Buffer] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-info-attach-option-body"/><a href="#test-info-attach-option-body" class="list-anchor">#</a>
    
    첨부 파일 본문입니다. [path](/api/class-testinfo.mdx#test-info-attach-option-path)와 상호 배타적입니다.
  - `contentType` [string] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-info-attach-option-content-type"/><a href="#test-info-attach-option-content-type" class="list-anchor">#</a>
    
    보고서에 올바르게 표시하기 위한 이 첨부 파일의 콘텐츠 유형입니다. 예를 들어 `'application/json'` 또는 `'image/png'`입니다. 생략하면 콘텐츠 유형은 [path](/api/class-testinfo.mdx#test-info-attach-option-path)를 기반으로 유추되거나 [string] 첨부 파일의 경우 `text/plain`, [Buffer] 첨부 파일의 경우 `application/octet-stream`으로 기본 설정됩니다.
  - `path` [string] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-info-attach-option-path"/><a href="#test-info-attach-option-path" class="list-anchor">#</a>
    
    첨부된 파일의 파일 시스템 경로입니다. [body](/api/class-testinfo.mdx#test-info-attach-option-body)와 상호 배타적입니다.

**Returns**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-info-attach-return"/><a href="#test-info-attach-return" class="list-anchor">#</a>

### outputPath {#test-info-output-path}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>testInfo.outputPath</x-search>

테스트가 임시 파일을 안전하게 넣을 수 있는 [testInfo.outputDir](/api/class-testinfo.mdx#test-info-output-dir) 내부의 경로를 반환합니다. 병렬로 실행되는 테스트가 서로 간섭하지 않도록 보장합니다.

```js
import { test, expect } from '@playwright/test';
import fs from 'fs';

test('example test', async ({}, testInfo) => {
  const file = testInfo.outputPath('dir', 'temporary-file.txt');
  await fs.promises.writeFile(file, 'Put some data to the dir/temporary-file.txt', 'utf8');
});
```

> `pathSegments`는 `testInfo.outputPath('relative', 'path', 'to', 'output')`와 같이 테스트 출력 디렉터리에 대한 경로 세그먼트를 허용합니다.
> 그러나 이 경로는 각 테스트의 [testInfo.outputDir](/api/class-testinfo.mdx#test-info-output-dir) 디렉터리(즉, `test-results/a-test-title`) 내에 있어야 합니다. 그렇지 않으면 오류가 발생합니다.

**Usage**

```js
testInfo.outputPath(...pathSegments);
```

**Arguments**
- `...pathSegments` [Array]&lt;[string]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-info-output-path-option-path-segments"/><a href="#test-info-output-path-option-path-segments" class="list-anchor">#</a>
  
  결과 경로의 끝에 추가할 경로 세그먼트입니다.

**Returns**
- [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-info-output-path-return"/><a href="#test-info-output-path-return" class="list-anchor">#</a>

### setTimeout {#test-info-set-timeout}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>testInfo.setTimeout</x-search>

현재 실행 중인 테스트의 타임아웃을 변경합니다. 0은 타임아웃이 없음을 의미합니다. [다양한 타임아웃](../test-timeouts.mdx)에 대해 자세히 알아보세요.

타임아웃은 일반적으로 [구성 파일](../test-configuration.mdx)에 지정되지만 특정 시나리오에서 타임아웃을 변경하는 것이 유용할 수 있습니다.

```js
import { test, expect } from '@playwright/test';

test.beforeEach(async ({ page }, testInfo) => {
  // 이 후크를 실행하는 모든 테스트의 타임아웃을 30초 연장합니다.
  testInfo.setTimeout(testInfo.timeout + 30000);
});
```

**Usage**

```js
testInfo.setTimeout(timeout);
```

**Arguments**
- `timeout` [number]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-info-set-timeout-option-timeout"/><a href="#test-info-set-timeout-option-timeout" class="list-anchor">#</a>
  
  밀리초 단위의 타임아웃입니다.

### expectedStatus {#test-info-expected-status}

현재 실행 중인 테스트의 예상 상태입니다. 이는 일반적으로 `'passed'`이지만 몇 가지 경우가 있습니다.
* [test.skip()](/api/class-test.mdx#test-skip)으로 건너뛴 테스트의 경우 `'skipped'`;

```js
import { test, expect } from '@playwright/test';

test.afterEach(async ({}, testInfo) => {
  if (testInfo.status !== testInfo.expectedStatus)
    console.log(`${testInfo.title} did not run as expected!`);
});
```

**Usage**

```js
testInfo.expectedStatus
```

**Type**
- "passed" | "failed" | "timedOut" | "skipped" | "interrupted"

### retry {#test-info-retry}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>testInfo.retry</x-search>

오류 후 테스트를 재시도할 때 재시도 횟수를 지정합니다. 첫 번째 테스트 실행은 [testInfo.retry](/api/class-testinfo.mdx#test-info-retry)가 0이고, 첫 번째 재시도는 1입니다. [재시도](../test-retries.mdx#retries)에 대해 자세히 알아보세요.

```js
import { test, expect } from '@playwright/test';

test.beforeEach(async ({}, testInfo) => {
  // 모든 후크 또는 픽스처에서 testInfo.retry에 액세스할 수 있습니다.
  if (testInfo.retry > 0)
    console.log(`Retrying!`);
});

test('my test', async ({ page }, testInfo) => {
  // 여기서는 재시도할 때 일부 서버 측 상태를 지웁니다.
  if (testInfo.retry)
    await cleanSomeCachesOnTheServer();
  // ...
});
```

**Usage**

```js
testInfo.retry
```

**Type**
- [number]
