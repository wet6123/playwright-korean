BrowserType은 특정 브라우저 인스턴스를 실행하거나 기존 인스턴스에 연결하는 메서드를 제공합니다. 다음은 Playwright를 사용하여 자동화를 수행하는 일반적인 예시입니다:

```js
const { chromium } = require('playwright');  // 또는 'firefox' 또는 'webkit'.

(async () => {
  const browser = await chromium.launch();
  const page = await browser.newPage();
  await page.goto('https://example.com');
  // 다른 작업들...
  await browser.close();
})();
```

## 메서드

### connect {#browser-type-connect}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browserType.connect</x-search>

이 메서드는 Node.js에서 `BrowserType.launchServer`를 통해 생성된 기존 브라우저 인스턴스에 Playwright를 연결합니다.

:::note

연결하는 Playwright 인스턴스의 메이저 및 마이너 버전은 브라우저를 실행하는 Playwright의 버전과 일치해야 합니다(1.2.3은 1.2.x와 호환됨).
:::

**사용법**

```js
await browserType.connect(wsEndpoint);
await browserType.connect(wsEndpoint, options);
```

**인자**
- `wsEndpoint` [string] <font size="2">v1.10에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-option-ws-endpoint"/><a href="#browser-type-connect-option-ws-endpoint" class="list-anchor">#</a>
  
  연결할 Playwright 브라우저 웹소켓 엔드포인트. 이 엔드포인트는 `BrowserServer.wsEndpoint`를 통해 얻을 수 있습니다.
- `options` [Object] *(선택사항)*
  - `exposeNetwork` [string] *(선택사항)* <font size="2">v1.37에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-option-expose-network"/><a href="#browser-type-connect-option-expose-network" class="list-anchor">#</a>
    
    이 옵션은 연결 중인 클라이언트에서 사용할 수 있는 네트워크를 연결 중인 브라우저에 노출합니다. 쉼표로 구분된 규칙 목록으로 구성됩니다.
    
    사용 가능한 규칙:
    1. 호스트명 패턴, 예: `example.com`, `*.org:99`, `x.*.y.com`, `*foo.org`.
    1. IP 리터럴, 예: `127.0.0.1`, `0.0.0.0:99`, `[::1]`, `[0:0::1]:99`.
    1. `<loopback>`으로 로컬 루프백 인터페이스와 일치: `localhost`, `*.localhost`, `127.0.0.1`, `[::1]`.
    
    일반적인 예시:
    1. `"*"` 모든 네트워크를 노출합니다.
    1. `"<loopback>"` localhost 네트워크를 노출합니다.
    1. `"*.test.internal-domain,*.staging.internal-domain,<loopback>"` 테스트/스테이징 배포 및 localhost를 노출합니다.
  - `headers` [Object]&lt;[string], [string]&gt; *(선택사항)* <font size="2">v1.11에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-option-headers"/><a href="#browser-type-connect-option-headers" class="list-anchor">#</a>
    
    웹 소켓 연결 요청과 함께 전송할 추가 HTTP 헤더. 선택사항입니다.
  - `logger` [Logger] *(선택사항)* <font size="2">v1.14에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-option-logger"/><a href="#browser-type-connect-option-logger" class="list-anchor">#</a>
    
    Playwright 로깅을 위한 로거 싱크. 선택사항입니다.
  - `slowMo` [number] *(선택사항)* <font size="2">v1.10에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-option-slow-mo"/><a href="#browser-type-connect-option-slow-mo" class="list-anchor">#</a>
    
    지정된 밀리초만큼 Playwright 작업을 늦춥니다. 진행 상황을 볼 수 있도록 유용합니다. 기본값은 0입니다.
  - `timeout` [number] *(선택사항)* <font size="2">v1.10에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-option-timeout"/><a href="#browser-type-connect-option-timeout" class="list-anchor">#</a>
    
    연결이 설정될 때까지 기다리는 최대 시간(밀리초). 기본값은 `0`(타임아웃 없음)입니다.

**반환값**
- [Promise]&lt;[Browser]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-return"/><a href="#browser-type-connect-return" class="list-anchor">#</a>

### connectOverCDP {#browser-type-connect-over-cdp}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9에 추가됨</font><x-search>browserType.connectOverCDP</x-search>

이 메서드는 Chrome DevTools Protocol을 사용하여 기존 브라우저 인스턴스에 Playwright를 연결합니다.

기본 브라우저 컨텍스트는 [browser.contexts()](/api/class-browser.mdx#browser-contexts)를 통해 접근할 수 있습니다.

:::note

Chrome DevTools Protocol을 통한 연결은 Chromium 기반 브라우저에서만 지원됩니다.
:::

:::note
이 연결은 [browserType.connect()](/api/class-browsertype.mdx#browser-type-connect)를 통한 Playwright 프로토콜 연결보다 상당히 낮은 충실도를 제공합니다. 문제가 발생하거나 고급 기능을 사용하려는 경우에는 [browserType.connect()](/api/class-browsertype.mdx#browser-type-connect)를 사용하는 것이 좋습니다.
:::

**사용법**

```js
const browser = await playwright.chromium.connectOverCDP('http://localhost:9222');
const defaultContext = browser.contexts()[0];
const page = defaultContext.pages()[0];
```

**인자**
- `endpointURL` [string] <font size="2">v1.11에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-over-cdp-option-endpoint-url"/><a href="#browser-type-connect-over-cdp-option-endpoint-url" class="list-anchor">#</a>
  
  연결할 CDP 웹소켓 엔드포인트 또는 http url. 예를 들어 `http://localhost:9222/` 또는 `ws://127.0.0.1:9222/devtools/browser/387adf4c-243f-4051-a181-46798f4a46f4`.
- `options` [Object] *(선택사항)*
  - `endpointURL` [string] *(선택사항)* <font size="2">v1.14에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-over-cdp-option-endpoint-url"/><a href="#browser-type-connect-over-cdp-option-endpoint-url" class="list-anchor">#</a>
    
    더 이상 사용되지 않음, 대신 첫 번째 인자를 사용하세요. 선택사항입니다.
  - `headers` [Object]&lt;[string], [string]&gt; *(선택사항)* <font size="2">v1.11에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-over-cdp-option-headers"/><a href="#browser-type-connect-over-cdp-option-headers" class="list-anchor">#</a>
    
    연결 요청과 함께 전송할 추가 HTTP 헤더. 선택사항입니다.
  - `logger` [Logger] *(선택사항)* <font size="2">v1.14에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-over-cdp-option-logger"/><a href="#browser-type-connect-over-cdp-option-logger" class="list-anchor">#</a>
    
    Playwright 로깅을 위한 로거 싱크. 선택사항입니다.
  - `slowMo` [number] *(선택사항)* <font size="2">v1.11에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-over-cdp-option-slow-mo"/><a href="#browser-type-connect-over-cdp-option-slow-mo" class="list-anchor">#</a>
    
    지정된 밀리초만큼 Playwright 작업을 늦춥니다. 진행 상황을 볼 수 있도록 유용합니다. 기본값은 0입니다.
  - `timeout` [number] *(선택사항)* <font size="2">v1.11에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-over-cdp-option-timeout"/><a href="#browser-type-connect-over-cdp-option-timeout" class="list-anchor">#</a>
    
    연결이 설정될 때까지 기다리는 최대 시간(밀리초). 기본값은 `30000`(30초)입니다. 타임아웃을 비활성화하려면 `0`을 전달하세요.

**반환값**
- [Promise]&lt;[Browser]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-connect-over-cdp-return"/><a href="#browser-type-connect-over-cdp-return" class="list-anchor">#</a>

### launch {#browser-type-launch}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browserType.launch</x-search>

브라우저 인스턴스를 반환합니다.

**사용법**

[ignoreDefaultArgs](/api/class-browsertype.mdx#browser-type-launch-option-ignore-default-args)를 사용하여 기본 인자에서 `--mute-audio`를 필터링할 수 있습니다:

```js
const browser = await chromium.launch({  // 또는 'firefox' 또는 'webkit'.
  ignoreDefaultArgs: ['--mute-audio']
});
```

> **Chromium 전용** Playwright는 Google Chrome 또는 Microsoft Edge 브라우저를 제어하는 데도 사용할 수 있지만, 함께 번들로 제공되는 Chromium 버전에서 가장 잘 작동합니다. 다른 버전에서 작동한다는 보장은 없습니다. [executablePath](/api/class-browsertype.mdx#browser-type-launch-option-executable-path) 옵션은 극도의 주의를 기울여 사용하세요.
>
> Google Chrome(Chromium이 아닌)을 선호하는 경우, [Chrome Canary](https://www.google.com/chrome/browser/canary.html) 또는 [Dev Channel](https://www.chromium.org/getting-involved/dev-channel) 빌드가 권장됩니다.
>
> Google Chrome 및 Microsoft Edge와 같은 일반 브라우저는 비디오 재생을 위한 독점 미디어 코덱이 필요한 테스트에 적합합니다. Chromium과 Chrome 간의 다른 차이점에 대해서는 [이 글](https://www.howtogeek.com/202825/what%E2%80%99s-the-difference-between-chromium-and-chrome/)을 참조하세요. Linux 사용자를 위한 일부 차이점은 [이 글](https://chromium.googlesource.com/chromium/src/+/lkgr/docs/chromium_browser_vs_google_chrome.md)에 설명되어 있습니다.

**인자**
- `options` [Object] *(선택사항)*
  - `args` [Array]&lt;[string]&gt; *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-args"/><a href="#browser-type-launch-server-option-args" class="list-anchor">#</a>
    
    :::warning
    
    커스텀 브라우저 인자는 Playwright 기능을 손상시킬 수 있으므로 자신의 책임하에 사용하세요.
    :::
    
    브라우저 인스턴스에 전달할 추가 인자. Chromium 플래그 목록은 [여기](https://peter.sh/experiments/chromium-command-line-switches/)에서 찾을 수 있습니다.
  - `channel` [string] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-channel"/><a href="#browser-type-launch-server-option-channel" class="list-anchor">#</a>
    
    브라우저 배포 채널.
    
    [새로운 헤드리스 모드](../browsers.mdx#chromium-new-headless-mode)를 사용하려면 "chromium"을 사용하세요.
    
    브랜드가 있는 [Google Chrome 및 Microsoft Edge](../browsers.mdx#google-chrome--microsoft-edge)를 사용하려면 "chrome", "chrome-beta", "chrome-dev", "chrome-canary", "msedge", "msedge-beta", "msedge-dev" 또는 "msedge-canary"를 사용하세요.
  - `chromiumSandbox` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-chromium-sandbox"/><a href="#browser-type-launch-server-option-chromium-sandbox" class="list-anchor">#</a>
    
    Chromium 샌드박싱을 활성화합니다. 기본값은 `false`입니다.
  - `devtools` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-devtools"/><a href="#browser-type-launch-server-option-devtools" class="list-anchor">#</a>
    
    :::warning[더 이상 사용되지 않음]
    대신 [디버깅 도구](../debug.mdx)를 사용하세요.
    :::
    
    
    **Chromium 전용** 각 탭에 대해 개발자 도구 패널을 자동으로 열지 여부. 이 옵션이 `true`이면 [headless](/api/class-browsertype.mdx#browser-type-launch-server-option-headless) 옵션이 `false`로 설정됩니다.
  - `downloadsPath` [string] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-downloads-path"/><a href="#browser-type-launch-server-option-downloads-path" class="list-anchor">#</a>
    
    지정된 경우, 수락된 다운로드는 이 디렉토리에 다운로드됩니다. 그렇지 않으면 임시 디렉토리가 생성되고 브라우저가 닫힐 때 삭제됩니다. 어떤 경우든 다운로드는 생성된 브라우저 컨텍스트가 닫힐 때 삭제됩니다.
  - `env` [Object]&lt;[string], [string] | [number] | [boolean]&gt; *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-env"/><a href="#browser-type-launch-server-option-env" class="list-anchor">#</a>
    
    브라우저에 표시될 환경 변수를 지정합니다. 기본값은 `process.env`입니다.
  - `executablePath` [string] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-executable-path"/><a href="#browser-type-launch-server-option-executable-path" class="list-anchor">#</a>
    
    번들로 제공된 것 대신 실행할 브라우저 실행 파일 경로. [executablePath](/api/class-browsertype.mdx#browser-type-launch-server-option-executable-path)가 상대 경로인 경우 현재 작업 디렉토리를 기준으로 해결됩니다. Playwright는 번들로 제공된 Chromium, Firefox 또는 WebKit에서만 작동하므로 자신의 책임하에 사용하세요.
  - `firefoxUserPrefs` [Object]&lt;[string], [string] | [number] | [boolean]&gt; *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-firefox-user-prefs"/><a href="#browser-type-launch-server-option-firefox-user-prefs" class="list-anchor">#</a>
    
    Firefox 사용자 환경 설정. [`about:config`](https://support.mozilla.org/en-US/kb/about-config-editor-firefox)에서 Firefox 사용자 환경 설정에 대해 자세히 알아보세요.
  - `handleSIGHUP` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-handle-sighup"/><a href="#browser-type-launch-server-option-handle-sighup" class="list-anchor">#</a>
    
    SIGHUP에서 브라우저 프로세스를 닫습니다. 기본값은 `true`입니다.
  - `handleSIGINT` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-handle-sigint"/><a href="#browser-type-launch-server-option-handle-sigint" class="list-anchor">#</a>
    
    Ctrl-C에서 브라우저 프로세스를 닫습니다. 기본값은 `true`입니다.
  - `handleSIGTERM` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-handle-sigterm"/><a href="#browser-type-launch-server-option-handle-sigterm" class="list-anchor">#</a>
    
    SIGTERM에서 브라우저 프로세스를 닫습니다. 기본값은 `true`입니다.
  - `headless` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-headless"/><a href="#browser-type-launch-server-option-headless" class="list-anchor">#</a>
    
    브라우저를 헤드리스 모드로 실행할지 여부. [Chromium](https://developers.google.com/web/updates/2017/04/headless-chrome) 및 [Firefox](https://developer.mozilla.org/en-US/docs/Mozilla/Firefox/Headless_mode)에 대한 자세한 내용을 확인하세요. [devtools](/api/class-browsertype.mdx#browser-type-launch-option-devtools) 옵션이 `true`가 아닌 경우 기본값은 `true`입니다.
  - `ignoreDefaultArgs` [boolean] | [Array]&lt;[string]&gt; *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-ignore-default-args"/><a href="#browser-type-launch-server-option-ignore-default-args" class="list-anchor">#</a>
    
    `true`인 경우, Playwright는 자체 구성 인자를 전달하지 않고 [args](/api/class-browsertype.mdx#browser-type-launch-server-option-args)의 인자만 사용합니다. 배열이 지정된 경우 주어진 기본 인자를 필터링합니다. 위험한 옵션이므로 주의해서 사용하세요. 기본값은 `false`입니다.
  - `logger` [Logger] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-logger"/><a href="#browser-type-launch-server-option-logger" class="list-anchor">#</a>
    
    Playwright 로깅을 위한 로거 싱크.

### launchPersistentContext {#browser-type-launch-persistent-context}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browserType.launchPersistentContext</x-search>

영구 브라우저 컨텍스트 인스턴스를 반환합니다.

[userDataDir](/api/class-browsertype.mdx#browser-type-launch-persistent-context-option-user-data-dir)에 위치한 영구 저장소를 사용하는 브라우저를 실행하고 유일한 컨텍스트를 반환합니다. 이 컨텍스트를 닫으면 브라우저가 자동으로 닫힙니다.

**사용법**

```js
await browserType.launchPersistentContext(userDataDir);
await browserType.launchPersistentContext(userDataDir, options);
```

**인자**
- `userDataDir` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-persistent-context-option-user-data-dir"/><a href="#browser-type-launch-persistent-context-option-user-data-dir" class="list-anchor">#</a>

[Chromium](https://chromium.googlesource.com/chromium/src/+/master/docs/user_data_dir.md#introduction) 및 [Firefox](https://developer.mozilla.org/en-US/docs/Mozilla/Command_Line_Options#User_Profile)에 대한 자세한 내용을 참조하세요. Chromium의 사용자 데이터 디렉토리는 `chrome://version`에서 볼 수 있는 "Profile Path"의 **상위** 디렉토리입니다. 대신 임시 디렉토리를 사용하려면 빈 문자열을 전달하세요.
- `options` [Object] *(선택사항)*

  - `args` [Array]&lt;[string]&gt; *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-persistent-context-option-args"/><a href="#browser-type-launch-persistent-context-option-args" class="list-anchor">#</a>

    `null` 값은 기본 프리셋에서 제외되어 뷰포트가 운영 체제에서 정의한 호스트 창 크기에 따라 달라지게 합니다. 이는 테스트 실행을 비결정적으로 만듭니다.
    :::

### launchServer {#browser-type-launch-server}

<font size="2" style={{position: "relative", top: "-20px"}}>v1.9 이전에 추가됨</font><x-search>browserType.launchServer</x-search>

브라우저 앱 인스턴스를 반환합니다. [browserType.connect()](/api/class-browsertype.mdx#browser-type-connect)를 통해 연결할 수 있으며, 이는 메이저/마이너 클라이언트/서버 버전이 일치해야 합니다(1.2.3은 1.2.x와 호환됨).

**사용법**

클라이언트가 연결할 수 있는 브라우저 서버를 실행합니다. 브라우저 실행 파일을 실행하고 나중에 연결하는 예:

```js
const { chromium } = require('playwright');  // 또는 'webkit' 또는 'firefox'.

(async () => {
  const browserServer = await chromium.launchServer();
  const wsEndpoint = browserServer.wsEndpoint();
  // 웹 소켓 엔드포인트를 나중에 연결하는 데 사용합니다.
  const browser = await chromium.connect(wsEndpoint);
  // 브라우저 인스턴스를 닫습니다.
  await browserServer.close();
})();
```

**인자**
- `options` [Object] *(선택사항)*
  - `args` [Array]&lt;[string]&gt; *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-args"/><a href="#browser-type-launch-server-option-args" class="list-anchor">#</a>
    
    :::warning
    
    커스텀 브라우저 인자는 Playwright 기능을 손상시킬 수 있으므로 자신의 책임하에 사용하세요.
    :::
    
    브라우저 인스턴스에 전달할 추가 인자입니다. Chromium 플래그 목록은 [여기](https://peter.sh/experiments/chromium-command-line-switches/)에서 찾을 수 있습니다.
  - `channel` [string] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-channel"/><a href="#browser-type-launch-server-option-channel" class="list-anchor">#</a>
    
    브라우저 배포 채널입니다.
    
    [새로운 헤드리스 모드](../browsers.mdx#chromium-new-headless-mode)를 사용하려면 "chromium"을 사용하세요.
    
    브랜드가 있는 [Google Chrome 및 Microsoft Edge](../browsers.mdx#google-chrome--microsoft-edge)를 사용하려면 "chrome", "chrome-beta", "chrome-dev", "chrome-canary", "msedge", "msedge-beta", "msedge-dev", 또는 "msedge-canary"를 사용하세요.
  - `chromiumSandbox` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-chromium-sandbox"/><a href="#browser-type-launch-server-option-chromium-sandbox" class="list-anchor">#</a>
    
    Chromium 샌드박싱을 활성화합니다. 기본값은 `false`입니다.
  - `devtools` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-devtools"/><a href="#browser-type-launch-server-option-devtools" class="list-anchor">#</a>
    
    :::warning[더 이상 사용되지 않음]
    대신 [디버깅 도구](../debug.mdx)를 사용하세요.
    :::
    
    
    **Chromium 전용** 각 탭에 대해 개발자 도구 패널을 자동으로 열지 여부입니다. 이 옵션이 `true`이면 [headless](/api/class-browsertype.mdx#browser-type-launch-server-option-headless) 옵션이 `false`로 설정됩니다.
  - `downloadsPath` [string] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-downloads-path"/><a href="#browser-type-launch-server-option-downloads-path" class="list-anchor">#</a>
    
    지정된 경우, 수락된 다운로드는 이 디렉토리에 다운로드됩니다. 그렇지 않으면 임시 디렉토리가 생성되고 브라우저가 닫힐 때 삭제됩니다. 어떤 경우든 다운로드는 생성된 브라우저 컨텍스트가 닫힐 때 삭제됩니다.
  - `env` [Object]&lt;[string], [string] | [number] | [boolean]&gt; *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-env"/><a href="#browser-type-launch-server-option-env" class="list-anchor">#</a>
    
    브라우저에 표시될 환경 변수를 지정합니다. 기본값은 `process.env`입니다.
  - `executablePath` [string] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-executable-path"/><a href="#browser-type-launch-server-option-executable-path" class="list-anchor">#</a>
    
    번들로 제공된 것 대신 실행할 브라우저 실행 파일 경로입니다. [executablePath](/api/class-browsertype.mdx#browser-type-launch-server-option-executable-path)가 상대 경로인 경우 현재 작업 디렉토리를 기준으로 해결됩니다. Playwright는 번들로 제공된 Chromium, Firefox 또는 WebKit에서만 작동하므로 자신의 책임하에 사용하세요.
  - `firefoxUserPrefs` [Object]&lt;[string], [string] | [number] | [boolean]&gt; *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-firefox-user-prefs"/><a href="#browser-type-launch-server-option-firefox-user-prefs" class="list-anchor">#</a>
    
    Firefox 사용자 환경 설정입니다. [`about:config`](https://support.mozilla.org/en-US/kb/about-config-editor-firefox)에서 Firefox 사용자 환경 설정에 대해 자세히 알아보세요.
  - `handleSIGHUP` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-handle-sighup"/><a href="#browser-type-launch-server-option-handle-sighup" class="list-anchor">#</a>
    
    SIGHUP에서 브라우저 프로세스를 닫습니다. 기본값은 `true`입니다.
  - `handleSIGINT` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-handle-sigint"/><a href="#browser-type-launch-server-option-handle-sigint" class="list-anchor">#</a>
    
    Ctrl-C에서 브라우저 프로세스를 닫습니다. 기본값은 `true`입니다.
  - `handleSIGTERM` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-handle-sigterm"/><a href="#browser-type-launch-server-option-handle-sigterm" class="list-anchor">#</a>
    
    SIGTERM에서 브라우저 프로세스를 닫습니다. 기본값은 `true`입니다.
  - `headless` [boolean] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-headless"/><a href="#browser-type-launch-server-option-headless" class="list-anchor">#</a>
    
    브라우저를 헤드리스 모드로 실행할지 여부입니다. [Chromium](https://developers.google.com/web/updates/2017/04/headless-chrome) 및 [Firefox](https://developer.mozilla.org/en-US/docs/Mozilla/Firefox/Headless_mode)에 대한 자세한 내용을 확인하세요. [devtools](/api/class-browsertype.mdx#browser-type-launch-option-devtools) 옵션이 `true`가 아닌 경우 기본값은 `true`입니다.
  - `host` [string] *(선택사항)* <font size="2">v1.45에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-host"/><a href="#browser-type-launch-server-option-host" class="list-anchor">#</a>
    
    웹 소켓에 사용할 호스트입니다. 선택 사항이며 생략하면 서버는 IPv6를 사용할 수 있는 경우 지정되지 않은 IPv6 주소(::)에서, 그렇지 않으면 지정되지 않은 IPv4 주소(0.0.0.0)에서 연결을 수락합니다. 특정 인터페이스를 선택하여 보안을 강화하는 것을 고려하세요.
  - `ignoreDefaultArgs` [boolean] | [Array]&lt;[string]&gt; *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-ignore-default-args"/><a href="#browser-type-launch-server-option-ignore-default-args" class="list-anchor">#</a>
    
    `true`인 경우, Playwright는 자체 구성 인자를 전달하지 않고 [args](/api/class-browsertype.mdx#browser-type-launch-server-option-args)의 인자만 사용합니다. 배열이 지정된 경우 주어진 기본 인자를 필터링합니다. 위험한 옵션이므로 주의해서 사용하세요. 기본값은 `false`입니다.
  - `logger` [Logger] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-logger"/><a href="#browser-type-launch-server-option-logger" class="list-anchor">#</a>
    
    Playwright 로깅을 위한 로거 싱크입니다.
  - `port` [number] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-port"/><a href="#browser-type-launch-server-option-port" class="list-anchor">#</a>
    
    웹 소켓에 사용할 포트입니다. 기본값은 사용 가능한 포트를 선택하는 0입니다.
  - `proxy` [Object] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-proxy"/><a href="#browser-type-launch-server-option-proxy" class="list-anchor">#</a>
    - `server` [string]
      
      모든 요청에 사용할 프록시입니다. HTTP 및 SOCKS 프록시가 지원됩니다(예: `http://myproxy.com:3128` 또는 `socks5://myproxy.com:3128`). 짧은 형식 `myproxy.com:3128`은 HTTP 프록시로 간주됩니다.
    - `bypass` [string] *(선택사항)*
      
      프록시를 우회할 선택적 쉼표로 구분된 도메인입니다(예: `".com, chromium.org, .domain.com"`).
    - `username` [string] *(선택사항)*
      
      HTTP 프록시가 인증을 요구하는 경우 사용할 선택적 사용자 이름입니다.
    - `password` [string] *(선택사항)*
      
      HTTP 프록시가 인증을 요구하는 경우 사용할 선택적 비밀번호입니다.
    
    네트워크 프록시 설정입니다.
  - `timeout` [number] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-timeout"/><a href="#browser-type-launch-server-option-timeout" class="list-anchor">#</a>
    
    브라우저 인스턴스가 시작될 때까지 기다리는 최대 시간(밀리초)입니다. 기본값은 `30000`(30초)입니다. 타임아웃을 비활성화하려면 `0`을 전달하세요.
  - `tracesDir` [string] *(선택사항)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-traces-dir"/><a href="#browser-type-launch-server-option-traces-dir" class="list-anchor">#</a>
    
    지정된 경우, 트레이스가 이 디렉토리에 저장됩니다.
  - `wsPath` [string] *(선택사항)* <font size="2">v1.15에 추가됨</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-server-option-ws-path"/><a href="#browser-type-launch-server-option-ws-path" class="list-anchor">#</a>
    
    브라우저 서버를 제공할 경로입니다. 보안상의 이유로 기본값은 추측할 수 없는 문자열입니다.
    
    :::warning
    
    `wsPath`를 알고 있는 프로세스나 웹 페이지(Playwright에서 실행 중인 페이지 포함)는 OS 사용자를 제어할 수 있습니다. 이런 이유로, 이 옵션을 사용할 때는 추측할 수 없는 토큰을 사용해야 합니다.
    :::
