`Suite`는 테스트 그룹입니다. Playwright Test의 모든 테스트는 다음 계층 구조를 형성합니다.
* 루트 스위트는 각 [FullProject]에 대한 하위 스위트를 가집니다.
  * 프로젝트 스위트 #1. 프로젝트의 각 테스트 파일에 대한 하위 스위트를 가집니다.
    * 파일 스위트 #1
      * [TestCase] #1
      * [TestCase] #2
      * [test.describe()](/api/class-test.mdx#test-describe) 그룹에 해당하는 스위트
        * 그룹 내의 [TestCase] #1
        * 그룹 내의 [TestCase] #2
      * < 더 많은 테스트 케이스 ... >
    * 파일 스위트 #2
    * < 더 많은 파일 스위트 ... >
  * 프로젝트 스위트 #2
  * < 더 많은 프로젝트 스위트 ... >

Reporter는 [reporter.onBegin()](/api/class-reporter.mdx#reporter-on-begin) 메서드에서 루트 스위트를 받습니다.

## Methods

### allTests {#suite-all-tests}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>suite.allTests</x-search>

[suite.tests](/api/class-suite.mdx#suite-tests)와 반대로 이 스위트와 그 하위 스위트의 모든 테스트 케이스 목록을 반환합니다.

**Usage**

```js
suite.allTests();
```

**Returns**
- [Array]&lt;[TestCase]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="suite-all-tests-return"/><a href="#suite-all-tests-return" class="list-anchor">#</a>

---

### entries {#suite-entries}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.44</font><x-search>suite.entries</x-search>

이 스위트에서 직접 정의된 테스트 케이스 및 스위트. 요소는 선언 순서대로 반환됩니다. [testCase.type](/api/class-testcase.mdx#test-case-type) 및 [suite.type](/api/class-suite.mdx#suite-type)을 사용하여 다양한 항목 유형을 구별할 수 있습니다.

**Usage**

```js
suite.entries();
```

**Returns**
- [Array]&lt;[TestCase] | [Suite]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="suite-entries-return"/><a href="#suite-entries-return" class="list-anchor">#</a>

---

### project {#suite-project}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>suite.project</x-search>

이 스위트가 속한 프로젝트의 구성 또는 루트 스위트의 경우 [void]입니다.

**Usage**

```js
suite.project();
```

**Returns**
- [FullProject] | [undefined]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="suite-project-return"/><a href="#suite-project-return" class="list-anchor">#</a>

---

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>suite.titlePath</x-search>

루트에서 이 스위트까지 제목 목록을 반환합니다.

**Usage**

```js
suite.titlePath();
```

**Returns**
- [Array]&lt;[string]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="suite-title-path-return"/><a href="#suite-title-path" class="list-anchor">#</a>

## Properties

### location {#suite-location}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>suite.location</x-search>

스위트가 정의된 소스 위치입니다. 루트 및 프로젝트 스위트에는 없습니다.

**Usage**

```js
suite.location
```

**Type**
- [Location]

---

### parent {#suite-parent}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>suite.parent</x-search>

부모 스위트, 루트 스위트에는 없습니다.

**Usage**

```js
suite.parent
```

**Type**
- [Suite]

---

### suites {#suite-suites}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>suite.suites</x-search>

자식 스위트. 스위트의 계층 구조는 [Suite]를 참조하십시오.

**Usage**

```js
suite.suites
```

**Type**
- [Array]&lt;[Suite]&gt;

---

### tests {#suite-tests}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>suite.tests</x-search>

스위트의 테스트 케이스. 이 스위트에서 직접 정의된 테스트 케이스만 목록에 있습니다. 중첩된 [test.describe()](/api/class-test.mdx#test-describe) 그룹에서 정의된 테스트 케이스는 하위 [suite.suites](/api/class-suite.mdx#suite-suites)에 나열됩니다.

**Usage**

```js
suite.tests
```

**Type**
- [Array]&lt;[TestCase]&gt;

---

### title {#suite-title}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font><x-search>suite.title</x-search>

스위트 제목.
* 루트 스위트의 경우 비어 있습니다.
* 프로젝트 스위트의 경우 프로젝트 이름입니다.
* 파일 스위트의 경우 파일 경로입니다.
* 그룹 스위트의 경우 [test.describe()](/api/class-test.mdx#test-describe)에 전달된 제목입니다.

**Usage**

```js
suite.title
```

**Type**
- [string]

---

### type {#suite-type}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.44</font><x-search>suite.type</x-search>

스위트의 유형을 반환합니다. 스위트는 `root` -> `project` -> `file` -> `describe` -> ...`describe` -> `test`와 같은 계층 구조를 이룹니다.

**Usage**

```js
suite.type
```

**Type**
- "root" | "project" | "file" | "describe"
