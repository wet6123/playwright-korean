## Introduction

Playwright를 사용하면 웹 페이지에서 발생하는 다양한 유형의 이벤트(예: 네트워크 요청, 자식 페이지 생성, 전용 워커 등)를 수신할 수 있습니다. 이벤트를 기다리거나 이벤트 리스너를 추가 또는 제거하는 등 이러한 이벤트를 구독하는 방법에는 여러 가지가 있습니다.

## Waiting for event

대부분의 경우 스크립트는 특정 이벤트가 발생할 때까지 기다려야 합니다. 다음은 일반적인 이벤트 대기 패턴입니다.

[page.waitForRequest()](/api/class-page.mdx#page-wait-for-request)를 사용하여 지정된 URL의 요청을 기다립니다.

```js
// goto 전에 요청을 기다리기 시작합니다. await가 없음에 유의하세요.
const requestPromise = page.waitForRequest('**/*logo*.png');
await page.goto('https://wikipedia.org');
const request = await requestPromise;
console.log(request.url());
```

팝업 창을 기다립니다.

```js
// 클릭하기 전에 팝업을 기다리기 시작합니다. await가 없음에 유의하세요.
const popupPromise = page.waitForEvent('popup');
await page.getByText('open the popup').click();
const popup = await popupPromise;
await popup.goto('https://wikipedia.org');
```

## Adding/removing event listener

때로는 이벤트가 임의의 시간에 발생하고 이벤트를 기다리는 대신 처리해야 합니다. Playwright는 이벤트 구독 및 구독 취소를 위한 기존 언어 메커니즘을 지원합니다.

```js
page.on('request', request => console.log(`Request sent: ${request.url()}`));
const listener = request => console.log(`Request finished: ${request.url()}`);
page.on('requestfinished', listener);
await page.goto('https://wikipedia.org');

page.off('requestfinished', listener);
await page.goto('https://www.openstreetmap.org/');
```

## Adding one-off listeners

특정 이벤트를 한 번만 처리해야 하는 경우 이를 위한 편리한 API가 있습니다.

```js
page.once('dialog', dialog => dialog.accept('2021'));
await page.evaluate("prompt('Enter a number:')");
```
