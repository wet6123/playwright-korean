## Introduction

Playwright는 로컬 머신에서 브라우저를 실행하는 대신 Selenium 4를 실행하는 [Selenium Grid Hub](https://www.selenium.dev/documentation/grid/)에 연결하여 **Google Chrome** 또는 **Microsoft Edge** 브라우저를 실행할 수 있습니다. 이 기능은 **실험적**이며 그에 따라 우선 순위가 지정됩니다.

:::warning

Playwright와 Selenium Grid Hub의 통합이 향후 중단될 위험이 있습니다. 사용하기 전에 위험과 이점을 신중하게 고려하십시오.

<details>

<summary>

<span style={{textTransform:'uppercase',fontSize:'smaller',fontWeight:'bold',opacity:'0.6'}}>More details</span>

</summary>

내부적으로 Playwright는 [Chrome DevTools Protocol](https://chromedevtools.github.io/devtools-protocol/) 웹소켓을 사용하여 브라우저에 연결합니다. Selenium 4는 현재 이 기능을 제공합니다. 그러나 이 기능은 [향후 변경될 수 있습니다](https://github.com/SeleniumHQ/selenium/issues/11590#issuecomment-1436113950). Selenium이 이 기능을 제거하면 Playwright는 더 이상 작동하지 않습니다.

</details>
:::

Playwright를 Selenium Grid에 연결하기 전에 그리드가 [Selenium WebDriver](https://www.selenium.dev/documentation/webdriver/)와 함께 작동하는지 확인하십시오. 예를 들어 [예제 중 하나](https://github.com/SeleniumHQ/selenium/tree/trunk/javascript/node/selenium-webdriver/example)를 실행하고 `SELENIUM_REMOTE_URL` 환경 변수를 전달합니다. 웹 드라이버 예제가 작동하지 않으면 Selenium 허브/노드/독립 실행형 출력에서 오류를 찾고 가능한 해결책을 위해 [Selenium 이슈](https://github.com/SeleniumHQ/selenium/issues)를 검색하십시오.

## Starting Selenium Grid

분산된 Selenium Grid를 실행하는 경우 Playwright는 브라우저에 연결할 수 있도록 selenium 노드가 접근 가능한 주소로 등록되어야 합니다. 예상대로 작동하는지 확인하려면 selenium 노드를 실행할 때 허브를 가리키는 `SE_NODE_GRID_URL` 환경 변수를 설정합니다.

```bash
# selenium 노드 시작
SE_NODE_GRID_URL="http://<selenium-hub-ip>:4444" java -jar selenium-server-<version>.jar node
```

## Connecting Playwright to Selenium Grid

Playwright를 **Selenium Grid 4**에 연결하려면 Selenium Grid Hub를 가리키는 `SELENIUM_REMOTE_URL` 환경 변수를 설정합니다. 이것은 Google Chrome 및 Microsoft Edge에서만 작동합니다.

```bash
SELENIUM_REMOTE_URL=http://<selenium-hub-ip>:4444 npx playwright test
```

코드를 변경할 필요 없이 테스트 하네스 또는 [browserType.launch()](/api/class-browsertype.mdx#browser-type-launch)를 평소대로 사용하십시오.

### Passing additional capabilities

그리드에서 추가 기능을 설정해야 하는 경우(예: 외부 서비스를 사용하는 경우) `SELENIUM_REMOTE_CAPABILITIES` 환경 변수를 설정하여 JSON 직렬화된 기능을 제공할 수 있습니다.

```bash
SELENIUM_REMOTE_URL=http://<selenium-hub-ip>:4444 SELENIUM_REMOTE_CAPABILITIES="{'mygrid:options':{os:'windows',username:'John',password:'secure'}}" npx playwright test
```

### Passing additional headers

그리드에서 추가 헤더를 설정해야 하는 경우(예: 클라우드에서 브라우저를 사용하기 위해 인증 토큰을 제공해야 하는 경우) `SELENIUM_REMOTE_HEADERS` 환경 변수를 설정하여 JSON 직렬화된 헤더를 제공할 수 있습니다.

```bash
SELENIUM_REMOTE_URL=http://<selenium-hub-ip>:4444 SELENIUM_REMOTE_HEADERS="{'Authorization':'Basic b64enc'}" npx playwright test
```

### Detailed logs

Playwright가 Selenium Grid에 어떻게 연결되는지 확인하려면 `DEBUG=pw:browser*` 환경 변수를 사용하여 실행합니다.

```bash
DEBUG=pw:browser* SELENIUM_REMOTE_URL=http://internal.grid:4444 npx playwright test
```

문제를 제기하는 경우 이 로그를 포함하십시오.

## Using Selenium Docker

Selenium Grid를 사용하는 쉬운 방법 중 하나는 공식 docker 컨테이너를 실행하는 것입니다. 자세한 내용은 [selenium docker images](https://github.com/SeleniumHQ/docker-selenium) 문서를 참조하십시오. 이미지 태깅 규칙에 대한 자세한 내용은 [자세히 읽어보십시오](https://github.com/SeleniumHQ/docker-selenium/wiki/Tagging-Convention#selenium-grid-4x-and-above).

### Standalone mode

다음은 selenium 독립 실행형을 실행하고 Playwright를 연결하는 예제입니다. 허브와 노드는 동일한 `localhost`에 있으며 허브를 가리키는 `SE_NODE_GRID_URL` 환경 변수를 전달합니다.

먼저 Selenium을 시작합니다.

```bash
docker run -d -p 4444:4444 --shm-size="2g" -e SE_NODE_GRID_URL="http://localhost:4444" selenium/standalone-chromium:latest
```

그런 다음 Playwright를 실행합니다.

```bash
SELENIUM_REMOTE_URL=http://localhost:4444 npx playwright test
```

### Hub and nodes mode

다음은 selenium 허브와 단일 selenium 노드를 실행하고 Playwright를 허브에 연결하는 예제입니다. 허브와 노드는 IP가 다르며 노드 컨테이너를 시작할 때 허브를 가리키는 `SE_NODE_GRID_URL` 환경 변수를 전달합니다.

먼저 허브 컨테이너와 하나 이상의 노드 컨테이너를 시작합니다.

```bash
docker run -d -p 4442-4444:4442-4444 --name selenium-hub selenium/hub:4.25.0
docker run -d -p 5555:5555 \
    --shm-size="2g" \
    -e SE_EVENT_BUS_HOST=<selenium-hub-ip> \
    -e SE_EVENT_BUS_PUBLISH_PORT=4442 \
    -e SE_EVENT_BUS_SUBSCRIBE_PORT=4443 \
    -e SE_NODE_GRID_URL="http://<selenium-hub-ip>:4444"
    selenium/node-chromium:4.25.0
```

그런 다음 Playwright를 실행합니다.

```bash
SELENIUM_REMOTE_URL=http://<selenium-hub-ip>:4444 npx playwright test
```

## Selenium 3

내부적으로 Playwright는 [Chrome DevTools Protocol](https://chromedevtools.github.io/devtools-protocol/) 웹소켓을 사용하여 브라우저에 연결합니다. Selenium 4는 이 기능을 제공하지만 Selenium 3는 그렇지 않습니다.

이는 Selenium 3가 최선을 다해 지원된다는 것을 의미하며, Playwright는 그리드 노드에 직접 연결하려고 시도합니다. 그리드 노드는 Playwright를 실행하는 머신에서 직접 액세스할 수 있어야 합니다.
