## Introduction

다음은 명령줄에서 사용할 수 있는 가장 일반적인 옵션입니다.
- 모든 테스트 실행
  
  ```bash
  npx playwright test
  ```
  
- 단일 테스트 파일 실행
  
  ```bash
  npx playwright test tests/todo-page.spec.ts
  ```
  
- 테스트 파일 집합 실행
  
  ```bash
  npx playwright test tests/todo-page/ tests/landing-page/
  ```
  
- 파일 이름에 `my-spec` 또는 `my-spec-2`가 있는 파일 실행
  
  ```bash
  npx playwright test my-spec my-spec-2
  ```
  
- my-spec.ts의 42번째 줄에 있는 테스트 실행
  
  ```bash
  npx playwright test my-spec.ts:42
  ```
  
- 제목으로 테스트 실행
  
  ```bash
  npx playwright test -g "add a todo item"
  ```
  
- 헤드리스 브라우저에서 테스트 실행
  
  ```bash
  npx playwright test --headed
  ```
  
- 특정 프로젝트에 대해 모든 테스트 실행
  
  ```bash
  npx playwright test --project=chromium
  ```
  
- [병렬화](./test-parallel.mdx) 비활성화
  
  ```bash
  npx playwright test --workers=1
  ```
  
- [리포터](./test-reporters.mdx) 선택
  
  ```bash
  npx playwright test --reporter=dot
  ```
  
- [Playwright Inspector](./debug.mdx)로 디버그 모드에서 실행
  
  ```bash
  npx playwright test --debug
  ```
  
- 기본 제공 감시 모드를 사용하는 대화형 UI 모드에서 테스트 실행 (미리보기)
  
  ```bash
  npx playwright test --ui
  ```
  
- 도움말 요청
  
  ```bash
  npx playwright test --help
  ```
  
## Reference

Playwright Test 옵션의 전체 집합은 [구성 파일](./test-use-options.mdx)에서 사용할 수 있습니다. 다음 옵션은 명령줄에 전달할 수 있으며 구성 파일보다 우선 순위가 높습니다.

<!-- // Note: packages/playwright/src/program.ts is the source of truth. -->

| Option | Description |
| :- | :- |
| Non-option arguments | 각 인수는 전체 테스트 파일 경로와 일치하는 정규식으로 처리됩니다. 패턴과 일치하는 파일의 테스트만 실행됩니다. `$` 또는 `*`와 같은 특수 기호는 `\`로 이스케이프해야 합니다. 많은 셸/터미널에서 인수를 따옴표로 묶어야 할 수도 있습니다. |
| `-c <file>` or `--config <file>` | 구성 파일 또는 선택적 "playwright.config.&#123;m,c&#125;?&#123;js,ts&#125;"가 있는 테스트 디렉터리입니다. 기본값은 현재 디렉터리의 `playwright.config.ts` 또는 `playwright.config.js`입니다. |
| `--debug` | Playwright Inspector로 테스트를 실행합니다. `PWDEBUG=1` 환경 변수 및 `--timeout=0 --max-failures=1 --headed --workers=1` 옵션의 단축키입니다. |
| `--fail-on-flaky-tests` | 모든 테스트가 불안정으로 플래그된 경우 실패합니다 (기본값: false). |
| `--forbid-only` | `test.only`가 호출되면 실패합니다 (기본값: false). CI에서 유용합니다. |
| `--fully-parallel` | 모든 테스트를 병렬로 실행합니다 (기본값: false). |
| `--global-timeout <timeout>` | 이 테스트 스위트가 밀리초 단위로 실행할 수 있는 최대 시간 (기본값: 무제한). |
| `-g <grep>` or `--grep <grep>` | 이 정규식과 일치하는 테스트만 실행합니다 (기본값: ".*"). |
| `-gv <grep>` or `--grep-invert <grep>` | 이 정규식과 일치하지 않는 테스트만 실행합니다. |
| `--headed` | 헤드리스 브라우저에서 테스트를 실행합니다 (기본값: 헤드리스). |
| `--ignore-snapshots` | 스크린샷 및 스냅샷 예상을 무시합니다. |
| `--last-failed` | 실패한 항목만 다시 실행합니다. |
| `--list` | 모든 테스트를 수집하여 보고하지만 실행하지는 않습니다. |
| `--max-failures <N>` or `-x` | 첫 번째 `N` 실패 후 중지합니다. `-x`를 전달하면 첫 번째 실패 후 중지됩니다. |
| `--no-deps` | 프로젝트 종속성을 실행하지 않습니다. |
| `--output <dir>` | 출력 아티팩트 폴더 (기본값: "test-results"). |
| `--only-changed [ref]` | 'HEAD'와 'ref' 사이에 변경된 테스트 파일만 실행합니다. 기본적으로 커밋되지 않은 모든 변경 사항을 실행합니다. Git만 지원합니다. |
| `--pass-with-no-tests` | 테스트가 발견되지 않아도 테스트 실행이 성공하도록 합니다. |
| `--project <project-name...>` | 지정된 프로젝트 목록의 테스트만 실행하고 '*' 와일드카드를 지원합니다 (기본값: 모든 프로젝트 실행). |
| `--quiet` | 표준 출력을 억제합니다. |
| `--repeat-each <N>` | 각 테스트를 `N`번 실행합니다 (기본값: 1). |
| `--reporter <reporter>` | 사용할 리포터 (쉼표로 구분, "dot", "line", "list" 또는 기타일 수 있음) (기본값: "list"). 사용자 지정 리포터 파일 경로를 전달할 수도 있습니다. |
| `--retries <retries>` | 불안정한 테스트에 대한 최대 재시도 횟수, 재시도하지 않으려면 0 (기본값: 재시도 없음). |
| `--shard <shard>` | 테스트를 샤딩하고 "current/all" 형식으로 지정된 선택한 샤드만 실행합니다 (예: "3/5"). |
| `--timeout <timeout>` | 테스트 시간 초과 임계값을 밀리초 단위로 지정하고 무제한은 0으로 지정합니다 (기본값: 30초). |
| `--trace <mode>` | 추적 모드를 강제 실행합니다. "on", "off", "on-first-retry", "on-all-retries", "retain-on-failure", "retain-on-first-failure"일 수 있습니다. |
| `--tsconfig <path>` | 가져온 모든 파일에 적용 가능한 단일 tsconfig 경로 (기본값: 가져온 각 파일에 대해 tsconfig를 별도로 찾습니다). |
| `--ui` | 대화형 UI 모드에서 테스트를 실행합니다. |
| `--ui-host <host>` | UI를 제공할 호스트입니다. 이 옵션을 지정하면 브라우저 탭에서 UI가 열립니다. |
| `--ui-port <port>` | UI를 제공할 포트입니다. 사용 가능한 포트는 0입니다. 이 옵션을 지정하면 브라우저 탭에서 UI가 열립니다. |
| `-u` or `--update-snapshots [mode]` | 실제 결과로 스냅샷을 업데이트합니다. 가능한 값은 "all", "changed", "missing" 및 "none"입니다. 플래그 없이 테스트를 실행하면 기본적으로 "missing"이 됩니다. 플래그를 사용하고 값을 지정하지 않고 테스트를 실행하면 기본적으로 "changed"가 됩니다. |
| `--update-source-method [mode]` | 실제 결과로 스냅샷을 업데이트합니다. 가능한 값은 "patch" (기본값), "3way" 및 "overwrite"입니다. "Patch"는 나중에 소스 코드를 업데이트하는 데 사용할 수 있는 통합 diff 파일을 만듭니다. "3way"는 소스 코드에 병합 충돌 마커를 생성합니다. "Overwrite"는 소스 코드를 새 스냅샷 값으로 덮어씁니다. |
| `-j <workers>` or `--workers <workers>` | 동시 작업자 수 또는 논리적 CPU 코어의 백분율, 단일 작업자에서 실행하려면 1을 사용합니다 (기본값: 50%). |
| `-x` | 첫 번째 실패 후 중지합니다. |
