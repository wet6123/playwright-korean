## 소개

Playwright Test는 엔드 투 엔드 테스트의 요구 사항을 수용하기 위해 특별히 만들어졌습니다. Playwright는 Chromium, WebKit 및 Firefox를 포함한 모든 최신 렌더링 엔진을 지원합니다. Windows, Linux 및 macOS에서 로컬 또는 CI에서, 헤드리스 또는 헤드 모드로 Android용 Google Chrome 및 Mobile Safari의 기본 모바일 에뮬레이션을 사용하여 테스트합니다.

Playwright를 설치하고 테스트를 생성하여 작동하는지 확인하십시오. 또는 [CLI](./intro.mdx)를 사용하여 테스트를 시작하고 실행할 수도 있습니다.

<LiteYouTube id="Xz6lhEzgI5I" title="Getting Started with Playwright in VS Code" />

## 설치

Playwright에는 Node.js로 테스트할 때 사용할 수 있는 VS Code 확장 프로그램이 있습니다. [VS Code Marketplace에서 설치](https://marketplace.visualstudio.com/items?itemName=ms-playwright.playwright)하거나 VS Code의 확장 프로그램 탭에서 설치하세요.

![VS Code extension for Playwright](https://github.com/microsoft/playwright/assets/13063165/cab54568-3168-4b3f-bf3d-854976594903)

설치가 완료되면 명령 패널을 열고 다음을 입력합니다.

```bash
Install Playwright
```

![install playwright](https://github.com/microsoft/playwright/assets/13063165/14e91050-24ab-4ff1-a37b-57d7c15e5c35)

**Test: Install Playwright**를 선택하고 테스트를 실행할 브라우저를 선택합니다. 이러한 브라우저는 나중에 [playwright.config](./test-configuration.mdx) 파일에서 구성할 수 있습니다. GitHub Actions 설정을 통해 [CI에서 테스트를 실행](./ci-intro.mdx)할지 여부를 선택할 수도 있습니다.

![choose browsers](https://github.com/microsoft/playwright/assets/13063165/c9e8a25a-e9e8-4419-aeb5-1b8ba58bd71d)

### 테스트 사이드바 열기

활동 모음에서 테스트 아이콘을 클릭하여 테스트 사이드바를 열 수 있습니다. 이렇게 하면 테스트 탐색기에 액세스할 수 있으며, 여기에는 프로젝트의 모든 테스트와 프로젝트, 설정, 도구 및 설정을 포함하는 Playwright 사이드바가 표시됩니다.

![Testing Sidebar](https://github.com/microsoft/playwright/assets/13063165/d203fe83-6015-4e7a-b816-35d373906b24)

## 테스트 실행

테스트 블록 옆에 있는 녹색 삼각형을 클릭하여 단일 테스트를 실행할 수 있습니다. Playwright는 테스트의 각 라인을 실행하고 완료되면 테스트 블록 옆에 녹색 체크 표시가 표시되고 테스트를 실행하는 데 걸린 시간이 표시됩니다.

<LiteYouTube id="mQmcIBMsc38" title="Getting Started with Playwright in VS Code" />

![run a single test](https://github.com/microsoft/playwright/assets/13063165/69dbccfc-4e9f-40e7-bcdf-7d5c5a11f988)

### 테스트 실행 및 브라우저 표시

테스트 사이드바에서 **Show Browsers** 옵션을 선택하여 테스트를 실행하고 브라우저를 표시할 수도 있습니다. 그런 다음 녹색 삼각형을 클릭하여 테스트를 실행하면 브라우저가 열리고 테스트를 통해 시각적으로 실행되는 것을 볼 수 있습니다. 모든 테스트에 대해 브라우저를 열어 두려면 이 옵션을 선택된 상태로 유지하고 브라우저가 열리지 않은 헤드리스 모드에서 테스트를 실행하려면 선택을 취소합니다.

![show browsers while running tests](https://github.com/microsoft/playwright/assets/13063165/9f231530-0c43-466a-b944-8cf5102f714a)

**Close all browsers** 버튼을 사용하여 모든 브라우저를 닫습니다.

### 모든 테스트 보기 및 실행

테스트 사이드바에서 모든 테스트를 보고 각 테스트를 클릭하여 테스트를 확장합니다. 실행되지 않은 테스트에는 옆에 녹색 체크 표시가 없습니다. 테스트 사이드바에서 테스트 위로 마우스를 가져가면 흰색 삼각형을 클릭하여 모든 테스트를 실행합니다.

![run all tests](https://github.com/microsoft/playwright/assets/13063165/348e18ff-f819-4caa-8f7e-f16c20724f56)

### 여러 브라우저에서 테스트 실행

Playwright 사이드바의 첫 번째 섹션은 프로젝트 섹션입니다. 여기에서 Playwright 구성 파일에 정의된 대로 모든 프로젝트를 볼 수 있습니다. Playwright를 설치할 때의 기본 구성은 Chromium, Firefox 및 WebKit의 3가지 프로젝트를 제공합니다. 첫 번째 프로젝트는 기본적으로 선택됩니다.

![Projects section in VS Code extension](https://github.com/microsoft/playwright/assets/13063165/58fedea6-a2b9-4942-b2c7-2f3d482210cf)

여러 프로젝트에서 테스트를 실행하려면 프로젝트 이름 옆에 있는 확인란을 선택하여 각 프로젝트를 선택합니다. 그런 다음 사이드바에서 또는 테스트 이름 옆에 있는 재생 버튼을 눌러 테스트를 실행하면 선택한 모든 프로젝트에서 테스트가 실행됩니다.

![Selecting projects to run tests on](https://github.com/microsoft/playwright/assets/13063165/6dc86ef4-6097-481c-9cab-b6e053ec7ea6)

테스트의 프로젝트 이름 옆에 있는 회색 재생 버튼을 클릭하여 특정 프로젝트에서 개별적으로 테스트를 실행할 수도 있습니다.

![Running a test on a specific project](https://github.com/microsoft/playwright/assets/13063165/d29a27ab-07b5-4ca6-b4d7-1ad6d44bf222)

### Trace viewer로 테스트 실행

더 나은 개발자 경험을 위해 **Show Trace Viewer** 옵션을 사용하여 테스트를 실행할 수 있습니다.

![run tests with trace viewer](https://github.com/microsoft/playwright/assets/13063165/fab8efa6-d5ef-496d-876d-c99e94e6a6b3)

이렇게 하면 테스트의 모든 동작을 단계별로 진행하고, 타임라인, 소스 코드 등을 탐색할 수 있는 테스트의 전체 추적이 열립니다.

![trace viewer](https://github.com/microsoft/playwright/assets/13063165/ee31a4fe-c786-4d4b-887e-2dcecacfba2c)

trace viewer에 대한 자세한 내용은 [Trace Viewer 가이드](./trace-viewer.mdx)를 참조하세요.

## 테스트 디버깅

VS Code 확장 프로그램을 사용하면 VS Code에서 바로 테스트를 디버그하고, 오류 메시지를 확인하고, 중단점을 만들고, 테스트를 실시간으로 디버그할 수 있습니다.

<LiteYouTube id="tJF7UhA59Gc" title="Debugging Playwright tests in VS Code" />

### 오류 메시지

테스트가 실패하면 VS Code는 예상된 내용, 수신된 내용 및 전체 호출 로그를 보여주는 편집기에서 바로 오류 메시지를 표시합니다.

![error messaging in vs code](https://github.com/microsoft/playwright/assets/13063165/3b8af12a-4805-4573-9d38-92055a0a7e75)

### 라이브 디버깅

VS Code에서 테스트를 실시간으로 디버그할 수 있습니다. `Show Browser` 옵션을 선택한 상태로 테스트를 실행한 후 VS Code에서 로케이터를 클릭하면 브라우저 창에서 강조 표시됩니다. Playwright는 존재하는 경우 강조 표시하고 결과가 두 개 이상 있는 경우 표시합니다.

![live debugging in vs code](https://github.com/microsoft/playwright/assets/13063165/7d236ebb-3d2d-4384-b73d-32a2b4e33b9e)

VS Code에서 로케이터를 편집할 수도 있으며 Playwright는 브라우저 창에서 변경 사항을 실시간으로 표시합니다.

### 디버그 모드에서 실행

중단점을 설정하려면 빨간색 점이 나타날 때까지 중단점을 설정할 줄 번호 옆을 클릭합니다. 실행할 테스트 옆에 있는 줄을 마우스 오른쪽 버튼으로 클릭하여 디버그 모드에서 테스트를 실행합니다.

![setting debug mode](https://github.com/microsoft/playwright/assets/13063165/31640629-efac-4cc7-b8b0-80ae18a3af83)

브라우저 창이 열리고 테스트가 실행되고 중단점이 설정된 위치에서 일시 중지됩니다. VS Code의 메뉴에서 테스트를 단계별로 진행하고, 테스트를 일시 중지하고, 테스트를 다시 실행할 수 있습니다.

![running in debug mode](https://github.com/microsoft/playwright/assets/13063165/b96a9f50-0f4d-49f9-a3d8-f093980a5673)

![choosing a profile for debugging](https://github.com/microsoft/playwright/assets/13063165/48c1b428-8dd2-4229-9eb5-24f7168db834)

디버깅에 대한 자세한 내용은 [Visual Studio Code에서 디버깅](https://code.visualstudio.com/docs/editor/debugging)을 참조하십시오.

### Trace viewer로 디버그

더 나은 개발자 경험을 위해 **Show Trace Viewer** 옵션을 사용하여 테스트를 디버그할 수 있습니다.

![run tests with trace viewer](https://github.com/microsoft/playwright/assets/13063165/fab8efa6-d5ef-496d-876d-c99e94e6a6b3)

이렇게 하면 테스트의 전체 추적이 열리며, 여기에서 각 동작을 단계별로 진행하고 동작 전후에 발생한 상황을 확인할 수 있습니다. DOM 스냅샷을 검사하고, 콘솔 로그, 네트워크 요청, 소스 코드 등을 볼 수도 있습니다.

![trace viewer](https://github.com/microsoft/playwright/assets/13063165/ee31a4fe-c786-4d4b-887e-2dcecacfba2c)

trace viewer에 대한 자세한 내용은 [Trace Viewer 가이드](./trace-viewer.mdx)를 참조하세요.

## 테스트 생성

CodeGen은 브라우저에서 작업을 수행할 때 테스트를 자동으로 생성하며 빠르게 시작하는 데 좋은 방법입니다. 브라우저 창의 뷰포트는 특정 너비와 높이로 설정됩니다. 뷰포트를 변경하거나 다른 환경을 에뮬레이션하려면 [구성 가이드](./test-configuration.mdx)를 참조하십시오.

<LiteYouTube id="5XIZPqKkdBA" title="Generating Playwright tests in VS Code" />

### 새 테스트 기록

테스트를 기록하려면 테스트 사이드바에서 **Record new** 버튼을 클릭합니다. 그러면 `test-1.spec.ts` 파일이 생성되고 브라우저 창이 열립니다. 브라우저에서 테스트할 URL로 이동하여 클릭하기 시작합니다. Playwright는 작업을 기록하고 VS Code에서 직접 테스트 코드를 생성합니다. 툴바에서 아이콘 중 하나를 선택한 다음 페이지의 요소를 클릭하여 확인할 어설션을 생성할 수도 있습니다. 다음 어설션을 생성할 수 있습니다.
* `'assert visibility'` 요소를 표시할지 여부를 어설션합니다.
* `'assert text'` 요소를 특정 텍스트가 포함되어 있는지 어설션합니다.
* `'assert value'` 요소를 특정 값이 있는지 어설션합니다.

기록이 완료되면 **cancel** 버튼을 클릭하거나 브라우저 창을 닫습니다. 그런 다음 `test-1.spec.ts` 파일을 검사하고 생성된 테스트를 볼 수 있습니다.

![record a new test](https://github.com/microsoft/playwright/assets/13063165/0407f112-e1cd-41e7-a05d-ae64e24d27ed)

### 커서에서 기록

테스트 파일의 특정 지점에서 기록하려면 테스트 사이드바에서 **Record at cursor** 버튼을 클릭합니다. 이렇게 하면 현재 커서 위치에서 기존 테스트에 작업이 생성됩니다. 테스트를 실행하고, 커서를 테스트 끝에 놓고, 테스트 생성을 계속할 수 있습니다.

![record at cursor](https://github.com/microsoft/playwright/assets/13063165/96933ea1-4c84-453a-acd7-22b4d3bde185)

### 로케이터 선택

**Pick locator** 버튼을 사용하여 [로케이터](./locators.mdx)를 선택하고 테스트 파일에 복사합니다. 그런 다음 브라우저에서 필요한 요소를 클릭하면 이제 VS Code의 **Pick locator** 상자에 나타납니다. 키보드에서 'enter' 키를 눌러 로케이터를 클립보드에 복사한 다음 코드의 아무 곳에나 붙여넣습니다. 취소하려면 'escape' 키를 누릅니다.

![pick locators](https://github.com/microsoft/playwright/assets/13063165/9a1b2da9-9ac7-4def-a9e0-f94770364fc2)

Playwright는 페이지를 살펴보고 최상의 로케이터를 파악하여 [역할, 텍스트 및 테스트 ID 로케이터](./locators.mdx)의 우선 순위를 지정합니다. 생성기가 로케이터와 일치하는 여러 요소를 찾으면 로케이터를 개선하여 복원력이 있고 대상 요소를 고유하게 식별하므로 로케이터로 인해 테스트가 실패하는 것에 대해 걱정할 필요가 없습니다.

## 프로젝트 종속성

[프로젝트 종속성](./test-projects.mdx)을 사용하여 다른 테스트에 종속된 테스트를 실행할 수 있습니다. 이는 웹사이트에 로그인하는 것과 같은 **설정** 테스트에 유용합니다.

### 설정 테스트 실행

구성 파일에 정의된 대로 Playwright 사이드바의 프로젝트 섹션에서 **setup** 프로젝트를 선택하여 설정 테스트를 실행합니다. 이렇게 하면 테스트 탐색기에서 **setup** 테스트에 액세스할 수 있습니다.

![setup tests in vscode](https://github.com/microsoft/playwright/assets/13063165/7a9eccd5-a5b3-431f-9eff-9b2971501e07)

**setup** 테스트에 종속된 테스트를 실행하면 **setup** 테스트가 먼저 실행됩니다. 테스트를 실행할 때마다 **setup** 테스트가 다시 실행됩니다.

![running setup tests in vscode](https://github.com/microsoft/playwright/assets/13063165/a54b3868-3f9f-4e74-8d42-a93443f099fc)

### 설정 테스트를 한 번만 실행

**setup** 테스트를 한 번만 실행하려면 Playwright 사이드바의 프로젝트 섹션에서 선택을 취소합니다. 이제 **setup** 테스트가 테스트 탐색기에서 제거되었습니다. **setup** 테스트에 종속된 테스트를 실행하면 더 이상 **setup** 테스트를 실행하지 않으므로 훨씬 빠르고 훨씬 더 나은 개발자 경험을 제공합니다.

![deselecting setup tests in vscode](https://github.com/microsoft/playwright/assets/13063165/ebc50e38-c98d-4538-82fe-ec08491f8487)

## 전역 설정

**전역 설정**은 첫 번째 테스트를 실행할 때 실행됩니다. 한 번만 실행되며 데이터베이스를 설정하거나 서버를 시작하는 데 유용합니다. Playwright 사이드바의 **Setup** 섹션에서 `Run global setup` 옵션을 클릭하여 **전역 설정**을 수동으로 실행할 수 있습니다. **전역 해제**는 기본적으로 실행되지 않습니다. `Run global teardown` 옵션을 클릭하여 수동으로 시작해야 합니다.

전역 설정은 테스트를 디버깅할 때 다시 실행됩니다. 이는 격리된 환경과 테스트 전용 설정을 보장하기 때문입니다.

![running global setup](https://github.com/microsoft/playwright/assets/13063165/bcf5fec7-2d7d-4cb9-a277-5f41e19b8d52)

## 다중 구성

프로젝트에 둘 이상의 playwright 구성 파일이 포함되어 있는 경우 먼저 Playwright 사이드바의 오른쪽 상단 모서리에 있는 기어 아이콘을 클릭하여 구성 파일을 전환할 수 있습니다. 이렇게 하면 프로젝트의 모든 구성 파일이 표시됩니다. 각 파일 옆에 있는 확인란을 선택하고 'ok' 버튼을 클릭하여 사용할 구성 파일을 선택합니다.

![Selecting a configuration file](https://github.com/microsoft/playwright/assets/13063165/ff9ff838-d27a-403d-b939-94e6c295e1d7)

이제 테스트 탐색기에서 모든 테스트에 액세스할 수 있습니다. 테스트를 실행하려면 파일 또는 프로젝트 이름 옆에 있는 회색 삼각형을 클릭합니다.

![Switching between configuration files](https://github.com/microsoft/playwright/assets/13063165/70930de5-0a0c-45e0-a6ee-b51f727f0e35)

모든 구성에서 모든 테스트를 실행하려면 테스트 탐색기 상단에 있는 회색 삼각형을 클릭합니다.

![Running all tests from all configurations](https://github.com/microsoft/playwright/assets/13063165/b3de4ce1-d311-4527-b2c7-b3e2f179a685)

작업할 구성 파일을 선택하려면 Playwright 사이드바에서 구성 파일 이름을 클릭하여 간단히 전환합니다. 이제 Record a test와 같은 도구를 사용하면 선택한 구성 파일에 대한 테스트가 기록됩니다.

![Recording a test for a specific configuration file](https://github.com/microsoft/playwright/assets/13063165/a8ecbcd1-fab8-4012-bdaa-428951f233a2)

Playwright 사이드바에서 구성 파일 이름을 클릭하여 구성을 쉽게 앞뒤로 전환할 수 있습니다.
