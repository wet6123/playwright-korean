## Introduction

프로젝트는 동일한 구성으로 실행되는 테스트의 논리적 그룹입니다. 프로젝트를 사용하여 서로 다른 브라우저 및 장치에서 테스트를 실행할 수 있습니다. 프로젝트는 `playwright.config.ts` 파일에서 구성되며, 구성이 완료되면 모든 프로젝트 또는 특정 프로젝트에서만 테스트를 실행할 수 있습니다. 프로젝트를 사용하여 동일한 테스트를 다른 구성으로 실행할 수도 있습니다. 예를 들어, 로그인 상태와 로그아웃 상태에서 동일한 테스트를 실행할 수 있습니다.

프로젝트를 설정하면 서로 다른 타임아웃 또는 재시도 횟수로 테스트 그룹을 실행하거나 스테이징 및 프로덕션과 같은 서로 다른 환경에 대해 테스트 그룹을 실행하고 패키지/기능별로 테스트를 분할하는 등의 작업을 수행할 수 있습니다.

## Configure projects for multiple browsers

**프로젝트**를 사용하면 chromium, webkit 및 firefox와 같은 여러 브라우저뿐만 아니라 Google Chrome 및 Microsoft Edge와 같은 브랜드 브라우저에서도 테스트를 실행할 수 있습니다. Playwright는 에뮬레이션된 태블릿 및 모바일 장치에서도 실행할 수 있습니다. 선택된 데스크톱, 태블릿 및 모바일 장치의 전체 목록은 [장치 매개변수 레지스트리](https://github.com/microsoft/playwright/blob/main/packages/playwright-core/src/server/deviceDescriptorsSource.json)를 참조하십시오.

```js
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },

    {
      name: 'firefox',
      use: { ...devices['Desktop Firefox'] },
    },

    {
      name: 'webkit',
      use: { ...devices['Desktop Safari'] },
    },

    /* Test against mobile viewports. */
    {
      name: 'Mobile Chrome',
      use: { ...devices['Pixel 5'] },
    },
    {
      name: 'Mobile Safari',
      use: { ...devices['iPhone 12'] },
    },

    /* Test against branded browsers. */
    {
      name: 'Microsoft Edge',
      use: {
        ...devices['Desktop Edge'],
        channel: 'msedge'
      },
    },
    {
      name: 'Google Chrome',
      use: {
        ...devices['Desktop Chrome'],
        channel: 'chrome'
      },
    },
  ],
});
```

## Run projects

Playwright는 기본적으로 모든 프로젝트를 실행합니다.

```bash
npx playwright test

Running 7 tests using 5 workers

  ✓ [chromium] › example.spec.ts:3:1 › basic test (2s)
  ✓ [firefox] › example.spec.ts:3:1 › basic test (2s)
  ✓ [webkit] › example.spec.ts:3:1 › basic test (2s)
  ✓ [Mobile Chrome] › example.spec.ts:3:1 › basic test (2s)
  ✓ [Mobile Safari] › example.spec.ts:3:1 › basic test (2s)
  ✓ [Microsoft Edge] › example.spec.ts:3:1 › basic test (2s)
  ✓ [Google Chrome] › example.spec.ts:3:1 › basic test (2s)
```

`--project` 명령줄 옵션을 사용하여 단일 프로젝트를 실행합니다.

```bash
npx playwright test --project=firefox

Running 1 test using 1 worker

  ✓ [firefox] › example.spec.ts:3:1 › basic test (2s)
```

VS Code 테스트 러너는 Chrome의 기본 브라우저에서 테스트를 실행합니다. 다른/여러 브라우저에서 실행하려면 테스트 사이드바에서 재생 버튼의 드롭다운을 클릭하고 다른 프로필을 선택하거나 **기본 프로필 선택**을 클릭하여 기본 프로필을 수정하고 테스트를 실행할 브라우저를 선택합니다.

## Configure projects for multiple environments

프로젝트를 설정하면 서로 다른 타임아웃 또는 재시도 횟수로 테스트 그룹을 실행하거나 서로 다른 환경에 대해 테스트 그룹을 실행할 수도 있습니다. 예를 들어, 2번의 재시도가 있는 스테이징 환경과 0번의 재시도가 있는 프로덕션 환경에 대해 테스트를 실행할 수 있습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  timeout: 60000, // Timeout은 모든 테스트 간에 공유됩니다.
  projects: [
    {
      name: 'staging',
      use: {
        baseURL: 'staging.example.com',
      },
      retries: 2,
    },
    {
      name: 'production',
      use: {
        baseURL: 'production.example.com',
      },
      retries: 0,
    },
  ],
});
```

## Splitting tests into projects

테스트를 프로젝트로 분할하고 필터를 사용하여 테스트 하위 집합을 실행할 수 있습니다. 예를 들어, 특정 파일 이름으로 모든 테스트와 일치하는 필터를 사용하여 테스트를 실행하는 프로젝트를 만들 수 있습니다. 그런 다음 특정 테스트 파일을 무시하는 다른 테스트 그룹을 가질 수 있습니다.

다음은 공통 타임아웃과 두 개의 프로젝트를 정의하는 예입니다. "Smoke" 프로젝트는 재시도 없이 테스트의 작은 하위 집합을 실행하고, "Default" 프로젝트는 재시도와 함께 다른 모든 테스트를 실행합니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  timeout: 60000, // Timeout은 모든 테스트 간에 공유됩니다.
  projects: [
    {
      name: 'Smoke',
      testMatch: /.*smoke.spec.ts/,
      retries: 0,
    },
    {
      name: 'Default',
      testIgnore: /.*smoke.spec.ts/,
      retries: 2,
    },
  ],
});
```

## Dependencies

종속성은 다른 프로젝트의 테스트를 실행하기 전에 실행해야 하는 프로젝트 목록입니다. 이는 전역 설정 작업을 구성하여 한 프로젝트가 먼저 실행되도록 하는 데 유용할 수 있습니다. 프로젝트 종속성을 사용하면 [테스트 리포터](./test-reporters.mdx)에 설정 테스트가 표시되고 [trace viewer](/trace-viewer.mdx)에 설정 추적이 기록됩니다. 검사기를 사용하여 설정 테스트 추적의 DOM 스냅샷을 검사하고 설정 내에서 [픽스처](./test-fixtures.mdx)를 사용할 수도 있습니다.

이 예에서 chromium, firefox 및 webkit 프로젝트는 setup 프로젝트에 종속됩니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  projects: [
    {
      name: 'setup',
      testMatch: '**/*.setup.ts',
    },
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
      dependencies: ['setup'],
    },
    {
      name: 'firefox',
      use: { ...devices['Desktop Firefox'] },
      dependencies: ['setup'],
    },
    {
      name: 'webkit',
      use: { ...devices['Desktop Safari'] },
      dependencies: ['setup'],
    },
  ],
});
```

### Running Sequence

종속성이 있는 테스트로 작업할 때 종속성이 항상 먼저 실행되고 이 프로젝트의 모든 테스트가 통과되면 다른 프로젝트가 병렬로 실행됩니다.

실행 순서:
1. 'setup' 프로젝트의 테스트가 실행됩니다. 이 프로젝트의 모든 테스트가 통과되면 종속 프로젝트의 테스트가 실행되기 시작합니다.

2. 'chromium', 'webkit' 및 'firefox' 프로젝트의 테스트가 함께 실행됩니다. 기본적으로 이러한 프로젝트는 최대 worker 제한에 따라 [병렬로 실행](./test-parallel.mdx)됩니다.

<img width="70%" style={{display: 'flex', margin: 'auto'}} alt="chromium, webkit and firefox projects depend on setup project" loading="lazy" src="https://user-images.githubusercontent.com/13063165/225937080-327b1e63-431f-40e0-90d7-35f21d7a92cb.jpg" />

종속성이 두 개 이상인 경우 이러한 프로젝트 종속성이 먼저 병렬로 실행됩니다. 종속성의 테스트가 실패하면 이 프로젝트에 의존하는 테스트는 실행되지 않습니다.

실행 순서:
1. 'Browser Login' 및 'DataBase' 프로젝트의 테스트가 병렬로 실행됩니다.
   - 'Browser Login' 통과
   - ❌ 'DataBase' 실패!
1. 'e2e tests' 프로젝트가 실행되지 않습니다!

<img width="70%" style={{display: 'flex', margin: 'auto'}} alt="Browser login project is blue, database is red and e2e tests relies on both" loading="lazy" src="https://user-images.githubusercontent.com/13063165/225938262-33c1b78f-f092-4762-a478-7f8cbc1e3b21.jpg" />

### Teardown

설정 프로젝트에 [testProject.teardown](/api/class-testproject.mdx#test-project-teardown) 속성을 추가하여 설정을 정리할 수도 있습니다. Teardown은 모든 종속 프로젝트가 실행된 후에 실행됩니다. 자세한 내용은 [teardown 가이드](./test-global-setup-teardown.mdx#teardown)를 참조하십시오.

### Test filtering

`--grep/--grep-invert` 또는 `--shard` [옵션](./test-cli.mdx#reference)을 사용하거나 [명령줄](./test-cli.mdx) 또는 [test.only()](./api/class-test.mdx#test-only)에서 테스트 파일 이름 필터를 지정하면 프로젝트 종속성 체인에서 가장 깊은 프로젝트의 테스트에만 적용됩니다. 즉, 일치하는 테스트가 프로젝트 종속성이 있는 프로젝트에 속하는 경우 Playwright는 필터를 무시하고 프로젝트 종속성의 모든 테스트를 실행합니다.
