## Migration principles

이 가이드는 [DOM Testing Library](https://testing-library.com/docs/dom-testing-library/intro/), [React Testing Library](https://testing-library.com/docs/react-testing-library/intro/), [Vue Testing Library](https://testing-library.com/docs/vue-testing-library/intro) 및 [Svelte Testing Library](https://testing-library.com/docs/svelte-testing-library/intro)에서 Playwright의 [Experimental Component Testing](./test-components)으로 마이그레이션하는 방법을 설명합니다.

:::note

브라우저에서 DOM Testing Library를 사용하는 경우(예: webpack으로 엔드 투 엔드 테스트를 번들링하는 경우) Playwright Test로 직접 전환할 수 있습니다. 아래 예제는 컴포넌트 테스트에 중점을 두지만, 엔드 투 엔드 테스트의 경우 테스트 중인 페이지를 열려면 `await mount`를 `await page.goto('http://localhost:3000/')`로 바꾸기만 하면 됩니다.
:::

## Cheat Sheet

| Testing Library                                                                 | Playwright                                                             |
| ------------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| [screen](https://testing-library.com/docs/queries/about#screen)                 | [page](./api/class-page) 및 [component](./api/class-locator)          |
| [queries](https://testing-library.com/docs/queries/about)                       | [locators](./locators)                                                 |
| [async helpers](https://testing-library.com/docs/dom-testing-library/api-async) | [assertions](./test-assertions)                                        |
| [user events](https://testing-library.com/docs/user-event/intro)                | [actions](./api/class-locator)                                         |
| `await user.click(screen.getByText('Click me'))`                                | `await component.getByText('Click me').click()`                        |
| `await user.click(await screen.findByText('Click me'))`                         | `await component.getByText('Click me').click()`                        |
| `await user.type(screen.getByLabelText('Password'), 'secret')`                  | `await component.getByLabel('Password').fill('secret')`                |
| `expect(screen.getByLabelText('Password')).toHaveValue('secret')`               | `await expect(component.getByLabel('Password')).toHaveValue('secret')` |
| `screen.getByRole('button', { pressed: true })`                                 | `component.getByRole('button', { pressed: true })`                     |
| `screen.getByLabelText('...')`                                                  | `component.getByLabel('...')`                                          |
| `screen.queryByPlaceholderText('...')`                                          | `component.getByPlaceholder('...')`                                    |
| `screen.findByText('...')`                                                      | `component.getByText('...')`                                           |
| `screen.getByTestId('...')`                                                     | `component.getByTestId('...')`                                          |
| `render(<Component />);`                                                        | `mount(<Component />);`                                                |
| `const { unmount } = render(<Component />);`                                    | `const { unmount } = await mount(<Component />);`                      |
| `const { rerender } = render(<Component />);`                                   | `const { update } = await mount(<Component />);`                       |

## Example

Testing Library:

```js
import React from 'react';
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';

test('sign in', async () => {
  // Setup the page.
  const user = userEvent.setup();
  render(<SignInPage />);

  // Perform actions.
  await user.type(screen.getByLabelText('Username'), 'John');
  await user.type(screen.getByLabelText('Password'), 'secret');
  await user.click(screen.getByRole('button', { name: 'Sign in' }));

  // Verify signed in state by waiting until "Welcome" message appears.
  expect(await screen.findByText('Welcome, John')).toBeInTheDocument();
});
```

Line-by-line migration to Playwright Test:

```js
const { test, expect } = require('@playwright/experimental-ct-react'); // 1

test('sign in', async ({ mount }) => { // 2
  // Setup the page.
  const component = await mount(<SignInPage />); // 3

  // Perform actions.
  await component.getByLabel('Username').fill('John'); // 4
  await component.getByLabel('Password').fill('secret');
  await component.getByRole('button', { name: 'Sign in' }).click();

  // Verify signed in state by waiting until "Welcome" message appears.
  await expect(component.getByText('Welcome, John')).toBeVisible(); // 5
});
```

마이그레이션 주요 사항 (Playwright Test 코드 스니펫의 인라인 주석 참조):
1. 컴포넌트 테스트의 경우 `@playwright/experimental-ct-react`(또는 -vue, -svelte)에서, 엔드 투 엔드 테스트의 경우 `@playwright/test`에서 모든 것을 가져옵니다.
2. 테스트 함수는 다른 테스트와 격리된 `page`와 이 페이지에서 컴포넌트를 렌더링하는 `mount`가 제공됩니다. 이들은 Playwright Test의 [유용한 fixtures](./api/class-fixtures) 중 두 가지입니다.
3. `render`를 [component locator](./locators)를 반환하는 `mount`로 바꿉니다.
4. 대부분의 작업을 수행하려면 [locator.locator()](/api/class-locator.mdx#locator-locator) 또는 [page.locator()](/api/class-page.mdx#page-locator)로 생성된 로케이터를 사용합니다.
5. 상태를 확인하려면 [assertions](./test-assertions)를 사용합니다.

## Migrating queries

`getBy...`, `findBy...`, `queryBy...`와 같은 모든 쿼리와 해당 다중 요소 대응 항목은 `component.getBy...` 로케이터로 대체됩니다. 로케이터는 항상 필요할 때 자동 대기 및 재시도를 수행하므로 올바른 방법을 선택하는 데 신경 쓸 필요가 없습니다. [list operation](./locators#lists)을 수행하려는 경우(예: 텍스트 목록을 어설션하는 경우) Playwright는 자동으로 다중 요소 작업을 수행합니다.

## Replacing `waitFor`

Playwright에는 조건이 충족될 때까지 자동으로 대기하는 [assertions](./test-assertions)가 포함되어 있으므로 일반적으로 명시적인 `waitFor`/`waitForElementToBeRemoved` 호출이 필요하지 않습니다.

```js
// Testing Library
await waitFor(() => {
  expect(getByText('the lion king')).toBeInTheDocument();
});
await waitForElementToBeRemoved(() => queryByText('the mummy'));

// Playwright
await expect(page.getByText('the lion king')).toBeVisible();
await expect(page.getByText('the mummy')).toBeHidden();
```

적절한 어설션을 찾을 수 없는 경우 [`expect.poll`](./test-assertions#expectpoll)을 대신 사용하십시오.

```js
await expect.poll(async () => {
  const response = await page.request.get('https://api.example.com');
  return response.status();
}).toBe(200);
```

## Replacing `within`

[locator.locator()](/api/class-locator.mdx#locator-locator) 메서드를 사용하여 다른 로케이터 안에 로케이터를 만들 수 있습니다.

```js
// Testing Library
const messages = document.getElementById('messages');
const helloMessage = within(messages).getByText('hello');

// Playwright
const messages = component.getByTestId('messages');
const helloMessage = messages.getByText('hello');
```

## Playwright Test Super Powers

Playwright Test를 사용하면 많은 것을 얻을 수 있습니다!
- 완전한 제로 구성 TypeScript 지원
- **모든 웹 엔진**(Chrome, Firefox, Safari)에서 **모든 인기 있는 운영 체제**(Windows, macOS, Ubuntu)에서 테스트 실행
- 여러 origins, [(i)frames](./api/class-frame), [tabs and contexts](./pages)에 대한 완전한 지원
- 여러 브라우저에서 병렬로 격리된 테스트 실행
- 내장된 테스트 [artifact collection](./test-use-options.mdx#recording-options)

또한 Playwright Test와 함께 제공되는 다음과 같은 ✨ 멋진 도구 ✨를 모두 얻을 수 있습니다.
- [Visual Studio Code integration](./getting-started-vscode.mdx)
- watch 모드가 완료된 시간 여행 경험으로 테스트를 디버깅하기 위한 [UI mode](./test-ui-mode.mdx).
- [Playwright Inspector](./debug.mdx#playwright-inspector)
- [Playwright Test Code generation](./codegen-intro.mdx)
- 사후 디버깅을 위한 [Playwright Tracing](./trace-viewer.mdx)
