## Introduction

Playwright Test는 다양한 요구 사항에 맞는 몇 가지 기본 제공 리포터와 사용자 지정 리포터를 제공하는 기능을 제공합니다. 기본 제공 리포터를 사용해 보는 가장 쉬운 방법은 `--reporter` [명령줄 옵션](./test-cli.mdx)을 전달하는 것입니다.

```bash
npx playwright test --reporter=line
```

더 많은 제어를 위해 [구성 파일](./test-configuration.mdx)에서 프로그래밍 방식으로 리포터를 지정할 수 있습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: 'line',
});
```

### Multiple reporters

여러 리포터를 동시에 사용할 수 있습니다. 예를 들어 멋진 터미널 출력을 위해 `'list'`를 사용하고 테스트 결과가 포함된 포괄적인 json 파일을 얻기 위해 `'json'`을 사용할 수 있습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: [
    ['list'],
    ['json', {  outputFile: 'test-results.json' }]
  ],
});
```

### Reporters on CI

로컬 및 CI에서 서로 다른 리포터를 사용할 수 있습니다. 예를 들어 간결한 `'dot'` 리포터를 사용하면 출력이 너무 많아지는 것을 방지할 수 있습니다. 이는 CI의 기본값입니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  // CI에는 간결한 'dot'을 사용하고, 로컬에서 실행할 때는 기본적으로 'list'를 사용합니다.
  reporter: process.env.CI ? 'dot' : 'list',
});
```

## Built-in reporters

모든 기본 제공 리포터는 실패에 대한 자세한 정보를 표시하며, 대부분 성공적인 실행에 대한 자세한 정도가 다릅니다.

### List reporter

List reporter가 기본값입니다(CI에서는 `dot` reporter가 기본값임). 실행되는 각 테스트에 대해 한 줄을 출력합니다.

```bash
npx playwright test --reporter=list
```

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: 'list',
});
```

테스트 실행 중간에 있는 출력 예제입니다. 실패는 마지막에 나열됩니다.

```bash
npx playwright test --reporter=list
Running 124 tests using 6 workers

 1  ✓ should access error in env (438ms)
 2  ✓ handle long test names (515ms)
 3  x 1) render expected (691ms)
 4  ✓ should timeout (932ms)
 5    should repeat each:
 6  ✓ should respect enclosing .gitignore (569ms)
 7    should teardown env after timeout:
 8    should respect excluded tests:
 9  ✓ should handle env beforeEach error (638ms)
10    should respect enclosing .gitignore:
```

다음 구성 옵션을 전달하여 단계 렌더링을 선택할 수 있습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: [['list', { printSteps: true }]],
});
```

List report는 다음 구성 옵션 및 환경 변수를 지원합니다.

| Environment Variable Name | Reporter Config Option| Description | Default
|---|---|---|---|
| `PLAYWRIGHT_LIST_PRINT_STEPS` | `printSteps` | 각 단계를 자체 행에 출력할지 여부입니다. | `false`
| `PLAYWRIGHT_FORCE_TTY` | | 라이브 터미널에 적합한 출력을 생성할지 여부입니다. 숫자가 지정되면 터미널 너비로도 사용됩니다. | 터미널이 TTY 모드인 경우 `true`, 그렇지 않으면 `false`입니다.
| `FORCE_COLOR` | | 색상이 지정된 출력을 생성할지 여부입니다. | 터미널이 TTY 모드인 경우 `true`, 그렇지 않으면 `false`입니다.

### Line reporter

Line reporter는 list reporter보다 간결합니다. 마지막으로 완료된 테스트를 보고하는 데 한 줄을 사용하고 실패가 발생하면 실패를 출력합니다. Line reporter는 진행 상황을 보여주지만 모든 테스트를 나열하여 출력을 스팸으로 보내지 않는 대규모 테스트 스위트에 유용합니다.

```bash
npx playwright test --reporter=line
```

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: 'line',
});
```

테스트 실행 중간에 있는 출력 예제입니다. 실패는 인라인으로 보고됩니다.

```bash
npx playwright test --reporter=line
Running 124 tests using 6 workers
  1) dot-reporter.spec.ts:20:1 › render expected ===================================================

    Error: expect(received).toBe(expected) // Object.is equality

    Expected: 1
    Received: 0

[23/124] gitignore.spec.ts - should respect nested .gitignore
```

Line report는 다음 구성 옵션 및 환경 변수를 지원합니다.

| Environment Variable Name | Reporter Config Option| Description | Default
|---|---|---|---|
| `PLAYWRIGHT_FORCE_TTY` | | 라이브 터미널에 적합한 출력을 생성할지 여부입니다. 숫자가 지정되면 터미널 너비로도 사용됩니다. | 터미널이 TTY 모드인 경우 `true`, 그렇지 않으면 `false`입니다.
| `FORCE_COLOR` | | 색상이 지정된 출력을 생성할지 여부입니다. | 터미널이 TTY 모드인 경우 `true`, 그렇지 않으면 `false`입니다.

### Dot reporter

Dot reporter는 매우 간결합니다. 성공적인 테스트 실행당 단일 문자만 생성합니다. CI의 기본값이며 출력을 많이 원하지 않는 경우에 유용합니다.

```bash
npx playwright test --reporter=dot
```

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: 'dot',
});
```

테스트 실행 중간에 있는 출력 예제입니다. 실패는 마지막에 나열됩니다.

```bash
npx playwright test --reporter=dot
Running 124 tests using 6 workers
······F·············································
```

실행된 각 테스트에 대해 해당 상태를 나타내는 한 문자가 표시됩니다.

| Character | Description
|---|---|
| `·` | Passed
| `F` | Failed
| `×` | Failed or timed out - and will be retried
| `±` | Passed on retry (flaky)
| `T` | Timed out
| `°` | Skipped

Dot report는 다음 구성 옵션 및 환경 변수를 지원합니다.

| Environment Variable Name | Reporter Config Option| Description | Default
|---|---|---|---|
| `PLAYWRIGHT_FORCE_TTY` | | 라이브 터미널에 적합한 출력을 생성할지 여부입니다. 숫자가 지정되면 터미널 너비로도 사용됩니다. | 터미널이 TTY 모드인 경우 `true`, 그렇지 않으면 `false`입니다.
| `FORCE_COLOR` | | 색상이 지정된 출력을 생성할지 여부입니다. | 터미널이 TTY 모드인 경우 `true`, 그렇지 않으면 `false`입니다.

### HTML reporter

HTML reporter는 웹 페이지로 제공할 수 있는 테스트 실행 보고서가 포함된 자체 포함 폴더를 생성합니다.

```bash
npx playwright test --reporter=html
```

기본적으로 일부 테스트가 실패하면 HTML 보고서가 자동으로 열립니다. Playwright 구성의 `open` 속성 또는 `PLAYWRIGHT_HTML_OPEN` 환경 변수를 통해 이 동작을 제어할 수 있습니다. 해당 속성에 사용할 수 있는 값은 `always`, `never` 및 `on-failure`(기본값)입니다.

HTML 보고서를 제공하는 데 사용되는 `host` 및 `port`를 구성할 수도 있습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: [['html', { open: 'never' }]],
});
```

기본적으로 보고서는 현재 작업 디렉터리의 `playwright-report` 폴더에 기록됩니다. `PLAYWRIGHT_HTML_OUTPUT_DIR` 환경 변수 또는 리포터 구성을 사용하여 해당 위치를 재정의할 수 있습니다.

구성 파일에서 옵션을 직접 전달합니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: [['html', { outputFolder: 'my-report' }]],
});
```

데이터 폴더에서 첨부 파일을 다른 위치에 업로드하는 경우 `attachmentsBaseURL` 옵션을 사용하여 html 보고서에 첨부 파일을 찾을 위치를 알릴 수 있습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: [['html', { attachmentsBaseURL: 'https://external-storage.com/' }]],
});
```

마지막 테스트 실행 보고서를 여는 빠른 방법은 다음과 같습니다.

```bash
npx playwright show-report
```

또는 사용자 지정 폴더 이름이 있는 경우:

```bash
npx playwright show-report my-report
```

HTML report는 다음 구성 옵션 및 환경 변수를 지원합니다.

| Environment Variable Name | Reporter Config Option| Description | Default
|---|---|---|---|
| `PLAYWRIGHT_HTML_OUTPUT_DIR` | `outputFolder` | 보고서를 저장할 디렉터리입니다. | `playwright-report`
| `PLAYWRIGHT_HTML_OPEN` | `open` | 브라우저에서 html 보고서를 열 때 `'always'`, `'never'` 또는 `'on-failure'` 중 하나입니다. | `'on-failure'`
| `PLAYWRIGHT_HTML_HOST` | `host` | 브라우저에서 보고서가 열릴 때 이 호스트 이름에 바인딩되어 제공됩니다. | `localhost`
| `PLAYWRIGHT_HTML_PORT` | `port` | 브라우저에서 보고서가 열릴 때 이 포트에서 제공됩니다. | `9323` 또는 `9323`을 사용할 수 없는 경우 사용 가능한 포트입니다.
| `PLAYWRIGHT_HTML_ATTACHMENTS_BASE_URL` | `attachmentsBaseURL` | `data` 하위 디렉터리의 첨부 파일이 업로드되는 별도의 위치입니다. 보고서와 `data`를 서로 다른 위치에 별도로 업로드하는 경우에만 필요합니다. | `data/`

### Blob reporter

Blob 보고서에는 테스트 실행에 대한 모든 세부 정보가 포함되어 있으며 나중에 다른 보고서를 생성하는 데 사용할 수 있습니다. 주요 기능은 [분할된 테스트](./test-sharding.mdx)에서 보고서를 병합하는 것을 용이하게 하는 것입니다.

```bash
npx playwright test --reporter=blob
```

기본적으로 보고서는 package.json 디렉터리 또는 현재 작업 디렉터리(package.json을 찾을 수 없는 경우)의 `blob-report` 디렉터리에 기록됩니다. 보고서 파일 이름은 [분할](./test-sharding.mdx)을 사용하는 경우 `report-<hash>.zip` 또는 `report-<hash>-<shard_number>.zip`과 같습니다. 해시는 명령줄 인수로 전달된 `--grep`, `--grepInverted`, `--project` 및 파일 필터에서 계산된 선택적 값입니다. 해시는 Playwright를 다른 명령줄 옵션으로 실행하면 실행 간에 서로 다르지만 안정적인 보고서 이름이 생성되도록 보장합니다. 출력 파일 이름은 구성 파일에서 재정의하거나 `'PLAYWRIGHT_BLOB_OUTPUT_FILE'` 환경 변수로 전달할 수 있습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: [['blob', { outputFile: `./blob-report/report-${os.platform()}.zip` }]],
});
```

Blob report는 다음 구성 옵션 및 환경 변수를 지원합니다.

| Environment Variable Name | Reporter Config Option| Description | Default
|---|---|---|---|
| `PLAYWRIGHT_BLOB_OUTPUT_DIR` | `outputDir` | 출력을 저장할 디렉터리입니다. 새 보고서를 쓰기 전에 기존 콘텐츠가 삭제됩니다. | `blob-report`
| `PLAYWRIGHT_BLOB_OUTPUT_NAME` | `fileName` | 보고서 파일 이름입니다. | `report-<project>-<hash>-<shard_number>.zip`
| `PLAYWRIGHT_BLOB_OUTPUT_FILE` | `outputFile` | 출력 파일의 전체 경로입니다. 정의된 경우 `outputDir` 및 `fileName`은 무시됩니다. | `undefined`

### GitHub Actions annotations

기본 제공 `github` reporter를 사용하여 GitHub Actions에서 실행할 때 자동 오류 주석을 가져올 수 있습니다.

다른 모든 reporter도 GitHub Actions에서 작동하지만 주석을 제공하지는 않습니다. 또한 매트릭스 전략으로 테스트를 실행하는 경우 스택 추적 오류가 곱셈되고 GitHub 파일 보기가 가려지므로 이 주석 유형을 사용하는 것은 권장되지 않습니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  // GitHub Actions CI의 경우 'github'를 사용하여 주석을 생성하고, 간결한 'dot'을 사용합니다.
  // 로컬에서 실행할 때는 기본적으로 'list'를 사용합니다.
  reporter: process.env.CI ? 'github' : 'list',
});
```

## Custom reporters

일부 reporter 메서드를 사용하여 클래스를 구현하여 사용자 지정 reporter를 만들 수 있습니다. [Reporter] API에 대해 자세히 알아보세요.

```js title="my-awesome-reporter.ts"
import type {
  FullConfig, FullResult, Reporter, Suite, TestCase, TestResult
} from '@playwright/test/reporter';

class MyReporter implements Reporter {
  onBegin(config: FullConfig, suite: Suite) {
    console.log(`Starting the run with ${suite.allTests().length} tests`);
  }

  onTestBegin(test: TestCase, result: TestResult) {
    console.log(`Starting test ${test.title}`);
  }

  onTestEnd(test: TestCase, result: TestResult) {
    console.log(`Finished test ${test.title}: ${result.status}`);
  }

  onEnd(result: FullResult) {
    console.log(`Finished the run: ${result.status}`);
  }
}

export default MyReporter;
```

이제 [testConfig.reporter](/api/class-testconfig.mdx#test-config-reporter)와 함께 이 reporter를 사용합니다.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  reporter: './my-awesome-reporter.ts',
});
```

또는 reporter 파일 경로를 `--reporter` 명령줄 옵션으로 전달합니다.

```bash
npx playwright test --reporter="./myreporter/my-awesome-reporter.ts"
```

다음은 자체 reporter를 작성할 때 살펴볼 수 있는 오픈 소스 reporter 구현의 짧은 목록입니다.
* [Allure Reporter](https://github.com/allure-framework/allure-js/tree/main/packages/allure-playwright)
* [Github Actions Reporter](https://github.com/estruyf/playwright-github-actions-reporter)
* [Mail Reporter](https://github.com/estruyf/playwright-mail-reporter)
* [ReportPortal](https://github.com/reportportal/agent-js-playwright)
* [Monocart](https://github.com/cenfun/monocart-reporter)
